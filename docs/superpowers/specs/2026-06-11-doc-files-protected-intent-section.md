# doc-files: захищена секція «Призначення» (Варіант B)

**Дата:** 2026-06-11
**Статус:** спека (рішення зафіксовано в інтерактивній сесії; до реалізації)
**Зв'язане:** `docs/doc-files-skill.md`, `docs/superpowers/specs/2026-06-11-doc-files-orchestrator-quality-recommendations.md`

## 1. Проблема

Кодові файли мають на початку рукотворний інформаційний блок (намір, контракт, приклади) — його пишуть агенти, коли створюють код. Приклад — `npm/lib/models.mjs` (формат значень, env-приклади, маршрутизація за префіксом, каскад `resolveModel`). Зараз `doc-files` згодовує цей header у LLM (`facts.header`) і **перефразовує** його в Огляд — високоцінний контракт деградує до переказу.

**Мета:** перенести цей блок у доку як **захищену людино-керовану секцію**, яку конвеєр генерації ніколи не перезаписує LLM-виходом, зберігає дослівно при регенерації, подає машинним секціям як контекст — і нагадує агенту тримати її актуальною через **наявний механізм rules**.

## 2. Ухвалені рішення

| # | Питання | Рішення |
|---|---|---|
| Ф1 | Розмежування захищеного регіону | **Конвенція заголовка**: `## Призначення`, межа — наступний `## ` (H2). Усередині дозволені `###`+ |
| Ф2а | Де живе правило-нагадування | **За мовами в наявних правилах**: js/mjs → `js-lint`, py → `python`, vue → `vue`; спільна логіка чека — крос-правильний lib |
| Ф2б | Семантика чека | **присутність=fail + свіжість=warn**; opt-in = наявність заголовка (§6) |
| Ф3 | Header у коді після переїзду | **Видалити повністю** |
| Ф4 | Обсяг | **Opt-in**: лише файли зі змістовним header-коментарем |
| Ф5 | Назва/позиція | `## Призначення`, **першою** секцією (після H1, до Огляду) |

## 3. Механіка генерації (`doc-files`)

**Конвенція секції.** `## Призначення` — зарезервований заголовок. Усе від нього до наступного `## ` — захищений регіон. Машинні секції — теж H2, тож не можуть опинитися «всередині».

**Порядок `assemble()`:**
```
# <stem>
## Призначення        ← дослівно з наявної доки (людський блок)
## Огляд              ← машинна (доповнює Призначення, не дублює)
## Поведінка
## Публічний API
## Гарантії поведінки
```

**Цикл регенерації:**
1. `generateDoc(file)` тепер читає **і наявну доку** (не лише джерело): парсить, вирізає блок `## Призначення` до наступного `##`, тримає його.
2. Генерує машинні секції як зараз (R1–R7).
3. Збирає назад, вставляючи захищений блок дослівно на фіксовану позицію.
4. Доки ще немає (missing) → блоку немає → генерація як раніше (для opt-in файлів першу затравку дає міграція, §5).

**Контекст для LLM.** Текст `## Призначення` подається в промпти Огляду/Поведінки як **read-only авторитетний контекст** із інструкцією «не повторюй дослівно, доповнюй» — щоб машинні секції узгоджувалися з контрактом і не дублювали його.

**Скоринг.** `scoreDoc` **виключає** ключ `призначення` з аналізу (людський блок — анкори/generic/суржик/internal-name до нього не застосовні). `parseSections` уже ріже по `##`, тож достатньо пропустити цей ключ.

**Джерело істини після міграції — дока.** `facts.header` для opt-in файлів стане порожнім (header видалено з коду) — Огляд і так генерується з Поведінки (R3).

## 4. Правило-нагадування (Ф2а/Ф2б)

Логіка чека — **спільний крос-правильний lib** (`npm/scripts/lib/doc-intent.mjs` за контрактом `scripts.mdc`: крос-правильна інфраструктура живе тут). Кожне мовне правило викликає його, звужено до свого глоба:

| Правило | Файли | Чек |
|---|---|---|
| `js-lint` (`npm/rules/js-lint/js/`) | `*.js`, `*.mjs` | `checkDocIntent(cwd, glob)` |
| `python` (`npm/rules/python/js/`) | `*.py` | те саме |
| `vue` (`npm/rules/vue/js/`) | `*.vue` | те саме |

**Opt-in = наявність заголовка (§6).** Обидві перевірки застосовуються лише до док, що **вже мають** `## Призначення`. Файли без заголовка — поза правилом.

**Дві перевірки в lib:**

1. **Присутність = fail.** Якщо дока має заголовок `## Призначення`, його тіло має бути **непорожнім**. Порожній (випадково стертий) блок → `fail` із підказкою заповнити. (Сам факт наявності заголовка = свідомий opt-in; після видалення header із коду намір живе тут.)
2. **Свіжість = warn.** Git-diff проти base-ref (патерн `changelog`/`consistency.mjs`): якщо **джерело** змінилося відносно бази, а **вміст блоку `## Призначення`** — ні (порівняння на рівні блоку через `git show <base>:<doc>`, не цілого файлу — машинні секції міняються щоразу) → `warn` «контекст писали проти старішого коду, переглянь `## Призначення`». Не fail — це нудж.

`.mdc`-контракт кожного правила доповнюється абзацом: «кодові файли з докою мають `## Призначення` — рукотворний намір/контракт; doc-files його не перезаписує; оновлюй при зміні поведінки».

## 5. Міграція (одноразово, opt-in)

Для кожного файлу зі змістовним header-коментарем — **атомарний порядок, що не губить намір**:
1. Витягти header (`extractFileHeader`) → записати/сидувати `## Призначення` у доці (демотувати вкладені `##`→`###`, щоб не ламати структуру). Цей крок і є opt-in (§6).
2. **Лише тоді** видалити header-блок із `.mjs`.

Ніколи не видаляти header до затравки доки. Міграція — окремий скрипт/режим (напр. `doc-files migrate-intent --from N --limit M`), порціями, з git diff на ревʼю.

## 6. Opt-in = наявність заголовка `## Призначення`

Ф4 (opt-in) × Ф2б (присутність=fail) узгоджено **без додаткового стану**: opt-in-сигнал — це сам факт наявності заголовка `## Призначення` в доці.

- Дока **має** `## Призначення` → opt-in: тіло мусить бути непорожнім (fail на спорожнення) + freshness=warn.
- Дока **без** заголовка → поза правилом, ні fail, ні warn.

Жодного frontmatter-прапора чи allowlist: блок самоописовий, нема що синхронізувати. Opt-in відбувається в момент, коли міграція (§5) або агент вперше додає `## Призначення`. «Присутність=fail» = «якщо заголовок є, не лишай його порожнім» (а не «кожен файл мусить мати»).

## 9. Ризики/межі

- Конвенція заголовка крихкіша за маркери: якщо агент вживе `## ` (H2) усередині блоку — межа обірветься. Пом'якшення: у `.mdc` явно «всередині `## Призначення` лише `###`+»; det-чек може попередити про H2 всередині.
- Block-level git-diff потребує читання base-версії доки (`git show`) — поза git (detached/перший коміт) freshness тихо пропускається (як changelog).
- Видалення header із коду робить `.mjs` «німим» при прямому читанні — свідомий компроміс (Ф3).
