# Огляд: інструменти й скіли @nitra/cursor

## Суть

Цей огляд висвітлює інструменти та скіли, які автоматизують підтримку якості коду та документації в проєкті. Тут описано механізми для забезпечення високого покриття тестами, автоматичної генерації документації, консистентності через лінтинг, а також оптимізації витрат на LLM за рахунок програмної верифікації.

## Автоматизація якості коду та тестів

Скіл `n-coverage-fix` підвищує мутаційне покриття шляхом детермінованого циклу генерації тестових кейсів, фокусуючись на неперевірених шляхах коду до досягнення стабільності показника. Скіл `n-fix` забезпечує повну автоматизацію приведення проєкту до стандартів шляхом поєднання діагностики інструментами та виправленням AI-агентом.

## Документація та консистентність

Механізм `doc-files` створює локально керовану поведінкову документацію для кожного файлу з використанням CRC32 для підтвердження актуальності. Централізований лінтинг через `eslint.config.js` забезпечує консистентність правил для JS та Vue-компонентів, коректно виключаючи з аналізу згенеровані артефакти.

## Оптимізація процесу роботи з AI

Для зменшення витрат токенів застосовується зміна парадигми — від LLM-інтерпретації правил до їхньої програмної верифікації через CLI-скрипти. Це дозволяє забезпечити дотримання конвенцій проєкту, отримуючи структурований результат від детермінованого виклику, замість множинних аналітичних запитів.
