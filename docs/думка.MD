# Думка: динамічний граф задач

Є директорія з проєктами. Кожен проєкт — задача для оркестратора.

Оркестратор розбиває задачу на граф: вузол або виконується атомарно, або динамічно розкладається у підграф під час виконання. Батьківський вузол не знає що всередині — він просто чекає "вирішено".

Граф орієнтований і ациклічний. Ребра несуть дані — вихід одного вузла стає входом наступного. Вихідних вузлів може бути кілька.

Файли — єдине сховище стану. Структура директорій і є граф. Вузол вирішено коли з'явився файл з результатом. Стан відновлюється скануванням.

Якщо вузол зламався — агент-інженер (поза графом) може змінити будь-який рівень, аж до кореня. Він бачить попередні спроби і не повторює їх. Час на вирішення обмежений — якщо не вкладається, передає вище.

Паралельне виконання через ізольовані воркспейси — злиття як єдина точка де перевіряється результат.
