Знайшов ключову несумісність у двох конвенціях:

**cursor `worktree add`**: користувач дає branch-ім'я → `.worktrees/<sanit>/` + `.md`-опис; гілка = user-provided (напр. `feat/my-feature`)

**mt `createWorktree`**: auto-naming `mt/<taskName>` → `.worktrees/<name>/`; гілка = `mt/<name>` (для task-graph)

Тобто перед тим, як делегувати — треба узгодити API. Є два підходи:

**А (спільна конвенція)** — `mt worktree add` підтримує cursor-контракт: `add <branch> "<опис>"`, `.worktrees/<sanit>/`, `.md`-файл. Внутрішньо mt може ще мати `mt worktree spawn <task>` для task-managed (auto-name). Cursor тоді делегує напряму без перекладу.

**Б (адаптер у cursor)** — `mt worktree` має свій API (task-oriented), cursor `worktree-cli.mjs` перекладає параметри. Більше зв'язаності — курс задоволений поки mt не змінює API.

Варіант **А** чистіший: mt стає canonical, cursor — тонка обгортка/fallback. Але треба додати `.md`-опис і `prune` у `mt worktree`.
