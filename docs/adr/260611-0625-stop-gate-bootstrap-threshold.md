# ADR: Anti-bootstrap threshold у Stop-гейтах документації

**Status:** Accepted
**Date:** 2026-06-11

## Context and Problem Statement

Stop-гейт (`doc-files check --git`) перевіряє, чи є серед змінених джерел файли з застарілими доками, і блокує завершення задачі. Якщо реалізувати гейт як «будь-яка кількість застарілих → exit 2 (block)», виникає bootstrap-deadlock: при першому масовому прогоні в репо з 200+ файлів без жодної доки — гейт блокує кожну задачу; щоб пройти гейт, потрібно спершу згенерувати доки; але генерація є окремою задачею, яку теж блокує гейт.

## Considered Options

* Поріг: якщо застарілих більше N (default 50, override `N_CURSOR_DOC_FILES_GATE_MAX`) — warning без блоку (exit 0); інакше — блок (exit 2).
* Строгий гейт: будь-яка кількість застарілих → завжди блокує.
* Нестрогий гейт: тільки попередження, ніколи не блокує.

## Decision Outcome

Chosen option: "Поріг з override через env", because bootstrap-deadlock — реальна ситуація при впровадженні: репо з 224 файлами і нульовими доками потрапляє у цикл «гейт блокує → генерація не відбувається → гейт блокує». Поріг 50 розриває цикл: перший масовий прогін (`doc-files gen`) проходить без блоку, а після нього кількість застарілих падає нижче порогу і гейт починає працювати в штатному режимі. Override (`N_CURSOR_DOC_FILES_GATE_MAX`) дозволяє репозиторіям з різним профілем налаштувати поріг під себе.

### Consequences

* Good, because система bootstrappiться без ручного відключення гейту або обходу хука.
* Good, because при N > 50 застарілих warning чітко сигналізує «тут великий накопичений борг — запусти `gen`» без блокування поточної задачі.
* Good, because патерн «anti-bootstrap threshold» масштабується на будь-який гейт, що застосовується рекурсивно до своїх власних артефактів (lint-гейти для нових правил, check-sum гейти для нових форматів).
* Bad, because при N ≤ 50 застарілих гейт суворий — є ризик блокування задач у репо з активними правками великих файлів; override знижує ризик.

## More Information

- Реалізація: `npm/skills/doc-files/js/docgen-scan.mjs` → `check --git` з `--max` (default `50`), env `N_CURSOR_DOC_FILES_GATE_MAX`.
- Дока: `npm/skills/doc-files/js/docs/docgen-scan.md`, `docs/doc-files-skill.md`.
- Bootstrap-прогін 2026-06-11: 224 файли, з яких 201 без доки — поріг дозволив пройти перший stamp-прогін без блоку; після міграції: 148 fresh / 76 stale (чесний борг).
