# ensureTool: авто-встановлення зовнішніх CLI-інструментів

**Status:** Accepted
**Date:** 2026-06-03

## Context and Problem Statement

Скрипти пакета `@nitra/cursor` залежать від зовнішніх CLI-інструментів (`conftest`, `hk`, `shellcheck`, `hadolint`, `opa`, `regal`, `kubeconform`, `kubescape`, `actionlint`, `dotenv-linter`). До цього рішення кожен call-site самостійно шукав бінарник через `resolveCmd`, а при невдачі — або замовчував, або друкував ad-hoc текстову підказку зі встановлення, що призводило до неодноманітної поведінки й «тихих» збоїв.

## Considered Options

- Єдиний seam `ensureTool(toolId)` з авто-встановленням через пакетний менеджер (brew/scoop) або GitHub Release binary (Linux / fallback)
- Залишити ad-hoc `resolveCmd` + текстові install-hints у кожному call-site (поточний стан)
- GitHub Release binary для всіх платформ (єдина пінована версія без пакетних менеджерів)

## Decision Outcome

Chosen option: "Єдиний seam `ensureTool(toolId)` з авто-встановленням через пакетний менеджер або GitHub Release binary", because це усуває дублювання install-логіки, дає однакову hard-fail поведінку в усіх сьютах і відповідає наявній архітектурі `lock-cache-dir.mjs`/`with-lock.mjs`.

### Consequences

- Good, because `withBinRemovedFromPath` автоматично виставляє `N_CURSOR_NO_AUTO_INSTALL=1` — тести не тягнуть реальний brew/curl.
- Good, because docker-run fallback прибрано: hadolint тепер виключно нативний бінарник через `ensureTool`, що спрощує контракт `lintDockerfileWithHadolint`.
- Bad, because winget не має `hk` і `conftest` (підтверджено через winget-pkgs) — Windows-шлях через Scoop не є ідеально безшовним: якщо Scoop відсутній, CLI падає на GitHub Release binary з додатковим кроком.
- Bad, because brew/scoop дають latest без пінінгу — різні члени команди можуть мати різні версії інструментів; команда свідомо прийняла цей компроміс.

## More Information

Per-platform matrix (зафіксована в `npm/scripts/lib/ensure-tool.mjs` та `scripts.mdc`):

| OS | Метод |
|---|---|
| macOS | `brew install <formula>` |
| Windows | `scoop install <app>` → fallback GitHub Release binary |
| Linux | GitHub Release binary (download + `chmod +x`) |

- `npm/scripts/lib/ensure-tool.mjs` — движок: реєстр `TOOLS` (інлайн), `fetchLatestVersion` через GitHub API, `installFromGithub` (підтримує `archive: false` для сирих бінарників: `opa`, `regal`, `hadolint`, `kubescape`; `tar.gz` для `kubeconform`, `hk`, `conftest`, `shellcheck`).
- Реєстр `TOOLS`: `hk`, `conftest`, `shellcheck`, `actionlint`, `dotenv-linter`, `opa`, `regal`, `hadolint`, `kubeconform`, `kubescape`.
- `N_CURSOR_NO_AUTO_INSTALL=1` — opt-out env для тестів і CI без мережі.
- `hk` особливість: `ensureHkInstall(hkBin)` — виконує `hk install` з CI-гардом (`process.env.CI`); викликається тільки з case `'fix'` у `npm/bin/n-cursor.js`.
- Winget відхилено: `jdx.mise` та `open-policy-agent.opa` є у winget, але не `hk` і не `conftest`. Scoop Main: `hk.json` (hk.jdx.dev), `conftest.json` (v0.68.2); `regal` відсутній у Scoop → GitHub Release fallback.
- `npm/scripts/utils/test-helpers.mjs` — `withBinRemovedFromPath` виставляє `N_CURSOR_NO_AUTO_INSTALL=1`.
- `npm/scripts/lib/tests/ensure-tool.test.mjs` — 6 unit-тестів seam'у.
- Мігровані call-sites: `rego/lint/lint.mjs` (opa, regal → `ensureTool`; conftest — м'який skip, свідомо), `docker/lib/docker-hadolint.mjs` (hadolint, docker-fallback видалено), `k8s/lint/lint.mjs` (kubeconform, kubescape).
- `npm/rules/docker/docker.mdc` — `HADOLINT_IMAGE`/`docker run` замінено на `ensureTool`.
- `.cursor/rules/scripts.mdc` — generic-секція «Зовнішні CLI-інструменти: резолв і авто-встановлення».
- Change-файли: `npm/.changes/1780483223111-67d0c7.md` (minor/Added), `npm/.changes/1780484101442-836c8a.md` (patch/Changed).
- Фінальний стан тестів: 2314 passed (2 skipped), 0 failed.

## Update 2026-06-03

Реєстр `TOOLS` розширено новими записами:
- `opa` — raw binary, `brew install opa`, `scoop install opa`
- `regal` — raw binary, `brew install regal`; Scoop відсутній → GitHub Release fallback
- `hadolint` — raw binary, `brew install hadolint`, `scoop install hadolint`
- `kubeconform` — tar.gz, `brew install kubeconform`, `scoop install kubeconform`
- `kubescape` — raw binary, `brew install kubescape`, `scoop install kubescape`

`installFromGithub` розширено на `archive: false` (download + `chmod +x` без розпакування tar) — сирі бінарники обробляються нарівні з архівами.

Docker-fallback видалено: `lintDockerfileWithHadolint` тепер викликає тільки нативний hadolint через `ensureTool`; `HADOLINT_IMAGE`/`docker run` видалено з коду, тестів і `docker.mdc`.

Мігровані call-sites: `rego/lint/lint.mjs` (opa, regal → `ensureTool`; conftest — м'який skip, свідомо), `docker/lib/docker-hadolint.mjs` (hadolint), `k8s/lint/lint.mjs` (kubeconform, kubescape).
