## ADR Реструктуризація `rules/<id>/js/` → `fix/` / `lint/` / `policy/`
**Контекст:** `rules/<id>/js/` змішувала два різних канали: JS-check-модулі (живлять `npx @nitra/cursor check`) і JS-оркестратори лінту (живлять `bun run lint-X`). Rego-політики у `policy/` логічно пов'язані з `check`-каналом, але написані іншою технологією. Потрібен однозначний принцип розміщення.
**Рішення/Процедура/Факт:** Технологія реалізації визначає директорію. `fix/<concern>/check*.mjs` — JS-діагностика для `n-cursor check`; `lint/lint.mjs` — JS-оркестратор для `bun run lint-X`; `policy/<concern>/*.rego` — Rego-діагностика (третій сиблінг поруч із fix/ та lint/, не вкладений у fix/). Виконано 3 фази (v1.11.9 → 1.11.14): dual-mode інфраструктура → переміщення 26 правил → прибирання legacy. Видалено `lint-conftest.mjs` (дублював `n-cursor check` для policy). Прибрано self-reference `devDependencies` у `npm/package.json`.
**Обґрунтування:** `policy/` під `fix/` (варіант B) або плоско (варіант C) змішував би rego і JS всупереч різним runner-ам (`runConftestBatch` vs dynamic `import`), namespace-конвенціям і ролі `target.json`.
