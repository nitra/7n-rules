# Runbook n-fix — автоматична перевірка і виправлення проєкту

**Status:** Accepted
**Date:** 2026-05-15

## Контекст

У монорепо `nitra/cursor` накопичується набір архітектурних правил у `.cursor/rules/`. Щоб переконатися, що проєкт відповідає цим правилам, і виправити відхилення, команда запускає скіл `/n-fix`.

## Рішення/Процедура

1. Запустити скіл через `/n-fix` або виклик `.cursor/skills/n-fix/SKILL.md`.
2. Скіл виконує діагностику: `npx @nitra/cursor check` — перевіряє всі активні правила (ADR-хуки, `settings.json`, `.gitignore`, lint-конфіги тощо).
3. За результатами перевірки скіл виправляє порушення відповідно до правил з `.cursor/rules/`.
4. Після виправлень запускається повторна перевірка для підтвердження чистого стану.

Ключові перевірки:
- `.claude/hooks/capture-decisions.sh` і `normalize-decisions.sh` збігаються з канонічними.
- `.claude/settings.json` містить Stop-hook із `bun run lint-conftest`.
- `.gitignore` покриває необхідні патерни.

## Обґрунтування

`npx @nitra/cursor check` є єдиним авторитетним джерелом стану відповідності: він виконує `check-*.mjs`-скрипти, що відповідають кожному правилу, і дає чіткий ✅/❌-статус перед будь-якими змінами.

## Розглянуті альтернативи

Не обговорювалися.

## Зачіпає

`.cursor/rules/` (активні правила), `.cursor/skills/n-fix/SKILL.md`, `.claude/hooks/`, `.claude/settings.json`, `.gitignore`.
