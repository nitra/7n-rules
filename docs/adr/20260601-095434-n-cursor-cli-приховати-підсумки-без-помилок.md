# `@nitra/cursor`: приховати підсумки Skills/Commands без помилок

**Status:** Accepted
**Date:** 2026-06-01

## Context and Problem Statement

`npx @nitra/cursor` виводив рядки `🧩 Skills: N скопійовано, 0 з помилками` та `⌨️ Commands: N скопійовано, 0 з помилками` навіть при повністю успішному виконанні, додаючи шум без корисної інформації.

## Considered Options

* Приховати секцію повністю, якщо `failCount === 0`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Приховати секцію повністю, якщо `failCount === 0`", because користувач явно попросив: «не відображались взагалі якщо там немає помилок».

### Consequences

* Good, because CLI-вивід стає лаконічнішим: підсумки показуються лише тоді, коли потребують уваги.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

Зміни в `npm/bin/n-cursor.js`: блок `syncSkills` (≈рядки 1358–1362) та аналогічний блок для `Commands` — якщо `failCount === 0`, `console.log` не викликається. Тест-файли `auto-skills.test.mjs` і `sync-claude-config.test.mjs` можуть потребувати оновлення assertions.

Додаткової інформації в transcript не зафіксовано.

## Update 2026-06-01

Переглянуте рішення: ховати не лише підсумковий рядок, а **цілий блок** (`⬇ … ✅` рядки + підсумок) для всіх трьох секцій (`🧩 Skills`, `⌨️ Commands`, `🥧 Pi skills`) — коли `fail === 0`.

Користувач явно відхилив варіант із частковим приховуванням (лише підсумок) і підтвердив, що `🥧 Pi skills` потрібно ховати так само.

Реалізація — `captureOutput(action)` у `npm/bin/n-cursor.js` (~рядок 1130): перехоплює `process.stdout.write`, `console.log`, `console.error` у внутрішній буфер; скидає в реальний stdout лише якщо `fail > 0`. Три `runSyncStep`-блоки (Skills, Commands, Pi skills) обгорнуто в `captureOutput`.

Верифікація: 65 тестів pass (`upgrade-nitra-cursor-and-install.test.mjs`, `sync-claude-config.test.mjs`); `oxlint npm/bin/n-cursor.js` → `exit=0`. Існуючі eslint-помилки (рядок 719 `unicorn/prefer-single-call`, рядок 1202 `jsdoc/escape-inline`) були на HEAD до патча — цим патчем не введені.

Локальний обхід re-exec: `NITRA_CURSOR_REEXEC=1 bun ./npm/bin/n-cursor.js` (зміна набирає чинності в `node_modules` лише після публікації нової версії пакету).
