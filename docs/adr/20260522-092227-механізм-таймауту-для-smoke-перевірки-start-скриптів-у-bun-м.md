## ADR Механізм таймауту для smoke-перевірки start-скриптів у bun-монорепо

## Context and Problem Statement
Для нового скіла `n-start-check`, що запускає `bun run start` у кожному воркспейсі монорепо, потрібно реалізувати таймаут: dev-сервери (наприклад, Vite) не завершуються самостійно, тому потрібен механізм переривання через grace-period (~12 с).

## Considered Options
* Фоновий процес + `sleep` + ручний `kill` (watchdog через `&`, `$!`, `kill -0`)
* `perl -e 'alarm shift; exec @ARGV'` — вбудований POSIX-таймаут

## Decision Outcome
Chosen option: "`perl` + `alarm`", because користувач запропонував цей підхід як чистіший (`perl -e 'alarm shift; exec @ARGV' 12 bun run start`), і він підтверджений під час ручного запуску скіла: `demo` дав `CODE=142` (SIGALRM = таймаут = OK), після чого порт 5173 залишився вільним — orphan-процесів немає.

### Consequences
* Good, because `exec` зберігає той самий PID, тож SIGALRM б'є по справжньому процесі, без ручного юглінгу `$PID` / `kill -0` / `wait`.
* Good, because код виходу сам класифікує результат: `142` → dev-сервер дожив до grace-period → **OK**; `0` → CLI-скрипт завершився чисто → **OK**; інше → крах → **FAIL**.
* Good, because transcript фіксує очікувану користь: `perl` доступний на macOS без додаткового встановлення, тоді як GNU `timeout` там відсутній у базовій поставці.
* Bad, because transcript не містить підтверджених негативних наслідків.
