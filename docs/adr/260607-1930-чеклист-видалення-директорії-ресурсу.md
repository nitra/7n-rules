**Status:** Accepted
**Date:** 2026-06-07

## Context and Problem Statement

Під час сесії потрібно було вирішити, чи безпечно видалити директорію `benchmarks/`, яка містила spike-код. Видалення без перевірки могло зламати `package.json` scripts і втратити незафіксовані зміни в самій директорії.

## Considered Options

- Перевірити залежності, зовнішні посилання і uncommitted changes перед видаленням.
- Видалити директорію без перевірки.
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Перевірити залежності та uncommitted changes перед видаленням", because transcript показав, що `benchmarks/` ще використовувалася скриптом `"benchmark": "bun benchmarks/runner-comparison/run.mjs"`, а також мала uncommitted changes у двох файлах.

### Consequences

- Good, because перевірка виявила активний `package.json` script до видалення директорії.
- Good, because uncommitted changes у `benchmarks/runner-comparison/demo/src/docs/promise-pool.md` та `benchmarks/tool-free/run.mjs` були явно помічені перед дією.
- Bad, because transcript не містить підтверджених негативних наслідків цього чеклисту.

## More Information

- Чеклист з transcript: `ls <dir>` для структури; `git log --all -- <dir>` для історії; `git diff HEAD -- <dir>` для uncommitted changes; `grep -r "<dir>" ... -l | grep -v <dir>` для зовнішніх посилань.
- Після перевірки було видалено `benchmarks/` і прибрано script `"benchmark": "bun benchmarks/runner-comparison/run.mjs"` з `package.json`.
- Користувач окремо зазначив, що change file для цього зробить сам.
