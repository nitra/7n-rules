# Пріоритет conftest-політик для задач, вирішуваних через OPA/Rego

**Status:** Accepted
**Date:** 2026-05-07

## Контекст

У проєкті використовується conftest/OPA для статичних перевірок структури файлів. Без явного правила асистент міг вирішувати задачі скриптами або іншими механізмами, навіть якщо є можливість реалізувати перевірку декларативно через Rego-політику.

## Рішення/Процедура/Факт

До `.cursor/rules/conftest.mdc` додано правило: якщо задачу можна вирішити через conftest-політику, саме цей підхід є обов'язковим. Зміна внесена редагуванням існуючого `.mdc`-файлу без створення нових файлів.

## Обґрунтування

Декларативні OPA/Rego-політики є версіонованими, легко тестованими та зрозумілими без знання конкретної мови скриптів. Запуск через `conftest` у CI є стандартним і self-contained. Уніфікація підходу зменшує кількість різнорідних механізмів перевірки у репозиторії.

## Розглянуті альтернативи

Не обговорювалися. Рішення продиктоване безпосередньо.

## Зачіпає

`.cursor/rules/conftest.mdc`, усі майбутні задачі, де вибір між скриптом/хуком і Rego-політикою.
