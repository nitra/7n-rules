## ADR Приховування блоків CLI-виводу синку при відсутності помилок

## Context and Problem Statement
CLI `@nitra/cursor` після кожного прогону виводив три розділи (`🧩 Skills:`, `⌨️ Commands:`, `🥧 Pi skills:`) разом з усіма рядками `⬇ … ✅`. Ці блоки засмічували вивід навіть за повністю успішного виконання без жодних помилок.

## Considered Options
* Ховати лише підсумкові рядки (`🧩 Skills: N скопійовано, 0 з помилками`), лишаючи `⬇ … ✅`
* Ховати цілі блоки (всі `⬇ … ✅` рядки + підсумок) за відсутності помилок

## Decision Outcome
Chosen option: "Ховати цілі блоки за відсутності помилок", because користувач явно скоригував початкову пропозицію (ховати лише підсумкові рядки) на повне приховування блоків, і вимога поширилась однаково на всі три секції (Skills, Commands, Pi skills).

### Consequences
* Good, because transcript фіксує очікувану користь: за успішного прогону `npx @nitra/cursor` вивід після правил одразу переходить до `📝 Оновлено AGENTS.md…` / `✨ Готово`.
* Bad, because при помилці весь буфер `⬇`-рядків (у т.ч. успішні) виводиться повністю — збільшується обсяг тексту на екрані порівняно з вибірковим виводом лише збійного рядка.

## More Information
* Файл із змінами: `npm/bin/n-cursor.js`
* Реалізація: доданий helper `captureOutput(action)` — перехоплює `process.stdout.write`, `console.log`, `console.error` у буфер; по завершенні дії — якщо `fail === 0`, буфер скидається мовчки; якщо `fail > 0`, буфер спочатку скидається в реальний stdout, потім друкується підсумковий рядок.
* `captureOutput` застосований до трьох `runSyncStep`-викликів: Skills (`syncSkills`), Commands (`syncCommands` + `syncLocalOnlySkillCommands`), Pi skills (`syncPiSkills` + `syncLocalOnlyPiSkills`).
* Перевірено: `bunx oxlint npm/bin/n-cursor.js` — без нових порушень; `bunx vitest run` — 65 тестів passed.
* Зміна набирає чинності лише після публікації нової версії пакету: `bun start` re-exec-ає у встановлений `node_modules/@nitra/cursor`, де зміни відсутні доки не вийде нова версія. Локальний обхід: `NITRA_CURSOR_REEXEC=1 bun ./npm/bin/n-cursor.js`.
