# `n-cursor lint` — агрегований лінт-оркестратор із per-rule таймінгом

**Status:** Accepted
**Date:** 2026-05-25

## Context and Problem Statement

Скіли `/n-fix` і `/n-lint` не показували, скільки часу займає кожне окреме правило. Кореневий `bun run lint` реалізовувався як `&&`-рядок у `package.json` без можливості зібрати тайминги на рівні платформи.

## Considered Options

- Скіл запускає кожне правило окремо (вимір через `date +%s`; N окремих процесів)
- Нова CLI-команда `n-cursor lint` — один агрегований прогін із таймінгами через ін'єкцію залежностей
- Гібрид: основний прогін одним викликом, окремі виклики з таймами лише для рядків із `❌`

## Decision Outcome

Chosen option: "Нова CLI-команда `n-cursor lint`", because зберігає один агрегований прогін без overhead N окремих запусків, дає точні тайминги через ін'єкцію залежностей (`spawnSyncFn`, `now`) всередині оркестратора.

### Consequences

- Good, because `npx @nitra/cursor lint` — єдина точка входу; кореневий `package.json` зводиться до `"lint": "n-cursor lint"`; тайминги з'являються автоматично.
- Good, because fail-fast і фіксований порядок скриптів гарантують детермінований вивід.
- Bad, because transcript не містить підтверджених негативних наслідків.
