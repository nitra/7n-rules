## ADR Ручна валідація вцілілих мутантів замість очікування Stryker

## Context and Problem Statement
Stryker у режимі `--incremental` зайняв 50+ хвилин і не завершився (incremental cache став invalid: `0 of 25053 mutant result(s) are reused`). Потрібно було перевірити, чи нові тести справді вбивають 4 survived мутанти в `coverage.mjs`, не чекаючи повного Stryker re-run.

## Considered Options
* Ручна мутація source-файлу + запуск target-тестів через `vitest run -t <pattern>`
* Дочекатися повного `bun run coverage` (Stryker з `--incremental`)

## Decision Outcome
Chosen option: "Ручна мутація source-файлу + запуск target-тестів через `vitest run -t <pattern>`", because Stryker завис через invalid incremental cache та конкуренцію CPU зі застарілими bun-процесами (PID 90242, 91321, 50+ год CPU), а ручна перевірка дає детерміновану відповідь без чекання.

### Consequences
* Good, because transcript фіксує очікувану користь: усі 4 мутанти (`if (opts.fix) → true`, `→ false`, `{ fix: false } → {}`, `→ { fix: true }`) підтверджено killed за <5 хв замість 50+.
* Bad, because COVERAGE.md не оновилося автоматично — агрегована статистика залишилась із попереднього запуску; потрібен окремий `bun run coverage` для оновлення файлу.

## More Information
