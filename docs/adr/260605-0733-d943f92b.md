## 1. Чи gemma3:4b оптимальна для цієї задачі?

**Ні — оптимальна для 8 GB M2, але не для задачі як такої.**

Задача docgen Tier 1 має дві рівнозначні вимоги:
- **Розуміння коду** (семантика алгоритмів, граничні випадки, API-інтерфейс)
- **Якісна українська** (технічний регістр, без кальок)

У 4B-класі є стеля: модель не тримає тонкі правила стилю (ми бачили витік сигнатур, stdlib, обрізання). Це не промпт-проблема — це ємність.

## 2. Де справжня якісна стеля?

Оціночна прогресія для цієї задачі (не виміряно, лише в нас 4B):

| Клас | Представники (2025) | Оцінка якості vs еталон |
|---|---|---|
| 4B (виміряно) | gemma3:4b | **~88%** |
| 8B | qwen3:8b, llama3.1:8b | ~92% |
| 12–14B | gemma3:12b, **qwen3:14b** | ~94% |
| 27–32B | gemma3:27b, **qwen3:32b**, qwen2.5:32b | ~96% |
| 70B+ | qwen2.5:72b, llama3.1:70b | ~97% |

**Ключовий стрибок** — між 4B і 12–14B (~6 п.п.): тут модель починає надійно тримати стильові правила навіть без підказки. Від 32B до 70B — лише ~1–2 п.п. (закон спадної дохідності).

**Найкращі кандидати для docgen** (українська × код × розмір):
- **qwen3:14b** (Qwen3, квітень 2025) — топ мультилінгвальний у своєму класі, ~9 GB q4
- **gemma3:12b** (Google, березень 2025) — та ж ДНК що 4b, але більший, ~8 GB q4
- **qwen3:32b** — near-SOTA для локальних моделей, ~20 GB q4

## 5. Рекомендація

**Оптимальний вибір для цієї задачі — `qwen3:14b` на RTX 4090 в хмарі.**

Чому:
- Якість ~94% (проти 88% gemma3:4b) — надійно тримає стиль без витоку реалізації
- **Вартість < $2** за повний прогін (проти $30+ Claude API чи 14+ год на локальній машині)
- RTX 4090 (24 GB) виходить за межі 8 GB M2 і дає concurrency 2 → ~3 год замість 14 год
- Один GPU, один сервер — просто підняти ollama + зупинити

**Якщо потрібна максимальна якість** (підтримка складних фреймворків, архітектурних нюансів):
- `qwen3:32b` на A100 40 GB → ~3 год, ~$4–5, ~96%

**Поточна 8 GB M2** лишається доречною лише для **окремих файлів** (PoC, spot-check), але **не для масового прогону** — надто повільно й без паралелізму.
