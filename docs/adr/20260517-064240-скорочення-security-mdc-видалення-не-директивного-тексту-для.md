## ADR Скорочення `security.mdc` — видалення не-директивного тексту для економії контексту

**Контекст:** Файл `npm/rules/security/security.mdc` містив 108 рядків, з яких значну частину займали пояснювальні блоки (мета-опис gitleaks, інструкції зі встановлення, посилання на `check-security.mjs`), що не впливають на поведінку правила під час його застосування агентом.

**Рішення/Процедура/Факт:** Файл переписано до 73 рядків. Видалено: мета-блок про активацію (`auto.md`, `disable-rules`), параграф про підвантаження AI-контексту, розділ «Мета» з описом gitleaks, розділ «Встановлення» з командами `brew`/`apt`/`pacman`, розділ «Перевірка» з посиланням на `npx … check security`, коментар у `.gitleaks.toml` про джерело дефолтних правил. Залишено тільки директивний контент: шаблон `scripts.lint-security` і `scripts.lint` у `package.json`, скелет `.gitleaks.toml` з `useDefault = true` і allowlist, опціональний GitHub Actions workflow з `fetch-depth: 0`.

**Обґрунтування:** MDC-правило потрапляє у контекст AI-агента і витрачає токени. Пояснювальні блоки (опис інструменту, інструкції встановлення для людини, повтор того, що описано в окремих check-скриптах) не змінюють того, що агент має зробити, але збільшують довжину промпта. Директивний текст — єдиний змістовно цінний вміст для агента.

**Розглянуті альтернативи:** не обговорювалися.

**Зачіпає:** `npm/rules/security/security.mdc`
