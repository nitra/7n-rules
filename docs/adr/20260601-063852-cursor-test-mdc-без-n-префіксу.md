# `cursor-test.mdc` без `n-`-префіксу як локальне правило репо

**Status:** Accepted
**Date:** 2026-06-01

## Context and Problem Statement

Потрібно додати правило для cursor-репо з конвенціями тестів (мова `describe/test`, Stryker-aware маркери, inline test-фабрики). Постало питання: `n-cursor-test.mdc` (за аналогією з керованими правилами) чи `cursor-test.mdc` (без `n-`-префіксу, як `conftest.mdc`, `dev-dep.mdc`).

## Considered Options

* `n-cursor-test.mdc` — за аналогією з пакетними правилами
* `cursor-test.mdc` — без `n-`-префіксу, як локальні правила репо

## Decision Outcome

Chosen option: "`cursor-test.mdc` без `n-`-префіксу", because синк `n-cursor` видаляє будь-який `.cursor/rules/n-*.mdc`, якого нема у `.n-cursor.json`; додавати `cursor-test` у `.n-cursor.json` не можна — copy-крок шукав би пакетне правило `npm/rules/cursor-test/`, якого нема.

### Consequences

* Good, because файл не буде автоматично видалено синком; відповідає конвенції локальних правил репо.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

Файл: `.cursor/rules/cursor-test.mdc` v1.0. Три секції: мова describe/test; Stryker-aware read-only маркери (`.n-cursor.json`, `COVERAGE.md`, `npm/reports/stryker/*.json`); inline test-фабрики (≤80 рядків). Механізм видалення: `removeOrphanManagedRuleFiles` у `npm/bin/n-cursor.js` (≈рядки 535–565).

У тій самій сесії: `npm/rules/js-lint-ci/meta.json` отримав `"auto": { "glob": ["**/*.mjs","**/*.cjs","**/*.js","**/*.jsx","**/*.ts","**/*.tsx"] }` — `js-lint-ci` тепер авто-активується за JS/TS-файлами; тести 61/61 pass. Кореневий `.n-cursor.json` відстане від авто-дискаверу до наступного синку.
