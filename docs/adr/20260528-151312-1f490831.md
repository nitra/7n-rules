## ADR 1: Підтримка glob-патернів у `resolveAllJsRoots`

## Context and Problem Statement

У monorepo з glob-патернами у `workspaces` (наприклад `"cf/*"`) функція `resolveJsRoot()` у `@nitra/cursor` обирала перший елемент масиву `workspaces` буквально, і оскільки glob не резолвився — fallback повертав корінь проєкту. Vitest запускався у `/Users/vitalii/www/nitra/ai`, де не було `vitest.config.js`, що призводило до помилки `ref is not defined`.

## Considered Options

* Розгортати glob-патерни у `resolveAllJsRoots()` через `node:fs/promises.glob`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Розгортати glob-патерни у `resolveAllJsRoots()` через `node:fs/promises.glob`", because `node:fs/promises.glob` вже використовується в `npm/scripts/lib/workspaces.mjs` і є доступним у Node 26.

### Consequences

* Good, because transcript фіксує очікувану користь: `resolveAllJsRoots()` повертає всі реальні workspace-директорії при glob-патернах, що усуває fallback до кореня.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Файл: `npm/scripts/utils/resolve-js-root.mjs`

## Context and Problem Statement

Функція `collect()` у `npm/rules/js-lint/coverage/coverage.mjs` (рядок 232) працювала з одним `jsRoot` — результатом `resolveJsRoot()`. У monorepo з кількома workspace-директоріями це означало, що покриття збиралося лише для одного workspace. `resolveAllJsRoots()` вже існувала поряд, але не використовувалася coverage-провайдером.

## Considered Options

* Переробити `collect()` і `detect()` для ітерації по `resolveAllJsRoots()` і агрегації lcov + mutation
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Переробити `collect()` і `detect()` для ітерації по `resolveAllJsRoots()` і агрегації lcov + mutation", because `resolveAllJsRoots()` вже існувала і надавала необхідну функціональність — потрібно було лише підключити її до coverage-провайдера.

### Consequences

* Good, because transcript фіксує очікувану користь: `bun run coverage` у monorepo більше не падає через неправильно обраний jsRoot; метрики агрегуються по всіх workspaces.
