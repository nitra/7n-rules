---

**Status:** Accepted
**Date:** 2026-06-16

## Context and Problem Statement

ADR-нормалізатор `npm/scripts/lib/adr/normalize-pipeline.mjs` вбудовував повний MADR-шаблон у системний промпт `GEN_SYS`: порядок заголовків, `**Status:** Accepted`, `**Date:** <…>`, фразу `Chosen option: "...", because ...` і fallback-рядки для відсутніх альтернатив чи додаткової інформації. JS потім перевіряв той самий формат regex-ами у `validateMadr`, утворюючи петлю «попроси модель про формат → перевір формат кодом → ретрай».

## Considered Options

- Pure JSON-fields: LLM повертає суворо типізовані поля без будь-якого Markdown, а JS збирає весь документ.
- Hybrid: LLM повертає зміст секцій у вигляді Markdown-прози зі збереженням inline code, а JS відповідає за весь каркас документа.
- Старий підхід: LLM повертає готовий MADR-документ цілком через контракт markdown-виходу в `GEN_SYS`.

## Decision Outcome

Chosen option: "Hybrid", because pure JSON-fields підхід у A/B-прогоні виявився lossy: LLM стирав inline-code backticks і колапсував rich More Information списки в один рядок, тоді як гібрид повернув backticks і повноту секцій при JS-owned каркасі.

### Consequences

- Good, because JS-функція `assembleMadr({title, date, sections})` гарантує канонічний MADR-формат незалежно від моделі.
- Good, because ретраї через форматний markdown-каркас мають зникнути, а `validateMadr` для `genMadr` стає майже зайвим.
- Good, because нові детерміновані функції `assembleMadr`, `normalizeSections` і `madrDate` покриті 8 юніт-тестами; transcript фіксує 21 passing test.
- Bad, because `MERGE_SYS` отримав аналогічну зміну для JS-збірки `## Update <date>`, але окремий A/B-прогін merge-гілки в transcript не зафіксований.

## More Information

- Змінені файли: `npm/scripts/lib/adr/normalize-pipeline.mjs`, `npm/scripts/lib/adr/tests/normalize-pipeline.test.mjs`.
- Нові експортовані функції: `assembleMadr`, `normalizeSections`, `madrDate`.
- A/B-команда: `AB_N=6 node npm/scripts/lib/adr/.ab-gen.mjs`; тимчасовий харнес після прогону видалено.
- A/B-результат із transcript: old=6/6 ok, new=6/6 ok; якість backticks і rich sections відновлена.
- Спростований pure JSON-fields варіант втрачав backtick-форматування для чернетки `260615-1601-авто-очищення-застарілих-правил-і-скілів-у-n-cursor-json-під.md`.
- `EDGE_SYS` і `KIND_SYS` залишені LLM-стадіями без змін, бо transcript описує їх як суб'єктивну семантичну класифікацію.

## Update 2026-06-16

- A/B-харнес `npm/scripts/lib/adr/.ab-gen.mjs` на 6 реальних чернетках показав 6/6 valid з однією спробою кожна та 5/5 IDENTICAL diff між старим і новим виходом після переходу на гібридний промпт.
- `MERGE_SYS` також переведено на JS-збірку заголовка `## Update <date>` через `genMerge`, щоб не вимагати від LLM форматного markdown-каркаса.
