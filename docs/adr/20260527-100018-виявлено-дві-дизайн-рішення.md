## ADR `pool: 'forks'` у vitest для ізоляції `process.chdir`

## Context and Problem Statement
43+ тест-файли використовують `withTmpCwd` (через `process.chdir`) — стандартна утиліта `npm/scripts/utils/test-helpers.mjs`. У default `pool: 'threads'` (shared memory + process) паралельні test workers ділять `process.cwd()`, що призводило до race condition: `git commit` у changelog consistency тесті потрапляв у реальний репо (rogue commit `8be4d5e init` з `Author: test <test@test>`).

## Considered Options
* `pool: 'forks'` у `vitest.config.js` — кожен test file у окремому OS-процесі
* Рефакторинг усіх 43 тест-файлів для CWD-injection без `process.chdir`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "`pool: 'forks'`", because one-liner зміна у `vitest.config.js` ізолює `process.chdir` без модифікації 43 існуючих тест-файлів.

### Consequences
* Good, because rogue commits зникли: повторний `bun run test` не змінює HEAD репо; vitest 1169/1171 passed (2 skipped — pre-existing flaky ENOTEMPTY у tmpdir cleanup).
* Bad, because `forks` повільніший за `threads` (окремий Node/Bun процес на кожен test file); transcript фіксує очікуване уповільнення, але конкретних мс не наведено.
