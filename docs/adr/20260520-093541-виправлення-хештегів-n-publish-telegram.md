# Виправлення суперечності про хештеги в n-publish-telegram

**Status:** Accepted
**Date:** 2026-05-20

## Context and Problem Statement

У `npm/skills/publish-telegram/SKILL.md` існувала суперечність: секція «Правила» вимагала 2–4 хештеги в кінці поста, тоді як шаблон і приклад ставили один хештег першим рядком. Суперечність виявлена під час першого реального запуску правила `feedback.mdc` — канал inline-резюме тертя спрацював за призначенням.

## Considered Options

- Хештег зверху (першим рядком), рівно 1 тег
- Хештеги в кінці поста, 2–4 теги
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Хештег зверху (першим рядком), рівно 1 тег", because користувач обрав цей варіант інтерактивно через `AskUserQuestion`; шаблон і приклад у файлі вже відповідали цьому варіанту.

### Consequences

- Good, because усунуто суперечність між прозовим текстом і шаблоном/прикладом — агент більше не матиме неоднозначності при генерації поста.
- Bad, because transcript не містить підтверджених негативних наслідків.
