# Архітектура changelog-гейту для крос-харнесного середовища

**Status:** Accepted
**Date:** 2026-06-03

## Context and Problem Statement
Правило `n-changelog.mdc` (з `alwaysApply: true`) реалізоване лише як текстова інструкція: агент може завершити сесію з npm-змінами без `change-файлу` і порушити правило без жодного технічного блокування. Pre-commit через `hk` не встановлений (`hk install` не виконано, `.git/hooks/pre-commit` порожній). Потрібен детермінований backstop, що спрацьовує незалежно від AI-харнесу (Claude Code, Cursor, pi.dev+ChatGPT).

## Considered Options
- Блокуючий `Stop`-hook у `.claude/settings.template.json` (`exit 2` → Claude Code блокує хід і повертає stderr агенту)
- `hk install` як pre-commit — крос-харнесний і модель-агностичний гейт; спрацьовує на `git commit` незалежно від харнесу
- Notify-попередження у pi.dev TS-extension через `agent_end` (async, не блокує)
- Автоматичний виклик `hk install` з CLI `npx @nitra/cursor` — щоб не вимагати ручного запуску в кожному репо

## Decision Outcome
Chosen option: "`hk install` як universal backstop", because `agent_end` у pi.dev є async і не дозволяє заблокувати хід (transcript: «Async, не блокує agent_end»); блокуючий `Stop`-hook діє лише в Claude Code; єдиний рівень, що неможливо обійти незалежно від харнесу й моделі — git pre-commit.

Рішення про автоматичний виклик `hk install` з `npx @nitra/cursor` на момент закінчення transcript ще обговорювалося і не зафіксоване як прийняте.

### Consequences
- Good, because pre-commit спрацьовує і для ручних змін, і для Claude, і для Cursor, і для pi+ChatGPT — зміна не проскочить у `git commit` без change-файлу.
- Bad, because блокуючий `Stop`-hook не транслюється в pi.dev через обмеження `agent_end`-події; там можливе лише notify-попередження.
- Bad, because `hk install` наразі не виконано — backstop неактивний на момент закінчення transcript.

## More Information
- `.cursor/hooks.json` — Cursor-рівень хуків; `agent_end` fire-and-forget, не блокуючий.
- `npm/.claude-template/settings.template.json` — канонічне джерело для `.claude/settings.json`; правки безпосередньо в `.claude/settings.json` зникають при наступному `sync-claude-config`.
- `npm/.pi-template/extensions/n-cursor-adr/index.ts` рядки 79–81 — коментар про async-семантику `agent_end`.
- `hk.pkl` — glob `npm/**`, правило `npm-changelog`; `check changelog` повертає ненульовий код при відсутньому change-файлі.
- `npm/scripts/sync-claude-config.mjs` — синкує `.claude-template/` → `.claude/`.
- Команда для активації backstop: `hk install` у корені репо.
- Наявний `n-changelog.mdc` вже містить блок `## STOP` (`alwaysApply: true`), але він покладається на текстову пам'ять агента, а не на детермінований harness-хук.
- Додаткової інформації про фінальний статус рішення про автоматичний виклик `hk install` з CLI в transcript не зафіксовано.

## Update 2026-06-03

### Канонічне розташування `.claude/` налаштувань

Файл `.claude/settings.json` генерується з `npm/.claude-template/settings.template.json` через `npm/scripts/sync-claude-config.mjs`. Правки безпосередньо в `.claude/settings.json` (включно зі Stop-гейтами для changelog) будуть перезаписані при наступному синку. Правильна точка редагування — `npm/.claude-template/settings.template.json` та `npm/.claude-template/hooks/`.

Структура: `npm/.claude-template/settings.template.json`, `npm/.claude-template/hooks/capture-decisions.sh`, `npm/.claude-template/hooks/normalize-decisions.sh`. Наявний `n-changelog.mdc` вже містить блок `## STOP` (`alwaysApply: true`), але він покладається на текстову пам'ять агента, а не на детермінований harness-хук. Конкретний Stop-гейт для changelog у `settings.template.json` на момент закінчення сесії додано не було — сесія завершилась на читанні файлу без commit змін.
