## ADR Per-rule `fix.mjs` entry-point з IoC та rename `fix/` → `js/`

## Context and Problem Statement

Правила `npm/rules/<id>/` не мали власної точки входу: CLI `n-cursor check` — єдиний спосіб їх запустити через convention-based discovery. Це унеможливлює debug одного правила напряму (`bun rules/abie/fix.mjs`), запуск через IDE Run button, CI per-rule matrix jobs та майбутню portability правила як самостійного пакету.

## Considered Options

* **Варіант A — Shim per rule:** CLI лишається авторитативним (convention-based discovery + центральний `runRule`); `<id>.mjs` — тонкий wrapper, що лише делегує `runStandardRule`.
* **Варіант B — Per-rule entry з справжньою IoC:** CLI перебирає каталоги та dynamic-import'ує `<id>/fix.mjs`; правило саме вирішує оркестрацію (за замовчуванням — `runStandardRule`, з правом відхилитись через `RuleContext` options у `run-standard-rule.mjs`).
* **Варіант C — Hybrid з fallback:** CLI спочатку перевіряє наявність `<id>/fix.mjs`, інакше fallback на convention-discovery.

## Decision Outcome

Chosen option: "Варіант B — Per-rule entry з справжньою IoC", because користувач підтвердив усі чотири use-cases (debug, IDE Run, CI per-rule, future portability), обрав Mandatory (кожне правило мусить мати власний entry-point), і явно відхилив Варіант A як "shim, а не справжню IoC". Конвенцію щодо назви файла визначено як `fix.mjs` (попри ризик плутанини з каталогом `fix/`, що усувається rename `fix/` → `js/`). Convention-drift контролюється через enumerated options у `RuleContext` у центральному `run-standard-rule.mjs` — локальна логіка в `rules/<id>/fix.mjs` заборонена.

### Consequences

* Good, because transcript фіксує очікувану користь: `bun npm/rules/abie/fix.mjs` запускає правило без CLI; CI отримує matrix `bun npm/rules/${{ matrix.rule }}/fix.mjs`; discovery CLI спрощується до `readdir(rules/) + exists(<id>/fix.mjs)`.
* Good, because transcript фіксує: rename `fix/` → `js/` усуває колізію між каталогом concerns і entry-point файлом; назва `js/` консистентна з `policy/` (розподіл за технологією, а не функцією).
* Bad, because Mandatory означає атомарну міграцію 30 правил в одній PR без перехідного fallback; існуючі `tests/*.test.mjs` з літеральними import-шляхами `rules/<id>/fix/<concern>/` потребують оновлення після rename.
