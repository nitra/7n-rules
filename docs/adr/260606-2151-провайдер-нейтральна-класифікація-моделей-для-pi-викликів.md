## ADR: Провайдер-нейтральна класифікація моделей для pi-викликів

## Context and Problem Statement

Скіли, що викликають `pi` для LLM-тиру, жорстко прив'язували назви до конкретного провайдера (`MODEL_HAIKU = 'claude-haiku-...'`). На машинах без anthropic-ключа виникала помилка `No API key found for anthropic` без пояснення що робити. Відсутність глобального контракту змушувала кожен скіл оголошувати свої env vars.

## Considered Options

* Зберегти per-skill env vars (поточний стан) з кращою документацією
* Глобальна класифікація `N_LOCAL_*/N_CLOUD_*` у shared модулі

## Decision Outcome

Chosen option: "Глобальна класифікація у `npm/lib/models.mjs`", because усуває дублювання env vars між скілами і дозволяє одноразово налаштувати тири в середовищі.

### Consequences

* Good, because кожен скіл імпортує `CLOUD_MIN`/`CLOUD_AVG` — не оголошує власних env vars.
* Bad, because `npm/lib/models.mjs` не містить підтверджених негативних наслідків.
