# Багаторівнева валідація GitHub Actions workflows у @nitra/cursor

**Status:** Accepted
**Date:** 2026-05-22

## Context and Problem Statement
Workflow-файли GitHub Actions у монорепо `@nitra/cursor` потребують кількох рівнів перевірки одночасно: синтаксичної коректності `uses:`/`run:`, security-аудиту pinning-тегів та відповідності канонічним версіям action, прийнятим у проєкті (наприклад, обов'язковий `actions/checkout@v6`). Один інструмент не покриває всі три шари.

## Considered Options
* Єдиний зовнішній лінтер (actionlint) для всіх перевірок
* Багаторівнева схема: actionlint + zizmor + conftest/Rego

## Decision Outcome
Chosen option: "Багаторівнева схема: actionlint + zizmor + conftest/Rego", because кожен інструмент покриває окремий шар: actionlint — синтаксична валідність `uses:`, zizmor — security/ref-pinning, conftest/Rego — канонічні версії та правила проєкту; `actions/checkout@v4` замість `@v6` буде зловлено саме Rego, тоді як actionlint не знає про проєктний канон версій.

### Consequences
* Good, because actionlint (`bunx github-actionlint`, команда `bun run lint-ga`) ловить синтаксичні помилки workflow та некоректні `uses:`-посилання.
* Good, because zizmor (`uvx zizmor --offline --collect=workflows .`, конфіг `.github/zizmor.yml`) перевіряє закріплення тегів (`@v4`, `@main`) з точки зору безпеки незалежно від версійної політики проєкту.
* Good, because Rego-полісі в `npm/rules/ga/policy/` разом із template-файлами та JS-перевірками фіксують канонічну версію (`checkout@v6`) і запускаються через `npx @nitra/cursor check ga`.
* Good, because JS cross-file перевірки (порядок кроків: checkout перед `setup-bun-deps`, `persist-credentials: false`) реалізовані в `check-ga.mjs` і доповнюють Rego-полісі там, де декларативний підхід Rego обмежений.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- `bun run lint-ga` → `n-cursor lint-ga` → `bunx github-actionlint` (синтаксис + shellcheck)
- `uvx zizmor --offline --collect=workflows .` з `.github/zizmor.yml` (`unpinned-uses` та ref-pin)
- `npx @nitra/cursor check ga` → `conftest` + `npm/rules/ga/policy/lint_ga/lint_ga.rego`
- Канонічні template-файли: `npm/rules/ga/policy/*/template/*.yml`
- JS-логіка: `npm/scripts/utils/gha-workflow.mjs`, `npm/rules/ga/lint/lint.mjs`
- JS cross-file: `check-ga.mjs` — порядок кроків, `persist-credentials: false`
