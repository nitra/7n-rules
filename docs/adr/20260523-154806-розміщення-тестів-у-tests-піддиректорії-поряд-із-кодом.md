## ADR Розміщення тестів у `tests/` піддиректорії поряд із кодом + правило `test` (канон)

## Context and Problem Statement
До цього моменту тести (`*.test.mjs`) у пакеті `@nitra/cursor` лежали **поряд із джерельним файлом**: для `dir/check.mjs` тест був `dir/check.test.mjs`. Це класична JS-конвенція, але має проблеми: каталог правила (`rules/<rule>/fix/<sub>/`) перевантажений — продуктивний код, тести, фікстури, інколи `__fixtures__/` лежать поруч; неможливо швидко окипропустити оком, які файли пакуються в npm tarball, а які — суто тестові. Конвенція пакета вже передбачає негативні globs у `package.json#files` (`!**/*.test.mjs`, `!**/__fixtures__/**`), але вони ловлять лише шаблон, не геометрію тек. Окремо: щоб така зміна не лишалась «джентльменською угодою», потрібен **програмний канон** — правило, яке репортує тест поза `tests/` як порушення.

## Considered Options
* Лишити статус-кво — тести поряд із файлами (`dir/X.test.mjs`).
* Перенести у глобальний `npm/tests/` (один кореневий каталог) — як у багатьох Python-проєктах.
* **Перенести у локальний `dir/tests/` поряд із кодом, який тестується + закріпити правилом `test` у `npm/rules/`.** Прийнятий варіант.

Назва піддиректорії: **`tests/`** (множина), узгоджена з top-level `npm/tests/` (integration-тести) та існуючим патерном у `npm-module.mdc` (де `tests/` і `test/` згадуються як test-style каталоги). Спочатку обрано однину `test/`, але після rev'ю — виправлено на `tests/` для одноманітності й закріплено правилом.

## Decision Outcome
Chosen option: "Локальний `dir/tests/` поряд із кодом + правило `test` у `npm/rules/test/`", because (1) тест залишається близько до коду — навігація `check.mjs ↔ tests/check.test.mjs` — один клік, (2) каталог правила тепер чіткіший: `check.mjs`, `<helper>.mjs`, `tests/`, `policy/`, `<template>.snippet.*`, замість миготіння тестів і фікстур серед робочого коду, (3) **візуальна сегрегація тестового коду** робить очевидним, що піде в tarball, а що ні, навіть без читання `package.json#files`, (4) **правило `test` (concern `location`)** перевіряє кожен `*.test.mjs` файл — басенейм батьківської директорії має бути `tests`; інакше fail з вказівкою куди перенести.

Переміщено (`git mv` для збереження історії):

- **73 sibling-тести** у `rules/...` і `scripts/...`: для кожного `dir/X.test.mjs` → `dir/tests/X.test.mjs` з автоматичним оновленням relative imports (`./X` → `../X`, `../X` → `../../X` через sed і placeholder-трюк).
- **3 integration-тести з `npm/tests/`** → залишилися у `npm/tests/` (перейменування каталогу не потрібне — він уже відповідав конвенції; depth не змінилася, relative imports без правок).
- **`npm/scripts/utils/__fixtures__/`** (3 піддиректорії `mdc-refs/`, `template/`, `inline-template/`) → `npm/scripts/utils/tests/__fixtures__/`; шляхи `join(HERE, '__fixtures__', ...)` у тестах залишаються коректними, бо `HERE` тепер `scripts/utils/tests/`, а фікстури теж там.
- **`npm/rules/nginx-default-tpl/fix/template/fixtures/`** → `npm/rules/nginx-default-tpl/fix/template/tests/fixtures/`; посилання з `npm/tests/check-rule-fixtures.test.mjs` оновлено.
- **Ручні фіксапи** для 4 HERE/`..` path patterns у тестах, які навігують up з тестового файлу до інших дир (sync-setup-bun-deps-action, inline-template-links, rules/adr/fix/hooks, rules/abie/utils/enabled) — кожному додано один зайвий `..`.
- **Реверти false-positive sed** у `rules/image-avif/fix/avif_generation/tests/check.test.mjs`: sed-регексп помилково модифікував рядки-фікстури у backtick template literals (`from './hero.png'`, `from './icon.svg'`, `from './imp.png'`), які є **вмістом Vue-фікстур**, а не реальними імпортами тесту. Реверт зроблено окремою sed-командою.

Створено **нове правило `test`** (`npm/rules/test/`):

- `test.mdc` — опис конвенції, перелік спеціальних випадків (root `tests/`, fixtures, test-helpers), tarball-захист через `files`.
- `fix/location/check.mjs` — програмний обхід дерева (`walkDir` зі стандартними skip-листом + `.n-cursor.json:ignore`); для кожного `*.test.mjs` басенейм батьківської директорії має бути `tests`.
- `fix/location/tests/check.test.mjs` — 6 одиничних тестів самого правила (eats own dogfood: лежить у `tests/`).
- `auto.md` — умова авто-додавання: «якщо у проекті є хоча б один файл `*.test.mjs`».
- Додано `"test"` у `.n-cursor.json:rules` репозиторію `@nitra/cursor`.
- Додано `.claude/worktrees` у `.n-cursor.json:ignore` — щоб правило не звітувало про знімки в git worktrees.

### Consequences
* Good, because у каталозі правила одразу видно «що — продакшен, що — тест» (порядок: `check.mjs`, `<helper>.mjs`, `policy/`, `template/...`, `tests/`).
* Good, because `package.json#files` негативні globs `!**/*.test.mjs` / `!**/__fixtures__/**` продовжують працювати без змін — recursive globs ловлять файли всередині `tests/` так само.
* Good, because Bun test discovery (`bun test`) знаходить файли рекурсивно — переміщення прозоре для test runner.
* Good, because конвенція захищена програмним правилом `test.location` — нові проєкти / контрибутори отримують fail з вказівкою куди перенести, а не «непомітну стилістичну претензію».
* Bad, because `import` шляхи у тестах стали глибшими на один рівень (`./X.mjs` → `../X.mjs`, `../X.mjs` → `../../X.mjs`); це разова ціна за чистіший layout.
* Bad, because автоматизація переміщення sed-ом мала false positives на рядках-фікстурах (приклад: image-avif Vue-фікстури); потрібна була ручна корекція. Уроки для майбутніх масових refactor'ів: не sed-ом, а ast-aware tool, або зробити sed більш консервативним (тільки початок рядка).

## More Information
Файли, що підтверджують рішення:
- `npm/scripts/utils/tests/*.test.mjs` (18 тестів), `npm/rules/<rule>/fix/<sub>/tests/*.test.mjs` (49 тестів), `npm/scripts/tests/*.test.mjs` (10 тестів), `npm/rules/<rule>/lint/tests/*.test.mjs` (5 тестів), `npm/rules/abie/utils/tests/*.test.mjs` (5 тестів), `npm/tests/*.test.mjs` (3 тести) — повний інвентар 76 тестів у новому layout + 1 тест нового правила = 77.
- `npm/scripts/utils/tests/__fixtures__/` — переміщені shared-фікстури.
- `npm/rules/nginx-default-tpl/fix/template/tests/fixtures/` — переміщені правило-специфічні фікстури.
- `npm/rules/test/{test.mdc,auto.md,fix/location/check.mjs,fix/location/tests/check.test.mjs}` — нове правило-канон.

Виконання:
- `bun test`: 837 pass, 2 fail (обидва pre-existing `with-lock` issues з commit `c4b9ea3`, не пов'язані з refactoring'ом).
- `npx @nitra/cursor check test`: `✅ Всі 77 файлів *.test.mjs у каталозі tests/ (test.mdc)`.
- `bun test rules/test`: 6/6 pass — тести нового правила.

## Update 2026-05-23

### `*_test.rego` залишаються поряд із полісі (OPA community-патерн)

Під час міграції 76 `*.test.mjs` у піддиректорії `tests/` тимчасово 69 `*_test.rego` також переміщено у `policy/<concern>/tests/`. Рішення скасовано: `*_test.rego` залишаються поряд із `.rego`-файлами відповідно до OPA/Conftest community-патерну — `conftest verify -p <dir>` рекурсивний, тому поведінка не змінюється. Правило `npm/rules/test/fix/location/check.mjs` перевіряє лише `*.test.mjs`; `*_test.rego` явно виключені. `test.mdc` v1.1 фіксує виняток окремою секцією. Кореневу директорію перейменовано з `test/` на `tests/` (28 директорій). `npm/rules/npm-module/fix/package_structure/check.mjs`: carve-out у `classifyPublishedFileAsTest` для rule-name сегмента `rules/test/...`. `.n-cursor.json`: додано `"test"` до `rules`, `"ignore": [".claude/worktrees"]`. Верифікація: `npx @nitra/cursor check test` → 77/77 `*.test.mjs` OK.
