## ADR Масове прибирання нерезультативного тексту з `.mdc`-правил

**Контекст:** В 26 `.mdc` файлах правил `npm/rules/*/` накопичився текст, який не впливає на поведінку AI або check-скриптів: секції `## Перевірка`, блоки `.vscode/extensions.json`, інструкції `brew install`, опис зовнішніх інструментів (hadolint, kubeconform тощо), міграційний контекст старих версій, дублі команд.

**Рішення/Процедура/Факт:** Виконано ручну інвентаризацію 26 файлів (4133 рядки, без `security.mdc`). Відповідно до вибору користувача («Все, окрім ризикових») — прибрано тільки однозначно нерезультативне, пропущено `vue.mdc`, `php.mdc`, `text.mdc` (потенційні суперечності з check-скриптами чи семантичні конфлікти) та borderline-блоки (`adr.mdc` ENV-таблиця, `k8s.mdc` install-steps у workflow YAML). Змінено 23 файли: видалено `## Перевірка` хвости (18 файлів), VS Code блоки (8 файлів), CLI install-підказки, опис зовнішніх інструментів, активаційну мету, дублі команд, міграційний контекст. Нові розміри: з 4133 → ~3945 рядків (−314 рядків, −7.6%).

**Обґрунтування:** `.mdc` є виключно AI-контекстом: Cursor/Claude Code підвантажує файл у промпт за `globs`-матчем. Текст типу "## Перевірка / npx @nitra/cursor check <id>" не впливає на те, що AI генерує; VS Code блоки не є директивами для AI-агента. Прибраний текст збільшував розмір промпту без корисного сигналу.

**Розглянуті альтернативи:** «Лише низькоризикові» (тільки Перевірка + VS Code + CLI install); «Всі, включно з ризиковими» (включно з vue/php/text); «Ніяких» — обрано «Все окрім ризикових».

**Зачіпає:** `npm/rules/*/` — 23 `.mdc` файли (rego, adr, style-lint, image-compress, image-avif, nginx-default-tpl, graphql, docker, bun, k8s, js-lint, ga, hasura, capacitor, changelog, npm-module, ci4, js-run, js-bun-db, js-bun-redis, tauri, js-mssql, abie).

## ADR Template-каталог у `npm/rules/<id>/` — merge-assertion і scaffold-канони

**Контекст:** `.mdc` файли правил поєднують AI-директиви (що генерувати) з конкретними канонічними фрагментами файлів (`package.json` scripts, `.gitleaks.toml`, workflow YAML тощо). Ці фрагменти дублюються між `.mdc` (prose) та `check.mjs`/`*.rego` (inline-літерали в логіці перевірки), що ускладнює синхронізацію при зміні канону.

**Рішення/Процедура/Факт:** Визначено архітектуру каталогу `template/` на рівні **концерну** (`policy/<concern>/template/` або `fix/<concern>/template/`). Три типи файлів у нативному форматі цільового файлу: `<target>.snippet.<ext>` (required subset), `<target>.deny.json` (заборонені шляхи), `<target>.contains.json` (substring-вимоги для рядкових полів). Повні файли-канони (`.gitleaks.toml`, workflow YAML) також живуть як `template/<target>.snippet.<ext>` у нативному форматі (TOML, YAML, JSON). Rego-полісі отримують template через `conftest --data` як `data.<rule>.<concern>.template`. JS-концерни використовують нові утиліти `loadTemplate` + `checkSnippet`/`checkDeny`/`checkContains`/`checkTextSubset`. Spec записано в `docs/superpowers/specs/2026-05-17-template-dir-design.md`. Деталізований план Phase 0+1 (infrastructure + security pilot) в `docs/superpowers/plans/2026-05-17-template-dir-phase-0-1.md`.

**Обґрунтування:** Concern-level layout (а не rule-level) обраний через реальні collision-cases: у `k8s` — 4 концерни на `*.yaml` glob та 2 на `kustomization.yaml`; у `npm-module` — 2 концерни на `package.json`. Rule-level мав би конфлікти для одного basename. Concern-level ізолює кожен концерн з власним template без перетину. Суфіксна конвенція (native-syntax files) обрана замість єдиного DSL-файлу `check.json` — файли залишаються в рідному форматі цільового файлу, AI може копіювати без перетворень.

**Розглянуті альтернативи:** Rule-level layout (відкинуто через collision); Expectations DSL yaml (відкинуто — новий синтаксис, складний для rego data); Snippet-only без deny/contains (відкинуто — не покриває substring та forbid-перевірки).

**Зачіпає:** `npm/scripts/utils/template.mjs` (новий), `npm/scripts/utils/run-conftest-batch.mjs`, `npm/scripts/check-mdc-template-refs.mjs` (новий), всі `fix/<concern>/check.mjs` і `policy/<concern>/*.rego` (поетапно), всі `.mdc` файли після Phase 6.
