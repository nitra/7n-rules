## ADR Зміна схеми релізу: change-файли замість ручного bump/CHANGELOG

## Context and Problem Statement
При паралельній роботі кількох агентів у воркtree одного монорепо кожен редагував `npm/package.json` і `CHANGELOG.md` безпосередньо; при злитті виникали merge-конфлікти в цих файлах. Потрібно було розв'язати цю проблему без зміни семантики Keep a Changelog і SemVer.

## Considered Options
* Change-файли per-workspace (`.changes/<timestamp>-<rand>.md`) — агрегація у CI
* Ручний bump + CHANGELOG (попередній підхід)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Change-файли per-workspace", because вони не конфліктують між паралельними агентами (кожен пише новий унікальний файл), а агрегацію (bump version, CHANGELOG, тег, push) делегують команді `n-cursor release` у CI.

### Consequences
* Good, because transcript фіксує очікувану користь: нуль merge-конфліктів у `package.json` / `CHANGELOG.md` при паралельній роботі; ручний bump лишається дозволеним (legacy/hotfix) — `check` приймає або change-файл, або вже підвищену version.
* Bad, because перший запуск CI без попереднього тегу `<name>@*` є no-op (fallback вимкнений на bootstrap) — перехідний реліз треба зробити вручну (що і зроблено: `@nitra/cursor@1.33.0` закомічено вручну).

- Plan: `docs/superpowers/plans/2026-05-29-n-cursor-release.md`

## ADR Fallback: bootstrap no-op замість синтезу з повної git-історії

## Context and Problem Statement
Команда `n-cursor release` при відсутності явних change-файлів вмикає fallback — синтез запису з git-комітів від останнього тегу `<name>@*`. На новому репо або після першого ручного релізу (коли тегів `<name>@*` ще нема) fallback брав би всю git-історію і генерував зайвий bump поверх вже виставленої вручну версії.

## Considered Options
* Bootstrap no-op: `synthesizeChangeFromCommits` повертає `null` якщо немає попереднього тегу
* Синтез з повної git-історії (`range = 'HEAD'`) — попередня реалізація

## Decision Outcome
Chosen option: "Bootstrap no-op", because без попереднього тегу немає надійної бази для delta-синтезу; перший автоматичний реліз після bootstrap був би подвійним bump'ом (ручна версія + згенерована зверху).

### Consequences
* Good, because transcript фіксує очікувану користь: перший CI-запуск після ручного `@nitra/cursor@1.33.0` — no-op, не створює `1.33.1` без change-файлів.
* Bad, because на абсолютно новому репо без жодного тегу `n-cursor release` нічого не зробить до першого ручного релізу — але це очікувана поведінка bootstrap.

## More Information
- Файл: `npm/rules/release/lib/fallback.mjs`
- Тест оновлено: `npm/rules/release/js/tests/fallback.test.mjs` — "без тегу пакета (bootstrap) → null"
- Коміт: `8b74ea1`

## ADR CI workflow для `n-cursor release`: локальний runner замість npx-реєстру

## Context and Problem Statement
Перша версія `npm-publish.yml` використовувала `npx @nitra/cursor release` і не мала кроку встановлення залежностей. У CI це давало `sh: 1: n-cursor: not found` — бінарник не злінкований без `bun install`, а `npx` тягнув би стару опубліковану версію з реєстру (chicken-and-egg для self-publishing репо).

## Considered Options
* `node npm/bin/n-cursor.js release` після `setup-bun-deps` — локальний runner
* `npx @nitra/cursor release` — тягне з реєстру

## Decision Outcome
Chosen option: "`node npm/bin/n-cursor.js release` після `setup-bun-deps`", because self-publishing репо не може надійно використовувати власний пакет із реєстру при релізі нової версії; локальний runner завжди бере актуальний код із HEAD.

### Consequences
* Good, because transcript фіксує очікувану користь: `n-cursor: not found` усунено; `smol-toml` та інші залежності доступні через `bun install`; git identity задана явно для `release-commit`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Файл: `.github/workflows/npm-publish.yml`
- Додано: `- uses: ./.github/actions/setup-bun-deps`, git config identity, `node npm/bin/n-cursor.js release`
- Видалено: `npx @nitra/cursor release`
- lint-ga exit=0, actionlint+zizmor чисто (коміт `8b74ea1`)
