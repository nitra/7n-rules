## ADR Автоматичний вибір вільної назви у `worktree add`

## Context and Problem Statement
Команда `npx @nitra/cursor worktree add "main-fix" "..."` падала з помилкою `fatal: a branch named 'main-fix' already exists`, якщо гілка вже існувала. CLI не перевіряв зайнятість назви перед викликом `git worktree add -b <branch>`, тому агент отримував збій і змушений був обирати альтернативну назву вручну — вже після падіння.

## Considered Options
* Перевіряти зайнятість назви до `git worktree add` і автоматично обирати вільну (`base`, `base2`, `base3`, …)
* Залишити поточну поведінку (падіння на колізії, вибір назви — відповідальність викликача)

## Decision Outcome
Chosen option: "Перевіряти зайнятість назви до `git worktree add` і автоматично обирати вільну", because колізія назви — детермінований і передбачуваний стан, тому CLI повинен сам визначати вільне ім'я замість того щоб перекладати це на агента-викликача.

### Consequences
* Good, because transcript фіксує очікувану користь: `worktree add` більше не падає на `already exists` — вибирає `base2`, `base3`, … без участі користувача.
* Bad, because transcript не містить підтверджених негативних наслідків.
