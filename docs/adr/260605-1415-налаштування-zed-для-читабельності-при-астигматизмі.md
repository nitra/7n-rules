## ADR Налаштування Zed для читабельності при астигматизмі

## Context and Problem Statement
Користувач має астигматизм і зіткнувся з проблемою злипання букв у редакторі Zed. Потрібно покращити читабельність коду без можливості безпосередньо керувати `letter-spacing` (якого у Zed немає як окремого параметра).

## Considered Options
* Шрифт з широким трекінгом (`JetBrainsMono Nerd Font Mono`) + збільшений `buffer_line_height` + вимкнені лігатури
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "JetBrainsMono Nerd Font Mono + buffer_line_height 1.7 + liga/calt false", because шрифт вже встановлений у системі, є широким та розбірливим, а комбінація з вимкненими лігатурами і збільшеним міжрядковим інтервалом замінює відсутній `letter-spacing`.

### Consequences
* Good, because transcript фіксує очікувану користь: символи (`->`, `=>`, `!=`) не зливаються у єдиний гліф, рядки отримують більший повітряний простір.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Змінено `~/.config/zed/settings.json`:
```json
{
"buffer_font_family": "JetBrainsMono Nerd Font Mono",
"buffer_font_size": 16,
"buffer_font_weight": 450,
"buffer_line_height": { "custom": 1.7 },
"buffer_font_features": { "liga": false, "calt": false }
}
```
Шрифт підтверджено через `system_profiler SPFontsDataType`. Zed застосовує зміни live при збереженні файлу.

## ADR Розмір шрифту в панелі агента Zed

## Context and Problem Statement
Після зміни шрифту редактора панель агента (вікно асистента) використовувала власні значення за замовчуванням: `agent_buffer_font_size: 12` і `agent_ui_font_size: null`. Шрифт поля вводу запиту користувача залишався дрібним (~12), що суперечило вимозі щодо читабельності.

## Considered Options
* Виставити `agent_ui_font_size` і `agent_buffer_font_size` явно, рівними `buffer_font_size` (16)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "agent_ui_font_size: 16, agent_buffer_font_size: 16", because ключі знайдено у бінарнику Zed через `strings`, панель агента має окремі параметри від редактора, і розмір має збігатися з `buffer_font_size` для однорідної читабельності.

### Consequences
* Good, because transcript фіксує очікувану користь: блоки коду та поле вводу в панелі агента стали однакового розміру з вікном редактора.
* Bad, because повне поле вводу запиту користувача (composer/message editor) за результатами сесії ще не отримало потрібного розміру — `agent_buffer_font_size` впливає лише на відповідь асистента та блоки коду.

## More Information
Ключі `agent_buffer_font_size` (default: 12) і `agent_ui_font_size` знайдено командою:
```sh
strings -n 8 /Applications/Zed.app/Contents/MacOS/zed | grep -iE "agent.*font_size"
```
Додано до `~/.config/zed/settings.json`:
```json
{
"agent_ui_font_size": 16,
"agent_buffer_font_size": 16
}
```
