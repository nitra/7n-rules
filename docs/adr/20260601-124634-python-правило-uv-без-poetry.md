# Нове правило `python` в `@nitra/cursor`: uv, Rego-first та без CLI-case

**Status:** Accepted
**Date:** 2026-06-01

## Context and Problem Statement

Пакет `@nitra/cursor` не мав правила для Python-проєктів. Потрібно додати правило за flat-concern-патерном (`meta.json`, `fix.mjs`, `js/tooling.mjs`, `lint/lint.mjs`, `policy/*/`, `*.mdc`), яке автоматично активується за наявністю `pyproject.toml`. При проєктуванні виникли чотири незалежних питання: вибір пакет-менеджера, розподіл логіки між Rego і JS, реєстрація CLI-підкоманди та режим запуску ruff.

## Considered Options

* uv + PEP 621 `[project]` — єдиний дозволений пакет-менеджер; `poetry.lock` / `[tool.poetry]` — явно заборонено
* Poetry (`[tool.poetry]`, `poetry.lock`) — відкинуто
* Rego для вмісту документів; JS лише для FS-existence та spawn CLI
* Перевіряти вміст документів у JS — відкинуто (порушує `conftest.mdc`)
* Не додавати `case 'lint-python'` у `n-cursor.js`; wire через `package.json#scripts.lint-python`
* Додати `case 'lint-python'` за зразком `lint-docker` — відкинуто
* `uv run ruff check --fix .` + `uv run ruff format .` — авто-fix, опційно (skip якщо ruff недоступний)
* `ruff check .` без `--fix` (read-only) — відкинуто

## Decision Outcome

Chosen option: "uv + PEP 621; Rego-first; без CLI-case; ruff auto-fix опційно", because:

- **uv**: замовник явно зафіксував «замість Poetry лише uv»; Poetry-артефакти відхиляються Rego-deny і JS-tooling.
- **Rego-first**: `conftest.mdc` (alwaysApply: true) явно вимагає пер-документні перевірки у `policy/`; JS — лише FS і spawn; патерн відтворює `php` і `rust`.
- **Без CLI-case**: зміна `n-cursor.js` і `build-agents-commands.mjs` (відстежувані файли) при uncommitted нових файлах активує `integration-repo-checks` (dirty-bump check), що суперечить `n-changelog.mdc`; `php` (еталон) так само не має субкоманди в CLI.
- **Ruff auto-fix**: користувач явно попросив «додати авто фікс руфом» після початкового перегляду.

### Consequences

* Good, because 17 conftest-тестів для `python`-policy зелені; JS-tooling 6/6; `bun run lint-rego` — 538 тестів, 0 порушень; `regal lint` — 158 файлів, 0 порушень.
* Good, because PR суто адитивний (лише нові untracked файли); `integration-repo-checks` чистий.
* Good, because Rego автоматично виявляє й відхиляє `[tool.poetry]`-секцію, унеможливлюючи drift у нових і мігрованих репозиторіях.
* Bad, because `npx @nitra/cursor lint-python` не спрацьовує без `scripts.lint-python` у `package.json` споживача — треба явне wiring (перевіряється Rego `policy/package_json/`).
* Neutral, because `uv`, `ruff`, `mypy` — не в `dependencies`/`devDependencies` пакета; лише в PATH середовища споживача.

## More Information

Структура `npm/rules/python/`:
- `meta.json`: `{ "auto": { "glob": "pyproject.toml" } }` — автоактивація правила.
- `js/tooling.mjs`: FS fail на `poetry.lock`/`poetry.toml`; вимога `pyproject.toml`, `uv.lock`, `package.json`, `lint-python.yml`.
- `lint/lint.mjs`: `uv run --frozen ruff check --fix .` → `uv run --frozen ruff format .` (обгорнуто у `uvToolAvailable('ruff', …)`); `uv run mypy .` (read-only).
- `policy/pyproject_toml/`: deny `[tool.poetry]` через `template/pyproject.toml.deny.toml`; вимога `project.name` + `project.version`.
- `policy/package_json/`: `scripts.lint-python` містить підрядок `bun` (через `template/package.json.contains.json`).
- `policy/lint_python_yml/`: `uses`-підмножина + `run`-підрядки — аналог `lint_rust_yml.rego`; канонічний CI з `astral-sh/setup-uv@v8.0.0` (версія узгоджена з `npm/rules/ga/policy/lint_ga/template/lint-ga.yml.snippet.yml:32`).
- `python.mdc`: міграційний гайд (`uv init`, перенесення метаданих у `[project]`, `uv lock`, `uv add --dev`); markdown-лінки на `template/` без дублювання fenced-блоків.
- Тест-каунт `npm/tests/fix-mjs-contract.test.mjs`: оновлено 34 → 36.
- Відкат tracked-файлів: `git checkout -- npm/bin/n-cursor.js npm/scripts/build-agents-commands.mjs`; початкова спроба додати `case 'lint-python'` активувала `integration-repo-checks` через uncommitted зміни у цих файлах.
- Change-файл: `npm/.changes/1780306777450-ca16ef.md` (bump minor, секція Added).
