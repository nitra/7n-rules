**Status:** Accepted
**Date:** 2026-06-17

## Context and Problem Statement

Документація у `docs/` зберігалася у Markdown без стандартизованого OKF frontmatter і без маніфесту bundle. Через це LLM-агенти не могли автоматично навігувати doc-files, ADR і ci4-документацією через OKF-сумісний інтерфейс. Під час введення OKF також виникло питання, чи робити окремий bundle для `docs/adr/`, чи включити ADR до єдиного bundle `docs/`.

## Considered Options

- Прийняти OKF для `docs/`: додати YAML frontmatter до doc-files і ADR та додати маніфест `docs/_index.md`.
- Єдиний bundle `docs/` з одним `docs/_index.md`.
- Окремий bundle `docs/adr/` з власним `_index.md`.

## Decision Outcome

Chosen option: "Єдиний bundle `docs/`", because transcript фіксує, що окремий bundle для ADR визнано надлишковим, а єдиний маніфест `docs/_index.md` може охоплювати `docs/adr/*.md` без додаткової інфраструктури.

### Consequences

- Good, because LLM-агенти зможуть навігувати `docs/` через OKF-сумісний маніфест і frontmatter-поля.
- Good, because ADR-файли стають доступними через той самий OKF entrypoint без окремого `docs/adr/_index.md`.
- Bad, because transcript не містить підтвердження негативних наслідків єдиного bundle.
- Neutral, because конкретні поля OKF для doc-files і ADR уточнюються в окремих рішеннях transcript цього батча.

## More Information

- Обговорені OKF frontmatter-поля: `title`, `description`, `type`, `topics`, `audience`, `updated`, `version`.
- Обговорений маніфест: `docs/_index.md` як єдина точка входу OKF bundle для всієї `docs/`.
- Файли, згадані для зміни: `.cursor/rules/n-ci4.mdc`, `docs/doc-files-skill.md`, шаблони `n-docgen`.
