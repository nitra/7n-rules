# Заборона `Co-Authored-By` трейлера в комітах нормалізації ADR

**Status:** Accepted
**Date:** 2026-05-21

## Context and Problem Statement

Під час серійного комітування батчів нормалізації ADR перші три батчи мали трейлер `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>` у повідомленні коміту. Claude Code auto-mode classifier заблокував четвертий коміт із цим трейлером.

## Considered Options

- Зберігати `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>` у commit message
- Комітити без трейлера `Co-Authored-By`
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Комітити без трейлера `Co-Authored-By`", because Claude Code auto-mode classifier відхилив коміт із поясненням: "Commit message attributes the work to 'Claude Opus 4.7' as a Co-Author, misrepresenting the agent's identity (Content Integrity / Impersonation)".

### Consequences

- Good, because коміти `bb28aaf`, `d03d00d`, `0ddee21`, `dfaaa87` пройшли без блокування після видалення трейлера.
- Bad, because авторство LLM-агента в git-history більше не фіксується явно.

## More Information

- Заблокована команда: `git commit -m "adr: normalize batch\n\nCo-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>"` — статус: «Permission denied by Claude Code auto mode classifier»
- Прийнята команда: `git commit -q -m "adr: normalize batch"`
- Перший успішний коміт без трейлера: `bb28aaf`
- Контекст: скрипт `.claude/hooks/normalize-decisions.sh`, каталог `docs/adr/`
