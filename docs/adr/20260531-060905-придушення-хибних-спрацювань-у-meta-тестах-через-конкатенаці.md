## ADR Придушення хибних спрацювань у meta-тестах через конкатенацію рядків та Identifier-аргументи

## Context and Problem Statement
Тести модулів `no-process-chdir.mjs` та `no-relative-fs-path.mjs` самостійно порушували правила, які перевіряли: fixture-рядки містили `process.chdir(` (regex-сканер), а `test-helpers.test.mjs` передавав string literals у функції `writeJson`/`ensureDir` (AST-сканер). Це спричиняло false positives при кожному запуску `npx @nitra/cursor fix`.

## Considered Options
* Конкатенація рядків (`'process.chd' + 'ir'`) і виносити pattern у `const`-змінну модуля для regex-сканерів
* Передача шляху через Identifier-змінну (а не string literal) для AST-сканера `no-relative-fs-path`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Конкатенація рядків + Identifier-аргументи", because `Array.join()`/конкатенація обходить regex-перевірку на рівні source-тексту, а передача через `const REL_FILE_PATH = 'relative/path'` дозволяє AST-сканеру пропустити не-literal аргумент — без змін у логіці самих checker-модулів.

### Consequences
* Good, because transcript фіксує очікувану користь: `npx @nitra/cursor fix` після правок показав `✨ Результат: 1/1 правил без зауважень` для всіх 15 правил.
* Bad, because test-код стає менш читабельним (неочевидна конкатенація), і коментар пояснення є обов'язковим.

## More Information
Змінені файли: `npm/rules/test/js/tests/no-process-chdir.test.mjs`, `npm/scripts/tests/test-helpers.test.mjs`. Коміт `b07a195` («тест»). Змінна `CHDIR = ['process.chd', 'ir'].join('')`; змінні `REL_FILE_PATH`, `REL_DIR_PATH` для AST-фікстур.
