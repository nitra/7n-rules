type: ADR
title: Файлова структура task-вузлів
description: Task-вузли зберігаються як директорії tasks/<node>/task.md з frontmatter і markdown-секціями.
---

**Status:** Accepted
**Date:** 2026-06-07

## Context and Problem Statement

Потрібно зберігати стан кількох повʼязаних робочих одиниць без централізованої БД. У репозиторії ще не було матеріалізованої директорії `tasks/`, але `docs/думка.MD` описував рекурсивний складений ОАГ із файловим сховищем стану.

## Considered Options

- Файлова система: один каталог на вузол `tasks/<node>/` з файлом `task.md`.
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Файлова система: один каталог на вузол `tasks/<node>/` з файлом `task.md`", because це відповідає описаній у `docs/думка.MD` архітектурі, де стан вузла визначається наявністю файлів у директорії.

### Consequences

- Good, because стан `waiting` визначається наявністю `task.md` без `run_*.md` і `outputs_*.md`, без зовнішнього реєстру.
- Good, because `## Done when` дає явні критерії перевірки для кожного вузла.
- Bad, because transcript не містить підтверджених негативних наслідків.
- Neutral, because transcript фіксує лише стартову структуру, а не повну реалізацію runner-а.

## More Information

Створені вузли:

- `tasks/ui-task-view/task.md` — UI-перегляд графу `n-cursor graph`, `budget_sec: 3600`.
- `tasks/coverage-skill-test/task.md` — перевірка `n-coverage-fix` після міграції на `pi`, `budget_sec: 1800`.
- `tasks/skills-orchestrator-migration/task.md` — міграція `npm/skills/` на JS-оркестратор, `budget_sec: 7200`.

Формат `task.md`: YAML frontmatter з `created_at`, `budget_sec`, опціонально `parent`, `deps`; markdown-секції `## Task`, `## Done when`, `## Inputs`.

Запуск вузла: `n-cursor graph run tasks/<name>`. Контекст структури: `docs/думка.MD`.

## Update 2026-06-07

Уточнено, що layout є прямою матеріалізацією специфікації з `docs/думка.MD`: кожен вузол має власну директорію `tasks/<node-name>/`, а `task.md` містить YAML-frontmatter (`created_at`, `budget_sec`, опціонально `parent`, `deps`) і секції `## Task`, `## Done when`, `## Inputs`.

Створені у сесії вузли:

- `tasks/ui-task-view/task.md` — UI-перегляд графу задач.
- `tasks/coverage-skill-test/task.md` — перевірка скілу `n-coverage-fix` після міграції `claude-agent-sdk` → `pi`.
- `tasks/skills-orchestrator-migration/task.md` — міграція `npm/skills/` на JS-оркестратор.

Transcript не фіксує фінальне рішення щодо назви окремого UI-проєкту.
