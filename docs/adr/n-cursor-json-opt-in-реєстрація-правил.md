# Opt-in реєстрація правил у .n-cursor.json

**Status:** Accepted
**Date:** 2026-05-08

## Контекст

Після додавання файлу `npm/mdc/rego.mdc` правило не з'явилось у списку завантажених при запуску `bun ./npm/bin/n-cursor.js`, хоча файл існував у директорії `npm/mdc/`.

## Рішення/Процедура/Факт

Наявність файлу в `npm/mdc/` є необхідною, але не достатньою умовою. Скрипт `n-cursor.js` завантажує лише ті правила, які явно перелічені у полі `"rules"` файлу `.n-cursor.json` у корені проєкту-споживача. Нове правило потрібно додати до масиву `"rules"` у `.n-cursor.json` пакету `@nitra/cursor` (щоб потрапити у дистрибуцію), а потім і в `.n-cursor.json` проєкту-споживача.

## Обґрунтування

Архітектура `@nitra/cursor` побудована на явній opt-in реєстрації правил — завантажується лише задеклароване, що дозволяє проєктам вибірково підключати правила без зайвого шуму.

## Розглянуті альтернативи

Автоматичне сканування всіх `.mdc`-файлів у `npm/mdc/` — неявно відхилено архітектурою (opt-in підхід).

## Зачіпає

`npm/mdc/rego.mdc`, `.n-cursor.json` (package-level та project-level), `npm/bin/n-cursor.js`
