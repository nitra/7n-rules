Найкращий варіант — **Chezmoi** (або просто dotfiles-репо в Git).

## Варіанти

### 1. Chezmoi (рекомендую)
- CLI-інструмент, який керує всіма dotfiles (`~/.config/**`, `~/.zshrc`, тощо)
- Зберігає в Git-репо (`~/.local/share/chezmoi`)
- На новому пристрої: `chezmoi init git@github.com:you/dotfiles.git && chezmoi apply`
- Підтримує шаблони, секрети (1Password/Bitwarden інтеграція), різні профілі (mac/linux)

### 2. Bare git-репо (мінімалістично)
```bash
git init --bare ~/.dotfiles
alias dotfiles='git --git-dir=$HOME/.dotfiles --work-tree=$HOME'
dotfiles add ~/.config/zed/settings.json
dotfiles commit -m "zed settings"
dotfiles remote add origin git@github.com:you/dotfiles.git
dotfiles push
```
Без жодних залежностей, але без шаблонів і шифрування.

### 3. Mackup (тільки macOS)
Синхронізує через iCloud/Dropbox/S3. Простіше, але прив'язаний до macOS і хмари.

Для одного файлу `settings.json` є ще простіший варіант:

**iCloud Drive** (або Dropbox): просто перенеси файл туди і постав symlink.

```bash
# один раз
mv ~/.config/zed/settings.json ~/Library/Mobile\ Documents/com~apple~CloudDocs/dotfiles/zed-settings.json
ln -s ~/Library/Mobile\ Documents/com~apple~CloudDocs/dotfiles/zed-settings.json ~/.config/zed/settings.json
```

На новому Mac — той самий symlink, і все. Жодного Git, жодного GitHub, файл живе в iCloud як звичайний файл.

---

Але якщо конфігів стане більше (Zed, Ghostty, `.zshrc`, ssh-config...) — тоді Git таки найкраще: він дає **історію** («коли я це зламав?») і **push на GitHub = безкоштовний backup + перенос за 1 команду**. Приватне репо безплатне, і це не «занадто» — це 3 команди один раз.

**Якщо зараз тільки Zed** — iCloud symlink, без зайвого. Можна мігрувати в Git пізніше.
