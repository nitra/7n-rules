## ADR Заміна stdin-скану kubescape на тимчасовий файл

## Context and Problem Statement
`kubescape scan -` у версії v4.0.8 не читає stdin — трактує `-` як шлях до файлу й завершується з `fatal: no resources found to scan`. `lint-k8s` (`npm/rules/k8s/lint/lint.mjs`) використовував саме цей підхід через `spawnSync` з `input: manifest` і аргументом `'-'`.

## Considered Options
* Передавати зібраний `kubectl kustomize` маніфест через stdin (`kubescape scan -`)
* Записувати маніфест у тимчасовий файл і передавати шлях (`kubescape scan <tmpfile>`)

## Decision Outcome
Chosen option: "Записувати маніфест у тимчасовий файл і передавати шлях", because `kubescape scan -` в v4.0.8 повертає `fatal: no resources found to scan` — stdin не підтримується; сканування через шлях до файлу (`kubescape scan /tmp/built.yaml`) повертає `All controls passed. No issues found` з exit 0.

### Consequences
* Good, because `lint-k8s` знову проходить для валідних kustomize-проєктів — підтверджено ручним тестом у `mktemp`-директорії з мінімальним Deployment + NetworkPolicy.
* Bad, because тимчасовий файл пишеться на диск при кожному kustomize-каталозі; transcript не містить підтверджених негативних наслідків щодо продуктивності або надійності очищення.
