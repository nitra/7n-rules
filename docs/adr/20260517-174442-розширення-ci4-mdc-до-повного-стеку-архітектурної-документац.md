## ADR Розширення `ci4.mdc` до повного стеку архітектурної документації

## Context and Problem Statement
Правило `npm/rules/ci4/ci4.mdc` (v1.0) описувало лише C4 model як основу архітектурної документації. Команда вирішила розширити правило до повного стеку: arc42 + Diátaxis + ADR + C4 як набір нотацій, доповнивши гібридною моделлю autogen-зон, MkDocs Material з collapsible-блоками та детермінованим валідатором.

## Considered Options
* Розширити `npm/rules/ci4/ci4.mdc` до повного стеку (arc42 + Diátaxis + ADR + autogen + MkDocs Material)
* Додати лише окремі секції без глобального refactoring
* Паралельно оновити `.cursor/rules/n-ci4.mdc`

## Decision Outcome
Chosen option: "Розширити тільки `npm/rules/ci4/ci4.mdc` до повного стеку", because користувач явно обрав скоуп «Розширити до повного стеку» з обмеженням «Тільки npm/rules/ci4/ci4.mdc»; `.cursor/rules/n-ci4.mdc` лишився незміненим.

### Consequences
* Good, because transcript фіксує очікувану користь: єдине правило тепер покриває весь lifecycle архітектурної документації — від ADR-template з YAML frontmatter до LLM-регенерації та детермінованого валідатора.
* Bad, because `.cursor/rules/n-ci4.mdc` не оновлено — між двома файлами правил може виникнути розсинхрон по контенту (transcript не містить підтверджених заходів проти цього).
