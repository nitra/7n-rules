# Інтерактивний генератор change-файлів `ch` у `@7n/7`

**Status:** Accepted
**Date:** 2026-06-02

## Context and Problem Statement

У проєкті `@7n/7` не було зручного способу генерувати `.changes/*.md`-файли (формат `n-cursor release`). Розглядалась альтернатива `@changesets/cli`.

## Considered Options

- Інтерактивний генератор як частина `@7n/7` CLI (гібрид: флаги → без промптів, відсутнє → питається)
- `@changesets/cli` (`changeset add`)
- Окремий standalone-пакет поза `@7n/7`

## Decision Outcome

Chosen option: "Інтерактивний генератор як частина `@7n/7` CLI", because `@changesets/cli` генерує несумісний формат — каталог `.changeset/` замість `.changes/`, `bump:` як маппінг пакет→бамп без поля `section:`, три підтверджені блокери несумісності. Standalone-пакет дублював би формат і споживача.

### Consequences

- Good, because формат `.changes/` залишається єдиним, сумісним із `n-cursor release`; жодного дублювання.
- Good, because гібридний режим (флаги → без TTY; відсутнє → промпт) підтверджений dogfood через `node npm/bin/n-7.js ch`.
- Bad, because `npx @7n/7 ch` запрацює ззовні лише після публікації пакета.

## More Information

`npm/ch.js` — функції `serializeChange`, `parseChArgs`, `collectChange`, `runCh`; DI-ін'єкція `prompt/now/rand/writeFile/cwd`. `npm/tests/ch.test.mjs` — 13 vitest-тестів. `npm/package.json` — bin `"7": "./bin/n-7.js"`. Несумісність `@changesets/cli` підтверджена в transcript: три блокери — каталог, схема frontmatter, відсутність `section:`.
