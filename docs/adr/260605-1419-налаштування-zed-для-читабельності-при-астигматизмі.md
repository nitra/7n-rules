## ADR Налаштування Zed для читабельності при астигматизмі

## Context and Problem Statement
Користувач з астигматизмом скаржився, що букви в Zed зливаються. Zed не має власного CSS-подібного `letter-spacing`, тому стандартний спосіб розріджування — обрати шрифт і підкрутити суміжні параметри.

## Considered Options
* Змінити лише `buffer_font_size` (збільшити існуючий шрифт)
* Перейти на JetBrainsMono Nerd Font Mono + `buffer_line_height` 1.7 + вимкнення лігатур + `buffer_font_weight` 450

## Decision Outcome
Chosen option: "JetBrainsMono Nerd Font Mono + комплекс параметрів читабельності", because шрифт вже був встановлений у системі, широкий по дизайну (замінює відсутній `letter-spacing`), а сукупність `buffer_line_height: {custom: 1.7}`, `buffer_font_features: {liga: false, calt: false}` і `buffer_font_weight: 450` усуває злипання як по горизонталі, так і по вертикалі.

### Consequences
* Good, because transcript фіксує очікувану користь: рядки розріджені, символи (`->`, `=>`) не зливаються в гліфи-лігатури, лінії букв стали товщими.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Змінено `~/.config/zed/settings.json`:
```json
{
"buffer_font_size": 16,
"buffer_font_family": "JetBrainsMono Nerd Font Mono",
"buffer_font_weight": 450,
"buffer_line_height": { "custom": 1.7 },
"buffer_font_features": { "liga": false, "calt": false }
}
```
Zed застосовує зміни live при збереженні файлу.

## ADR Різні pt-розміри для mono та sans у панелі агента Zed

## Context and Problem Statement
Після виставлення `agent_ui_font_size: 16` (рівно як `buffer_font_size`) текст у панелі агента візуально виглядав дрібнішим, ніж код у редакторі. Причина: проза в панелі агента рендерується sans-serif шрифтом, а редактор — моноширинним; при однаковому розмірі в pt sans має меншу видиму висоту гліфів.

## Considered Options
* Виставити `agent_ui_font_size` = 16 (ідентично `buffer_font_size`)
* Виставити `agent_ui_font_size` = 18, `agent_buffer_font_size` = 16

## Decision Outcome
Chosen option: "`agent_ui_font_size: 18`, `agent_buffer_font_size: 16`", because sans-serif шрифт потребує +2pt понад mono, щоб виглядати однаково на екрані; блоки коду в агенті рендеруються тим само моношрифтом, тому їм достатньо 16pt як у редакторі.

### Consequences
* Good, because transcript фіксує очікувану користь: після підняття до 18 проза і блоки коду в панелі агента виглядають порівняного розміру з текстом редактора.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Ключі знайдено через `strings` у бінарному файлі `/Applications/Zed.app/Contents/MacOS/zed`; стандартне значення `agent_buffer_font_size` — 12. Фінальні значення у `~/.config/zed/settings.json`:
```json
{
"agent_ui_font_size": 18,
"agent_buffer_font_size": 16
}
```
