# Перейменування `utils/` → `lib/` для domain-bound модулів у `npm/rules/` та `npm/scripts/`

**Status:** Accepted
**Date:** 2026-05-24

## Context and Problem Statement

У монорепо `npm/rules/<rule>/utils/` (10 каталогів) і `npm/scripts/utils/` містилися два різних типи модулів: generic helpers (прийнятні для окремого npm-пакету) і domain-bound модулі (читають конфіг проєкту, оркеструють правила, інтегрують зовнішні бінарі). Правило `js-lint.mdc` вже визначало семантику — `utils/` для generic, `lib/` для domain-bound — але структура каталогів їй не відповідала.

## Considered Options

* Перейменувати `utils/` → `lib/` лише в `rules/`; `scripts/` чіпати окремим PR
* Все одразу: 10 каталогів `rules/*/utils/` + спліт `scripts/utils/` + переміщення `redis-imports.mjs` у відповідне правило

## Decision Outcome

Chosen option: "Все одразу: rules + scripts split", because користувач явно обрав повний скоуп в одному PR; відкладати `scripts/` означало б продовжувати порушення тієї самої конвенції.

### Consequences

* Good, because `utils/` тепер містить лише справді generic файли (9 файлів у `scripts/utils/`), а domain-логіка живе в `lib/` — відповідно до задекларованої семантики `js-lint.mdc`.
* Good, because `redis-imports.mjs` переміщено у `npm/rules/js-bun-redis/lib/`, симетрично до `bunyan-imports.mjs` та `vue-forbidden-imports.mjs` — принцип co-location з правилом.
* Good, because доданий guard-концерн `npm/rules/js-lint/js/utils_imports.mjs` автоматично ловить майбутні порушення (relative `..`-імпорт з `utils/`).
* Bad, because ~220 рядків імпортів і ~26 внутрішніх шляхів потребували глобального оновлення через `perl -pi`.
* Neutral, because `git mv` зберігає git-історію файлів.

## More Information

Перейменування `rules/*/utils/` → `lib/` (10 правил): `abie`, `changelog`, `docker`, `graphql`, `js-bun-db`, `js-lint`, `js-mssql`, `js-run`, `rust`, `vue`.

Переміщено до `npm/scripts/lib/` (19 файлів): `check-mdc-template-refs.mjs`, `check-reporter.mjs`, `discover-check-rules-from-cursor.mjs`, `discover-checkable-rules.mjs`, `generated-markdown.mjs`, `gha-workflow.mjs`, `inline-template-links.mjs`, `list-rule-ids.mjs`, `load-cursor-config.mjs`, `read-n-cursor-config-lite.mjs`, `resolve-target-files.mjs`, `run-conftest-batch.mjs`, `run-lint-step.mjs`, `run-rule-cli.mjs`, `run-rule.mjs`, `run-standard-lint.mjs`, `run-standard-rule.mjs`, `template.mjs`, `workspaces.mjs`.

Залишилися у `npm/scripts/utils/` (9 generic): `ast-scan-utils.mjs`, `find-package-json-paths.mjs`, `pass.mjs`, `resolve-cmd.mjs`, `test-helpers.mjs`, `walk-cache.mjs`, `walkDir.mjs`, `with-lock.mjs`, `worktree-fingerprint.mjs`.

Guard-концерн `npm/rules/js-lint/js/utils_imports.mjs`: парсить static/dynamic imports через `oxc-parser` (`parseAsync`), падає на відносних `..`-імпортах з `utils/`-файлів (крім `tests/`); запускається як частина `bun run n-cursor fix js-lint`. `js-lint.mdc` доповнено абзацом про автоматичну перевірку.

Зовнішні правки: `npm/scripts/auto-rules.mjs` (рядки 19, 24–25), `npm/scripts/rename-yaml-extensions.mjs:17`. Коміти: `0397e19` (refactor), `2ef7d0f` (fix).
