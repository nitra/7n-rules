## ADR Автоматичний вибір вільної назви у `worktree add`

## Context and Problem Statement

`npx @nitra/cursor worktree add "main-fix" "…"` падав із `fatal: a branch named 'main-fix' already exists`, якщо гілка вже існувала (наприклад, від попереднього запуску). Агент мусив вручну перевіряти вільні назви і вводити їх у команду — замість того, щоб CLI вирішував колізію сам.

## Considered Options

* Перевірити вільну назву вручну в агенті/скрипті й передати у `worktree add`
* Додати до `worktree add` CLI авто-вибір першої вільної назви (`base → base2 → base3 …`)

## Decision Outcome

Chosen option: "Додати до `worktree add` CLI авто-вибір першої вільної назви", because користувач явно вказав: «потрібно спочатку перевіряти, а потім уже агенту пропонувати назву яка спрацює» — тобто вирішення колізії має бути всередині CLI, а не на стороні агента.

### Consequences

* Good, because `worktree add` більше не падає на `fatal: a branch named '…' already exists` — CLI сам обирає `base2`, `base3` і друкує `ℹ️ гілка/worktree "…" уже існує — обрано "…"`.
* Bad, because transcript не містить підтверджених негативних наслідків.
