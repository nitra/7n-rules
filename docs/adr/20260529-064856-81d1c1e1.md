У VSCode аналогічний стек виглядає так:

## Вбудовано в VSCode

- **Markdown Preview** `cmd+shift+v` / `ctrl+k v` (side-by-side) — GFM + таблиці, code blocks; Mermaid **не рендерить** з коробки
- **`cmd+click`** по `[link](file.md)` — перехід без розширень
- **Outline** у Explorer — заголовки поточного файлу
- **`ctrl+p`** — швидке відкриття `.md` файлів

## Розширення-аналоги

| Потреба | VSCode | Zed |
|---|---|---|
| Wiki-links + backlinks + file-navigation | **Foam** або **Dendron** | marksman |
| LSP marksman | **marksman** (розширення в marketplace) | marksman |
| Mermaid у preview | **Markdown Preview Mermaid Support** | вбудовано |
| Admonitions / Material `!!! note` | **Markdown Preview Enhanced** | ❌ теж не рендерить |
| KaTeX/Math | **Markdown Preview Enhanced** або **Math Preview** | з розширенням |

## Практична різниця

**Foam** — найближчий аналог до `marksman` у Zed: `[[wiki-links]]`, graph view, backlinks panel, автокомпліт посилань між `.md`-файлами. Підходить для Obsidian-vault-стилю поверх `docs/`.

**Markdown Preview Enhanced** — якщо потрібен максимум рендерингу без сервера: Mermaid, KaTeX, custom admonitions через CSS, PlantUML (через сервер), навіть деякі VitePress `:::` контейнери через налаштування.

## Головне обмеження — однакове

І в Zed, і у VSCode framework-specific синтаксис (`!!! note`, `::: tip`, MDX-компоненти) **не рендериться** без збирача. Тому правило те ж саме: **portable Markdown + Mermaid** у docs/, а framework-specific — мінімально і тільки там де треба.

## Висновок

VSCode через **Foam + marksman + Markdown Preview Mermaid Support** дає практично ідентичний DX до Zed. Різниця тільки одна: Zed має Mermaid з коробки, VSCode потребує розширення.
