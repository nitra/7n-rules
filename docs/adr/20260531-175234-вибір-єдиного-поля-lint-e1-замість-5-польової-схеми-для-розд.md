## ADR Стратегія discard застарілих worktree: перевірка «чи вміст вже в main»

## Context and Problem Statement
Наприкінці сесії на машині залишилось 6 worktree від різних паралельних сесій (`cursor-ci-bump`, `feat-skill-meta`, `keen-swanson-f7dff6`, `coverage-fix`, `n-coverage-fix`, `stryker-incremental`). Для кожної треба було вирішити — зберегти, змерджити чи відкинути, без ризику втратити незбережену роботу.

## Considered Options
* Перевірити кожну worktree: `git diff origin/main...branch --stat`, `git diff main branch --name-only --diff-filter=A`, наявність незакомічених файлів — тоді discard/merge/keep
* Видалити всі без аналізу
* Зберегти всі без аналізу
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Ітеративний аналіз кожної worktree з порівнянням проти main", because підхід дозволяє виявити, чи унікальна робота гілки вже інтегрована в main іншим шляхом, перш ніж безповоротно видаляти.

### Consequences
* Good, because transcript фіксує очікувану користь: `cursor-ci-bump` виявився застарілим (coverage-classify вже в main), `feat-skill-meta` — аналогічно (worktree-конвенція вже в main); всі 6 видалено без втрати актуальної роботи з основного потоку.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
