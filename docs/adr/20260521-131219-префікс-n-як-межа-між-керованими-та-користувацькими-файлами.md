## ADR Префікс `n-` як межа між керованими та користувацькими файлами в `.cursor/`

## Context and Problem Statement
CLI `@nitra/cursor` синхронізує правила та скіли в `.cursor/rules/` і `.cursor/skills/` проєкту з пакетом, але проєктна директорія також може містити **власні** правила та скіли, що не мають відношення до пакета. Потрібен механізм, щоб CLI знав, які файли він може перезаписувати або видаляти, а які залишати без змін.

## Considered Options
* Префікс `n-` для файлів, керованих CLI; без префікса — власність проєкту
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Префікс `n-` для файлів, керованих CLI", because `npx @nitra/cursor` перезаписує лише `n-*.mdc` і `n-*/SKILL.md` (за списком у `.n-cursor.json`), видаляє лише `n-*`-orphan'и; файли без префікса (наприклад `conftest.mdc`, `dev-dep.mdc`, `mdc-check/`) CLI не чіпає.

### Consequences
* Good, because ручні правила (`*.mdc` без `n-`) і власні скіли (директорії без `n-`) зберігаються між запусками `npx @nitra/cursor` і автоматично потрапляють в індекси `CLAUDE.md` / `AGENTS.md` через `listProjectRulesMdcFiles`, яка сканує **всі** `*.mdc` без фільтрації за префіксом.
* Bad, because будь-які ручні правки всередині `n-*.mdc` або `n-*/SKILL.md` **перезаписуються** при наступному синку — канонічне джерело для них лише в `npm/rules/` і `npm/skills/` пакета.

## More Information
- `npm/bin/n-cursor.js:489–496` — `listProjectRulesMdcFiles` сканує весь `RULES_DIR` без фільтра за префіксом.
- `CLAUDE.md` і `AGENTS.md` **повністю** перегенеруються кожним запуском `npx @nitra/cursor`; ручні правки в цих файлах не зберігаються.
- Шаблон статичного тексту для `AGENTS.md` — `npm/AGENTS.template.md` у пакеті.
- Конфігурація керованих id правил і скілів — `.n-cursor.json` у споживчому проєкті, секції `rules` і `skills`.
