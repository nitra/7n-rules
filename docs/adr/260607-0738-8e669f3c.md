## Що реалізовано

`npm/skills/docgen/js/docgen-gen.mjs` тепер автоматично вибирає backend залежно від оточення:

```
checkOllama() → true   →  ollamaOrchestrated / ollamaOneShot  (швидко ~50s/файл)
checkOllama() → false  →  piOrchestrated / piOneShot          (universal fallback)
```

**Ключові зміни:**
- `checkOllama()` — `fetch(OLLAMA_HOST/api/tags, timeout=3s)`, кешується на процес
- `OLLAMA_HOST` env var (default: `http://localhost:11434`) — сумісний з kubeai в K8s
- `ollamaModelName()` — strips `ollama/` prefix: `ollama/gemma3:4b` → `gemma3:4b` для HTTP API
- CLI тепер показує `[tier1 ollama-orchestrated]` або `[tier1 pi-orchestrated]`

**Сценарії:**

| Середовище | `checkOllama()` | Backend |
|---|---|---|
| Локально (ollama є) | true | ollama HTTP orchestrated |
| CI / без GPU | false | pi orchestrated → cloud |
| K8s + kubeai | true (`OLLAMA_HOST=http://kubeai.svc:11434`) | ollama HTTP orchestrated |
| K8s без GPU | false | pi orchestrated → cloud |
