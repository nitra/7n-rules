## ADR Додавання мінімальної підтримки `.rs` у `doc-files`

## Context and Problem Statement
Скіл `doc-files` виявляв і документував лише `.js`, `.mjs`, `.ts`, `.vue`, `.py` файли. У проєкті `/Users/vitaliytv/www/nitra/task` є Rust-код у `app/src-tauri/`, і потрібно було генерувати документацію для цих файлів тим самим механізмом.

## Considered Options
* Мінімальна підтримка: додати `.rs` до `SOURCE_EXTENSIONS` і використовувати вже існуючий `oneShotDoc` path (той самий що Vue/Python — LLM отримує весь файл без AST-парсингу)
* Повноцінна підтримка: новий `units-rs.mjs` з парсингом `pub fn`/`pub struct`/`impl` через `tree-sitter-rust` або regex і додатковий диспатч у `units.mjs`

## Decision Outcome
Chosen option: "Мінімальна підтримка через `oneShotDoc`", because достатньо одного рядка у `docgen-scan.mjs` і зміни в `docgen-ignore.mjs`; existing `oneShotDoc` branch вже обробляє unsupported-розширення без додаткового коду. Повна AST-підтримка відкладена до появи значної кількості `.rs` файлів у проєкті.

### Consequences
* Good, because transcript фіксує очікувану користь: 3 Rust-файли (`build.rs`, `lib.rs`, `main.rs`) успішно задокументовано, `✓ OK: 3 ✗ Err: 0`.
* Bad, because документація для Rust генерується без розбивки на юніти — LLM бачить весь файл як єдиний блок; для великих `*.rs` файлів із багатьма `pub fn`/`impl` якість може бути гіршою ніж для JS/TS.

## More Information

- `npm/skills/doc-files/js/docgen-ignore.mjs:7` — `**/target/**` додано до `DOCGEN_IGNORE_GLOBS` (виключає Cargo build-артефакти — сотні авто-генерованих `.rs` у `target/`)
