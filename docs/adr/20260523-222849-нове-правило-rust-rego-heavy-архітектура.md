# Нове правило `rust` — rego-heavy архітектура

**Status:** Accepted
**Date:** 2026-05-23

## Context and Problem Statement

Проєкти на Rust (Tauri, stand-alone CLI, workspace-крейти) у монорепо не мали стандартизованих перевірок форматера (`rustfmt`) та лінтера (`cargo clippy`), канонічного скрипту `lint-rust`, CI workflow і VSCode-розширень. Постала необхідність додати правило `rust` до пакету `@nitra/cursor` — аналог `js-lint`/`style-lint`.

## Considered Options

- A. Rego-heavy — чотири rego-policy-пакети (`package_json`, `vscode_extensions`, `vscode_settings`, `lint_rust_yml`); `js/tooling/check.mjs` — gating і orchestration через `runConftestBatch`; за зразком `js-lint`/`style-lint`
- B. JS-heavy — лише один rego-пакет `vscode_extensions`; `package.json` і workflow перевіряються JS-логікою (як у `tauri`)
- C. Гібрид — rego для `package_json` і `vscode_extensions`, CI-workflow — JS-логікою

## Decision Outcome

Chosen option: "A. Rego-heavy (за зразком js-lint/style-lint)", because дрейф від конвенції при JS-heavy підході створить технічний борг при майбутніх розширеннях (наприклад, `cargo deny`, MSRV); паритетність з `js-lint`/`style-lint` полегшує обслуговування.

### Consequences

- Good, because структура правила повністю відповідає патерну `style-lint`; всі документи перевіряються окремими rego-пакетами з тестами і template-сніпетами.
- Good, because ідентифікатор — `rust` (паритет з `vue`, `tauri`, `capacitor`), а не `rust-lint` — коротко і без дублювання суфікса.
- Good, because `rust-lang.rust-analyzer` перенесено з `tauri.policy.vscode_extensions` до `rust.policy.vscode_extensions`; `tauri` вимагає лише `tauri-apps.tauri-vscode`; Tauri-проєкт отримує `rust-analyzer` транзитивно через auto-trigger за `Cargo.toml`.
- Good, because канонічні `rustfmt.toml`/`clippy.toml` не вимагаються — defaults достатні; авто-виправка файлів у `fix.mjs` відсутня (нуль прецедентів у проєкті).
- Bad, because більше boilerplate-файлів, ніж у мінімальному підході (варіант B).

## Update 2026-05-23

### Ідентифікатор правила: `rust` (не `rust-lint`)

Обрано парність з `vue`, `tauri`, `capacitor` — правила, що описують технологію повністю, а не лише її lint-аспект. Папка `npm/rules/rust/`, `.cursor/rules/n-rust.mdc`, `npm/scripts/auto-rules.mjs` (`'rust'` в `AUTO_RULE_ORDER`). Спека: `docs/superpowers/specs/2026-05-23-rust-rule-design.md`.

### Тригер auto-правила: наявність `Cargo.toml`

Обрано `Cargo.toml` у корені або workspace як FS-маркер (аналог `capacitor.config.json`). Реалізація: fact `hasCargoToml` у `gatherProjectFacts` в `auto-rules.mjs` + запис `{ enabled: facts.hasCargoToml, id: 'rust' }`. Файл: `npm/rules/rust/js/applies/check.mjs`.

### Канонічна команда `lint-rust`

`cargo fmt --all && cargo clippy --fix --allow-staged --allow-dirty --all-targets --all-features && cargo clippy --all-targets --all-features -- -D warnings` — тристадійний підхід для локального dev (`--allow-staged --allow-dirty` для брудного дерева). CI-workflow використовує `--check` без `--fix` (навмисна відмінність). Rego-policy `rust.package_json.deny` перевіряє наявність підрядків `cargo fmt`, `cargo clippy --fix`, `cargo clippy` + `-D warnings`. Template: `npm/rules/rust/policy/package_json/template/package.json.contains.json`.

### CI-workflow: `dtolnay/rust-toolchain@stable` + `Swatinem/rust-cache@v2`

`dtolnay/rust-toolchain` — підтримуваний world-standard action; `Swatinem/rust-cache@v2` кешує `target/` і `registry`, оскільки `dtolnay/rust-toolchain` самостійно не кешує їх. Rego `rust.lint_rust_yml` перевіряє послідовність steps: `actions/checkout@v6` → `dtolnay/rust-toolchain@stable` (з `components: rustfmt, clippy`) → `Swatinem/rust-cache@v2` → fmt-step → clippy-step. Template: `npm/rules/rust/policy/lint_rust_yml/template/lint-rust.yml.snippet.yml`.

### Рефакторинг `tauri`: `rust-lang.rust-analyzer` перенесено у правило `rust`

Tauri-проєкт завжди містить `src-tauri/Cargo.toml`, тому правила `rust` і `tauri` активні одночасно. `tauri.policy.vscode_extensions` тепер вимагає лише `tauri-apps.tauri-vscode`; `rust.policy.vscode_extensions` — єдине джерело правди для `rust-lang.rust-analyzer` + `tamasfe.even-better-toml`. Для Tauri-проєктів без `Cargo.toml` у FS-walk (нетиповий сценарій) `rust-analyzer` більше не перевірятиметься — але маркер `rust` визначений саме через `Cargo.toml`. Змінені файли: `npm/rules/tauri/policy/vscode_extensions/vscode_extensions.rego`, `vscode_extensions_test.rego`, `npm/rules/tauri/tauri.mdc`.

## Update 2026-05-24

**Ідентифікатор правила: `rust` замість `rust-lint`.** Правило іменується `rust` (а не `rust-lint`), бо воно описує технологічний стек (як `abie`, `tauri`, `capacitor`), а не лише лінтинг; `rust-lint` — надмірна специфікація. `AUTO_RULE_ORDER` розміщено між `rego` і `security`; `auto.md` = «якщо в проєкті є хоч один `Cargo.toml`».

## Update 2026-05-24

**Реалізація правила `rust` — ключові рішення:**

- **`required:true` + `missingMessage` у `target.json`**: замість окремого `js/tooling.mjs` з `existsSync`-перевірками три `target.json` (для `package_json`, `vscode_extensions`, `lint_rust_yml`) отримали `"required": true` + `"missingMessage"`. Логіка обробки вже реалізована в `runPolicyConcern` (`npm/scripts/utils/run-rule.mjs`) — дублювання усунено.

- **Flat concern layout як першопрохідець**: правило `rust` будувалось одразу у flat-структурі (`js/applies.mjs`, `js/tests/applies.test.mjs`), оскільки міграція вже була виконана і `discover-checkable-rules.mjs` v1.13.90+ шукає `rules/<id>/js/<concern>.mjs`.

- **Вилучення Rust-розширень з `tauri.vscode_extensions`**: `rust-lang.rust-analyzer` і `tamasfe.even-better-toml` перенесено у `rust/policy/vscode_extensions`; `tauri` залишає лише `tauri-apps.tauri-vscode`. Tauri-проєкт завжди має `src-tauri/Cargo.toml`, що автоматично активує правило `rust` — дублювання вимог усунено.

- **Авто-детект `hasCargoToml`**: `auto-rules.mjs` використовує факт `hasCargoToml` — рекурсивний walker з пропуском `node_modules`, `.git`, `.next`, `.turbo`. Та сама утиліта `npm/rules/rust/utils/has-cargo-toml.mjs` використовується в `js/applies.mjs` — єдиний алгоритм для auto-detect і applies-gate. Тести: 2 нових сценарії з `withTmpCwd`/`ensureDir`/`writeFile` у `npm/scripts/tests/auto-rules.test.mjs`.

Змінені файли: `npm/rules/rust/policy/*/target.json`, `npm/rules/tauri/policy/vscode_extensions/vscode_extensions.rego`, `npm/rules/tauri/tauri.mdc`, `npm/scripts/auto-rules.mjs`. 10 комітів запушено на `origin/main`.
