# Нове правило `rust` — rego-heavy архітектура

**Status:** Accepted
**Date:** 2026-05-23

## Context and Problem Statement

Проєкти на Rust (Tauri, stand-alone CLI, workspace-крейти) у монорепо не мали стандартизованих перевірок форматера (`rustfmt`) та лінтера (`cargo clippy`), канонічного скрипту `lint-rust`, CI workflow і VSCode-розширень. Постала необхідність додати правило `rust` до пакету `@nitra/cursor` — аналог `js-lint`/`style-lint`.

## Considered Options

- A. Rego-heavy — чотири rego-policy-пакети (`package_json`, `vscode_extensions`, `vscode_settings`, `lint_rust_yml`); `js/tooling/check.mjs` — gating і orchestration через `runConftestBatch`; за зразком `js-lint`/`style-lint`
- B. JS-heavy — лише один rego-пакет `vscode_extensions`; `package.json` і workflow перевіряються JS-логікою (як у `tauri`)
- C. Гібрид — rego для `package_json` і `vscode_extensions`, CI-workflow — JS-логікою

## Decision Outcome

Chosen option: "A. Rego-heavy (за зразком js-lint/style-lint)", because дрейф від конвенції при JS-heavy підході створить технічний борг при майбутніх розширеннях (наприклад, `cargo deny`, MSRV); паритетність з `js-lint`/`style-lint` полегшує обслуговування.

### Consequences

- Good, because структура правила повністю відповідає патерну `style-lint`; всі документи перевіряються окремими rego-пакетами з тестами і template-сніпетами.
- Good, because ідентифікатор — `rust` (паритет з `vue`, `tauri`, `capacitor`), а не `rust-lint` — коротко і без дублювання суфікса.
- Good, because `rust-lang.rust-analyzer` перенесено з `tauri.policy.vscode_extensions` до `rust.policy.vscode_extensions`; `tauri` вимагає лише `tauri-apps.tauri-vscode`; Tauri-проєкт отримує `rust-analyzer` транзитивно через auto-trigger за `Cargo.toml`.
- Good, because канонічні `rustfmt.toml`/`clippy.toml` не вимагаються — defaults достатні; авто-виправка файлів у `fix.mjs` відсутня (нуль прецедентів у проєкті).
- Bad, because більше boilerplate-файлів, ніж у мінімальному підході (варіант B).

## More Information

- Структура: `npm/rules/rust/{rust.mdc,auto.md,fix.mjs,js/applies/check.mjs,js/tooling/check.mjs,policy/package_json/,policy/vscode_extensions/,policy/vscode_settings/,policy/lint_rust_yml/}`
- Ідентифікатор: `rust`; auto-trigger: `якщо в проєкті є файл Cargo.toml`; fact `hasCargoToml` у `npm/scripts/auto-rules.mjs`; `'rust'` у `AUTO_RULE_ORDER`
- Канонічна команда `lint-rust` (локально): `cargo fmt --all && cargo clippy --fix --allow-staged --allow-dirty --all-targets --all-features && cargo clippy --all-targets --all-features -- -D warnings`
- CI (без `--fix`): `cargo fmt --all -- --check` + `cargo clippy --all-targets --all-features -- -D warnings`
- CI steps: `actions/checkout@v6` → `dtolnay/rust-toolchain@stable` (components: `rustfmt, clippy`) → `Swatinem/rust-cache@v2` → lint steps
- VSCode extensions: `rust-lang.rust-analyzer`, `tamasfe.even-better-toml`
- VSCode settings: `[rust].editor.defaultFormatter: rust-lang.rust-analyzer`, `[rust].editor.formatOnSave: true`, `rust-analyzer.check.command: clippy`
- Зміни `tauri`: `npm/rules/tauri/policy/vscode_extensions/vscode_extensions.rego` — прибрати `rust-lang.rust-analyzer`; `tauri.mdc` — додати примітку про делегування Rust-вимог правилу `rust`
- Синхронізація: `npx @nitra/cursor` копіює `npm/rules/rust/rust.mdc` → `.cursor/rules/n-rust.mdc`; рядок `@.cursor/rules/n-rust.mdc` у `cursor/CLAUDE.md`
- Spec: `docs/superpowers/specs/2026-05-23-rust-rule-design.md` (commit `2730c02`)
- Plan: `.claude/plans/rust-rule.md`

## Update 2026-05-23

Деталь реєстрації: fact `hasCargoToml` вже виявляється у `npm/scripts/auto-rules.mjs` через `walkDir` (рядки 300, 308) без нового коду — реєстрація зводиться до двох рядків: `'rust'` у `AUTO_RULE_ORDER` (між `'rego'` і `'security'`) та `{ enabled: facts.hasCargoToml, id: 'rust' }` у `enabledRules`. `auto.md`: `якщо в проєкті є файл Cargo.toml`.

## Update 2026-05-23

Підтверджено під час реалізації: `Swatinem/rust-cache@v2` кешує `target/` та registry і є окремим кроком між `dtolnay/rust-toolchain@stable` і lint steps. `fix.mjs` правила `rust` — виключно entry-point для `runStandardRule` без авто-виправки файлів (відповідає нульовому прецеденту autofix у проєкті). Флаги `--allow-staged --allow-dirty` у локальному `cargo clippy --fix` потребують пояснення у `rust.mdc`, щоб не виглядати підозріло.
