## ADR gemma3:4b потребує --no-tools для text-mode у pi CLI

## Context and Problem Statement

Під час побудови tool-free harness виявлено, що `pi` CLI завжди додає `tools` поле в API-запит навіть при `--mode text`. `gemma3:4b` (Ollama) повертає `400 does not support tools` і відмовляється виконати запит. `gemma4:4b` приймає той самий формат без помилки.

## Considered Options

* Додати `--no-tools` прапор до виклику `pi` для `gemma3:4b`
* Замінити `gemma3:4b` на `gemma4:4b` (підтримує tools API формат)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Додати `--no-tools` прапор до виклику `pi` для `gemma3:4b`", because підтверджено прямим тестом: `pi -p '...' --model gemma3:4b --no-session --no-tools --mode text` повертає коректний JSON; `gemma4:4b` як fallback також допустимий.

### Consequences

* Good, because transcript фіксує очікувану користь: `gemma3:4b` з `--no-tools` коректно відповів `{"recommendations":["arr.marksman","dbaeumer.vscode-eslint","Vue.volar","tsandall.opa"]}` на тестовому промпті.
* Bad, because навіть з коректним прапором warm inference для реальних промптів (~500 chars, 9-entry JSON) займає 78–90 s на M2 8GB; локальна модель не придатна для production config-фіксів.

## More Information

- Команда: `pi -p "$PROMPT" --provider ollama --model gemma3:4b --no-session --no-tools --mode text`
- Тест: промпт 332 chars, 3-entry JSON — відповідь коректна, total time 3:50 (230 s, переважно cold-start після вивантаження `gemma4:4b`)
- Warm prewarm (32–40 s), після якого inference для 9-entry JSON file все одно timeout (90 s)
- `gemma4:4b` у `--mode text` без `--no-tools`: приймає запит, але також timeout ~90 s для реальних промптів
- Файл harness: `benchmarks/tool-free/run.mjs` — константи `MODEL = 'gemma3:4b'`, `MODEL_NO_TOOLS = true`
