## ADR Декларативна міграція правил через `target.json` + concern-based JS-розпил

**Контекст:** Правила `@nitra/cursor` мали монолітні `rules/<id>/js/check.mjs`, які всередині викликали `runConftestBatch` з hardcoded `namespace`. Таблиця `TARGETS` у `lint-conftest.mjs` (~280 рядків) дублювала інформацію про те, які файли фідити в яку rego-полісі. Масштабування потребувало редагувати три місця одночасно: JS-check, lint-conftest таблицю і rego.

**Рішення/Процедура/Факт:** Впроваджено `rules/<id>/policy/<concern>/target.json`-маніфест поруч із кожним `.rego`. CLI `check <id>` через `discover-checkable-rules.mjs` + `run-rule.mjs` + `resolve-target-files.mjs` сам резолвить файли і запускає `conftest`. Паралельно JS-монолітні `js/check.mjs` розпилюються на `js/<concern>/check.mjs`-концерни — наприклад, `abie/js/check.mjs` (1152 рядки) → 6 концернів + 8 shared utils. Shared стан (`findK8sYamlFiles`, `collectDeploymentDirs`) виноситься у `utils/k8s-tree.mjs` з module-level кешем. `lint-conftest.mjs` переписано на `discoverCheckableRules` замість hardcoded `TARGETS`. Мігровано 22+ правила (46 нових `target.json`). Виправлено два системних баги: (1) `namespace` для kebab-id правил не конвертувався у snake — rego відкидав матч; (2) picomatch масив із `!neg`-елементами повертав `true` майже всюди — позитивні/негативні глоби тепер розділяються і застосовуються послідовно.

**Обґрунтування:** Одне джерело правди для таргетів: `target.json` живе поруч із `.rego`, тому при додаванні нової полісі редагується один файл. Concern-based JS-структура зменшує зв'язність і дозволяє CLI застосовувати applies-гейт на рівні rule (один `js/applies/check.mjs`) або concern (окремий `js/<concern>/check.mjs` зі своїм `applies()`). `YAGNI` щодо `conftest.combine`: поле прибрано зі schema і коду — додамо при першому юзкейсі.

**Розглянуті альтернативи:** (1) Залишити hardcoded `TARGETS` у `lint-conftest.mjs` і не вводити `target.json` — відхилено: два місця для однієї зміни, немає self-describing структури. (2) Один великий `target.json` на рівні `rules/<id>/` замість per-concern — відхилено: неможливо мати різні `files` і `missingMessage` для різних полісі одного правила. (3) `conftest.combine: true` для збирання всіх полісі в один виклик conftest — відкладено як YAGNI.

**Зачіпає:** `npm/scripts/utils/resolve-target-files.mjs`, `discover-checkable-rules.mjs`, `run-rule.mjs`; `npm/scripts/lint-conftest.mjs`; `npm/rules/abie/js/` (6 нових концернів), `npm/rules/abie/utils/` (8 модулів); `npm/rules/<22 правил>/policy/<concern>/target.json` (46 нових файлів); `npm/schemas/target.json` (JSON Schema); `npm/bin/n-cursor.js`; `npm/package.json` (picomatch, version bump 1.10→1.11); `npm/CHANGELOG.md`.
