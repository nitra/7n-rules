## ADR CLI-скан + агент-генерація для `n-docgen`

## Context and Problem Statement
Користувач запитав, як запустити генерацію документації через `npx`. Відповідь виявила, що CLI надає лише сканери, а запис `.md`-файлів виконує Claude-агент через skill `/n-docgen`.

## Considered Options
* CLI виконує лише `scan`/`modules` (data extraction), агент виконує генерацію
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "CLI = сканер, агент = генератор", because `npx @nitra/cursor docgen scan` та `npx @nitra/cursor docgen modules` повертають JSON-зрізи без токенів Claude; сам диспатч субагентів і запис `.md`-файлів виконується лише через skill `/n-docgen` у Claude-агенті.

### Consequences
* Good, because CLI-сканування не витрачає токенів Claude і дає детермінований JSON-список файлів/модулів.
* Bad, because transcript не містить підтверджених негативних наслідків.
