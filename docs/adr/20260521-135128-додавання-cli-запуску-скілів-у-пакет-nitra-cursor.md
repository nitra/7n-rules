## ADR Додавання CLI запуску скілів у пакет `@nitra/cursor`

## Context and Problem Statement
Скіли `@nitra/cursor` (`npm/skills/<id>/SKILL.md`) були доступні лише у проєктах, які синхронізували правила через `n-cursor sync`. Запустити скіл у довільному репозиторії без встановлення пакета як `devDependency` не було можливості.

## Considered Options
* Додати CLI безпосередньо до існуючого пакета `@nitra/cursor` (підкоманда `skill` у `n-cursor`)
* Окремий пакет `@nitra/skills` (описаний у запиті користувача як аналог)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Додати CLI безпосередньо до існуючого пакета `@nitra/cursor`", because скіли вже лежать у `npm/skills/<id>/SKILL.md` цього пакета, тому окремий пакет — зайвий індirection; `npx @nitra/cursor skill …` дозволяє `npx`-запуск без встановлення.

### Consequences
* Good, because transcript фіксує очікувану користь: запуск `npx @nitra/cursor skill taze` з довільного репо без `devDependencies` і без `n-cursor sync`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Новий модуль: `npm/scripts/skills-cli.mjs` (функції `listSkillIds`, `normalizeSkillId`, `buildSkillPrompt`, `runSkillsCli`)
- Підкоманда `skill` у `npm/bin/n-cursor.js`
- Тести: `npm/scripts/skills-cli.test.mjs`
- Скіли читаються з `npm/skills/<id>/SKILL.md` установленого пакета; контекст CWD: `package.json`, `tsconfig.json`, `.n-cursor.json`

## ADR Спрощений UX підкоманди `skill` до чотирьох форматів

## Context and Problem Statement
Перша реалізація CLI мала п'ять форматів (`skill list`, `skill prompt <id>`, `skill <id>`, `skill claude <id>`, `skill cursor <id>`) і два додаткових bin (`n-skills`, `n-claude`). Паралельно була додана підкоманда `claude` з «інвертованим» порядком аргументів. Користувач попросив спростити до чотирьох конкретних форматів.

## Considered Options
* Зберегти всі формати (`prompt`, `claude-first` bin `n-claude`, `n-skills`)
* Спростити до чотирьох форматів: `skill list`, `skill <id> [task]`, `skill cursor <id> [task]`, `skill claude <id> [task]`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Спростити до чотирьох форматів", because користувач явно визначив бажані формати і попросив прибрати зайве; `skill prompt` замінено на `skill <id>` (коротша форма).

### Consequences
* Good, because transcript фіксує очікувану користь: прибрано `skill prompt`, `skill claude` як окрему підкоманду `n-cursor`, bin `n-skills` і bin `n-claude`; залишилась єдина точка входу — підкоманда `skill` у `n-cursor`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Видалено: `npm/bin/n-skills.js`, `npm/bin/n-claude.js`
- Видалено: функція `runClaudeFirstSkillsCli`, підкоманда `claude` у `n-cursor.js`
- Видалено: `skill prompt` як окрема підкоманда (замінена на `skill <id>`)
- `npm/package.json` `bin`: залишено лише `n-cursor`
- Версія пакета: `1.13.72`

## ADR Заборона bin з іменем `claude` у `@nitra/cursor`

## Context and Problem Statement
Під час розробки «claude-first» UX виникло питання, чи можна додати bin з іменем `claude` до пакета `@nitra/cursor`, щоб команда в терміналі виглядала як `claude taze`.

## Considered Options
* Додати bin `claude` у `package.json`
* Не додавати, запропонувати alias у `~/.zshrc`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Не додавати bin `claude`", because ім'я `claude` збігається з Anthropic Claude Code CLI; після `npm i -g` обидва бінарники боролися б за одну команду в `PATH`.

### Consequences
* Good, because transcript фіксує очікувану користь: відсутність конфлікту з Anthropic Claude Code CLI.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Альтернатива для локального використання: alias у `~/.zshrc` (`claude() { npx -p @nitra/cursor n-claude "$@"; }`)
- Bin `n-claude` також видалено під час фінального спрощення (версія `1.13.72`)
