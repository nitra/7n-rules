**Status:** Accepted
**Date:** 2026-06-14

## Context and Problem Statement

Логіка `doc-files` жила у `npm/skills/doc-files/js/`, але skill не має rule-слотів для policy-каналу, GA-workflow і `package.json`-перевірок через Rego. Через це `lint-doc` лишався підкомандою без повного CI-енфорсу застарілості документації.

Потрібно було зберегти агентний workflow для генерації документації, але перенести перевірювану логіку в rule-структуру, де доступні `meta.json`, `.mdc`, `fix.mjs`, `lint/lint.mjs` і агрегаторний `js/lint.mjs`.

## Considered Options

* Перейменувати правило на `doc-files` і лишити тонкий скіл.
* Зробити `doc-files` виключно скілом і прибрати правило `doc`.
* Реалізувати rego-policy і GA-workflow одразу в межах цього завдання.
* Свідомо відкласти rego-policy і GA-workflow до стабілізації lint-consolidation.
* Нові top-level команди `lint-doc-files` / `fix-doc-files` із deprecated-аліасом `doc-files <sub>`.
* Інструментувати `callLlm` через module-level лічильник для пер-файлового LLM-таймінгу.
* Використовувати `N_LOCAL_MIN_MODEL` як стандартний env для мінімального локального tier.
* Використовувати `N_CURSOR_DOCGEN_MODEL` як docgen-specific env.

## Decision Outcome

Chosen option: "Перейменувати правило на `doc-files`, лишити тонкий скіл і використовувати `N_LOCAL_MIN_MODEL`", because лише правило має потрібний policy-канал для майбутнього CI-енфорсу, а тонкий скіл зберігає агентний workflow; стандартний `N_LOCAL_MIN_MODEL` уже підтримується `resolveModel('min')` і працює для всіх skill-ів мінімального tier.

### Consequences

* Good, because правило отримує повний набір слотів: `meta.json`, `.mdc`-канон, `fix.mjs`, `lint/lint.mjs` і `js/lint.mjs` для агрегатора.
* Good, because `lint-doc-files` / `fix-doc-files` дають явні top-level CLI-команди з окремими lock-ключами і hook-маркерами.
* Good, because пер-файловий таймінг показує частку LLM і оркестрації, наприклад `53.8s (llm 53.8s/5 calls, orch 0.0s)`.
* Good, because `N_LOCAL_MIN_MODEL=omlx/gemma-4-e2b-it-4bit` був перевірений через `env -i` без `N_CURSOR_DOCGEN_MODEL` і генерація завершилась успішно.
* Bad, because rego-policy і GA-workflow для `doc-files` відкладені, тому споживачі ще не мають автоматичного енфорсу наявності `lint-doc-files` у `package.json` і workflows.
* Bad, because dogfooding gap: опублікований `node_modules/@nitra/cursor` у цьому ж репо ще міг не містити нових команд, тому локальний `package.json` тимчасово мусив вказувати на локальне джерело.
* Neutral, because transcript фіксує виправлення прихованого бага зі стороннім `*/` у JSDoc `js/lint.mjs`, але не подає його як окреме архітектурне рішення.
