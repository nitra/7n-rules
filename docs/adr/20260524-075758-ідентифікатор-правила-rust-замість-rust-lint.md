## ADR Ідентифікатор правила — `rust` замість `rust-lint`

## Context and Problem Statement
При проєктуванні нового правила для Rust-проєктів виникло питання: як іменувати правило — `rust` або `rust-lint`, враховуючи конвенцію інших правил (`js-lint`, `style-lint`).

## Considered Options
* `rust-lint` — аналогія з `js-lint`, `style-lint`
* `rust` — коротший ідентифікатор, що описує технологію, а не дію

## Decision Outcome
Chosen option: "`rust`", because правило описує технологічний стек (як `abie`, `tauri`, `capacitor`), а не тільки лінтинг; `rust-lint` — надмірна специфікація.

### Consequences
* Good, because `AUTO_RULE_ORDER` і discovery зберігають лаконічність; `auto.md` стає `якщо є Cargo.toml` без прив'язки до типу перевірки.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- `npm/rules/rust/auto.md` — маркер `Cargo.toml`
- `npm/scripts/auto-rules.mjs`: `AUTO_RULE_ORDER` між `rego` і `security`
