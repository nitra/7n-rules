# COVERAGE.md: пропуск рядка «Разом» при єдиному провайдері покриття

**Status:** Accepted
**Date:** 2026-05-30

## Context and Problem Statement

Коли в проєкті активний лише один провайдер покриття (наприклад, тільки `JS`), таблиця в `COVERAGE.md` містила два ідентичні рядки: рядок провайдера та рядок `**Разом**`. Підсумковий рядок у такому разі дублює дані і не несе додаткової інформації.

## Considered Options

* Завжди додавати рядок `**Разом**`, незалежно від кількості провайдерів (попередня поведінка)
* Додавати `**Разом**` тільки коли кількість рядків провайдерів ≥ 2

## Decision Outcome

Chosen option: "Додавати `**Разом**` тільки коли рядків ≥ 2", because підсумок має цінність лише при агрегації кількох провайдерів; єдиний рядок провайдера і є підсумком.

### Consequences

* Good, because `COVERAGE.md` з одним провайдером більше не містить ідентичних рядків.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- `npm/rules/test/coverage/coverage.mjs`: пропуск виклику `buildTotalsRow(rows)` коли `rows.length < 2`.
- `npm/rules/test/coverage/tests/coverage.test.mjs`: тест «агрегує дані одного провайдера» перевіряє відсутність рядка `**Разом**`; прибрано `expect(md).toContain('**Разом**')` з utf8-тесту.
- `COVERAGE.md` у корені репо вручну очищено від дублікату до наступного прогону `n-cursor coverage`.
- Зміна записана як patch bump: `npm/.changes/1780122298610-380204.md`.
- Всі 57 тестів `coverage.test.mjs` пройшли після змін.
