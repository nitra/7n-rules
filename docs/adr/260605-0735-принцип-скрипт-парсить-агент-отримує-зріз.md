## ADR Принцип «скрипт парсить — агент отримує зріз»

## Context and Problem Statement
Кілька скілів (`coverage-fix`, `taze`, `start-check`) інструктували LLM-агента самостійно читати великі артефакти (COVERAGE.md ~2.76 МБ ≈ 700K токенів, `package.json.taze-bak`, glob-воркспейси) і виконувати детерміновану роботу (парсинг JSON, semver-класифікацію, розгортання glob), яку можна виконати у скрипті за 0 LLM-токенів.

## Considered Options
* Залишити детермінований парсинг у SKILL.md-інструкціях для агента
* Виносити детерміновану роботу в read-only CLI-підкоманди (`n-cursor <skill> <verb>`); агент отримує компактний `index` або `slice`

## Decision Outcome
Chosen option: "CLI-екстрактор: read-only `n-cursor <skill> index|slice`", because аналіз реальних метрик показав ≈350× різницю токенів (7.4 КБ проти 2.76 МБ для `coverage-fix`), а детермінована класифікація (regex, semver, glob) не є когнітивною роботою LLM. Правило зафіксовано в `.cursor/rules/scripts.mdc` v1.13 як «🔴 ВИСОКИЙ ПРІОРИТЕТ».

### Consequences
* Good, because transcript фіксує очікувану користь: оркестратор `coverage-fix` читає 2K токенів замість 700K; субагент отримує `slice` ~10 КБ на один файл.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- `.cursor/rules/scripts.mdc` v1.13 — нова секція «Детермінований парсинг у скіла: у CLI, не в LLM» із переліком red-flags і еталонів
- Пам'ять `feedback_script_parses_agent_gets_slice.md` + `project_skill_extractor_refactor_backlog.md`
- Черга: done — `coverage-fix`, `taze`, `start-check`; pending — `lint`, `fix`, `llm-patch`, `docgen`, `publish-telegram`

## ADR `n-cursor taze diff` — semver-класифікація major без LLM

## Context and Problem Statement
Крок 3 скіла `n-taze` казав агенту порівняти `package.json.taze-bak` і `package.json` (по всіх воркспейсах) та визначити, для яких пакетів змінилася «перша значуща цифра semver». Це чиста JSON-операція з детермінованою логікою (caret-семантика: `1.x→2.x`, `0.4.x→0.5.x`, `0.0.3→0.0.4`).

## Considered Options
* Агент вручну diff-ить бекап і класифікує major-зміни
* `n-cursor taze diff` повертає `{major:[{workspace,pkg,from,to}], minorPatch, totalChanged, comparedWorkspaces}`

## Decision Outcome
Chosen option: "`n-cursor taze diff`", because semver-класифікація — детермінований алгоритм; LLM має займатись читанням CHANGELOG breaking-changes і рефакторингом коду, а не JSON-порівнянням.

### Consequences
* Good, because transcript фіксує smoke: root без бекапів воркспейсів → `totalChanged:0, comparedWorkspaces:1`; 12/12 тестів зелені.
* Bad, because transcript не містить підтверджених негативних наслідків.

## ADR `n-cursor start-check scan|run` — детерміноване виявлення воркспейсів і spawn

## Context and Problem Statement
Кроки 1–4 скіла `n-start-check` давали агенту інструкції вручну читати `package.json#workspaces`, розгортати glob-патерни, читати кожен `package.json`, запускати процеси з таймаутом (через `perl -e 'alarm'`), перевіряти exit-коди і парсити лог на маркери готовності. Це ~80% детермінована shell/Node-логіка.

## Considered Options
* Агент виконує всі ці кроки вручну через Bash/Read
* `n-cursor start-check scan` повертає `[{workspace, name, hasStart, startCmd, type}]`; `run <ws>` повертає `{workspace, status, exitCode, ready, firstError, logTail, sideEffects}`

## Decision Outcome
Chosen option: "`scan`/`run` CLI з ін'єкцією `spawnImpl`", because розгортання glob, класифікація `server`/`cli` (за маркерами `vite`, `next`, `listen`, тощо), cross-platform spawn із таймаутом через `spawnSync({timeout})` і парс лога — детерміновані; LLM має лише діагностувати FAIL за готовим звітом.

### Consequences
* Good, because transcript фіксує smoke `scan` на реальному монорепо: root→cli, demo→server, npm→cli; 15/15 тестів зелені; oxlint чисто.
* Bad, because regex готовності (`READY_RE`) потребує ретельного підбору: початковий варіант з `\b` не матчив `Local:` (двокрапка — не word-boundary) — баг виявлено і виправлено в цій же сесії.

## More Information
- `npm/skills/start-check/js/check.mjs` — `classifyStartType`, `parseStartLog`, `scanStartWorkspaces`, `runWorkspaceStart`, `runStartCheckCli`; `spawnImpl` для тестованості
- `npm/skills/start-check/js/tests/check.test.mjs` — 15 кейсів
- `npm/skills/start-check/SKILL.md` кроки 1–4 замінено на `scan`/`run`; крок 3 (відкат) лишено з `sideEffects` із відповіді `run`
- `npm/.changes/260605-0731.md` (Added, minor)
