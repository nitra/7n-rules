## ADR Систематичне скорочення `.mdc`-правил через видалення нерезультативного тексту

**Контекст:** Файли правил у `npm/rules/**/*.mdc` з часом накопичили значний пласт пояснювального тексту — мета-блоки активації, інструкції встановлення CLI (`brew`, `apt-get`), VS Code блоки налаштувань, розділи `## Перевірка` з посиланням на `npx … check <rule>`, описи того, що робить зовнішній інструмент (hadolint, kubeconform, gitleaks тощо). Це збільшувало контекст при підвантаженні правил без впливу на кінцевий результат їх застосування агентом.

**Рішення/Процедура/Факт:** Виконана систематична чистка 24 файлів у `npm/rules/`: від `security.mdc` (ініційована задача) до `abie.mdc`, `js-mssql.mdc`, `tauri.mdc`. Алгоритм — читати файл, видаляти класи нерезультативного тексту (перелічені нижче), зберігати весь канон. Три файли (`vue.mdc`, `php.mdc`, `text.mdc`) навмисно пропущені через ризик непомітного руйнування або суперечностей у канонічному вмісті. Видалено ~314 рядків (~7,6%) із загальних 4133.

**Обґрунтування:** Правило — це директива для AI-агента, а не довідник оператора. Текст «встанови regal через `brew`» або «розширення VS Code для NGINX» не змінює жодного файлу проєкту при застосуванні правила, але збільшує кількість токенів, які необхідно обробити. Мета — залишити лише те, що формує канон: конкретні значення, версії, структури файлів, заборони, шаблони.

**Розглянуті альтернативи:**
- Прибирати лише найгрубіші нерезультативні блоки (низький ризик, малий ефект) — відхилено на користь ширшого скорочення.
- Повне переписування правил — не обговорювалось (надлишкова трансформація).
- Переміщення VS Code / install-блоків у окрему документацію — не обговорювалось.
