## ADR Уніфікований публічний інтерфейс `fix` як автономний оркестратор

## Context and Problem Statement
Команда `npx @nitra/cursor fix` виконувала лише детерміністичну перевірку (check-only mode), а для автономного виправлення існував окремий публічний команд `fix-run`. Разом з тим внутрішній режим `fix --json` слугував субпроцесовим transport для orchestrator і T0-auto, але при цьому був частиною публічного інтерфейсу CLI.

## Considered Options
* Залишити `fix` як check, `fix-run` як окремий orchestrator
* Зробити `fix` самим orchestrator, прибрати `fix --json` з публічного API, замінити на приватну команду `_fix-check`

## Decision Outcome
Chosen option: "`fix` = orchestrator, `_fix-check` = internal", because команда `npx @nitra/cursor fix` повинна вирішувати задачу повністю автономно (convergence-loop: check → T0-auto → LLM via pi → recheck), а не бути лише перевіркою. `fix-run` виключено як дублікат (deprecated alias → `fix`). `fix --json` прибрано з публічного API; його функцію виконує внутрішня команда `_fix-check` (underscore = private), яку викликають лише `orchestrator.mjs` і `t0.mjs` через subprocess.

### Consequences
* Good, because transcript фіксує очікувану користь: `npx @nitra/cursor fix` виконує повний цикл виправлення без участі агента; `fix-run` і `fix --json` більше не плутають публічний API.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Змінені файли: `npm/bin/n-cursor.js` (`case 'fix'` → завжди `runOrchestratorCli`; новий `case '_fix-check'`; `fix-run` deprecated), `npm/skills/fix/js/t0.mjs` (`fix --json` → `_fix-check`), `npm/skills/fix/js/orchestrator.mjs` (використовує `_fix-check` внутрішньо). Команди тесту: `bun npm/bin/n-cursor.js fix js-run 2>&1`, `bun npm/bin/n-cursor.js _fix-check js-run 2>/dev/null | python3 -c "..."`.

## ADR C1 pattern у `llm-worker.mjs`: script збирає контекст → pi → script застосовує

## Context and Problem Statement
Попередня реалізація `llm-worker.mjs` використовувала Anthropic SDK напряму (`ANTHROPIC_API_KEY` + tool-use) для виправлення порушень. Це прив'язувало LLM-tier до конкретного провайдера й унеможливлювало використання власних API-ключів користувача через `pi`.

## Considered Options
* Залишити Anthropic SDK з `ANTHROPIC_API_KEY` і tool-use
* C1 pattern: script читає файли й будує контекст → `pi -p "..." --no-session --mode text --no-tools` → script парсить відповідь і записує файл цілком

## Decision Outcome
Chosen option: "C1 pattern через `pi`", because всі LLM-виклики мають іти через `pi`, де кожен користувач самостійно налаштовує ключі доступу; SDK-залежність видаляється повністю.

### Consequences
* Good, because transcript фіксує очікувану користь: GPT-5 через pi (subscription default, без `--model` flag) успішно виправив `process.env.TEST_KEY` → `env.TEST_KEY` за одну ітерацію; e2e підтверджено.
* Bad, because без tool-use LLM не може динамічно запитувати файли — скрипт мусить завчасно вилучити весь контекст (файли з порушення + `.mdc` правило). Якщо path extraction не спрацює — LLM поверне помилку "file contents not provided".

## More Information
Файл: `npm/skills/fix/js/llm-worker.mjs`. Функції: `extractFilePaths(output)` (workspace-aware: `[npm] skills/foo.mjs` → `npm/skills/foo.mjs`), `callPi(prompt, model)` (omits `--model` коли `model === ''`), `parseResponse(text)`. Env vars: `N_CURSOR_FIX_MODEL_HAIKU`, `N_CURSOR_FIX_MODEL_SONNET` (default `''` = pi subscription default = GPT-5). Команда перевірки: `bun npm/bin/n-cursor.js fix js-run 2>&1`.

## ADR Опускання `--model` у `pi` для використання subscription default

## Context and Problem Statement
При виклику `pi --model gpt-5.3-codex-spark` (або будь-якого gpt-5.x) pi v0.78.0 повертає `No API key found for azure-openai-responses`, бо всі явні назви моделей роутяться через Azure endpoint, для якого немає ключа. При цьому `pi` без `--model` використовує subscription default (GPT-5 через `openai-codex` OAuth) і працює коректно.

## Considered Options
* Передавати конкретну назву моделі через `--model gpt-5.3-codex-spark`
* Не передавати `--model` взагалі (порожній рядок = subscription default)

## Decision Outcome
Chosen option: "Не передавати `--model` коли модель порожня", because `pi --model <name>` завжди роутить через azure-openai-responses незалежно від наявних credentials; без `--model` pi використовує subscription routing і credentials з `~/.pi/agent/auth.json`.

### Consequences
* Good, because transcript фіксує очікувану користь: `pi -p '{"ok":true}' --mode text --no-tools` без `--model` повернув `{"ok":true}`; GPT-5 виправив порушення.
* Bad, because Neutral, because transcript не містить підтвердження наслідку — поведінка може змінитися при оновленні `pi`.

## More Information
Реалізація: `callPi(prompt, model)` у `npm/skills/fix/js/llm-worker.mjs`: `const modelArgs = model ? ['--model', model] : []`. Дефолти: `MODEL_HAIKU = env.N_CURSOR_FIX_MODEL_HAIKU ?? ''`, `MODEL_SONNET = env.N_CURSOR_FIX_MODEL_SONNET ?? ''`. Версія pi: 0.78.0. Auth: `~/.pi/agent/auth.json` містить `openai-codex` credentials.
