**Status:** Accepted
**Date:** 2026-06-09

## Context and Problem Statement

Після правки джерела `npm/rules/npm-module/npm-module.mdc` потрібно було оновити inline snapshot `.cursor/rules/n-npm-module.mdc`. Дефолтний запуск `bun ./npm/bin/n-cursor.js` виконав self-upgrade до 5.0.0, ADR-нормалізацію, оновлення skills і `CLAUDE.md`, тобто створив побічні зміни, які довелося відкочувати через `git checkout --`.

## Considered Options

- Запустити `n-cursor` sync і прийняти всі побічні ефекти.
- Написати мінімальний одноразовий скрипт, що викликає тільки `inlineTemplateLinks` для одного правила.

## Decision Outcome

Chosen option: "Написати мінімальний одноразовий скрипт", because дефолтний sync self-upgrades `@nitra/cursor` з registry, якщо specifier у кореневому `package.json` є plain semver, а також запускає ADR-нормалізацію і перезаписує skills/CLAUDE.md, що небажано для точкової правки одного правила.

### Consequences

- Good, because diff `.cursor/rules/n-npm-module.mdc` містить лише потрібні зміни без зайвих side effects.
- Bad, because одноразовий скрипт потребує знання внутрішнього API `inlineTemplateLinks` з `npm/scripts/lib/inline-template-links.mjs` і не є офіційним інтерфейсом.
- Neutral, because transcript не містить підтвердження інших наслідків.

## More Information

Одноразовий скрипт `/tmp/regen-npm-module.mjs` імпортував:

```js
import { inlineTemplateLinks } from '/Users/vitalii/www/nitra/cursor/npm/scripts/lib/inline-template-links.mjs'
```

Скрипт записав результат у `.cursor/rules/n-npm-module.mdc` і був видалений після виконання.

Причина self-upgrade: функція `upgradeNitraCursorToLatestAndBunInstall` у `npm/bin/n-cursor.js:108` спрацьовує, коли specifier у кореневому `package.json` — plain semver (`^3.22.0`), а не `workspace:`, `file:` або `link:`.

У цьому ж transcript також зафіксовано вже покрите clean-ADR рішення `2026-06-09-release-step-portable-n-cursor-binary.md`: release step у `npm-publish.yml` має викликати `n-cursor release` з PATH замість `node npm/bin/n-cursor.js release`.
