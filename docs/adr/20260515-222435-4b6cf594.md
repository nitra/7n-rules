## ADR syncSkills: auto.md більше не копіюється у .cursor/skills/
**Контекст:** При синхронізації скілів у `.cursor/skills/` функція `syncSkills` копіювала всі файли з `npm/skills/<id>/`, включно з `auto.md`. Файл `auto.md` є службовим мета-файлом лише для CLI-сторони (`scripts/auto-skills.mjs`) і не повинен потрапляти у каталог проєкту.
**Рішення/Процедура/Факт:** У `npm/bin/n-cursor.js::syncSkills` (~рядок 751) додано перевірку `if (file === 'auto.md') continue` — файл `auto.md` пропускається при ітерації по вмісту скілу перед копіюванням. Header-коментар CLI оновлено. Версію пакету підвищено `1.11.7 → 1.11.8`, до `npm/CHANGELOG.md` додано секцію `[1.11.8]`.
**Обґрунтування:** `auto.md` — це механізм автовизначення скілів на стороні `@nitra/cursor` CLI; його вміст не є частиною тіла скілу і не має бути присутнім у репозиторії-споживачі. Користувач явно зазначив, що лише `SKILL.md` має синхронізуватись.
**Розглянуті альтернативи:** Автоматичне видалення вже синхронізованих `auto.md` при наступному синку — розглядалось у першій версії, але прибрано на прохання користувача; видалення лишається ручним.
**Зачіпає:** `npm/bin/n-cursor.js` (функція `syncSkills`), `npm/package.json` (версія), `npm/CHANGELOG.md`, всі каталоги `.cursor/skills/n-*/` (більше не отримуватимуть `auto.md`).
