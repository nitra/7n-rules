## ADR Скорочення CLI-виводу `n-cursor`: приховування цілих блоків без помилок

## Context and Problem Statement
`npx @nitra/cursor` виводив три секції прогресу — `🧩 Skills:`, `⌨️ Commands:`, `🥧 Pi skills:` — включно з рядками `⬇ id → path ... ✅` навіть тоді, коли синхронізація пройшла без жодної помилки. Це захаращувало вивід зайвою інформацією при штатному запуску.

## Considered Options
* Ховати лише підсумковий рядок (`🧩 Skills: N скопійовано, 0 з помилками`), лишаючи `⬇ … ✅` рядки.
* Ховати цілий блок (і `⬇ … ✅`, і підсумок) для всіх трьох секцій, коли `fail === 0`.

## Decision Outcome
Chosen option: "Ховати цілий блок для всіх трьох секцій", because користувач явно відхилив варіант із частковим приховуванням і підтвердив, що приховувати треба `🥧 Pi skills:` так само.

Реалізовано через допоміжну функцію `captureOutput` у `npm/bin/n-cursor.js`: вона перехоплює `process.stdout.write`, `console.log` та `console.error` у внутрішній буфер і скидає його у реальний stdout лише якщо `fail > 0`.

### Consequences
* Good, because при штатному запуску CLI-вивід компактний — видно лише правила та підсумок `✨ Готово`.
* Bad, because перехоплення `process.stdout.write` глобально — крихкий патч: будь-який код усередині `captureOutput` не зможе писати у stdout безпосередньо; потребує обережності при розширенні логіки всередині блоків.
