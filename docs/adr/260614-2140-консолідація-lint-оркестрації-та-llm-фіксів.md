description: Lint-оркестрацію перенесено у rules/lint, CI переведено в read-only, /n-fix видалено, а detect-only фікси винесено у спільне LLM-ядро.
---

**Status:** Accepted
**Date:** 2026-06-14

## Context and Problem Statement

Lint-підсистема мала кілька повʼязаних проблем: CI-воркфлоу могли мутувати дерево через auto-fix команди, `/n-fix` існував як делегат без власної логіки, lint-оркестратор жив у `scripts/` замість rule-centric структури, detect-only інструменти не мали спільного механізму LLM-автофіксу, а внутрішні `_fix-check`/`fix-t0` запускалися через subprocess-round-trip.

## Considered Options

- Залишити CI з мутуючими lint-командами, `/n-fix` як alias, `scripts/lint-cli.mjs` на місці, ad-hoc фіксери для кожного tool і internal subcommands.
- Перевести CI на read-only, видалити `/n-fix`, перенести оркестратор у `rules/lint/`, виділити спільне ядро `llm-fix-apply` / `llmLintFix` і замінити internal subcommands прямими функціями.

## Decision Outcome

Chosen option: "Перевести lint-підсистему на rule-centric read-only/fix архітектуру зі спільним LLM-ядром", because transcript фіксує, що CI не має мутувати дерево, `/n-fix` не мав власної логіки, spec §7 вимагала `rules/lint/`, а дублювання parse/read/apply логіки для detect-only tools було небажаним.

### Consequences

- Good, because CI лише детектує порушення, `lint` зареєстровано як правило, `/n-lint` стає єдиною точкою входу, а cspell отримує per-tool omlx-фіксер через спільне ядро.
- Good, because `_fix-check` і `fix-t0` більше не потребують subprocess-spawn для внутрішніх фаз.
- Bad, because transcript фіксує потребу синхронно оновити канони workflow/snippet/test/mdc; без цього тести-канони ламались.
- Neutral, because transcript не містить підтверджених негативних наслідків видалення `/n-fix`, перенесення оркестратора або inline internal subcommands.

## Update 2026-06-14

- Абсолютну заборону паралельних запусків ESLint/lint-ланцюжків розслаблено: паралельний lint дозволений для дизʼюнктних наборів файлів.
- Whole-tree прогони того самого корпусу (`bun run lint`, `n-cursor lint --full`) лишаються серіалізованими.

## Update 2026-06-14

- Додатково зафіксовано, що `_fix-check` і `fix-t0` більше не доступні як CLI subcommands після inline-рефакторингу; transcript класифікує це як breaking-зміну.
- `llm-lint-fix.mjs` було виправлено відповідно до `js-run`: пряме `process.env.N_CURSOR_FIX_MODEL` замінено на `env` з `node:process`.
- `jscpd`, `trufflehog` і `knip` не ввімкнено як LLM autofix у цьому рішенні: transcript фіксує ризики dynamic-usage false positives, деструктивність dedup і потребу людського рішення для секретів.
