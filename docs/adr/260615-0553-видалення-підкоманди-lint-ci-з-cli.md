## ADR Видалення підкоманди `lint-ci` з CLI

## Context and Problem Statement
`n-cursor lint-ci` існував як чистий аліас для `runLint({ full: true, readOnly: true })` — ідентичний `lint --read-only --full`. Жоден реальний workflow, CI-конфіг чи кореневий `package.json` його не викликав; посилання лишались лише в документаційних рядках (`js-lint-ci.mdc`, `n-cursor.md`, CHANGELOG). Наявність окремої підкоманди збільшувала поверхню CLI без відповідного приросту функціональності.

## Considered Options
* Залишити `lint-ci` як задокументований аліас
* Видалити `lint-ci` і направити CI на `lint --read-only --full`

## Decision Outcome
Chosen option: "Видалити `lint-ci` і направити CI на `lint --read-only --full`", because ціль — мінімальна поверхня CLI; аліас не надавав жодної унікальної поведінки.

### Consequences
* Good, because transcript фіксує очікувану користь: менша поверхня CLI, менше коду в `switch`, немає двох шляхів до одного результату.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Видалено `lint-ci` з переліку в `default`-помилці (`npm/bin/n-cursor.js`).
