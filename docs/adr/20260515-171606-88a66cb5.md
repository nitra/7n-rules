## ADR Concern-based JS structure для правил: розпил `js/check.mjs` → `js/<concern>/check.mjs`

**Контекст:** Кожне правило CLI `@nitra/cursor` мало монолітний файл `rules/<id>/js/check.mjs`, що поєднував кілька несполучених перевірок (file system, DNS, YAML-парсинг, patch-валідація тощо). Файли досягали 1 000+ рядків, утрудняли локалізацію помилок і унеможливлювали незалежне кешування результатів концернів.

**Рішення/Процедура/Факт:** Усі 25 вшитих правил переведено на структуру `rules/<id>/js/<concern>/check.mjs`:
- `adr→hooks`, `bun→layout`, `capacitor→platforms`, `changelog→consistency`, `docker→lint`, `ga→workflows`, `graphql→tooling`, `hasura→internal_urls`, `image-avif→avif_generation`, `image-compress→package_setup`, `js-bun-db→safety`, `js-bun-redis→imports`, `js-lint→tooling`, `js-mssql→deps`, `js-run→runtime`, `k8s→manifests`, `nginx-default-tpl→template`, `npm-module→package_structure`, `php→tooling`, `style-lint→tooling`, `tauri→tooling`, `text→formatting`, `vue→packages`, `rego→applies`, `abie→(6 концернів)`.
- `abie` — єдине правило з кількома концернами: `applies`, `firebase_hosting`, `hc_pairing`, `ua_node_selector`, `ua_http_route`, `env_dns`; shared-state (`findK8sYamlFiles`, `collectDeploymentDirs`) — module-level кеш у `utils/k8s-tree.mjs`.
- `scripts/utils/discover-checkable-rules.mjs` переписано: subdir-concerns мають пріоритет; legacy-fallback (`js/check.mjs` без підкаталогу) прибрано повністю після завершення міграції.
- `scripts/lint-conftest.mjs` переписано: hardcoded `TARGETS`-таблиця (~280 рядків) замінена на `discoverCheckableRules` + читання `policy/<concern>/target.json`.
- Побічно виправлено: `adr/js/hooks` — `BUNDLED_HOOKS_DIR` (неправильна глибина `../`), `js-lint/js/tooling` — `OXLINT_CANONICAL_JSON_PATH`/`KNIP_CANONICAL_JSON_PATH` (шляхи вказували на неіснуючий `utils/`), `ga/js/lint.mjs` — внутрішній імпорт після переіменування.

**Обґрунтування:** Concern-based структура дзеркалить `policy/<concern>/` side-by-side, дозволяє CLI запускати й кешувати концерни незалежно, тестувати кожен концерн ізольовано (unit без запуску повного `check()`), і унеможливлює «blob-файл» антипатерн. Видалення legacy-fallback — обовʼязковий cleanup після завершення міграції, інакше discovery залишає «зяючу шпарину» для регресій.

**Розглянуті альтернативи:** Залишити flat `check.mjs` із внутрішнім розбиттям на іменовані функції — відхилено, бо не дає незалежного прогону і не усуває проблему розміру файлу.

**Зачіпає:** `npm/rules/*/js/` (усі 25 правил), `npm/scripts/utils/discover-checkable-rules.{mjs,test.mjs}`, `npm/scripts/utils/run-rule.{mjs,test.mjs}`, `npm/scripts/lint-conftest.mjs`, `npm/tests/integration-repo-checks.test.mjs`, `npm/tests/check-empty-trees.test.mjs`.
