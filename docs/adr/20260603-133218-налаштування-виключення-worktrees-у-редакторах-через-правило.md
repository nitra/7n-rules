## ADR Налаштування виключення `.worktrees` у редакторах через правило `n-text`

## Context and Problem Statement
Директорії `.worktrees/` та `.claude/worktrees/` потрапляють у файловий індекс VSCode і Zed, засмічуючи пошук і навігацію проєктом. Потрібно централізовано встановлювати відповідні exclusion-налаштування при виконанні `n-fix`.

## Considered Options
* Розширити наявний `snippet` у `npm/rules/text/policy/vscode_settings/template/settings.json.snippet.json` — додати `search.exclude` та `files.exclude` для VSCode; створити новий concern `zed_settings` із власним `target.json` + `snippet` для Zed.
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Розширення VSCode snippet + новий Zed concern у правилі `n-text`", because користувач явно задав конкретні JSON-блоки для обох редакторів і зазначив, що вони мають застосовуватися через `n-fix` механізм правила `text`.

VSCode-блоки:
```json
{
"search.exclude": { "**/.worktrees/**": true, "**/node_modules/**": true },
"files.exclude":  { "**/.worktrees/**": true }
}
```
Zed-блок (`file_scan_exclusions`):
```json
["**/.git","**/.svn","**/.hg","**/.DS_Store","**/Thumbs.db",
"**/node_modules","**/.worktrees","**/.claude/worktrees"]
```

### Consequences
* Good, because виключення потрапляють у `.vscode/settings.json` та `.zed/settings.json` автоматично при `n-fix`, без ручного редагування кожного проєкту.
* Bad, because transcript не містить підтверджених негативних наслідків; повна реалізація Zed-concern потребуватиме нового `target.json` і, можливо, `.rego`-файлу — transcript зафіксував лише початок дослідження `template.mjs`.

## More Information
* Наявний VSCode concern: `npm/rules/text/policy/vscode_settings/template/settings.json.snippet.json`, `npm/rules/text/policy/vscode_settings/target.json`
* Orchestration entry point: `npm/rules/text/fix.mjs` → `runStandardRule` → `runConftestBatch` per `policy/<concern>/target.json`
* Механізм snippet-driven перевірки описаний у `npm/scripts/lib/template.mjs` та `npm/scripts/lib/run-rule.mjs` (рядок ~63: "Snippet-driven перевірка концерну")
* Правило `n-text` активується автоматично (`alwaysApply: true`, version `1.30`)
