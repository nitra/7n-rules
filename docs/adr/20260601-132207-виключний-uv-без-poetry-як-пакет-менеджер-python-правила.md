## ADR Без case 'lint-python' у n-cursor.js (суто адитивна реєстрація)

## Context and Problem Statement
При додаванні нового правила `python` постало питання: реєструвати підкоманду `lint-python` у `npm/bin/n-cursor.js` (як `lint-docker`, `lint-k8s`) або обмежитися `package.json#scripts.lint-python` на стороні споживача — як у правила `php`.

## Considered Options
* Додати `case 'lint-python'` до `n-cursor.js` + `runLintPython` import (tracked-файл зміниться)
* Не додавати: споживач викликає через `bun run lint-python → bun …/rules/python/lint/lint.mjs` (PR залишається суто адитивним — лише нові untracked-файли)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Не додавати case до n-cursor.js", because зміна `n-cursor.js` і `build-agents-commands.mjs` — відстежуваних файлів — при uncommitted-стані активує `integration-repo-checks` (npm-module dirty-bump check), що вимагає ручного бампу `version`, забороненого `n-changelog.mdc`. Правило `php` — еталон — так само не має субкоманди в CLI; достатньо `package.json#scripts.lint-python`.

### Consequences
* Good, because PR залишається суто адитивним (лише `npm/rules/python/**` + change-файл); `integration-repo-checks` чистий.
* Bad, because `npx @nitra/cursor lint-python` не спрацьовує з коробки — споживач мусить мати `scripts.lint-python` у своєму `package.json` (перевіряється Rego `policy/package_json/`).

## More Information
- Початкова спроба додати `import { runLintPython }` і `case 'lint-python'` зробила `integration-repo-checks` червоним із повідомленням: «Незакомічені зміни під npm/ (npm/bin/n-cursor.js, npm/scripts/build-agents-commands.mjs), але "version" у npm/package.json лишився 3.4.1».
- Рішення: `git checkout -- npm/bin/n-cursor.js npm/scripts/build-agents-commands.mjs`.
- Авторитетне правило: `n-changelog.mdc` — version бампить лише CI через change-файли; ручний bump ламає `fix changelog`.
- Перевіряється через Rego: `npm/rules/python/policy/package_json/template/package.json.contains.json` — `scripts.lint-python` повинен містити підрядок `bun`.
