## ADR Конвенція розширень нових JS-файлів: `.mjs`/`.cjs` замість `.js`

## Context and Problem Statement
Node.js і Bun визначають тип модуля (ESM vs CJS) за `package.json#type`, якщо файл має розширення `.js`. Це означає, що однаково названий `.js`-файл трактується по-різному залежно від пакета, в якому він лежить. Потрібна конвенція, що зробить тип модуля однозначним незалежно від `package.json`.

## Considered Options
* Явні розширення `.mjs`/`.cjs` для нових файлів, `.js` — залишити для існуючих
* Автоматична git-aware перевірка (фейлити на `.js` у `git status untracked/added`)
* Просто warn на всі `.js` без фейлу
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Явні розширення `.mjs`/`.cjs` для нових файлів, `.js` — залишити для існуючих", because правило (a) — статична інструкція в `.mdc` без git-aware перевірки — єдиний реалістичний підхід: stateless-скан не розрізняє новий файл від існуючого, а варіант (b) крихкий.

### Consequences
* Good, because тип модуля нового файлу (ESM vs CJS) буде однозначним без читання `package.json#type`, що усуває неочевидну залежність від контексту пакета.
* Bad, because правило не піддається автоматичній перевірці — `fix js-lint` не ловитиме порушення; дотримання покладається виключно на агента/розробника.

## More Information
- Правило розміщується в `js-lint/js-lint.mdc` (version: `1.29` → `1.30`); опис додається як нова секція «Розширення нових файлів — `.mjs` / `.cjs`, не `.js`».
- Стосується backend і frontend (`src/`, тести `*.test.*`, `scripts/`, `src/conn/`).
- Існуючі `.js` лишаються без змін — масова міграція не передбачена.
- Приклади у самому `js-lint.mdc` мають бути оновлені на `.mjs`, але перед редагуванням потрібна перевірка чекерів: `js-run/js/tests/runtime/tests/check-fixture.test.mjs` хардкодить `src/conn/pg-write.js` і `src/conn/mssql-write.js` — ці фікстури конфліктуватимуть із новою конвенцією, якщо їх не оновити одночасно.
- Конфіги тулінгу (`eslint.config.js`, `vitest.config.js`, `vite.config.js`) та`.cjs` — явно дозволені й не підпадають під заборону.
