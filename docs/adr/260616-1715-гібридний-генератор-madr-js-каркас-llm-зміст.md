**Status:** Accepted
**Date:** 2026-06-16

## Context and Problem Statement

ADR-нормалізатор `npm/scripts/lib/adr/normalize-pipeline.mjs` вбудовував повний MADR-шаблон у системний промпт `GEN_SYS`: порядок заголовків, `**Status:** Accepted`, `**Date:**`, форму `Chosen option: "...", because ...` і fallback-рядки. Потім JS перевіряв той самий формат regex-ами у `validateMadr`, створюючи петлю «попросити модель про формат → перевірити формат кодом → ретраїти при розбіжності».

## Considered Options

- Pure JSON-fields: LLM повертає суворо типізовані поля без Markdown, а JS збирає все.
- Hybrid: LLM повертає зміст секцій у вигляді Markdown-прози зі збереженням inline code, а JS відповідає за весь MADR-каркас.
- Старий підхід: LLM повертає готовий MADR-документ цілком через контракт `GEN_SYS`.

## Decision Outcome

Chosen option: "Hybrid", because A/B-прогін показав, що pure JSON-fields є lossy: модель стирала inline-code backticks і колапсувала rich `More Information` у плаский рядок, тоді як hybrid повернув backticks і повноту секцій при JS-owned каркасі.

### Consequences

- Good, because `assembleMadr({ title, date, sections })` гарантує канонічний MADR-формат незалежно від моделі.
- Good, because форматні ретраї для `genMadr` стають непотрібними або майже непотрібними, бо заголовки, `Status`, `Date` і fallback-рядки генерує JS.
- Good, because `MERGE_SYS` отримав аналогічну зміну: заголовок `## Update <date>` тепер будує JS у `genMerge`.
- Good, because нові детерміновані функції `assembleMadr`, `normalizeSections`, `madrDate` покриті 8 юніт-тестами, а загалом 21 тест проходить.
- Bad, because для merge-гілки окремий A/B-прогін у transcript не зафіксований; перевірка була через загальний лінт і юніт-тести.
- Neutral, because `EDGE_SYS` і `KIND_SYS` залишені LLM-стадіями без змін як семантична класифікація.

## More Information

Змінені файли: `npm/scripts/lib/adr/normalize-pipeline.mjs`, `npm/scripts/lib/adr/tests/normalize-pipeline.test.mjs`.

Нові export-функції: `assembleMadr`, `normalizeSections`, `madrDate`.

A/B-харнес: `npm/scripts/lib/adr/.ab-gen.mjs`; команда прогону: `AB_N=6 node npm/scripts/lib/adr/.ab-gen.mjs`; тестова команда: `cd npm && npx vitest run scripts/lib/adr/`.
