## ADR Розслаблення заборони паралельного ESLint (диз'юнктні файли)

## Context and Problem Statement
До цього рішення CLAUDE.md містив абсолютну заборону паралельних запусків ESLint/lint-ланцюжків. Під час сесії виникла потреба запускати lint паралельно по різних (диз'юнктних) наборах файлів, і було визнано, що заборона надто широка.

## Considered Options
* Залишити повну заборону паралельних запусків lint
* Дозволити паралельний lint на диз'юнктних файлах, серіалізувати лише whole-tree прогони того самого корпусу

## Decision Outcome
Chosen option: "Дозволити паралельний lint на диз'юнктних файлах", because паралельний lint по різних файлах не конфліктує і не перевантажує диск/CPU; серіалізація потрібна лише для повних прогонів того самого корпусу (`bun run lint`, `n-cursor lint --full`).

### Consequences
* Good, because transcript фіксує очікувану користь: знято зайве обмеження, що блокувало ефективну паралельну роботу агента.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Змінено: `CLAUDE.md` (секція «Лінт і ESLint»), `npm/skills/lint/SKILL.md`. Коміт `4ceb657e`.

## ADR CI lint у read-only режимі (lint-text і lint-style без авто-фіксу)

## Context and Problem Statement
CI-воркфлоу `lint-text.yml` і `lint-style.yml` мутували дерево (markdownlint `--fix`, `stylelint --fix`), тоді як після впровадження осі `--read-only` у lint-оркестраторі CI повинен лише перевіряти, а не виправляти.

## Considered Options
* Залишити `bun run lint-text` і `npx stylelint '**/*.{css,scss,vue}' --fix` у CI
* Перевести CI на `n-cursor lint-text --read-only` і `npx stylelint '**/*.{css,scss,vue}'` (без `--fix`)

## Decision Outcome
Chosen option: "Перевести CI на read-only команди", because CI не повинен мутувати дерево; `n-cursor` — workspace-symlink на локальне джерело, тому `--read-only` доступний без додаткових кроків.

### Consequences
* Good, because transcript фіксує очікувану користь: CI більше не здійснює авто-змін.
* Bad, because зміна потребувала синхронного оновлення канону у кількох точках: `formatting.mjs`, `lint_text_test.rego`, `lint_style_yml_test.rego`, snippet-шаблони, `style-lint.mdc`, дзеркало `n-text.mdc` — інакше тести-канони ламались.

## More Information
Файли: `.github/workflows/lint-text.yml`, `.github/workflows/lint-style.yml`, `npm/rules/text/policy/lint_text/template/lint-text.yml.snippet.yml`, `npm/rules/text/js/formatting.mjs`, `npm/rules/text/policy/lint_text/lint_text_test.rego`, `npm/rules/style-lint/policy/lint_style_yml/lint_style_yml_test.rego`, `npm/rules/style-lint/style-lint.mdc`, `.cursor/rules/n-text.mdc`. Коміт `0d3eaeac`.
