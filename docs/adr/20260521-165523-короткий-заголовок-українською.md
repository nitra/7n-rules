## ADR Машинна перевірка заборони vitest та jsdom у vue package.json

## Context and Problem Statement
Секція «Тестування» правила `vue.mdc` забороняла `vitest` і `jsdom`, але ця заборона існувала лише як текст — `npx @nitra/cursor check vue` не ловив порушень, тому `n-fix` не бачив і не виправляв їх. Потрібно зробити перевірку машинно-верифікованою.

## Considered Options
* Додати `deny`-блоки до наявного `package_json.rego`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Додати `deny`-блоки до наявного `package_json.rego`", because rego-policy вже обробляє `package.json` у Vue-пакетах і наявний helper `all_dependency_names` покриває обидві секції залежностей.

### Consequences
* Good, because transcript фіксує очікувану користь: `opa test` — 6/6 PASS, `regal lint` — 0 violations, ручний `conftest` підтверджує deny для `vitest`+`jsdom` і 0 deny без них.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Змінені файли: `npm/rules/vue/policy/package_json/package_json.rego`, `npm/rules/vue/policy/package_json/package_json_test.rego`. Команди перевірки: `opa test npm/rules/vue/policy/package_json -v`, `regal lint npm/rules/vue/policy/package_json`, `conftest test path/to/package.json -p npm/rules/vue/policy -n vue.package_json`. `happy-dom` обов'язковою не зроблено — rego по `package.json` не може визначити наявність component/DOM-тестів. Окреме рішення: на запит «дозволити jsdom для DOM-тестів» обрано «лишити повну заборону» — `jsdom` deny беззастережно.
