## ADR Заміна локального хелпера `sleep` на іменований імпорт з `node:timers/promises`

## Context and Problem Statement
У файлах `npm/scripts/utils/with-lock.mjs` та `npm/scripts/utils/tests/with-lock.test.mjs` була присутня локальна функція-хелпер `function sleep(ms) { return new Promise(r => setTimeout(r, ms)) }` / `const sleep = ms => new Promise(r => setTimeout(r, ms))`. ## Considered Options
* Замінити локальний хелпер `sleep` на `import { setTimeout as sleep } from 'node:timers/promises'`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Замінити локальний хелпер `sleep` на `import { setTimeout as sleep } from 'node:timers/promises'`", because правило `js-run` в `npx @nitra/cursor fix` вимагає використання стандартного Node.js API `node:timers/promises` замість inline-реімплементації.

### Consequences
* Good, because `npx @nitra/cursor fix js-run` проходить без зауважень після заміни.
* Good, because transcript фіксує очікувану користь: усі 8 тестів `bun test with-lock.test.mjs` пройшли після рефакторингу.
