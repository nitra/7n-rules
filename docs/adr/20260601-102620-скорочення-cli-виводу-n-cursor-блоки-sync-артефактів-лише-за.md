Саме цей сеанс описаний у задачі. Ось ADR:

## ADR Скорочення CLI-виводу `n-cursor`: блоки sync-артефактів — лише за наявності помилок

## Context and Problem Statement
CLI-інструмент `@nitra/cursor` виводив під час успішного прогону три великих блоки: `🧩 Skills`, `⌨️ Commands`, `🥧 Pi skills` — разом із усіма рядками `⬇ … ✅`. При відсутності помилок ці рядки не несуть корисної інформації і захаращують термінальний вивід.

## Considered Options
* Показувати весь вивід блоків завжди (поточна поведінка).
* Показувати лише підсумковий рядок (`🧩 Skills: N скопійовано, 0 з помилками`), приховуючи рядки `⬇`.
* Приховувати увесь блок цілком (рядки `⬇` + підсумок), якщо `fail === 0`.

## Decision Outcome
Chosen option: "Приховувати увесь блок цілком, якщо `fail === 0`", because користувач явно вказав, що при відсутності помилок потрібно не відображати ні підсумковий рядок, ні рядки `⬇`, і що це стосується всіх трьох блоків (`🧩 Skills`, `⌨️ Commands`, `🥧 Pi skills`).

### Consequences
* Good, because transcript фіксує очікувану користь: вивід `bun start` / `npx @nitra/cursor` стає значно коротшим при успішному прогоні.
* Bad, because при буферизованому виведенні перехоплюється весь stdout/console під час дії — якщо додається новий тип виводу всередині цих блоків, він за замовчуванням не відобразиться при успіху.

## More Information
* Реалізовано у `npm/bin/n-cursor.js`: доданий helper `captureOutput(action)` (рядки ~1130+), що буферизує stdout і console; три виклики `runSyncStep` обгорнуто в `captureOutput` і виводять буфер лише при `fail > 0`.
