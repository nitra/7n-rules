## ADR Збереження часового префікса чернетки в імені clean-ADR

## Context and Problem Statement
Скрипт `normalize-decisions.sh` при операції `rewrite` перейменовував чернетку з іменем `YYYYMMDD-HHMMSS-<sid>.md` у файл `<slug>.md` без часового префікса. Через це при повторній нормалізації або перегенерації slug-а clean-файл міг змінити ім'я, що спричиняло зайві зміни в git-history та посилання на файл ставали недійсними.

## Considered Options
* Зберігати `YYYYMMDD-HHMMSS-` префікс чернетки в імені clean-файлу → `YYYYMMDD-HHMMSS-<slug>.md`
* Залишати лише `<slug>.md` без часового префікса (попередня поведінка)

## Decision Outcome
Chosen option: "Зберігати `YYYYMMDD-HHMMSS-` префікс чернетки в імені clean-файлу", because користувач явно вказав: «залишай дату та час на початку файлу, щоб назва файлів рідше змінювалась».

### Consequences
* Good, because transcript фіксує очікувану користь: ім'я clean-файлу стабільне між нормалізаціями — slug змінюється, але дата/час лишаються незмінними, тому git-посилання не ламаються.
* Bad, because transcript не містить підтверджених негативних наслідків.
