## ADR Спостережувані сліди як основний сигнал для аналізу поведінки агентів

## Context and Problem Statement
Потрібно аналізувати поведінку агентів у різних інструментах (Claude Code, Codex, Cursor), щоб знаходити точки тертя й покращувати правила/скіли проєкту. Основна ідея — використовувати reasoning-блоки моделей, але вони доступні не скрізь.

## Considered Options
* Reasoning-блоки (`thinking`) з JSONL-транскриптів
* Спостережувані сліди: послідовності tool-calls, помилки, retry-ланцюги, token-cost

## Decision Outcome
Chosen option: "Спостережувані сліди", because reasoning у Codex зашифрований провайдером (`encrypted_content`), а в Cursor зберігається лише у закритому SQLite без lifecycle-hooks — тобто reasoning фізично недоступний у двох з трьох інструментів. Спостережувані сліди присутні в усіх трьох інструментах і об'єктивніші: дія сталась чи ні, незалежно від самозвіту моделі.

### Consequences
* Good, because transcript фіксує очікувану користь: сигнали портативні, детектуються кодом без LLM (регулярки/підрахунки), і кожен патерн прямо мапиться на конкретну правку в `.mdc`/`SKILL.md`/allowlist.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Дев'ять сигнальних категорій, ідентифікованих у transcript: (1) серії однотипних tool-calls з дрібними правками, (2) помилкові виходи інструментів, (3) retry-after-failure, (4) співвідношення розвідка/виконання, (5) permission-prompts і відмови, (6) token_count/тривалість, (7) наявність звернення до правил перед порушуючою дією, (8) обсяг дифу проти кількості кроків, (9) покинуті/перезапущені ходи. Claude Code пише `tool_use`/`tool_result` у `~/.claude/projects/<proj>/*.jsonl`; Codex пише `function_call`/`function_call_output`/`token_count` у `~/.codex/sessions/.../rollout-*.jsonl`.

## ADR Cursor виключений зі збору агентних слідів

## Context and Problem Statement
При проєктуванні аналізатора поведінки агентів постало питання, які інструменти включати: Claude Code, Codex і Cursor.

## Considered Options
* Включити Cursor через скрапінг SQLite (`state.vscdb`)
* Виключити Cursor зі списку

## Decision Outcome
Chosen option: "Виключити Cursor зі списку", because формат `state.vscdb` недокументований і крихкий, reasoning зберігається тільки як summary, а lifecycle-hooks відсутні — нема куди «причепити» збір. Рішення ухвалене явно користувачем у transcript: «Cursor прибираємо зі списку».

### Consequences
* Good, because transcript фіксує очікувану користь: простіший двоканальний pipeline (Claude Code + Codex).
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Додаткової інформації в transcript не зафіксовано.

## ADR Проєктно-локальний колектор для агрегації глобальних логів інструментів

## Context and Problem Statement
Claude Code зберігає транскрипти у глобальній директорії `~/.claude/projects/<закодований-cwd>/` з авто-очисткою через 30 днів (`cleanupPeriodDays`). Codex зберігає сесії у `~/.codex/sessions/РРРР/ММ/ДД/` без розділення за проєктом — усі проєкти змішані, атрибуція лише через `session_meta.cwd` та `session_meta.git` всередині кожного файлу. Пряме читання глобальних директорій призводить до аналізу даних стороніх проєктів.

## Considered Options
* Читати глобальні директорії напряму під час аналізу
* Переодичний колектор, що фільтрує за `cwd`/`git` і зберігає зрізи у проєктно-локальному сховищі

## Decision Outcome
Chosen option: "Переодичний колектор, що фільтрує за `cwd`/`git` і зберігає зрізи у проєктно-локальному сховищі", because пряме читання глобальних директорій включає дані сторонніх проєктів (у transcript наведений конкретний приклад: Codex-сесія з `cwd: /Users/vitalii/www/nitra/k8s` потрапила б до аналізу `nitra/cursor`). Проєктно-локальний append-лог також переживає 30-денну авто-очистку Claude Code.

### Consequences
* Good, because transcript фіксує очікувану користь: аналіз залишається проєктно-специфічним, історія довша за 30 днів, дані живуть поруч із кодом, який покращується.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Ключ атрибуції для Codex: `session_meta.cwd` та `session_meta.git` у `rollout-*.jsonl`. Для Claude Code атрибуція не потрібна — директорія вже специфічна для проєкту (`~/.claude/projects/-Users-vitalii-www-nitra-cursor/`). Ліміт очистки Claude Code: `cleanupPeriodDays` (значення не встановлене у проєкті → діє дефолт 30 днів). Існуючі хуки `capture-decisions.sh` та `normalize-decisions.sh` у `.claude/hooks/` реалізують аналогічний pattern для ADR-рішень.
