## ADR Кодування поля `auto` у `meta.json` — збереження літерала `"завжди"`

## Context and Problem Statement
При канонізації Spec A (міграція `auto.md` → `meta.json`) паралельна сесія вписала в spec і проміжний план поле `auto: "always"` (англ.). Парсер `auto-skills.mjs` порівнює значення з константою `ALWAYS_LITERAL = 'завжди'` (укр.), тому зміна рядка без синхронного рефакторингу коду сломила б авто-активацію скілів без видимих тестових помилок.

## Considered Options
* Зберегти `"завжди"` (укр.) — 1:1 з константою парсера
* Змінити на `"always"` (англ.) — вирівняти з англомовною кодовою базою (варіант паралельної сесії)

## Decision Outcome
Chosen option: "Зберегти `\"завжди\"` (укр.)", because зміна літерала без одночасного рефакторингу `auto-skills.mjs` (`ALWAYS_LITERAL`) зламала б парсинг без падіння тестів; мінімальний безпечний шлях — тримати `meta.json` і парсер синхронізованими.

### Consequences
* Good, because `auto-skills.mjs` продовжує коректно розпізнавати авто-скіли без додаткових змін у runtime-коді.
* Bad, because `meta.json` містить нестандартний (неангломовний) рядок; transcript не містить підтверджених негативних наслідків поза цим естетичним моментом.

## More Information
Константа в `npm/scripts/auto-skills.mjs` (рядок з `ALWAYS_LITERAL`). Канонічний spec: `docs/superpowers/specs/2026-05-31-skill-meta-json-worktree-design.md`, коміт `79e38d1`. Виправлення застосовано при відхиленні варіанту 1 і виборі варіанту 2 у цій сесії.

## ADR Видалення `auto.md` без fallback при міграції на `meta.json`

## Context and Problem Statement
Spec A вимагала замінити плоскі `auto.md` у `npm/skills/<id>/` структурованим `meta.json`. Питання — чи залишити `auto.md` як deprecated-fallback із warning для зворотної сумісності, чи видалити повністю.

## Considered Options
* Видалити `auto.md` повністю — єдине джерело правди
* Зберегти `auto.md` як fallback з попередженням (варіант паралельної сесії)

## Decision Outcome
Chosen option: "Видалити `auto.md` повністю", because два паралельні формати для одного поля створюють подвійне джерело правди; `check`-концерн явно перевіряє відсутність `auto.md` як умову валідності скіла.

### Consequences
* Good, because `check`-перевірка стає детермінованою: `auto.md` = помилка, `meta.json` = норма.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Обидва скасованих плани (видалені в коміті `79e38d1`): `docs/superpowers/plans/2026-05-31-auto-md-to-meta-json.md`, `docs/superpowers/plans/2026-05-31-skill-meta-json-worktree.md`. Остаточний план: `docs/superpowers/plans/2026-05-31-skill-meta-json-worktree.md`, коміт `f53b097`.

## ADR Дизайн CLI-команди `n-cursor worktree` для крос-платформної роботи з worktree

## Context and Problem Statement
Антропік-специфічний інструмент `EnterWorktree` кладе worktree у `.claude/worktrees/` і не доступний у Cursor IDE. Потрібен один інструмент, що однаково працює в Claude Code, Cursor і терміналі, реалізує конвенцію `.worktrees/` з `n-worktrees.mdc` і атомарно керує інвентарним файлом-описом.

## Considered Options
* Підкоманда CLI `n-cursor worktree` + тонкий skill (форма C)
* Тільки CLI без skill (форма A)
* Тільки skill / markdown-правило (форма B — вже є `n-worktrees.mdc`, але залежить від дисципліни агента)

## Decision Outcome
Chosen option: "Форма C — CLI + тонкий skill", because CLI гарантує ідентичну поведінку в будь-якому середовищі; skill залишається як точка входу для агента, щоб він викликав команду, а не виконував кроки вручну. Логіка створення інвентарного `.worktrees/<branch>.md` переноситься в tool.

### Consequences
* Good, because transcript фіксує очікувану користь: агент в Claude і Cursor виконує одну команду замість ручних кроків; інвентарний файл завжди створюється за конвенцією.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Підкоманди: `add <branch>`, `remove <branch>`, `list`, `prune` (набір D3). Інтерфейс: один обов'язковий аргумент `<branch>`; слеш у назві гілки санітизується в дефіс для пласкої структури (варіант E-slash-a): `feat/skill-meta` → `.worktrees/feat-skill-meta/` + `.worktrees/feat-skill-meta.md`. Конвенція `n-worktrees.mdc`: `.cursor/rules/n-worktrees.mdc`. Інвентарний файл gitignored через `.worktrees/` у `.gitignore`, не потрапляє в git-історію. Команда лягає в диспетчер `npm/bin/n-cursor.js` поряд з `case 'skill'`, `case 'change'` тощо.
