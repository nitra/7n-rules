## ADR Переміщення regexp-літералів на рівень модуля

## Context and Problem Statement
В `npm/rules/bun/fix/layout/check.mjs` та `npm/scripts/utils/inline-template-links.mjs` / `template.mjs` regexp-вирази конструювалися всередині функцій через `new RegExp(variable)` або `new RegExp(String.raw`...`)` на кожному виклику. Лінтер `e18e/prefer-static-regex` позначив це як помилку.

## Considered Options
* Перемістити regexp до модуль-рівневих констант (static literals)
* Додати `eslint-disable` для порушень
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Перемістити regexp до модуль-рівневих констант", because `bun run lint` відмовляв з exit code 1 через `e18e/prefer-static-regex`, і єдиний спосіб задовольнити правило без `eslint-disable` — винести літерали в константи поза функцією.

### Consequences
* Good, because `bun run lint` повертає exit code 0 після виправлення.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Файли змінено: `npm/rules/bun/fix/layout/check.mjs` (додано `WHITESPACE_RE` і `LINT_CHAIN_PART_RE` поза `lintChainHasScript`), `npm/scripts/utils/inline-template-links.mjs`, `npm/scripts/utils/template.mjs`. Правило лінтера: `e18e/prefer-static-regex`.

## ADR Заміна `any` на `unknown` у JSDoc-типах

## Context and Problem Statement
В JSDoc-анотаціях кількох файлів використовувався тип `any` (`@param {*}` та `{ snippet?: any, deny?: any, contains?: any }`), що спричиняло попередження від ESLint / oxlint під час `bun run lint-js`.

## Considered Options
* Замінити `any` / `{*}` на `unknown` у всіх відповідних JSDoc-параметрах
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Замінити `any` / `{*}` на `unknown`", because `unknown` — більш точний тип у TypeScript-сумісному JSDoc і усуває попередження без відключення правила.

### Consequences
* Good, because `bun run lint` виходить з кодом 0 після заміни.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Файли змінено: `npm/scripts/utils/template.mjs` (поле типу `TemplateMap`), `npm/rules/k8s/fix/manifests/check.mjs` (параметри `handleResourceDescriptorPathRef` і `walk`). Попередньо у файлах були також дубльовані `/* @param */` + `/** @param {*} */` блоки — їх злито в одну коректну JSDoc-анотацію.

* Перефразувати текст, замінивши невідомі слова загальновживаними
* Додати слова до кастомного cspell-словника проєкту
* Інші варіанти в transcript не обговорювалися.
