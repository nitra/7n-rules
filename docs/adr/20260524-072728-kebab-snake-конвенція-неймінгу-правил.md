# Подвійна конвенція неймінгу: kebab-case зовнішнє, snake_case для концернів і Rego

**Status:** Accepted
**Date:** 2026-05-24

## Context and Problem Statement

У `npm/rules/` файли та директорії іменуються двома різними стилями — kebab-case і snake_case. Відсутність явного пояснення розподілу спричиняла враження хаотичного різнобою. Необхідно зафіксувати, що подвійний стиль є навмисним і мотивованим технічними обмеженнями Rego.

## Considered Options

- Єдиний kebab-case для всіх файлів і директорій
- Єдиний snake_case для всіх файлів і директорій
- Розподіл: kebab-case для зовнішнього рівня, snake_case для концернів і Rego
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Розподіл: kebab-case для зовнішнього рівня, snake_case для концернів і Rego", because Rego-ідентифікатори (`package`, змінні, `deny`/`allow`) не можуть містити дефіс — він парситься як оператор віднімання (`package npm-module.package_json` є синтаксичною помилкою); discovery-логіка (`listJsConcerns`, `runConftestBatch`) вимагає, щоб ім'я `js/<concern>.mjs` збігалося 1:1 з `package <rule>.<concern>` у Rego; kebab-case для id правила відповідає полю `id` у `.n-cursor.json:rules`.

Розподіл ролей:

| Що | Стиль | Приклади |
|---|---|---|
| Каталог правила | kebab-case | `js-bun-db/`, `nginx-default-tpl/` |
| `.mdc` файл правила | kebab-case | `js-bun-db.mdc` |
| Концерн `js/<concern>.mjs` / `policy/<concern>/` | snake_case | `internal_urls.mjs`, `vscode_settings/` |
| Rego-пакети та тести | snake_case | `hasura_httproute.rego`, `hpa_pdb_test.rego` |
| Helpers у `utils/` | kebab-case з префіксом | `docker-mirror.mjs`, `vue-forbidden-imports.mjs` |
| Інфра-скрипти / тести | kebab-case | `walk-cache.test.mjs`, `auto-rules.mjs` |

### Consequences

- Good, because ім'я `js/<concern>.mjs` і `policy/<concern>/<concern>.rego` збігаються 1:1, що забезпечує автоматичний discovery без додаткового маппінгу
- Good, because kebab-case для id правила відповідає полю `id` у `.n-cursor.json:rules`, зберігаючи консистентність конфігурації
- Bad, because transcript не містить підтверджених негативних наслідків

## More Information

Конвенція задокументована в `.cursor/rules/scripts.mdc` (рядок ~34). Граматичне обмеження Rego: identifier — `[a-zA-Z_][a-zA-Z0-9_]*`; дефіс не входить у множину допустимих символів (OPA Policy Language — Lexical conventions).

Перевірка порушень:

```
find ./npm/rules -maxdepth 1 -mindepth 1 -type d -name "*_*"
find ./npm/rules/*/utils -maxdepth 1 -type f -name "*_*.mjs"
```

Обидві команди повернули порожній вивід — порушень не зафіксовано.
