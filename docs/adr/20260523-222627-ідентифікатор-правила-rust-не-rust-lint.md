* A — rego-heavy: три окремі rego-policy-пакети (`package_json`, `vscode_extensions`, `lint_rust_yml`), `check.mjs` лише gating + `runConftestBatch`
* C — гібрид: rego для `package_json` і `vscode_extensions`, workflow перевіряється JS-логікою в `check.mjs`
* B — JS-heavy: лише один rego-пакет `vscode_extensions`, решта — JS у `check.mjs`

Chosen option: "A — rego-heavy", because user обрав повну паритетність з `js-lint`/`style-lint`; ці правила вже мають однакову тріаду `package_json + vscode_extensions + <lint>_yml`, і дрейф створює технічний борг при майбутніх змінах перевірок.

### Consequences
* Good, because transcript фіксує очікувану користь: єдиний UX для всіх lint-правил, rego-тести генерують template-сніпети, конфіги переглядаються policy-інструментами.

## ADR Канонічна команда `lint-rust`: fmt-fix + clippy-fix + clippy -D warnings

## Context and Problem Statement
Необхідно визначити, яка команда буде канонічним вмістом `scripts.lint-rust` у `package.json` Rust-проєктів. Ключові параметри: чи застосовувати авто-виправлення (`--fix`), та наскільки жорстко обробляти warnings.

## Considered Options
* `fmt --check` + `clippy -D warnings` — лише перевірка, без авто-fix (жорсткий, без виправлень)
* `fmt` (with fix) + `clippy --fix` + `clippy -D warnings` — форматування з виправленням + авто-fix clippy + hard-fail на warnings

## Decision Outcome
Chosen option: "`fmt` (with fix) + `clippy --fix` + `clippy -D warnings`", because user обрав цей варіант явно; канонічна команда: `cargo fmt --all && cargo clippy --fix --allow-dirty --allow-staged --all-targets --all-features && cargo clippy --all-targets --all-features -- -D warnings`.

### Consequences
* Good, because команда виконує авто-виправлення перед жорстким fail-on-warnings, що зменшує ручну роботу розробника.
* Bad, because `--allow-dirty --allow-staged` дозволяє запускати fix на uncommitted змінах — потенційно перезаписує непідготовлені правки. transcript не містить підтвердження цього як проблеми.

## More Information
Rego у `policy/package_json/package_json.rego` перевіряє: `contains(scripts["lint-rust"], "cargo")`. Template: `npm/rules/rust/policy/package_json/template/package.json`. Скрипт запускається через `bun run lint-rust` (хост — root `package.json`).
