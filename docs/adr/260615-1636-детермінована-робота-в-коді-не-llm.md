type: ADR
title: Детермінована робота в коді, не в LLM
description: Обчислювані та механічні частини npm-підсистем мають виконуватися JS-кодом, а LLM лишається для творчих або семантичних рішень.
---

**Status:** Accepted

**Date:** 2026-06-15

## Context and Problem Statement

У npm-підсистемах `doc-files`, `adr`, `lint` і `coverage-classify` LLM-виклики виконували одночасно творчу та детерміновану роботу. Детермінована частина є механічною, обчислюваною кодом і не потребує нейромережі; її виконання в prompt підвищує вартість і знижує передбачуваність, особливо за температури більше нуля.

## Considered Options

- Залишити статус-кво: LLM виконує творчі й детерміновані частини в одному prompt.
- Вилучити детерміновану роботу в JS-код, лишивши LLM лише для творчих завдань.

## Decision Outcome

Chosen option: "Вилучити детерміновану роботу в JS-код", because користувач явно поставив задачу прибрати детерміновану роботу від LLM, а аналіз transcript підтвердив наявність таких ділянок у `npm/rules/lint/js/orchestrate.mjs`, `npm/rules/adr/js/hooks.mjs`, `npm/scripts/coverage-classify/index.mjs`, `npm/scripts/lib/adr/normalize-pipeline.mjs` та docgen-пайплайні.

### Consequences

- Good, because детермінована логіка стає стабільною, тестованою і дешевшою.
- Bad, because transcript не містить підтверджених негативних наслідків.
- Neutral, because transcript зафіксував лише напрям і перелік підсистем для аналізу; повний список конкретних ділянок субагентів був обрізаний.

## More Information

Під аналіз потрапили `npm/rules/doc-files/js/`, `npm/rules/adr/js/hooks.mjs`, `npm/scripts/lib/adr/normalize-pipeline.mjs`, `normalize-cli.mjs`, `npm/rules/lint/js/orchestrate.mjs`, `npm/scripts/coverage-classify/index.mjs`, `prompt.mjs`, `verdict-schema.mjs`, `npm/scripts/coverage-fix.mjs`, `coverage-fix-extract.mjs`.

Також досліджувались skills `.cursor/skills/n-adr-normalize/SKILL.md`, `.cursor/skills/n-llm-patch/SKILL.md` і `.cursor/skills/n-lint/SKILL.md`, які делегують роботу субагентам.
