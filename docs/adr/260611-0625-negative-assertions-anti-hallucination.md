# ADR: Негативні твердження у system-промпті як анти-галюцинаційний прийом

**Status:** Accepted
**Date:** 2026-06-11

## Context and Problem Statement

LLM при генерації документації на файли без кешування, мережевих викликів або складних паттернів систематично галюцинує «загальновживані» факти: «Кешує результати для швидкодії», «Звертається до зовнішнього API», «Використовує EventEmitter». Пост-перевірка (скорер, критик) ловить частину таких помилок, але вже після генерації. Промпт із загальним «Не вигадуй факти» теж не відсікає конкретних патернів.

## Considered Options

* Включати до system-промпту явні негативні твердження для кожного відомого «false fact»: `"Кешування: НЕМАЄ — не згадуй кеш"`, `"Мережа: ні"`.
* Покладатись лише на загальне правило «описуй лише наявне».
* Обробляти виключно пост-фактум: критик → перепис.

## Decision Outcome

Chosen option: "Явні негативні твердження у `factsSummary`", because LLM надає перевагу конкретному забороненому твердженню над абстрактним правилом; «Кешування: НЕМАЄ» в промпті прямо перекриває шаблон, який модель «хоче» вставити. Маркери поведінки витягуються детерміновано з коду (`extractFacts`: `caches`, `network`, `readOnly`, `failSafe`), і `factsSummary()` перетворює їх на позитивні або негативні твердження для конкретного файлу. Це персоналізований анти-галюцинаційний блок.

### Consequences

* Good, because галюцинація відсікається структурно, ще до генерації — не потрібно «ловити» її в скорері.
* Good, because твердження точні: якщо `markers.caches === false`, у промпт іде `"Кешування: НЕМАЄ — не згадуй кеш"`; якщо `true` — `"Кешує: так (це важливо для документа)"`.
* Good, because той самий прийом масштабується поза docgen: будь-який pipeline, де є відомі класи галюцинацій, може включати їхнє явне спростування у system-промпт.
* Bad, because список маркерів в `extractFacts` неповний — нові класи галюцинацій вимагають нових маркерів і нових негативних тверджень.
