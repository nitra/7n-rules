## ADR Гібридний генератор MADR: JS-каркас + LLM-зміст секцій

## Context and Problem Statement
ADR-нормалізатор (`npm/scripts/lib/adr/normalize-pipeline.mjs`) вбудовував повний MADR-шаблон у системний промпт `GEN_SYS`: порядок заголовків, `**Status:** Accepted`, `**Date:** <…>`, шаблон фрази `Chosen option: "...", because ...`, fallback-рядки `"Інші варіанти не обговорювалися."` / `"Додаткової інформації не зафіксовано."`. JS потім перевіряв той самий формат regex-ами у `validateMadr` — класична петля «попроси модель про формат → перевір формат кодом → ретрай».

## Considered Options
* **Pure JSON-fields**: LLM повертає суворо типізовані поля без будь-якого Markdown — JS збирає всe.
* **Hybrid (обрано)**: LLM повертає зміст секцій у вигляді Markdown-прози (зберігаючи inline code), JS відповідає за весь каркас (заголовки, Status, Date, fallback-фрази).
* Старий підхід — LLM повертає готовий MADR-документ цілком (контракт на markdown-вихід через `GEN_SYS`).

## Decision Outcome
Chosen option: "Hybrid", because чистий JSON-fields підхід (перша ітерація рефактору) виявився **lossy** при A/B-прогоні: LLM зтирав inline-code backticks (`` `npx @nitra/cursor` `` → plain text) і колапсував rich More Information списки в один рядок. Гібрид повернув backticks і повноту секцій при тому, що каркас залишився виключно в JS.

### Consequences
* Good, because JS-функція `assembleMadr({title, date, sections})` тепер гарантує канонічний MADR-формат (заголовки, Status, Date, fallback-рядки) незалежно від моделі — `validateMadr` стає майже зайвим і ретраї за формат зникають.
* Good, because A/B-прогін по 6 реальних чернетках показав: old=6/6 ok(6 спроб), new=6/6 ok(7 спроб) — надійність на рівні старого підходу; якість (backticks, rich sections) відновлена.
* Good, because нові детерміновані функції (`assembleMadr`, `normalizeSections`, `madrDate`) покриті 8 юніт-тестами, всього 21 тест pass.
* Bad, because `MERGE_SYS` (`genMerge`) отримав аналогічну зміну (заголовок `## Update <date>` тепер будує JS), але A/B для merge-гілки окремо не проводився — перевірено лише через загальний лінт і юніт-тести.
