# flow release ws-інференс, рецензент Read, detectLevel та DEFAULT_GATES

**Status:** Accepted
**Date:** 2026-06-02

## Context and Problem Statement

Чотири покращення flow dispatcher з однієї сесії (c893caa2): (1) `flow release` без `--ws` клав change-файл у корінь монорепо замість `npm/.changes/`; (2) рецензент `flow review` видавав нефальсифіковні findings «з diff не видно» без перевірки referenced-файлів; (3) `detectLevel` давав L0 при підрядку `fix`, навіть якщо опис містив сигнали складності; (4) незавершений merge лишив маркери конфлікту у `reviewer.mjs` — потрібно обрати `DEFAULT_GATES`.

## Considered Options

- **ws inference:** явний `--ws` від користувача _vs_ інференс із `collectChangedFilesSince(state.base_commit)` + `getMonorepoProjectRootDirs`, fail-soft при помилці
- **reviewer Read:** промпт «лише diff» _vs_ дозволити й зобов'язати `Read` для верифікації cross-file тверджень
- **detectLevel:** без змін _vs_ `COMPLEXITY_GUARD_KEYS` скасовують L0 при сигналах складності
- **DEFAULT_GATES:** `[lint]` (HEAD) _vs_ `[lint, 'coverage --changed']` (feat-бік); для `rust/coverage/coverage.mjs` — HEAD (новіший API)

## Decision Outcome

Chosen option (ws): "інферувати `--ws` із git-diff", because `base_commit` вже у flow-стані; fail-soft гарантує деградацію до дефолту.

Chosen option (reviewer): "зобов'язати Read", because subagent вже мав Read у allowed-tools — змінено лише текст промпта.

Chosen option (detectLevel): "COMPLEXITY_GUARD_KEYS", because мінімально інвазивний; не ламає існуючі L0/L3 тести.

Chosen option (DEFAULT_GATES): "feat-бік для reviewer.mjs/flow.mdc, HEAD для rust/coverage.mjs", because feat зберігає safety net з інкрементальним scope; rust/coverage HEAD новіший (diffPath, baseline:skip API).

### Consequences

- Good, because change-файл автоматично у `<ws>/.changes/`; 27 тестів commands, 14 тестів review зелені.
- Good, because `flow init "fix mdc checker"` → L1; 330 тестів по сліду merge зелені.
- Bad, because кілька змінених subworkspace → `exit 1`, потрібен явний `--ws`.
- Bad, because `' spec'` (з пробілом) не ловить `spec` на початку рядка у detectLevel.
- Neutral, because Stryker local-bin fix (та сама сесія): `runStryker` мігрував на прямий запуск `bin/stryker.js` через `package.json#bin` (shebang node); `vitest-runner` знаходиться як сусід core у `node_modules/@stryker-mutator/`.

## More Information

**reviewer:** `review.mjs` — `reviewerPrompt`; тести `READ_ACCESS_RE`, `VERIFY_RE`. Коміт `b0308d4`, гілка `flow-review-read-access`.

**detectLevel:** `level.mjs` — `COMPLEXITY_GUARD_KEYS`: `['mdc','rego','policy','правил','rules','checker','check-',' spec','інваріант','invariant','кілька файлів','декілька','multiple files','суперечн','конфлікт','conflict','порушен','violation']`. Коміт `489d3e0`.

**DEFAULT_GATES:** `reviewer.mjs` DEFAULT_GATES = `['lint','coverage --changed']`. Коміт `c091708`.

**Stryker:** `npm/rules/js-lint/coverage/coverage.mjs` — `resolveLocalStrykerBin`: `createRequire(import.meta.url).resolve('@stryker-mutator/core/package.json')` → dirname → `bin.stryker`. Коміт `373ce42`.

## Update 2026-06-02

**Виправлення inline `--ws=value`:** Перша перевірка `rest.includes('--ws')` не ловила форму `--ws=value` (з `=` замість пробілу) — виявлено під час `flow review` та виправлено в тій самій сесії.

**WIP change-file stubs та WAL-логування:** `flow run --autonomous` виконував `defaultCommit` без change-файлу → наступний `verify` падав на changelog-consistency; `runPostToolUseFixCli` змінював файли без запису у WAL. Рішення: `defaultCommit` викликає `ensureWipChangeFiles` перед комітом (WIP-стаби у змінених workspace-ах, замінюються при `flow release`); `runPostToolUseFixCli` логує `hook_fix` у flow WAL. Обмеження: `ensureWipChangeFiles` покриває лише workspace-и глибини 1.

Файли: `npm/scripts/dispatcher/lib/active.mjs` (`ensureWipChangeFiles`, `defaultCommit`), `npm/scripts/post-tool-use-fix.mjs` (`runPostToolUseFixCli`). 219 тестів green. Гілка `flow-hook-coordination`.
