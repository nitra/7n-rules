## ADR Трекінг зв'язку spec→plan та статусу реалізації у plans

## Context and Problem Statement
У `docs/superpowers/specs/` поле `Статус:` виникло стихійно: ключ плаває між `**Статус:**`, `Status:`, `Статус:` (без bold), значення — довільний текст (`draft`, `Approved`, `Узгоджено (brainstorming)` тощо). У `docs/superpowers/plans/` статусного поля немає взагалі. Неможливо машиночитано відрізнити спеку без плану від спеки, для якої план уже створено, а план «в роботі» від «реалізованого в коді».

## Considered Options
* (A) Ручна конвенція — фіксований формат (наприклад, YAML frontmatter `status:` + `plan:` у specs; `status:` + `implemented:` у plans), що заповнюється вручну; опціонально — validation-правило у `n-cursor`.
* (B) Автоматично-похідний статус — `n-cursor` сам визначає наявність плану (за slug) і стан реалізації (всі чекбокси пройдені / запис у CHANGELOG / лінк на commit).
* (C) Гібрид — поля заповнюються вручну, а `n-cursor` будує зведену таблицю зі сканування цих полів.

## Decision Outcome
Chosen option: не обрано — brainstorming перервано до вибору механізму. Вимога сформульована: додати посилання spec→plan у specs (щоб відрізняти спеки ще без плану) та атрибут стану реалізації у plans.

### Consequences
* Good, because transcript фіксує очікувану користь: машиночитана прив'язка spec↔plan спростить навігацію і дасть змогу будувати зведений стан прогресу по фічах.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Поточний стан зафіксовано: `docs/superpowers/specs/*.md` мають нестандартизоване `Статус:`/`Status:`, у `docs/superpowers/plans/*.md` статусного поля немає. Superpowers v5.1.0 (`~/.claude/plugins/cache/claude-plugins-official/superpowers/5.1.0/skills/brainstorming/SKILL.md`) — шаблон `brainstorming` не диктує статусного поля. Кеш плагіна перезаписується при оновленні — патч superpowers нестійкий, тому будь-яке розширення має бути локальним. Brainstorming перервано питанням про `claude plugin uninstall superpowers@claude-plugins-official`.
