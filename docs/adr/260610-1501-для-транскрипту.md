# Дизайн: wire-log thinking та спостережуваних слідів omlx-викликів

**Дата:** 2026-06-10 · **Власник:** @vitaliytv · **Статус:** Draft

## Мета

Зафіксувати **кожен HTTP-виклик до omlx** разом із: повним response-тілом (`usage`, `finish_reason`), витягнутим `<think>…</think>`, спостережуваними слідами (latency, retries, ok/err), контекстом скіла (який скіл, файл, крок).

## Контекст

`npm/lib/omlx.mjs:callOmlx` — єдиний wire-чокпойнт. Зараз повертає лише `choices[0].message.content` — все інше губиться. Над ним два шари з контекстом: `docgen-gen.mjs:callOmlxMessages` (знає skill/file/step) і `fix/llm-worker.mjs` (знає skill/worker).

## Scope

**In:** `npm/tmp/` (gitignored, JSONL), зміни в `callOmlx` + `callOmlxMessages` + `llm-worker.mjs`, схема запису, `opts.context` контракт, `<think>` extraction, `.gitignore`.
**Out:** LLM-аналізатор, ротація логів, pi/Anthropic провайдери, зміна публічного API callOmlx.

## Рішення

### 1. Сховище: `npm/tmp/omlx-wire.jsonl`
- Поряд із `npm/lib/` і `npm/skills/`, gitignored, не в npm publish.
- Append-only, один рядок = один `callOmlx`-виклик незалежно від success/fail.

### 2. Схема JSONL-запису
```json
{
"ts": "<ISO 8601 UTC>",
"requestId": "<8-hex>",
"context": { "skill": "docgen|fix|null", "file": "...|null", "step": "E1|null" },
"request": { "url": "...", "model": "...", "temperature": 0.2, "maxTokens": 4096, "messages": [...] },
"response": {
"ok": true, "statusCode": 200,
"body": { /* повний JSON від сервера */ },
"content": "<raw content зі <think>>",
"thinking": "<витягнутий thinking або null>",
"finishReason": "stop|null",
"usage": { "promptTokens": 0, "completionTokens": 0, "totalTokens": 0 }
},
"meta": { "attempt": 1, "retries": 0, "elapsedMs": 0, "error": null }
}
```

### 3. `opts.context` контракт
`callOmlx(messages, model, opts)` отримує `opts.context = {skill, file, step}`. Caller передає сам — без глобальних змінних. Якщо не передано — `null` у всіх полях.

### 4. `<think>` extraction
- `response.content` — завжди сирий (зі `<think>`).
- `response.thinking` — regex `/^<think>([\s\S]*?)<\/think>/` усередині wire-логера; `null` якщо тегів нема.
- Caller отримує `content` без змін — поведінка `callOmlx` не міняється.

### 5. Де записувати
Після `return content` або `throw` (фінальна помилка) — `fs.appendFileSync` синхронно. Таймер від першого `attempt`. При помилці: `ok: false`, `error: err.message`.

### 6. Init каталогу
`fs.mkdirSync(dir, { recursive: true })` при першому записі — без окремого init-скрипта.

## Структура після реалізації
```
npm/lib/omlx.mjs              ← +wire-log
npm/skills/docgen/js/docgen-gen.mjs  ← +opts.context
npm/skills/fix/js/llm-worker.mjs     ← +opts.context
npm/tmp/omlx-wire.jsonl       ← новий, gitignored
```

## Обмеження
- **Розмір**: ~2–10 КБ/запис; ротація поза scope.
- **Приватність**: `messages` містять вихідний код, але `npm/tmp/` gitignored.
- **Sync**: `appendFileSync` — прийнятно для CLI/скілів, не для сервера.

```markdown
## ADR Логування omlx wire-trace та thinking

## Context and Problem Statement
Всі прямі HTTP-виклики до omlx-сервера проходять через `npm/lib/omlx.mjs:callOmlx`, але функція повертає лише текст відповіді, а `usage`, `finish_reason`, latency, retry-count і reasoning (`<think>`) губляться. Це унеможливлює накопичення даних для аналізу якості скілів.

## Considered Options
* Wire-log у `npm/tmp/omlx-wire.jsonl` з інструментуванням на рівні `callOmlx` + skill-контекст від callers.
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Wire-log у `npm/tmp/omlx-wire.jsonl`", because transcript явно визначив: wire-log (не aggregated trace), `tmp/` поряд із кодом, інструментування на двох рівнях (`callOmlx` + callers), `response.content` зберігати raw і одразу парсити `<think>` у окреме поле `thinking`.

### Consequences
* Good, because transcript фіксує очікувану користь: накопичення сирих даних для LLM-аналізу «де скіли недовизначені» та «де модель борсається» без зміни поведінки `callOmlx` для callers.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Wire-функція: `npm/lib/omlx.mjs:callOmlx` (рядки 56–102), `spawnSync('curl', ...)`.
- Callers: `npm/skills/docgen/js/docgen-gen.mjs:callOmlxMessages`, `npm/skills/fix/js/llm-worker.mjs`.
- Спека: `docs/specs/2026-06-10-omlx-wire-log-design.md`.
- `<think>` regex: `/^<think>([\s\S]*?)<\/think>/`.
- `npm/tmp/` — gitignored, append-only, не в npm publish.
```
