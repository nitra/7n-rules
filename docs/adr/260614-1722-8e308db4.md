Підсумок: **`/v1/models/fallback` фактично не існує** як HTTP-ендпоінт — OpenAPI-парсер раніше збрехав через дублікати ключів у відповіді. Реальний спосіб один — **`settings.json`**.

## Як увімкнути fallback

У `~/.omlx/settings.json` замінити:
```json
"model_fallback": false
```
на:
```json
"model_fallback": {
"enabled": true,
"mappings": {
"*": "gemma-4-e4b-it-OptiQ-4bit"
}
}
```

Зберегти, потім **перезапустити сервер** — omlx читає settings.json при старті.

Після цього будь-який запит з невідомим `model` (зокрема старий `gemma-4-e2b-it-4bit` з хардкоду в коді) тихо перенаправиться на `e4b-OptiQ-4bit`.

> Wildcard `"*"` — загальний catch-all; можна точніше: `{"mlx-community/*": "gemma-4-e4b-it-OptiQ-4bit"}`.
