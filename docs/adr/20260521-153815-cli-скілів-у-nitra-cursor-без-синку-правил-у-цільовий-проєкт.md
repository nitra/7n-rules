## ADR CLI скілів у `@nitra/cursor` без синку правил у цільовий проєкт

## Context and Problem Statement
Скіли (`lint`, `taze`, `publish-telegram` тощо) були доступні лише в репозиторіях із синхронізованими `.cursor/skills/` через `npx @nitra/cursor` (без аргументів). У сторонніх проєктах скіли були відсутні, хоча інструкції вже містяться в npm-пакеті. Потрібна можливість запускати скіли з будь-якого каталогу без встановлення `devDependency` і без синку правил.

## Considered Options
* Нова підкоманда `skill` в існуючому bin `n-cursor`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "нова підкоманда `skill` в існуючому bin `n-cursor`", because це дозволяє не публікувати окремий пакет і не плодити нові bin-файли — достатньо одного `npx @nitra/cursor skill …`.

### Consequences
* Good, because transcript фіксує очікувану користь: запуск скілів у будь-якому репо через `npx` без `devDependency` і без копіювання `SKILL.md` вручну.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- `npm/scripts/skills-cli.mjs` — логіка `list`, побудови промпту, виклику `cursor`/`claude`
- `npm/bin/n-cursor.js` — case `'skill'` → `runSkillsCli(args)`
- `npm/scripts/skills-cli.test.mjs` — unit-тести на `listSkillIds`, `normalizeSkillId`, `buildSkillPrompt`, `runSkillsCli`
- Промпт включає `skills/<id>/SKILL.md`, завдання та контекст CWD: `package.json`, `tsconfig.json`, `.n-cursor.json`

## ADR Спрощення UX skill-CLI до чотирьох канонічних форм

## Context and Problem Statement
Після першої реалізації CLI скілів з'явилася надмірна кількість форм запуску: `skill prompt <id>`, `skill claude <id>`, `skill cursor <id>`, `skill <id>`, а також окремі bins `n-skills` та `n-claude` і "claude-first" підкоманда `claude <id>` у `n-cursor`. Це ускладнювало документацію і ментальну модель.

## Considered Options
* Залишити всі форми (`prompt`, `claude`-first, `n-skills`, `n-claude`)
* Спростити до чотирьох форм під єдиною підкомандою `skill`

## Decision Outcome
Chosen option: "спростити до чотирьох форм під єдиною підкомандою `skill`", because користувач явно визначив саме ці чотири формати як бажані, а решта — видалено як надлишкові.

### Consequences
* Good, because transcript фіксує очікуваний результат: єдина точка входу `skill`, без `prompt`, без `n-skills`/`n-claude`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Канонічні форми після спрощення:
```bash
npx @nitra/cursor skill list
npx @nitra/cursor skill taze
npx @nitra/cursor skill cursor taze
npx @nitra/cursor skill cursor taze "онови залежності"
```
Видалено: `skill prompt …`, підкоманда `claude` у `n-cursor`, bins `n-skills` (`npm/bin/n-skills.js`) та `n-claude` (`npm/bin/n-claude.js`), поле `"n-skills"` і `"n-claude"` з `"bin"` у `npm/package.json`.
Версія пакета після спрощення: `1.13.72`.
