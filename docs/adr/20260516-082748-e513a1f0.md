## ADR Реструктуризація `rules/<id>/js/` → `fix/` / `lint/` / `policy/` (принцип «технологія визначає директорію»)

**Контекст:** Директорія `rules/<id>/js/` змішувала два різних канали виклику: JS-checks (запускає `npx @nitra/cursor check`) і JS-оркестратори лінту (запускає `bun run lint-X`). Це ускладнювало навігацію та не давало змоги visually відрізнити діагностику від лінт-infrastructure.

**Рішення/Процедура/Факт:**
- Уведено принцип: **технологія реалізації визначає директорію**, незалежно від того, який канал CLI викликає результат.
- `rules/<id>/fix/<concern>/check*.mjs` — JS-діагностика, яку запускає `npx @nitra/cursor check`.
- `rules/<id>/lint/lint.mjs` + помічники — JS-оркестратори, яких запускає `bun run lint-X` через `n-cursor lint-X`.
- `rules/<id>/policy/<concern>/*.rego` — Rego-діагностика, яку теж запускає `npx @nitra/cursor check`, але **policy/ є третім сиблінгом поруч із fix/ і lint/**, а не вкладений у fix/.
- Виконано 3 фази: 1) dual-mode у `discover-checkable-rules.mjs` і `run-rule.mjs`; 2) переміщення всіх 26 правил; 3) прибирання legacy `js/`-коду з інфраструктури.
- Видалено `npm/scripts/lint-conftest.mjs` — дублював `npx @nitra/cursor check` окремим `bun run lint-conftest` каналом. Єдиний канал для policy/rego-діагностики тепер — `n-cursor check`.

**Обґрунтування:** Поміщення `policy/` під `fix/policy/` (варіант B) або плоско в `fix/<concern>/` (варіант C) порушувало б ізоляцію rego-ecosystem: rego і JS мають різні runner-и (`runConftestBatch` vs dynamic `import`), різну namespace-конвенцію, різну роль `target.json`. Окремий `policy/`-сиблінг зберігає цю відмінність на рівні файлової системи, не ховаючи її всередині `fix/`.

**Розглянуті альтернативи:**
- Варіант B — `fix/policy/<concern>/`: відкинуто через зайвий рівень nesting і розчинення rego-блоку всередині JS-структури.
- Варіант C — `fix/<concern>/<name>.rego` поруч із `check.mjs`: відкинуто через ризик колізії імен, якщо у правила є і JS-концерн, і rego-концерн з однаковим ім'ям (`cspell`, `markdownlint`…).
- Залишити `policy/` і окремий `lint-conftest` канал: відкинуто як джерело плутанини — перевірки `n-cursor check` і `bun run lint` мали частковий overlap без чіткої межі.

**Зачіпає:** `npm/scripts/utils/discover-checkable-rules.mjs`, `npm/scripts/utils/run-rule.mjs`, `npm/bin/n-cursor.js`, `knip.json`, `npm/README.md`, `.cursor/rules/conftest.mdc`, `.cursor/rules/scripts.mdc`, всі `npm/rules/*/fix/`, `npm/rules/*/lint/`, `npm/rules/*/policy/`; версії `1.11.9` → `1.11.14`.
