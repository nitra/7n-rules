## ADR Нормалізація Markdown під час генерації замість lint-зачистки

## Context and Problem Statement
`npx @nitra/cursor` генерував `AGENTS.md` і `CLAUDE.md` з подвійними порожніми рядками (MD012): у `AGENTS.md` — через `expandMustacheSection` з inner-шаблоном `\n{{name}}\n` і `join('')` між елементами; у `CLAUDE.md` — через стик секцій, кожна з яких починалась/завершувалась порожнім рядком. Виправлення покладалось на наступний lint-прохід, що змушувало робити вручну мінімальний diff замість повної регенерації.

## Considered Options
* Виправити генератор: trim inner-шаблону + `join('\n')` + `collapseMultipleBlankLines()` безпосередньо при генерації
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Виправити генератор", because усуває першопричину MD012 під час генерації і прибирає залежність від lint-проходу для зачистки `AGENTS.md` / `CLAUDE.md`.

### Consequences
* Good, because `npx @nitra/cursor` після зміни одразу дає lint-чисті файли без ручного втручання.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Новий модуль: `npm/scripts/utils/generated-markdown.mjs` — функції `collapseMultipleBlankLines`, `expandMustacheSection`, `formatGeneratedMarkdownLines`, `renderAgentsTemplate`
- Тести: `npm/scripts/utils/generated-markdown.test.mjs`
- Змінено: `npm/bin/n-cursor.js` — імпорт утиліт замість inline-логіки; `collapseMultipleBlankLines` нормалізує `\n{3,}` → `\n\n`
- Версія: `npm/package.json` → `1.13.60`, запис у `npm/CHANGELOG.md` (`### Fixed`)

## ADR Виокремлення логіки генерації Markdown у спільний утилітний модуль

## Context and Problem Statement
Логіка розгортання Mustache-секцій і збирання рядків `CLAUDE.md` була розміщена inline у `npm/bin/n-cursor.js`. Це ускладнювало тестування та унеможливлювало повторне використання нормалізації Markdown поза CLI-точкою входу.

## Considered Options
* Виокремити у `npm/scripts/utils/generated-markdown.mjs` з покриттям unit-тестами
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Виокремити у `generated-markdown.mjs`", because дозволяє ізольовано тестувати `expandMustacheSection`, `collapseMultipleBlankLines` і `renderAgentsTemplate` без запуску CLI.

### Consequences
* Good, because transcript фіксує очікувану користь: unit-тести в `generated-markdown.test.mjs` проходять після рефакторингу.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- `npm/scripts/utils/generated-markdown.mjs` — публічні функції: `collapseMultipleBlankLines`, `expandMustacheSection`, `formatGeneratedMarkdownLines`, `renderAgentsTemplate`
- `npm/bin/n-cursor.js` — видалена дублікат-функція `expandMustacheSection`, додано імпорт із нового модуля
- Тест-команда: `bun test scripts/utils/generated-markdown.test.mjs`
