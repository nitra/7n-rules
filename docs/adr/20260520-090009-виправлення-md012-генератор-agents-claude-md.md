# Виправлення MD012 у генераторі AGENTS.md / CLAUDE.md

**Status:** Accepted
**Date:** 2026-05-20

## Context and Problem Statement

`npx @nitra/cursor` генерував `AGENTS.md` і `CLAUDE.md` з подвійними порожніми рядками (порушення MD012), покладаючись на наступний lint-прохід для зачистки. Два джерела проблеми: в `AGENTS.md` — `expandMustacheSection` використовував inner-шаблон `\n{{name}}\n` і `join('')` між елементами; у `CLAUDE.md` — стик `buildClaudeLintParallelismSectionLines()` (завершувалася `''`) і `buildClaudeSkillsSectionLines()` (починалася з `''`) давав `\n\n\n` перед `## Skills`. Inline-логіка в `npm/bin/n-cursor.js` унеможливлювала unit-тестування без запуску CLI.

## Considered Options

- Виправити генератор: нормалізація MD012 + перенесення логіки у окремий модуль з unit-тестами
- Залишити поточну поведінку (генератор → окремий lint-прохід для зачистки)

## Decision Outcome

Chosen option: "Виправити генератор: нормалізація MD012 + окремий модуль", because lint-прохід не повинен компенсувати артефакти генератора; рефакторинг у окремий модуль підтверджений запуском тестів і smoke-перевіркою результату.

### Consequences

- Good, because `AGENTS.md` і `CLAUDE.md` більше не містять подвійних порожніх рядків після генерації — окремий lint-крок для зачистки не потрібен.
- Good, because функції `collapseMultipleBlankLines`, `expandMustacheSection`, `renderAgentsTemplate`, `formatGeneratedMarkdownLines` покриті unit-тестами без запуску CLI.
- Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Новий модуль `npm/scripts/utils/generated-markdown.mjs`:
  - `collapseMultipleBlankLines(text)` — замінює `\n{3,}` на `\n\n`
  - `expandMustacheSection(template, section, items, prop)` — trim inner-шаблону + `join('\n')` між елементами
  - `renderAgentsTemplate(templateText, mdcBasenames, skillItems, commandItems)` — готовий вміст `AGENTS.md`
  - `formatGeneratedMarkdownLines(lines)` — збирає рядки `CLAUDE.md` з фінальним `\n`, застосовує `collapseMultipleBlankLines`
- `npm/bin/n-cursor.js`: inline-логіка замінена імпортом з нового модуля; дублікат `expandMustacheSection` видалений
- Тести: `npm/scripts/utils/generated-markdown.test.mjs` (bun:test); константа `TRIPLE_OR_MORE_NEWLINES` винесена статично (правило `e18e/prefer-static-regex`)
- Версія: `npm/package.json` → `1.13.60`, запис у `npm/CHANGELOG.md` (секція `### Fixed`)
