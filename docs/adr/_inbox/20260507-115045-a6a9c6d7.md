## ADR Розподіл відповідальності `lint-image` / `check image` та хеш-інвалідація AVIF

**Контекст:** У `@nitra/cursor` скрипт `lint-image` містив прапорець `--avif`, що плодило AVIF-артефакти під час кожного локального lint-запуску. Одночасно `check image` виконував лише перевірку, а не фактичний rewrite посилань у `.vue`/`.html`. Крім того, зміна вихідного PNG/JPEG не призводила до перегенерації вже наявного `.avif`, що залишало застарілі файли.

**Рішення/Процедура/Факт:**
1. `lint-image` у `package.json` — канонічна форма `npx @nitra/minify-image --src=. --write` (без `--avif`); присутність `--avif` є помилкою перевірки.
2. `npx @nitra/cursor check image` виконує повний триетапний pipeline:
- **Крок 1** — `npx @nitra/minify-image --src=. --write --avif` (best-effort, пропускається через `NITRA_CURSOR_NO_AVIF_RUN=1`).
- **Крок 2** — in-place rewrite посилань у `.vue`/`.html`: статичні `src="…png"`, `:src="…png"` (без JS-виразів), `import x from '…png'`; динамічні вирази, `data-src=` та SVG — пропускаються. Rewrite відбувається тільки якщо `.avif`-двійник реально існує; `write-then-fail` (вже переписані файли не відкочуються).
- **Крок 3** — orphan-cleanup: видаляє `.avif` без живих посилань у `.vue`/`.html`; пропускає opt-out пакети; пропускає `dist/`, `build/`, `android/`, `ios/`, `.output/`, `.nuxt/`, `.cache/`.
3. Хеш-кеш `.n-image-avif.tsv` (committed, рядки `<rel-path>\t<sha1>`): перед Кроком 1 для кожного raster-файла, поряд з яким є `.avif`, порівнюється поточний sha1 з кешем; якщо не збігається — `.avif` видаляється, щоб `@nitra/minify-image` перегенерував його. Кеш оновлюється в кінці check.
4. Резолвер шляхів `resolveImageCandidates` повертає впорядкований список кандидатів: для абсолютних `/x.png` — `<pkg>/public/x.png` → `<pkg>/x.png` → `<cwd>/x.png`; для відносних з `/` у середині — відносно файла-джерела + через `<pkg>/public/`; без жодного `/` — порожній список (alias-resolver невідомий, пропускається без fail).

**Обґрунтування:** Розмежування lint (лише оптимізація, ніяких нових форматів) та check (повний rewrite-pipeline) дозволяє `bun run lint` залишатися детермінованим і не плодити `.avif` поза контролем. Хеш-інвалідація усуває сценарій, коли оновлений PNG лишає застарілий `.avif` у git. Пріоритетний шлях через `public/` покриває стандарт Vite/Quasar, де `/path` відповідає `public/path`, а не кореню ОС.

**Розглянуті альтернативи:** Залишити `--avif` у `lint-image` (відхилено — lint не має сайд-ефектів у вигляді нових файлів); mtime-based інвалідація замість sha1 (відхилено — mtime непередбачуваний після git checkout).

**Зачіпає:** `.cursor/rules/n-image.mdc` (v1.4→1.6), `npm/mdc/image.mdc`, `npm/scripts/check-image.mjs` (функції `resolveImageCandidates`, `checkVueAvifImportsInPackage`, `cleanupOrphanAvifs`, `deleteStaleAvifs`, `sha1File`, hash-cache I/O), `npm/tests/check-image.test.mjs`, `npm/package.json` (1.8.184→1.8.195), `npm/CHANGELOG.md`, кореневий `package.json` (видалено `--avif` з `lint-image`).
