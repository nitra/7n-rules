## ADR Rule-centric структура директорій у npm-пакеті @nitra/cursor

**Контекст:** Пакет `@nitra/cursor` мав розпорошену структуру: текст правила у `npm/mdc/`, rego-поліси у `npm/policy/`, js-скрипти у `npm/scripts/`, умова автоактивації у `npm/bin/auto-rules.md`. Видалення правила вимагало ручного пошуку і видалення файлів у чотирьох різних місцях.

**Рішення/Процедура/Факт:** Вся власність правила переїжджає в `npm/rules/{rule}/`: `.mdc`-файл (`{rule}/{rule}.mdc`), умова автоактивації (`{rule}/auto.md`), rego-поліси (`{rule}/policy/`), js-скрипти (`{rule}/js/check.mjs`, `run.mjs`, `lint.mjs`). `npm/mdc/` перейменовано на `npm/rules/`. `npm/policy/` видалено. `npm/bin/auto-rules.md` розбито на окремі `rules/*/auto.md`. `bin/n-cursor.js` оновлює константи (`BUNDLED_RULES_DIR`), `getAvailableCheckRules()`, `getCheckScript()` і статичні імпорти `run-*`/`lint-*`. `auto-rules.mjs` агрегує умови з `rules/*/auto.md` замість `bin/auto-rules.md`. `lint-conftest.mjs` шукає rego у `rules/*/policy/`. `package.json#files` замінює `mdc` і `policy` на `rules`. Внутрішній snake_case у rego-namespace збережено (`image_avif/`, `js_bun_db/`).

**Обґрунтування:** Ціль формулював сам користувач: «якщо видалити цю директорію, то все правило і всі його атрибути очистяться» — єдина точка повного видалення правила без залишків.

**Розглянуті альтернативи:** Часткова реструктуризація лише `npm/scripts/` (зберігаючи `policy/` і `mdc/` окремо) — відкинута, оскільки не досягає мети атомарного видалення правила.

**Зачіпає:** `npm/mdc/` (→ `npm/rules/`), `npm/policy/` (видаляється), `npm/scripts/check-*.mjs`, `npm/scripts/run-*.mjs`, `npm/scripts/lint-ga.mjs`, `npm/scripts/lint-rego.mjs`, `npm/scripts/auto-rules.mjs`, `npm/scripts/lint-conftest.mjs`, `npm/bin/n-cursor.js`, `npm/package.json`

## Knowledge Шаблон заміни перекладів у `@nitra/tf` / `@nitra/tfm` / `@nitra/tfm-node`

**Контекст:** Скіл `npm/skills/abie-clean/SKILL.md` (секція 5) описує очистку проєкту від ru-середовища; переклади раніше не охоплювались явним прикладом.

**Рішення/Процедура/Факт:** До скілу додано розділ «Переклади» з двома паттернами. Для `@nitra/tf` (webpack) — замінювати значення у словнику `tr: { 'Привіт': 'Привет' }` → `{ 'Привіт': 'Hello' }`. Для `@nitra/tfm` — замінювати значення ключа `ru:` у мультимовному словнику: `{ Так: { ru: 'Да' } }` → `{ Так: { ru: 'Yes' } }`. Якщо англійська вже присутня поряд із `ru:` — прибирати `ru:`-запис.

**Обґрунтування:** Переклади є частиною ru-очистки (`abie-clean`), але без прикладів залишали неоднозначність щодо того, який запис видаляти (ключ `ru:` з мовою чи сам ключ словника).

**Розглянуті альтернативи:** Не обговорювалися — правило діє однозначно: значення `ru:` замінюється відповідним англійським перекладом.

**Зачіпає:** `npm/skills/abie-clean/SKILL.md` (секція 5), проєкти з `@nitra/tf`, `@nitra/tfm`, `@nitra/tfm-node`
