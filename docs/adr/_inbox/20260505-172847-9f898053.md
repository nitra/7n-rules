## ADR Скіп frontend-пакетів у `js-run.mdc` та `check-js-run.mjs`

**Контекст:** Правило `js-run.mdc` мало `alwaysApply: true` і пропонувало заміну `process.env.X` → `import { env } from 'node:process'`. LLM-агент застосував це до `site/src/main.js` у frontend-пакеті (Vue/Vite SPA), де `node:process` resolve'иться у `undefined` у браузерному бандлі — результат: `TypeError: Cannot read properties of undefined` у продакшн-збірці. Паралельно `check-js-run.mjs` сканував всі workspace-пакети, включно з frontend, і репортив хибні порушення, які frontend-розробник не міг коректно усунути.

**Рішення/Процедура/Факт:**
1. **`npm/mdc/js-run.mdc`** (v1.1 → v1.2): додано секцію **«Область застосування»** одразу після frontmatter — явно описує, що правило стосується виключно backend Node.js workspace-пакетів; у блоці `## process.env` додано callout із посиланням на цю секцію. Frontend-маркер: наявність `vite` у `devDependencies`.
2. **`npm/scripts/check-js-run.mjs`**: додано хелпер `packageJsonHasViteDevDependency(pkgJson)` та ранній вихід у `checkWorkspacePackage` — якщо пакет є frontend (vite у devDeps), весь скан `process.env`, `#conn/*`, OTEL configmap пропускається з `passFn`-повідомленням. Перевірка bunyan-залежності у `package.json` виконується раніше і залишається.
3. **`npm/tests/check-js-run-fixture.test.mjs`**: 2 нові тести — vite-пакет з прямим `process.env.NODE_ENV` → `check()` повертає 0; non-vite пакет із тим же кодом → повертає 1.
4. **`npm/package.json`**: версія `1.8.179` → `1.8.180`.
5. **`npm/CHANGELOG.md`**: запис `## [1.8.180] - 2026-05-05` з описом обох змін і передумовою — інцидент у `abie/b2b site/`.

**Обґрунтування:** Семантика frontend-маркера (`vite` у `devDependencies`) вже існувала в `auto-rules.mjs` (`packageJsonLacksViteDevDependency`) для вирішення, чи додавати правило `js-run` на рівні монорепо. Використано ту саму семантику в `check-js-run.mjs` для симетрії: одне правило — один маркер. Повний skip (а не частковий) обрано як простіший підхід, оскільки bunyan, `#conn/*`-аліаси та OTEL configmap неактуальні для frontend-пакетів за визначенням.

**Розглянуті альтернативи:** Частковий skip — пропускати лише скан `process.env` і conn-imports, але залишати перевірку bunyan-imports і OTEL configmap. Відхилено як надмірно ускладнений без практичної вигоди: bunyan у frontend не використовується, frontend `k8s/base/configmap.yaml` — це nginx/static, а не OTEL.

**Зачіпає:** `npm/mdc/js-run.mdc`, `npm/scripts/check-js-run.mjs`, `npm/tests/check-js-run-fixture.test.mjs`, `npm/package.json`, `npm/CHANGELOG.md`; публічний API: поведінка `npx @nitra/cursor check js-run` для монорепо із Vite-пакетами.
