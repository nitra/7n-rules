## ADR Правило обов'язкового CHANGELOG для воркспейсів Bun-монорепо

**Контекст:** У проєкті вже діє вимога до `CHANGELOG.md` для npm-модуля (`n-npm-module.mdc`). Постала потреба поширити аналогічну вимогу на звичайні (не npm) субпакети Bun-монорепо — з окремим `CHANGELOG.md` у кожному воркспейсі, без єдиного спільного файлу.

**Рішення/Процедура/Факт:** Заплановано нове правило (`.cursor/rules/` + `npm/mdc/`), яке вимагає: кожен воркспейс Bun-монорепо повинен мати власний `CHANGELOG.md` у форматі [Keep a Changelog](https://keepachangelog.com/uk/1.1.0/) (українська мова, новіші версії зверху, записи виду `## [версія] - YYYY-MM-DD` з секціями `### Added`, `### Changed`, `### Fixed`, `### Removed`). Злиття в `main` без оновлення відповідного `CHANGELOG.md` — заборонено. Реалізацію перевірки планується додати до `npm/scripts/check-*.mjs` за аналогією з `check-npm-module.mjs`.

**Обґрунтування:** Підхід взятий з наявної практики npm-module: вимога на рівні правила примусово забезпечує документування змін. Роздільні файли на воркспейс — бо монорепо може містити незалежно релізовані пакети з різними версіями.

**Розглянуті альтернативи:** Єдиний спільний `CHANGELOG.md` на кореневому рівні — відхилено: не відповідає семантиці незалежних воркспейсів у монорепо.

**Зачіпає:** `.cursor/rules/` (нове `.mdc`-правило), `npm/mdc/` (нова MDC-копія), `npm/scripts/` (новий `check-*.mjs`), CI/CD pipeline (Gate перед merge в `main`).
