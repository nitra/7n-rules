## Knowledge Зручний синтаксис паузи через `node:timers/promises`
**Контекст:** У кодових сесіях зустрічались паузи через громіздку конструкцію `await new Promise(resolve => setTimeout(resolve, ms))`, яку слід замінювати на лаконічнішу.
**Рішення/Процедура/Факт:** Використовувати `setTimeout` з `node:timers/promises` — `import { setTimeout } from 'node:timers/promises'` та `await setTimeout(ms)` — замість ручного обгортання у `new Promise`.
**Обґрунтування:** Стандартна Node.js API (починаючи з v15) надає `promisified` версії таймерів у `node:timers/promises`. Це коротше, читабельніше і не потребує самостійного промісу.
**Розглянуті альтернативи:** Не обговорювалися — правило задане безпосередньо користувачем.
**Зачіпає:** Будь-який Node.js / Bun код у проєкті, де є затримки через `setTimeout` у `async` функціях.
