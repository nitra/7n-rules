## ADR Автоматична LLM-нормалізація ADR-чернеток у `docs/adr/`

**Контекст:** Stop-hook `capture-decisions.sh` накопичив 156 сирих чернеток у `docs/adr/_inbox/`, жодна з яких не була промотована вручну до структурованого ADR — бо ручна промоція занадто трудомістка, а автоматики не існувало.

**Рішення/Процедура/Факт:**
- Видаляється окремий каталог `_inbox/`; усі ADR-файли (сирі і канонічні) живуть безпосередньо в `docs/adr/`.
- Маркером «ще не нормалізовано» є наявність `session:` у YAML frontmatter: поки поле присутнє — файл сирий (`20260515-090910-88a66cb5.md`); після нормалізації frontmatter знімається і файл перейменовується у `<slug>.md`.
- Stop-hook рахує файли з `session:` у frontmatter; при досягненні порогу `ADR_PROMOTE_THRESHOLD` (default `30`) асинхронно запускається LLM-нормалізатор.
- LLM повертає JSON-операції: `rewrite` (переписати, зняти frontmatter, новий slug), `delete` (тривіальний зміст або дублікат), `merge-into` (зміст вже покрито іншим файлом — дописати деталі туди, цей видалити).
- Кластеризація реалізується двофазно: (a) перший виклик LLM → тільки JSON-кластери `[{title, sources:[...]}]`, (b) окремий LLM-виклик на кожен кластер → фінальний ADR-текст.
- **Зміни не комітяться автоматично** — скрипт залишає модифіковані/видалені/нові файли в робочому дереві. Розробник переглядає `git diff` / `git status` і сам вирішує, що прийняти.

**Обґрунтування:** Відмова від `_inbox/` усуває штучний бар'єр «коли і куди переносити». Frontmatter-маркер — вже наявна метадата, дармовий сигнал без додаткових полів. Поріг-тригер дає природне «вікно стабілізації» (рішення зафіксовані в кількох сесіях, перш ніж синтезуватися). Відсутність авто-коміту зберігає людський контроль: git diff стає review-інтерфейсом без додаткового інструментарію.

**Розглянуті альтернативи:**
- Continuous-промоція на кожен Stop-hook — відхилена: сесія може ще змінюватись, плодить ADR-и з пів-сирих даних.
- Окремий каталог `_inbox/_promoted/` як буфер після промоції — відхилений: зайвий шар структури; git history і так зберігає оригінальні файли.
- Авто-коміт batch `"adr: normalize"` — відхилений користувачем: розробник хоче бачити `git diff` і самостійно приймати рішення.
- Однофазний LLM-виклик (30 файлів → N ADR-ів одразу) — збережено як fallback, але двофазний (JSON-кластеризація → окремий синтез) рекомендовано як надійніший.

**Зачіпає:** `docs/adr/` (структура каталогу), `npm/.claude-template/hooks/capture-decisions.sh` (trigger-логіка), новий скрипт `npm/.claude-template/hooks/normalize-decisions.sh` (майбутній), `.cursor/rules/n-adr.mdc` (документація правила), `.n-cursor.json` (конфігурація через `"adr"` / `"adr-normalize"` прапори).
