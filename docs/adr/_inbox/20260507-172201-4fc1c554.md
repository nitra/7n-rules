## ADR Міграція перевірок `@nitra/cursor` на conftest / Rego

**Контекст:** Перевірки структури GitHub Actions workflow-файлів (зокрема `validateCleanGaWorkflows`) реалізовані на JavaScript у `npm/scripts/check-ga.mjs`. Для стандартизації policy-логіки та покращення читабельності й розширюваності вирішено частину перевірок перенести у декларативний Policy Language (OPA Rego) та запускати через `conftest`.

**Рішення/Процедура/Факт:** Створено PoC-порт перевірки `validateCleanGaWorkflows` у файлі `npm/policy/ga/clean-ga-workflows.rego`. Правила записуються у вигляді `deny contains msg if { … }` — `conftest` читає YAML-файл як `input` і матчить їх. Запуск локально: `conftest test .github/workflows/clean-ga-workflows.yml -p npm/policy/ga`. У `npm/scripts/lint-ga.mjs` вже інтегровано виклик `conftest test` як один із кроків pipeline (після `github-actionlint` та `uvx zizmor`). Підготовлено Telegram-публікацію про міграцію для команди (розробники + менеджери).

**Обґрунтування:** Rego дозволяє виражати policy-логіку декларативно й більш стисло, ніж еквівалентний JS. `conftest` — стандартний інструмент для policy-as-code у YAML/JSON-конфігах; він добре відомий DevOps-спільноті. Довгострокова ціль — зменшити JS-специфічну логіку перевірок і перейти на перевірену policy-платформу (OPA/Rego), що спрощує аудит і розширення правил.

**Розглянуті альтернативи:** Збереження перевірок повністю у JS (`check-ga.mjs`) — явно не обговорювалося як альтернатива; поточна міграція відбувається поступово (PoC-підхід) без повного переписування одразу.

**Зачіпає:** `npm/policy/ga/clean-ga-workflows.rego` (новий файл), `npm/scripts/lint-ga.mjs` (інтеграція `conftest`), `npm/scripts/check-k8s.mjs`, `.cursor/rules/conftest.mdc` (нове правило), `docs/adr/_inbox/` (зафіксовані ADR-и сесії).
