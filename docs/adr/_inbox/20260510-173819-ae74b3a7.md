## ADR: Rego-першим при реалізації пер-документних перевірок k8s

**Контекст:** У проєкті `@nitra/cursor` існував великий JS-скрипт `check-k8s.mjs` (~6580 рядків) з ~50 exported `*Violation`-функціями, що перевіряють Kubernetes-маніфести. Водночас зʼявилася інфраструктура `conftest` / OPA, і частина перевірок була перенесена в `npm/policy/k8s/manifest/manifest.rego`, але більшість пер-документних правил залишалась тільки у JS.

**Рішення/Процедура/Факт:** Проведено аналіз `check-k8s.mjs` і класифікацію всіх функцій на три категорії (pure per-document, path-conditional, cross-file/FS/autofix). Прийнято рішення перенести пер-документний шар у Rego та встановити правило «Rego першим» як стандарт проєкту:
- Розширено `npm/policy/k8s/manifest/manifest.rego` (Deployment cpu+memory у `resources.requests`, Hasura image pin, canonical topologySpreadConstraints, заборона namespace у kustomize-managed файлах).
- Створено 10 нових rego-пакетів: `gateway`, `kustomization`, `svc_yaml`, `svc_hl_yaml`, `base_manifest`, `base_kustomization`, `kustomize_managed`, `hasura_configmap`, `hasura_httproute`, `hpa_pdb`.
- Для кожного пакету написано `*_test.rego` (сумарно 93 тести, усі проходять через `conftest verify`).
- Оркестрація: `lint-conftest.mjs` розширено новими TARGETS, `lint-rego.mjs` — опційний `conftest verify`.
- `.cursor/rules/conftest.mdc` (alwaysApply) доповнено секцією «Пріоритет: Rego першим» з декізія-деревом.
- JS-шар (`check-k8s.mjs`) залишається авторитетним для cross-file / FS / autofix; rego — швидкий gate для одиничного маніфесту.

**Обґрунтування:** Rego-полісі мають унікальні переваги для single-document перевірок: явна декларативна семантика, незалежне версіонування тестів, підтримка IDE через OPA extension, можливість запускати через `conftest test` поза JS-оркестратором. Перенесення зменшує coupling між логікою перевірки та Node.js-рантаймом і дає додатковий шар захисту (два незалежні механізми).

**Розглянуті альтернативи:** Залишити всі перевірки у JS (відхилено — rego дає кращу ізоляцію); повністю відмовитись від JS і перенести навіть cross-file логіку в rego (відхилено — OPA не має доступу до FS і не підтримує autofix).

**Зачіпає:** `npm/policy/k8s/**`, `npm/scripts/lint-conftest.mjs`, `npm/scripts/lint-rego.mjs`, `npm/scripts/check-k8s.mjs` (cross-file частина залишена), `.cursor/rules/conftest.mdc`, `npm/CHANGELOG.md`, `npm/package.json` (1.8.222).
