## ADR Міграція перевірки невикористаних залежностей з `depcheck` на `knip`

**Контекст:** Проєкт використовував `depcheck` для перевірки невикористаних залежностей — окремим кроком у `js-run.mdc` та у CI (крок `npx depcheck` у `npm-publish.yml`). Необхідно замінити `depcheck` на `knip` і консолідувати перевірку в правилі `js-lint.mdc` (разом з oxlint/eslint/jscpd), а в CI заборонити `depcheck` полісі.

**Рішення/Процедура/Факт:**
- `npm/mdc/js-lint.mdc` + `.cursor/rules/n-js-lint.mdc` (v1.17 → v1.18): додано секцію «knip», `bunx knip` включено в канонічний `lint-js` скрипт (`bunx oxlint --fix && bunx eslint --fix . && bunx jscpd . && bunx knip`) та у приклад CI workflow; задокументований канон `knip.json` з `ignoreDependencies: ["graphql"]`.
- `npm/mdc/js-run.mdc` + `.cursor/rules/n-js-run.mdc` (v1.6 → v1.7): прибрана секція «depcheck у GitHub Actions з path-фільтром».
- `npm/mdc/ga.mdc` + `.cursor/rules/n-ga.mdc` (v1.8 → v1.9): додано правило «`depcheck` не використовувати у `.github/workflows/*.yml` — мігровано на `knip`».
- `npm/policy/ga/workflow_common/workflow_common.rego`: `deny`-правило на наявність `depcheck` у будь-якому `run:`-кроці workflow.
- `npm/policy/js_lint/package_json/package_json.rego` та `lint_js_yml.rego`: оновлено `canonical_lint_js` та перевірку на присутність `bunx knip` у CI.
- `npm/scripts/check-js-run.mjs`: прибрано `checkDepcheckInWorkflows`, `readAllWorkflowFiles`, відповідний блок перевірки.
- `npm/scripts/utils/depcheck-workflow.mjs`: файл видалено повністю.
- `npm/scripts/check-js-lint.mjs`: оновлено `CANONICAL_LINT_JS`, додано `checkKnipConfig` (перевірка наявності `knip.json` з `graphql` у `ignoreDependencies`).
- `.github/workflows/npm-publish.yml`: прибрано крок `npx depcheck`.
- `.github/workflows/lint-js.yml`: додано `bunx knip` до CI-кроку «Eslint».
- `.github/workflows/lint-ga.yml`: додано крок `Install conftest` (conftest потрібен для `runConftestBatch` у `check-ga.mjs`).
- `package.json` (root): `lint-js` оновлено з `&& bunx knip`.
- `knip.json` (root): новий файл з `ignoreDependencies`, `ignoreBinaries`, `workspaces.npm.entry` та `ignore` для цього монорепо.
- `npm/package.json`: 1.9.4 → 1.9.5; `npm/CHANGELOG.md`: запис `[1.9.5]`.

**Обґрунтування:** `knip` є сучаснішим інструментом і покриває більше категорій (невикористані файли, re-exports, бінарники) порівняно з `depcheck`. Перевірка залежностей логічно належить до lint-фази (разом з іншими статичними аналізаторами), а не до окремого CI-кроку в `js-run`. `depcheck` потребував окремого path-scoped workflow кроку, що ускладнювало правила і розповсюджувало логіку по різних місцях. Конфіг `ignoreDependencies: ["graphql"]` потрібен, бо `graphql` є peer-залежністю пакетів (`@nitra/graphql-request`) і не імпортується напряму в проєкті.

**Розглянуті альтернативи:** Не обговорювалися — перехід на `knip` та розміщення в `js-lint` задані користувачем явно.
