## ADR Правило «depcheck у path-scoped GitHub Actions» для backend-пакетів

**Контекст:** Backend-пакети в монорепо мають власні GitHub Actions workflow з `paths:`-фільтром (наприклад, `cron-jobs/refund-loyalty-points/**`). Без явного кроку `depcheck` у такому workflow невикористані або відсутні залежності пакета залишаються непоміченими на CI.

**Рішення/Процедура/Факт:**

- У `npm/mdc/js-run.mdc` (v1.2 → v1.3) додано секцію **«depcheck у GitHub Actions з path-фільтром»**: якщо `on.push.paths` / `on.pull_request.paths` в `.github/workflows/*.yml` містять лише шляхи одного workspace-пакета, у job обовʼязковий крок `npx depcheck --ignores="graphql,bun"` з `working-directory: <rootDir>`; `graphql` і `bun` є мінімальним обовʼязковим набором ignores, але список може бути розширений.
- Створено `npm/scripts/utils/depcheck-workflow.mjs` — утиліта, що визначає, чи всі `paths:`-записи workflow ведуть у один каталог пакета, і перевіряє наявність відповідного `depcheck`-кроку з правильним `working-directory` та достатніми `--ignores`.
- У `npm/scripts/check-js-run.mjs` інтегровано виклик `checkDepcheckInWorkflows` для кожного workspace-пакета; workflow зчитуються один раз через `gha-workflow.mjs`.
- До `npm/tests/check-js-run-fixture.test.mjs` додано 9 нових тест-кейсів: відсутні workflow, глобальні paths, path-scoped без depcheck, неправильний `working-directory`, відсутні або неповні `--ignores`, валідні розширені ignores.
- `.github/workflows/npm-publish.yml` доповнено кроком `npx depcheck --ignores="graphql,bun,bun:test,@nitra/cursor"` для відповідності власному правилу.
- Версія пакета bumped до `1.8.184`, CHANGELOG оновлено.

**Обґрунтування:** Автоматична перевірка гарантує, що будь-який backend-пакет із path-scoped CI не пропустить проблеми із залежностями. `graphql` та `bun` є обовʼязковими в ignores, бо вони використовуються в середовищі виконання Bun/GraphQL але не відображаються як прямий `import` у коді.

**Розглянуті альтернативи:** не обговорювалися.

**Зачіпає:** `npm/mdc/js-run.mdc`, `.cursor/rules/n-js-run.mdc`, `npm/scripts/utils/depcheck-workflow.mjs` (новий файл), `npm/scripts/check-js-run.mjs`, `npm/tests/check-js-run-fixture.test.mjs`, `.github/workflows/npm-publish.yml`, `npm/package.json`, `npm/CHANGELOG.md`.
