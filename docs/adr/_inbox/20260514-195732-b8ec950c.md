## ADR Реорганізація `npm/` на rule-centric структуру `npm/rules/`

**Контекст:** Артефакти одного правила (`@nitra/cursor`) були розпорошені по трьох директоріях: `npm/mdc/` (текст правила), `npm/policy/` (Rego), `npm/scripts/` (JS-скрипти) та `npm/bin/auto-rules.md` (умови авто-активації). Це ускладнювало видалення, аудит і навігацію по правилах.

**Рішення/Процедура/Факт:** Ввести `npm/rules/{rule}/` як єдиний каталог на правило. Всередині: `{rule}.mdc`, `auto.md` (умова авто-активації), опційні `policy/` (Rego) та `js/` (JS-скрипти, без суфікса правила у назвах файлів — `check.mjs`, `lint.mjs`, `run.mjs`). Назви каталогів — kebab-case. Стара `npm/mdc/` і `npm/policy/` зникають; `npm/scripts/` залишається виключно для спільної інфраструктури. `bin/auto-rules.md` замінюється `rules/*/auto.md`, які читаються `auto-rules.mjs` динамічно.

**Обґрунтування:** Видалення правила = видалення одного каталогу без залишків. Зрозуміла локальність: все, що стосується правила, в одному місці. Узгоджується з наявною структурою `npm/policy/`, яка вже була директорно-організована.

**Розглянуті альтернативи:** Реорганізація тільки `npm/scripts/` без переносу `mdc/` і `policy/` — відхилена як половинчаста, оскільки не досягає атомарності «видалення = повне видалення правила».

**Зачіпає:** `npm/mdc/` (видаляється), `npm/policy/` (видаляється), `npm/scripts/check-*.mjs`, `npm/scripts/lint-ga.mjs`, `npm/scripts/lint-rego.mjs`, `npm/scripts/run-*.mjs`, `npm/scripts/auto-rules.mjs`, `npm/scripts/lint-conftest.mjs`, `npm/bin/n-cursor.js`, `npm/bin/auto-rules.md` (видаляється), `npm/package.json` (`files` + `scripts`).
