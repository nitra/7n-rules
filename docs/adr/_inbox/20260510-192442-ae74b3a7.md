## Skill Priority

When multiple skills could apply, use this order:

1. **Process skills first** (brainstorming, debugging) - these determine HOW to approach the task
2. **Implementation skills second** (frontend-design, mcp-builder) - these guide execution

"Let's build X" → brainstorming first, then implementation skills.
"Fix this bug" → debugging first, then domain-specific skills.

## ADR Міграція пер-документних rego-правил у `check-*.mjs` за патерном «rego-authoritative»

**Контекст:** Пер-документні JS-предикати в `check-abie.mjs` і rego-пакети в `npm/policy/abie/` дублювали одну й ту саму логіку. Виявлений bug (`health_check_policy.rego` читав `spec.config` замість `spec.default.config`) показав, що розбіжність між JS і rego може жити непомічено без механізму крос-перевірки.

**Рішення/Процедура/Факт:**
- Введено патерн **«rego-authoritative»**: JS-предикати, що дублюються в rego, видаляються з `check-*.mjs`. Orchestrator-функція в JS замість власної логіки викликає `runConftestBatch` (новий `npm/scripts/utils/run-conftest-batch.mjs`), а cross-file логіка лишається в JS.
- `check-abie.mjs`: видалено `abieBaseHttpRouteHostnamesErrors`, `deploymentDocumentHasAbieBasePreemNodeSelector`, `parseCleanMergedIgnoreBranches`, `ignoreBranchesIncludesRequired`, `validateAbieHcPolicy` та пов'язані хелпери. Orchestrator-функції делегують conftest 4 пакетам: `abie.base_deployment_preem`, `abie.clean_merged_ignore_branches`, `abie.health_check_policy`, `abie.http_route_base`.
- `check-ga.mjs`: на початку `check()` додано `runAllGaRego()` — 5 викликів `runConftestBatch` (4 per-workflow + 1 `ga.workflow_common`). `lint-ga.mjs` спрощено: прибрано `CONFTEST_TARGETS`, `runConftestStep`, `runConftestWorkflowCommon`; функція `runLintGaCli` тепер `async` і завершується викликом `await checkGa()`.
- `npm/scripts/utils/run-conftest-batch.mjs`: батчений виклик `conftest test --output json`, повертає структурований список порушень; hard-fail якщо `conftest` відсутній у PATH.
- Додано 4 нові rego-пакети з `_test.rego`: `abie.base_deployment_preem` (8 тестів), `abie.clean_merged_ignore_branches` (7 тестів), розширено `abie.health_check_policy` (10 тестів, виправлено `spec.default.config`), `abie.http_route_base` отримав перший `_test.rego` (10 тестів). Усього 128 rego-тестів.

**Обґрунтування:** Два джерела істини (JS + rego) накопичують дрейф без видимого сигналу — bug у `health_check_policy.rego` це довів. Rego-authoritative виключає дублювання; batched conftest (~80-150ms на namespace незалежно від кількості файлів) не дає помітного сповільнення. Крос-чек тест (`cross-check-rego-abie.test.mjs`) підтвердив drift-детекцію до видалення JS-функцій, після чого його видалено як надлишковий (JS-сторони більше немає).

**Розглянуті альтернативи:**
- **Plan A — «JS authoritative + rego-mirror»** (лишити обидва, додати крос-чек тести): відхилено як таке, що зберігає дублювання і вимагає постійної підтримки двох реалізацій.
- **Plan C — JS-проксі через `opa eval`**: відхилено — аналогічне сповільнення, складніші тести, половинчасте рішення.
- **Спавн на кожен файл окремо** (~50-100ms per file): відхилено на користь batched-спавну на namespace.

**Зачіпає:** `npm/scripts/check-abie.mjs`, `npm/scripts/check-ga.mjs`, `npm/scripts/lint-ga.mjs`, `npm/scripts/utils/run-conftest-batch.mjs` (новий), `npm/bin/n-cursor.js`, `npm/policy/abie/base_deployment_preem/`, `npm/policy/abie/clean_merged_ignore_branches/`, `npm/policy/abie/health_check_policy/`, `npm/policy/abie/http_route_base/`, `npm/tests/check-abie.test.mjs`, `npm/tests/check-ga.test.mjs`, `npm/tests/lint-ga.test.mjs`, `.cspell.json`.
