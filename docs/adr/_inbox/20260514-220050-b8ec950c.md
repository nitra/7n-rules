## ADR Реорганізація `@nitra/cursor`: rule-centric структура `npm/rules/`

**Контекст:** Файли одного правила були розпорошені по чотирьох директоріях (`npm/mdc/`, `npm/policy/`, `npm/scripts/`, `npm/tests/`), що ускладнювало навігацію та не давало можливості «видалити правило одним кроком» без ризику залишити рештки.

**Рішення/Процедура/Факт:** Кожне правило переміщено в єдину директорію `npm/rules/{rule}/` із підструктурою: `{rule}.mdc` — текст правила; `policy/` — Rego-поліси (раніше `npm/policy/{snake}/`); `js/check.mjs`, `js/run.mjs`, `js/lint.mjs` — скрипти (раніше `npm/scripts/check-*.mjs`, `run-*.mjs`, `lint-*.mjs`); `js/*.test.mjs` — тести, співрозташовані з джерелами; `auto.md` — умова автоактивації (розрізана з колишніх `npm/bin/auto-rules.md` / `auto-skills.md`). Скіли отримали паралельну структуру `npm/skills/{skill}/SKILL.md + auto.md + js/`. Директорії `npm/mdc/`, `npm/policy/`, `npm/bin/auto-rules.md`, `npm/bin/auto-skills.md` видалено. `bin/n-cursor.js` переключено на `BUNDLED_RULES_DIR = rules/`, нові резолвери `.mdc` і `check.mjs`; `lint-conftest.mjs` і `run-conftest-batch.mjs` — на `rules/{rule}/policy/`. `package.json#files`: `mdc`/`policy` прибрано, `rules` додано, негативні glob-патерни для тестів і fixtures. Версія 1.9.21 → 1.9.23. Правило зафіксовано в `.cursor/rules/scripts.mdc` v1.5.

**Обґрунтування:** Атомарність: видалення `npm/rules/{rule}/` повністю очищає правило без решток у чотирьох місцях. Локальність: усі артефакти правила — в одному місці, що спрощує навігацію, рев'ю та додавання нових правил. Однорідність зі структурою `npm/policy/` (вже мала per-rule каталоги) розширена на всі типи артефактів. Тести поряд із джерелами — ідіоматичний Go/Rust-підхід для монорепо-пакету.

**Розглянуті альтернативи:** 1. Зберегти `npm/mdc/` як назву кореневого каталогу — відхилено, назва вже не відображала вміст (не тільки `.mdc`). 2. Тримати тести в `npm/tests/` — відхилено на користь co-location; залишились тільки 3 крос-правильні файли. 3. Зберегти повне ім'я файла в каталозі (`check-docker.mjs`) — відхилено, ім'я правила вже несе директорія. 4. Зберегти `npm/skills/` незмінним — відхилено, скіли отримали ту саму структуру (auto.md + js/).

**Зачіпає:** `npm/rules/` (26 правил), `npm/skills/` (7 скілів), `npm/bin/n-cursor.js`, `npm/scripts/lint-conftest.mjs`, `npm/scripts/utils/run-conftest-batch.mjs`, `npm/scripts/auto-rules.mjs`, `npm/scripts/auto-skills.mjs`, `npm/package.json` (fields: `files`, `version`, `scripts.test`), `npm/CHANGELOG.md`, `.cursor/rules/scripts.mdc`, `docs/superpowers/specs/2026-05-14-npm-rules-restructure-design.md`.
