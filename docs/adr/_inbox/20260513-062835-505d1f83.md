## ADR Видалення дубльованих JS-констант канону на користь Rego-полісі
**Контекст:** У `check-*.mjs` скриптах були JS-функції (`CANONICAL_LINT_JS`, `isAllowedRootDevDependency`, `checkVscodeStylelint`, `checkExtensionsRecommendation`, `checkVscodeNginx`), які дублювали логіку, вже перенесену до `npm/policy/<rule>/**/*.rego`. Підтримка обох призводила до дрифту: одна зміна потребувала правки в двох місцях одночасно.
**Рішення/Процедура/Факт:** Для кожного виявленого дубля: (1) видалено JS-функцію/константу; (2) залишено або створено авторитетну rego-полісі; (3) JS-оркестратор викликає rego через `runConftestBatch` там, де потрібна умовна перевірка (graphql, nginx); (4) для безумовних правил (style-lint) rego зареєстровано глобально в TARGETS у `lint-conftest.mjs`.
**Обґрунтування:** Єдине джерело правди запобігає дрифту між JS і rego. При додаванні нового прапора (`--no-config-hints`) було достатньо змінити одне місце — rego-рядок. До рефакторингу одна зміна потребувала правки у JS-константі + rego-константі + тестах обох.
**Розглянуті альтернативи:** Залишити JS як авторитетне джерело і видалити rego — не розглядалося; rego краще підходить для декларативної перевірки структури JSON/YAML файлів.
**Зачіпає:** `npm/scripts/check-js-lint.mjs`, `npm/scripts/check-bun.mjs`, `npm/scripts/check-style-lint.mjs`, `npm/scripts/check-graphql.mjs`, `npm/scripts/check-nginx-default-tpl.mjs`, `npm/scripts/check-js-run.mjs`, `npm/scripts/lint-conftest.mjs`, `npm/policy/js_lint/package_json/`, `npm/policy/bun/package_json/`, `npm/policy/style_lint/vscode_extensions/`, `npm/policy/style_lint/vscode_settings/`, `npm/policy/graphql/vscode_extensions/`, `npm/policy/nginx_default_tpl/vscode_extensions/`, `npm/policy/nginx_default_tpl/vscode_settings/`, `npm/policy/js_run/jsconfig/`, `npm/policy/image_avif/package_json/`

## ADR Два патерни активації rego-полісі: глобальний TARGETS і JS-оркестратор
**Контекст:** `lint-conftest.mjs` фільтрує полісі лише за активними правилами репо (`.n-cursor.json`), але не вміє визначати умовні per-workspace або per-file контексти (наприклад: «лише якщо є `gql\`…\``», «лише якщо є `default.conf.template`», «лише якщо пакет — backend з `src/`»).
**Рішення/Процедура/Факт:** Встановлено два паттерни: (A) **Глобальний TARGETS** — для безумовних правил; rego-полісі реєструються у масиві `TARGETS` у `lint-conftest.mjs` з `rule: '<name>'` для фільтра за `.n-cursor.json`. (B) **JS-оркестратор** — для умовних правил; JS-скрипт `check-<rule>.mjs` спочатку перевіряє умову (сканує файли, визначає тип пакета), потім викликає `runConftestBatch` з відповідним `policyDir` і namespace. Rego-полісі не реєструються у `lint-conftest.mjs` TARGETS.
**Обґрунтування:** graphql-полісі на `.vscode/extensions.json` без умови `gql` дала б false-positive у кожному проєкті без GraphQL; `js_run.jsconfig` перевіряє лише backend-пакети з `src/`, а `demo/` — Vite-frontend і правомірно виключається.
**Розглянуті альтернативи:** Додати до TARGETS з умовою через `walk.match` — неможливо: `walk.match` оперує лише шляхами файлів, не вмістом дерева.
**Зачіпає:** `npm/scripts/lint-conftest.mjs` (TARGETS), `npm/scripts/check-graphql.mjs`, `npm/scripts/check-nginx-default-tpl.mjs`, `npm/scripts/check-js-run.mjs`, `npm/policy/graphql/`, `npm/policy/nginx_default_tpl/`, `npm/policy/js_run/jsconfig/`

## ADR Прапор `--no-config-hints` у канонічному `lint-js` скрипті
**Контекст:** `bunx knip` виводив інформаційну секцію «Configuration hints» (`Remove from ignoreDependencies`) щоразу під час `bun run lint`, засмічуючи вивід без реального порушення правил.
**Рішення/Процедура/Факт:** Додано `--no-config-hints` до канонічного рядка: `bunx oxlint --fix && bunx eslint --fix . && bunx jscpd . && bunx knip --no-config-hints`. Оновлено в: `npm/scripts/check-js-lint.mjs` (`CANONICAL_LINT_JS`), `npm/policy/js_lint/package_json/package_json.rego` (`canonical_lint_js`), `package.json` root (`scripts.lint-js`), `npm/mdc/js-lint.mdc` + дзеркало `.cursor/rules/n-js-lint.mdc` (обидва блоки прикладу), `.github/workflows/lint-js.yml`. Rego-deny для `bunx knip` у lint-js.yml лишився сумісним (перевірка `contains`, підрядок присутній).
**Обґрунтування:** knip не мав цих підказок у `ignoreDependencies` і правомірно шумів; прапор зупиняє шум без ігнорування реальних перевірок.
**Розглянуті альтернативи:** Додати `graphql` до `ignoreDependencies` у `knip.json` — неправильно: це маскує справжню проблему, а правило `js-lint.mdc` прямо забороняє `ignoreDependencies ∋ "graphql"`.
**Зачіпає:** `package.json` (scripts.lint-js), `npm/scripts/check-js-lint.mjs`, `npm/policy/js_lint/package_json/package_json.rego`, `npm/mdc/js-lint.mdc`, `.cursor/rules/n-js-lint.mdc`, `.github/workflows/lint-js.yml`
