## ADR Документування проєкту `@nitra/cursor` через CI4-модель

**Контекст:** Монорепозиторій `@nitra/cursor` (npm-пакет, CLI, GitHub Action, Claude-хуки, набір `.mdc`-правил) не мав формального опису архітектури — не зрозуміло, які компоненти існують, хто зовнішні актори і як вони взаємодіють. Планувалося також двостороннє посилання між архітектурою та ADR-рішеннями.

**Рішення/Процедура/Факт:** Прийнято рішення описати проєкт у форматі C4-моделі (перейменованого на **CI4**) з такою структурою файлів:
```
docs/ci4/
├── README.md         # індекс + інструкція кросс-лінкування з ADR
├── 01-context.md     # L1: зовнішні актори і межа системи
├── 02-containers.md  # L2: CLI (subcommands: check, stop-hook, rename-yaml-extensions), Rule Sync, AGENTS Builder, GH Action
├── 03-components.md  # L3: внутрішні модулі кожного контейнера з якорями
├── 04-code.md        # L4: drill-in для Check Runner і AGENTS Builder
└── decisions.md      # таблиця "елемент CI4 ↔ суміжні ADR"
```
Формат діаграм — **Mermaid у Markdown** (нативний рендер GitHub/Cursor/VS Code). Межа системи — тільки пакет `@nitra/cursor` (`npm/`); `demo/`, CI, npm registry, цільові репозиторії та AI-агенти (Cursor, Claude Code) лишаються зовнішніми. Описуються всі 4 рівні C4 (L4 — для Check Runner і AGENTS Builder). Стабільні якорі в файлах CI4 (`## Check Runner {#check-runner}`) дозволяють ADR-файлам посилатися на конкретні елементи архітектури.

**Обґрунтування:** Окремі файли замість одного `c4-model.md` — бо стабільні URL (`docs/ci4/02-containers.md#check-runner`) не ламаються при редагуванні сусідніх секцій; `decisions.md` як центральна cross-reference таблиця простіша у підтримці ніж розкидані inline-лінки; назва `ci4` замість `c4` — власна конвенція проєкту; Mermaid обрано бо проєкт text-first (markdownlint, cspell) і рендеринг без зовнішніх інструментів.

**Розглянуті альтернативи:**
- Один файл `docs/ci4-model.md` — відхилено (URL-якорі нестабільні при переписуванні розділів)
- Structurizr DSL — відхилено (додаткова залежність Docker/CLI, надмірно для одного пакету)
- PlantUML — відхилено (не рендериться нативно на GitHub без CI-екшну)
- Mermaid + Structurizr паралельно — відхилено як overkill

**Зачіпає:** `docs/ci4/` (новий каталог), `docs/adr/_inbox/*.md` (додаються зворотні посилання на CI4), конвенція іменування архітектурних документів у монорепо `@nitra/cursor`
