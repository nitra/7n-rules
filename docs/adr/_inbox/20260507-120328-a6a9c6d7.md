## Патч для `@nitra/minify-image` — відстеження version-прив'язки AVIF через `avifHash`

### Проблема (баг у `npm/src/index.js`)

Поточна логіка `tryCacheHit` (рядки ~408–425) не відрізняє "AVIF відсутній" від "AVIF є, але застарів":

```
if (avifPath && !existsSync(avifPath)) → генерує AVIF
```

**Сценарій збою:**

1. `image1.png` оброблено з `--write --avif` → `image1.png.avif` створено; у `.n-minify-image.tsv` запис: `image1.png\tH_old\t...`.
2. Розробник оновлює `image1.png` (новий вміст).
3. Запуск `--write` **без** `--avif`: sha1 змінився → MISS → файл перестиснутий, TSV оновлено → `image1.png\tH_new\t...`. AVIF не торкається (avifPath = null).
4. Запуск `--write --avif`: sha1(поточний файл) = H_new = cached → **HIT**. `existsSync(avifPath)` = true → **AVIF не перегенерується**, хоча сформований зі старого оригіналу.

### Рішення: 5-та колонка `avifHash` у `.n-minify-image.tsv`

`avifHash` — sha1 файлу **на момент генерації AVIF**. Якщо `avifHash ≠ hashEntry.hash` → джерело змінилось → AVIF треба перегенерувати.

### Конкретні зміни у `npm/src/index.js`

**1. Тип `hashCache` — додати поле `avifHash`**

JSDoc типу об'єкта:

```js
// було:
// Map<string, { hash: string, originalSize: number, size: number }>
// стало:
// Map<string, { hash: string, originalSize: number, size: number, avifHash?: string }>
```

**2. `parseHashLine` (рядок ~149) — читати 5-ту колонку**

```js
// було:
const parseHashLine = (cols, cache) => {
const [path, hash, originalSize, size] = cols
if (!path || !hash || !size) return
const sizeNum = Number(size)
cache.set(path, { hash, originalSize: Number(originalSize) || sizeNum, size: sizeNum })
}

// стало:
const parseHashLine = (cols, cache) => {
const [path, hash, originalSize, size, avifHash] = cols
if (!path || !hash || !size) return
const sizeNum = Number(size)
cache.set(path, {
hash,
originalSize: Number(originalSize) || sizeNum,
size: sizeNum,
avifHash: avifHash || undefined,
})
}
```

**3. `readTsv4` — прийняти і 5-колонкові рядки**

Функція `readTsv4` зараз скіпає рядки де `cols.length !== 4`. Треба прийняти 4 або 5 колонок:

```js
// було:
if (cols.length === 4) parseLine(cols, cache)

// стало:
if (cols.length === 4 || cols.length === 5) parseLine(cols, cache)
```

**4. `saveHashCache` (рядок ~208) — серіалізувати `avifHash`**

```js
// було:
.map(([path, { hash, originalSize, size }]) => `${path}\t${hash}\t${originalSize}\t${size}`)

// стало:
.map(([path, { hash, originalSize, size, avifHash }]) => {
const base = `${path}\t${hash}\t${originalSize}\t${size}`
return avifHash ? `${base}\t${avifHash}` : base
})
```

**5. `tryCacheHit` (рядок ~402) — перевіряти `avifHash` на HIT**

```js
// було (fast-path):
if (avifPath && !existsSync(avifPath)) {
await writeAvif(readFileSync(imagePath), avifPath, imagePath)
}

// стало (fast-path):
if (avifPath && (!existsSync(avifPath) || hashEntry.avifHash !== hashEntry.hash)) {
const src = readFileSync(imagePath)
await writeAvif(src, avifPath, imagePath)
hashEntry.avifHash = hashEntry.hash
}
```

Те саме виправлення у **slow-path** блоці (рядки ~423–425) — `hashEntry` тут вже є у hashCache:

```js
// було (slow-path):
if (avifPath && !existsSync(avifPath)) {
await writeAvif(buf, avifPath, imagePath)
}

// стало (slow-path):
if (avifPath && (!existsSync(avifPath) || hashEntry.avifHash !== hashEntry.hash)) {
await writeAvif(buf, avifPath, imagePath)
hashEntry.avifHash = hashEntry.hash
}
```

**6. `processOne` — MISS path (рядок ~487) — зберегти `avifHash` після генерації**

У блоці де `hashCache.set(relPath, {...})` виконується оновлення кешу після MISS:

```js
// було:
hashCache.set(relPath, {
hash: hashBuffer(onDiskBytes),
originalSize: existingOriginal,
size: stat.size,
})

// стало:
const newHash = hashBuffer(onDiskBytes)
hashCache.set(relPath, {
hash: newHash,
originalSize: existingOriginal,
size: stat.size,
avifHash: avifPath ? newHash : hashCache.get(relPath)?.avifHash,
})
```

Якщо `--avif` увімкнений: AVIF щойно згенерований → `avifHash = newHash`.
Якщо `--avif` вимкнений: зберігаємо попереднє значення `avifHash` (якщо є), щоб не втратити зв'язок між AVIF і його поточним джерелом — але якщо джерело змінилося (MISS), попередній `avifHash ≠ newHash` → наступний запуск з `--avif` побачить розбіжність і перегенерує.

### Тести (Bun test)

Додати до наявного тест-файлу (або нового `tests/avif-staleness.test.mjs`):

**A. Базовий випадок: AVIF генерується при першому запуску з `--avif`**

```
withTmpCwd → writeFile('a.png', pngBytes) → run(--write --avif)
→ expect existsSync('a.png.avif') toBe true
→ expect TSV 5th column for 'a.png' toBe sha1(compressedA)
```

**B. Зміна джерела без `--avif` → наступний `--avif` перегенерує**

```
withTmpCwd →
1) writeFile('a.png', pngBytesV1) → run(--write --avif)
→ save content of a.png.avif as avifV1
2) writeFile('a.png', pngBytesV2) → run(--write)       ← без --avif
3) run(--write --avif)
→ expect a.png.avif content ≠ avifV1                ← перегенеровано
→ expect TSV avifHash === sha1(currentCompressedA)
```

**C. Повторний `--avif` без змін → AVIF не перезаписується**

```
withTmpCwd →
1) writeFile('a.png', pngBytes) → run(--write --avif)
→ const mtimeAfterFirstRun = statSync('a.png.avif').mtimeMs
2) run(--write --avif)
→ expect statSync('a.png.avif').mtimeMs toBe mtimeAfterFirstRun  ← файл не чіпали
```

**D. Зворотна сумісність: існуючий 4-колонковий TSV читається без помилок**

```
writeFileSync('.n-minify-image.tsv', 'a.png\tHASH\t1000\t800\n')
→ TSV парсується, hashEntry.avifHash === undefined
→ run(--write --avif) → AVIF created (avifHash undefined → treated same as missing)
→ TSV has 5 columns after run
```
