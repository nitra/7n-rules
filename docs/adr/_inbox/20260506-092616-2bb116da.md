## ADR Обов'язковий крок `depcheck` у GitHub Actions з `paths:`-фільтром

**Контекст:** У backend-проєктах workspace-пакети запускаються окремими GitHub Actions workflows з `paths:`-фільтром (наприклад, `cron-jobs/refund-loyalty-points/**`). Без явної вимоги цей крок перевірки залежностей міг бути відсутній, що залишало невикористані або незадекларовані залежності непоміченими.

**Рішення/Процедура/Факт:** До правила `js-run.mdc` додано нову секцію **«depcheck у GitHub Actions з path-фільтром»**: якщо workflow має `paths:`-тригер, що обмежує запуск одним workspace-пакетом, у ньому обов'язково повинен бути крок `npx depcheck --ignores="graphql,bun"` з `working-directory` цього пакета. Список `--ignores` може бути розширений, але `graphql` та `bun` — обов'язкові. Файли змінено: `npm/mdc/js-run.mdc` (v1.2 → v1.3), синхронізовано до `.cursor/rules/n-js-run.mdc`, версія пакета `npm/package.json` підвищена до 1.8.182, `npm/CHANGELOG.md` оновлено.

**Обґрунтування:** Користувач явно вказав, що перевірка залежностей через `depcheck` є обов'язковою для кожного path-scoped workflow у backend-проєктах. Ігноровані пакети `graphql` і `bun` є частиною runtime-оточення, а не прямими залежностями проєкту, тому їх не слід вважати зайвими.

**Розглянуті альтернативи:** Не обговорювалися.

**Зачіпає:** `npm/mdc/js-run.mdc`, `.cursor/rules/n-js-run.mdc`, `npm/package.json`, `npm/CHANGELOG.md`; публічне правило `@nitra/cursor` для всіх backend Node.js workspace-пакетів.
