## ADR Заборона `new Promise` обгортки для `setTimeout` у Node.js backend-пакетах

**Контекст:** У backend-коді проєкту траплялись паузи, реалізовані через `await new Promise(resolve => setTimeout(resolve, ms))` — багатослівний патерн, що не відповідає сучасним можливостям Node.js і є зайвим boilerplate.

**Рішення/Процедура/Факт:** До `npm/mdc/js-run.mdc` (v1.3 → v1.4) додано секцію **«Паузи через setTimeout»**, яка забороняє `await new Promise(resolve => setTimeout(resolve, ms))` і вимагає натомість:

```js
import { setTimeout } from 'node:timers/promises';
await setTimeout(ms);
```

Версію пакета `@nitra/cursor` бампнуто `1.8.185 → 1.8.186`, відповідний запис додано до `npm/CHANGELOG.md`.

**Обґрунтування:** `node:timers/promises` є стандартним Node.js API (з v15), що повертає справжній `Promise` без ручної обгортки. Патерн коротший, безпечніший і відповідає стилю, закріпленому в правилах проєкту для всіх backend workspace-пакетів.

**Розглянуті альтернативи:** Збереження правила у пам'яті Claude (відкинуто — правило має бути у кодовій базі як частина `.mdc`, а не лише в контексті асистента).

**Зачіпає:** `npm/mdc/js-run.mdc`, `npm/package.json`, `npm/CHANGELOG.md`; правило `alwaysApply: true` поширюється на всі backend Node.js workspace-пакети проєкту.
