## ADR Новий скіл `n-llm-patch`: read-only генератор крос-сесійних промптів

**Контекст:** При передачі завдань між Claude-сесіями (різні проєкти) доводилось вручну збирати контекст — копіювати `package.json`, конфіги, описувати обмеження. Не було жодного уніфікованого способу отримати самодостатній промпт для іншого агента.

**Рішення/Процедура/Факт:** Спроектовано скіл `/n-llm-patch <опис>` — read-only команда, що аналізує CWD і збирає один markdown-блок у чаті, готовий до вставки в іншу Claude/Cursor-сесію. Шаблон: нормалізоване завдання → контекст проєкту (`name`, `version`, стек, tree `-L 2`) → повний вміст релевантних конфіг-файлів (дрібних) → список кроків → обмеження → команди для перевірки. Скіл нічого не пише в репо; `/tmp` дозволено лише за потреби. Джерело файлів — виключно локальний CWD (не npm registry tarball).

**Обґрунтування:** Цільова LLM потребує самодостатнього промпту без доступу до вихідного репо. Локальний CWD обраний як єдине джерело — у ньому вже є актуальні файли, що бачить і автор. Read-only обмеження підкреслює, що скіл лише діагностує, не змінює.

**Розглянуті альтернативи:** `npm pack` + завантаження tarball із registry (відхилено — завантажує опубліковану версію, не локальну; повільніше); клонування git (надмірно).

**Зачіпає:** `npm/skills/llm-patch/SKILL.md` (каркас, підлягає наповненню), `.cursor/skills/n-llm-patch/SKILL.md` (майбутнє дзеркало)

## ADR `llm-patch` додано як always-on скіл у `auto-skills.mjs`

**Контекст:** Скіл `llm-patch` уже фігурував у `npm/bin/auto-skills.md` з позначкою «завжди», але `npm/scripts/auto-skills.mjs` та тести його ігнорували — розбіжність між специфікацією і реалізацією.

**Рішення/Процедура/Факт:** `'llm-patch'` додано до `AUTO_SKILL_ORDER` (між `lint` та `publish-telegram`, алфавітно) та `ALWAYS_ON_SKILLS` у `auto-skills.mjs`. Оновлено тест у `npm/tests/auto-skills.test.mjs` — `ALL_SKILLS` і перевірка always-on набору. Версія пакета підвищена `1.8.217 → 1.8.218`, додано запис у `npm/CHANGELOG.md`. Перевірки `check changelog` та `check npm-module` пройшли чисто; 8 тестів — `pass`.

**Обґрунтування:** Always-on скіли (`fix`, `lint`, `publish-telegram`, `llm-patch`) не мають умов за rules — вони корисні в будь-якому проєкті. Алфавітний порядок у масивах підтримує консистентність зі стилем існуючих записів.

**Розглянуті альтернативи:** Не обговорювалися; рішення продиктоване наявним записом у `auto-skills.md`.

**Зачіпає:** `npm/scripts/auto-skills.mjs`, `npm/tests/auto-skills.test.mjs`, `npm/package.json`, `npm/CHANGELOG.md`
