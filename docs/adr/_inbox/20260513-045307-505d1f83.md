## ADR Оформлення npm-модуля: компактний пакет, rego-first перевірки, knip

**Контекст:** Пакет `@nitra/cursor` публікував зайві файли (тести, rego-юніт-тести), мав `devDependencies` у `npm/package.json`, не мав явного `"files"` whitelist. Перевірки структурних полів JSON писалися у JS, а не у Rego, де вони мають бути.

**Рішення/Процедура/Факт:** Версія `npm-module.mdc` піднята до `1.11` з трьома новими вимогами компактності (`"files"` обов'язковий, без `devDependencies`, тести/fixtures поза опублікованим деревом; виняток — `*_test.rego` публікується разом із policy). Перевірки `files`/`devDependencies` перенесені у Rego (`npm_module.npm_package_json`) з тестами (`npm_package_json_test.rego`). JS-чек (`check-npm-module.mjs`) зберіг лише `checkNoTestsInPublishedFiles` (FS walk + oxc-parser AST). `knip.json` налаштовано: `graphql` у `ignoreDependencies`. `build-agents-commands.mjs` генерує рядок `bunx knip` у AGENTS.md. `conftest.mdc` та `npm/CLAUDE.md` отримали гостріший Rego-first stop-блок для запобігання написання пер-документних перевірок у JS.

**Обґрунтування:** Максимально компактний модуль → `npm install` тягне тільки runtime; rego-first → пер-документні deny живуть у полісі, JS — тільки там де потрібен FS/AST. Knip у AGENTS.md — щоб агенти знали про цей інструмент перевірки.

**Розглянуті альтернативи:** Виносити `*_test.rego` з `policy/` у `npm/tests/policy/` (фізичний виняток) — відхилено через необхідність рефакторингу conftest-конфігу; залишення перевірок `files`/`devDependencies` у JS — відхилено після виявлення ляпу, відповідає rego-first правилу.
