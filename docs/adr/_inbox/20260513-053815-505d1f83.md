## ADR Міграція пер-документних перевірок `npm/package.json` у Rego + ліквідація JS-дублікатів

**Контекст:** У `check-npm-module.mjs` та `check-js-lint.mjs` накопичилися JS-функції (`checkPackageCompactness`, `CANONICAL_LINT_JS`, `isCanonicalLintJs`, `nitraEslintConfigMeetsMinVersion`), що перевіряли структурні властивості `package.json` — хоча проєкт уже мав rego-шар (`npm/policy/`) для таких перевірок. Виникло «два джерела істини»: при будь-якій зміні канону (напр., додавання `--no-config-hints`) доводилося правити і JS, і rego.

**Рішення/Процедура/Факт:**
1. **`npm/policy/npm_module/npm_package_json/npm_package_json.rego`** — додано два `deny`-правила: обов'язковий непорожній `files`-whitelist (3 варіанти помилки) і заборона `devDependencies`; `npm_package_json_test.rego` — happy-path + 7 негативних кейсів через `json.patch`.
2. **`npm/policy/js_lint/package_json/package_json.rego`** — `canonical_lint_js` включає `bunx knip --no-config-hints`; додано новий `package_json_test.rego` з 16 test-кейсами (total 157/157 pass).
3. **`check-npm-module.mjs`** — видалено `checkPackageCompactness` та її виклик; лишився тільки `checkNoTestsInPublishedFiles` (FS walk + AST — rego не вміє).
4. **`check-js-lint.mjs`** — видалено dead exports: `CANONICAL_LINT_JS`, `isCanonicalLintJs`, `normalizeLintJsScript`, `WHITESPACE_RE`, `nitraEslintConfigMeetsMinVersion`; додано коментар-посилання на rego.
5. **`eslint.config.js`** — `.claude/worktrees/**` додано до `ignores`, щоб eslint не лінтив stale git worktree.
6. **`knip.json`** — прибрано 4 зайві `ignoreDependencies`/`ignoreBinaries` hints; `graphql` залишено явно (`ignoreDependencies`) за рішенням автора.
7. **`npm/scripts/utils/knip-canonical.json`** — новий файл-канон для `knip.json` проєкту-споживача (9 категорій false-positives, `entry`/`project`/`ignore`/`ignoreDependencies`/`ignoreBinaries`); `checkKnipConfig` у `check-js-lint.mjs` перевіряє лише **наявність** файлу і автокопіює канон при відсутності.
8. **`npm/mdc/js-lint.mdc`** + дзеркало **`n-js-lint.mdc`** — версія `1.19 → 1.21`; оновлено секцію knip, скрипти і CI-сніпети.
9. **`conftest.mdc`** і **`npm/.claude-template/npm-CLAUDE.md`** — додано STOP-блок «перш ніж дописувати JS-чек — перевір rego-пакет».

**Обґрунтування:** Prego-first архітектура `npm/policy/`: пер-документні перевірки (структура полів, значення ключів, формати) належать rego — один канонічний код, одне місце правки, авто-тести через `conftest verify`. JS-чеки (`check-*.mjs`) залишаються тільки там, де потрібен FS-обхід кількох файлів, AST-парсинг або cross-file логіка, недоступна для rego.

**Розглянуті альтернативи:** Залишити JS-дублікат як є (відкидається: дрифт одразу проявився при додаванні `--no-config-hints`); перенести весь чек у rego повністю (відкидається: `checkNoTestsInPublishedFiles` вимагає рекурсивного FS-обходу + AST-аналізу JS-файлів, що не підходить для rego).

**Зачіпає:** `npm/policy/npm_module/npm_package_json/`, `npm/policy/js_lint/package_json/`, `npm/scripts/check-npm-module.mjs`, `npm/scripts/check-js-lint.mjs`, `npm/scripts/utils/knip-canonical.json`, `eslint.config.js`, `knip.json`, `npm/mdc/js-lint.mdc`, `.cursor/rules/n-js-lint.mdc`, `.cursor/rules/conftest.mdc`, `npm/.claude-template/npm-CLAUDE.md`, `npm/CHANGELOG.md`, `npm/package.json` (версії `1.9.5 → 1.10.2`).
