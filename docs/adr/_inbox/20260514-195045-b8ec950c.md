## ADR Реорганізація `npm/` довкола правил — rule-centric структура

**Контекст:** `npm/` росте, і правило як концепція «розмазане» по кількох несуміжних директоріях: `.mdc`-файл у `mdc/`, Rego-полісі у `policy/{rule}/`, JS-скрипти у `scripts/check-{rule}.mjs` тощо, умови авто-активації у `bin/auto-rules.md`. Видалення правила вимагає ручного пошуку та видалення артефактів у декількох місцях.

**Рішення/Процедура/Факт:** Запропоновано перехід до rule-centric монолітної структури: кожне правило отримує власну директорію `npm/mdc/{rule}/`, усередині якої зосереджено всі його артефакти:
- `npm/mdc/{rule}/rule.mdc` (або `{rule}.mdc` поруч із директорією — уточнюється)
- `npm/mdc/{rule}/policy/` — вміст, що переїжджає з `npm/policy/{rule}/`
- `npm/mdc/{rule}/js/` — відповідні файли з `npm/scripts/` (`check.mjs`, `lint.mjs`, `run.mjs` тощо, без rule-суфіксу в імені файлу)
- умова авто-активації — фрагмент поруч, в тій-же директорії або окремий `auto.md`

Крос-правильна інфраструктура (`cli-entry.mjs`, `auto-rules.mjs`, `lint-conftest.mjs`, `utils/` тощо) лишається у плоскому корені `npm/scripts/`.

Найменування каталогів правил — kebab-case (узгоджено з поточними іменами файлів). Файли всередині каталогу — без rule-суфіксу (директорія вже несе namespace).

**Обґрунтування:** Rule-centric структура дозволяє видалити правило одним `rm -rf npm/mdc/{rule}/` — без пошуку по `policy/`, `scripts/`, `bin/`. Полегшує онбординг: всі частини правила видимі одним `ls`. Відповідає принципу high cohesion / low coupling для одиниць «правило».

**Розглянуті альтернативи:**
1. Реорганізація лише `npm/scripts/` в підкаталоги за правилом (без переміщення `policy/` та `mdc/`) — частковий виграш, проблема залишається.
2. Збереження повного rule-суфіксу у файлах (`{rule}/check-{rule}.mjs`) — відхилено на користь лаконічного варіанту A (прибрати суфікс).
3. snake_case для директорій (як у поточному `policy/`) — відхилено на користь kebab-case.

**Зачіпає:** `npm/mdc/`, `npm/policy/`, `npm/scripts/`, `npm/bin/auto-rules.md`, `npm/bin/auto-skills.md`, `npm/bin/n-cursor.js` (динамічний `readdir` по `check-*.mjs` потребує оновлення), тести, що імпортують `../scripts/check-*.mjs` напряму.
