## ADR Конвенції нейминга та експортів для файлів у `src/conn/`

**Контекст:** Файли підключень до БД та зовнішніх GraphQL-сервісів у `src/conn/` не мали формальних вимог до назви та форми експорту, що призводило до неоднозначних імен (`pg.js` без ознаки read/write) і `export default`, ускладнюючи автоматичну перевірку і рефакторинг.

**Рішення/Процедура/Факт:**
- У `npm/mdc/js-run.mdc` (версія 1.5 → 1.6) додано два нові підрозділи:
- **«Нейминг файлів у `src/conn/`»** — обов'язкові префікси: `ql-<id>` для GraphQL, `pg-(read|write)[-<id>]` для PostgreSQL, `mysql-(read|write)[-<id>]` для MySQL; якщо з назви env-змінної не зрозуміло read чи write — аналізувати операції в коді (`INSERT`/`UPDATE`/`DELETE`/DDL → `write`, інакше → `read`).
- **«Експорти у файлах `src/conn/`»** — `export default` заборонений; єдиний дозволений варіант — іменований експорт у camelCase від basename (`ql-smart.js` → `export const qlSmart`).
- Створено `npm/scripts/utils/conn-file-rules.mjs` — утиліта AST-перевірки: валідує basename файла regex-ом, знаходить `export default` (порушення), збирає всі іменовані `export const`, порівнює з очікуваним camelCase-іменем.
- `npm/scripts/check-js-run.mjs` розширено новою функцією `checkConnFileRules`, яка обходить усі файли conn-каталога (крім `index.*`) та агрегує порушення через `conn-file-rules.mjs`.
- Розділ «Перевірка» у `.mdc` оновлено: `npx @nitra/cursor check js-run` тепер явно описує перевірку basename, відсутності `export default` і відповідності імені експорту.
- `npm/package.json` → 1.8.208; `npm/CHANGELOG.md` доповнено записом.

**Обґрунтування:** Чіткий нейминг одразу відповідає на питання «куди йде запит і з якими правами» без читання коду. Іменований експорт, рівний basename у camelCase, дозволяє автоматично верифікувати відповідність і уникає плутанини при barrel-ре-експорті.

**Розглянуті альтернативи:** Не обговорювалися; підхід продиктований користувачем.

**Зачіпає:** `npm/mdc/js-run.mdc`, `npm/scripts/check-js-run.mjs`, `npm/scripts/utils/conn-file-rules.mjs` (новий файл), `npm/package.json`, `npm/CHANGELOG.md`; публічний CLI-контракт `npx @nitra/cursor check js-run`.
