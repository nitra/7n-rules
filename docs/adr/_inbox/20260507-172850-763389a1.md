## ADR Пріоритет conftest-політик для задач, вирішуваних через OPA/Rego

**Контекст:** У проєкті використовується conftest/OPA для статичних перевірок. Без явного правила асистент міг вирішувати задачі скриптами або іншими механізмами, навіть якщо є можливість реалізувати перевірку декларативно через Rego-політику.

**Рішення/Процедура/Факт:** У файл `.cursor/rules/conftest.mdc` додано правило: якщо задачу можна вирішити через conftest-політику, саме цей підхід є обов'язковим. Зміна внесена редагуванням існуючого `.mdc`-файлу без створення нових файлів.

**Обґрунтування:** Декларативні OPA/Rego-політики дають перевірки, які запускаються в CI через `conftest`, є версіонованими, легко тестованими та зрозумілими без знання конкретної мови скриптів. Уніфікація підходу зменшує кількість різнорідних механізмів перевірки в репозиторії.

**Розглянуті альтернативи:** Не обговорювалися; рішення продиктоване користувачем напряму.

**Зачіпає:** `.cursor/rules/conftest.mdc`, усі майбутні задачі, де assistant обирає між скриптом/хуком і Rego-політикою.
