## ADR k8s `$schema` modeline — опційна, `file:` заборонений як заглушка

**Контекст:** Механізм авто-виправлення в `check-k8s.mjs` додавав рядок `# yaml-language-server: $schema=file:.` до K8s YAML-файлів, для яких не існує публічної JSON-схеми. Такий fallback не давав жодної реальної валідації й вводив в оману редактор.

**Рішення/Процедура/Факт:** У `check-k8s.mjs` логіка перевірки першого рядка змінена: modeline `# yaml-language-server: $schema=…` обов'язкова **лише** для тих `apiVersion`/`kind`, для яких існує надійний публічний URL (kustomization, yannh/kubernetes-json-schema, datree CRDs-catalog тощо). Якщо URL не знайдено — файл без modeline вважається коректним (pass, не fail). Будь-яке значення `$schema=file:…` тепер є помилкою. Правило `k8s.mdc` оновлено відповідно (пункт «Немає надійного публічного URL — не вигадуй URL і **не** використовуй `$schema=file:…`»). Версія пакета підвищена до `1.9.2`, changelog оновлено.

**Обґрунтування:** `file:` заглушка не валідує маніфест, але дає хибне відчуття покриття. Краще явно не мати modeline, ніж мати непрацюючу. `lint-k8s` (kubeconform з `--ignore-missing-schemas`) покриває валідацію навіть без modeline.

**Розглянуті альтернативи:** Залишити поточну поведінку з `file:.` — відкинуто, бо не дає жодної практичної користі. Генерація локальних схем (CRD → JSON Schema) — не обговорювалася.

**Зачіпає:** `npm/scripts/check-k8s.mjs` (логіка перевірки modeline та авто-виправлення), `npm/mdc/k8s.mdc` (правило для LLM/агентів), `npm/package.json` (версія 1.9.2), `npm/CHANGELOG.md`.
