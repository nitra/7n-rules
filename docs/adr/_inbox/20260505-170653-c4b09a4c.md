## ADR Заборона Node-нативних імпортів у Vue SFC

**Контекст:** Vue компоненти (`.vue` SFC) не повинні безпосередньо імпортувати Node-нативні модулі (наприклад, `import { setTimeout as sleep } from 'node:timers/promises'`), оскільки браузерне середовище їх не підтримує — такий код ламає бандлинг або провалюється в рантаймі.

**Рішення/Процедура/Факт:** Додано новий сканер у `npm/scripts/utils/vue-forbidden-imports.mjs`, який виявляє імпорти Node-нативних модулів (як з префіксом `node:`, так і bare-ім'я типу `fs`, `path`, `timers/promises` тощо) у `.vue` файлах через **oxc-parser**. Сканер підключено до `npm/scripts/check-vue.mjs` як окремий прохід по `.vue` SFC. Тести додано у `npm/tests/vue-forbidden-imports.test.mjs` та `npm/tests/check-rule-fixtures.test.mjs`. Правило задокументовано в `npm/mdc/vue.mdc` та `.cursor/rules/n-vue.mdc` (версія 1.4 → 1.5). Версія пакета: 1.8.177 → 1.8.178.

**Обґрунтування:** Node-нативні модулі є несумісними з браузером — їх не повинно бути у фронтенд-коді Vue компонентів. Автоматична перевірка через `npx @nitra/cursor check vue` запобігає появі таких імпортів у кодовій базі без ручного review.

**Розглянуті альтернативи:** Не обговорювалися; підхід через oxc-parser вже використовувався для аналогічного сканування заборонених `vue`-імпортів, тому його природно розширили для Node-модулів.
