## ADR Канонізація патерну «Rego-authoritative + JS-orchestrator» для `check-*.mjs`

**Контекст:** Пер-документні правила перевірки (abie.mdc, ga.mdc) існували одночасно в JS-функціях (`check-abie.mjs`, `check-ga.mjs`) і в rego-пакетах (`npm/policy/abie/`, `npm/policy/ga/`). Це призводило до дрифту: реальний bug — `health_check_policy.rego` читав `spec.config.httpHealthCheck` тоді як JS перевіряв `spec.default.config.httpHealthCheck`, і розбіжність місяцями лишалася непоміченою.

**Рішення/Процедура/Факт:**
- Видалено дубльовані JS-предикати з `check-abie.mjs` (`abieBaseHttpRouteHostnamesErrors`, `deploymentDocumentHasAbieBasePreemNodeSelector`, `parseCleanMergedIgnoreBranches`, `ignoreBranchesIncludesRequired`, `validateAbieHcPolicy` тощо).
- Створено `npm/scripts/utils/run-conftest-batch.mjs` — утиліта для батч-виклику `conftest test` з JSON-output; hard-fail якщо `conftest` не у PATH.
- Кожен `check-<rule>.mjs::check()` тепер починає з виклику `runConftestBatch` для відповідних rego-пакетів, далі — JS cross-file логіка.
- `lint-<rule>.mjs` спрощено до: preflight → external tools (actionlint, zizmor) → `await checkX()`.
- Зафіксовано у `.cursor/rules/conftest.mdc` (alwaysApply) як декізія-дерево та «червоні прапори» з забороною плодити JS-копії rego.

**Обґрунтування:** Пер-документне правило має існувати в одному місці. JS-дублі коштують підтримки і створюють тихий дрифт (підтверджено реальним багом). Batched `conftest test` дає ~80-150ms на namespace незалежно від кількості файлів — прийнятна ціна. Cross-file логіка (DNS-схеми між env-файлами, JSON6902-патчі, кросс-document Service↔HCP пейринг) лишається в JS, де rego обмежений.

**Розглянуті альтернативи:**
- Plan A — «JS authoritative + rego-mirror» + golden cross-check тести: реалізовано тимчасово (`cross-check-rego-abie.test.mjs`, 25 тестів), але відкинуто — усуває симптом (видимість дрифту), а не причину (дублювання).
- Plan C — JS-проксі через `opa eval` per-call: повільніше (спавн на документ замість на namespace), складніше для unit-тестів; відкинуто.
- Plan B (обрано) — rego авторитетне, JS викликає conftest через `runConftestBatch`.

**Зачіпає:** `npm/scripts/check-abie.mjs`, `npm/scripts/check-ga.mjs`, `npm/scripts/lint-ga.mjs`, `npm/bin/n-cursor.js`, `npm/scripts/utils/run-conftest-batch.mjs` (новий), `npm/tests/check-ga.test.mjs`, `npm/tests/lint-ga.test.mjs`, `.cursor/rules/conftest.mdc`

## ADR Нові rego-пакети abie з `_test.rego` і виправлення `health_check_policy`

**Контекст:** 47 rego-файлів поза `k8s/` не мали `_test.rego`. Пер-документні правила abie.mdc для Deployment, clean-merged-branch workflow та HTTPRoute hostnames існували лише в JS і не були доступні для IDE-фідбеку через `conftest`.

**Рішення/Процедура/Факт:**
- Додано `npm/policy/abie/base_deployment_preem/` — Deployment у `…/k8s/.../base/…` має `spec.template.spec.nodeSelector.preem: true` або `"true"`. 8 rego-тестів.
- Додано `npm/policy/abie/clean_merged_ignore_branches/` — workflow `clean-merged-branch.yml` має `with.ignore_branches` з токенами `dev`, `ua`, `ru` (case-insensitive). 7 rego-тестів.
- Переписано `npm/policy/abie/health_check_policy/health_check_policy.rego`: виправлено шлях `spec.config` → `spec.default.config`; додано перевірки `apiVersion == networking.gke.io/v1`, `metadata.name`, `targetRef.kind == Service`, exact match `targetRef.name == "<hcp.name>-hl"` (а не лише `endswith "-hl"`). 10 rego-тестів.
- Додано `npm/policy/abie/http_route_base/http_route_base_test.rego` — 10 тестів для пакета, що раніше не мав жодного.
- Оновлено `npm/scripts/lint-conftest.mjs` (TARGETS table): 2 нові abie-таргети, уточнено `policyDir` для існуючих.
- Оновлено `npm/mdc/abie.mdc`: розділ «Швидкий gate через conftest (Rego)» з мапою пакетів.

**Обґрунтування:** IDE-фідбек через `tsandall.opa` працює лише при наявності rego-пакету. Без `_test.rego` зміни в rego можуть мовчки зламати правило. Виявлений дрифт `spec.config` vs `spec.default.config` демонструє необхідність тестів — вони б зловили це одразу.

**Розглянуті альтернативи:** не обговорювалися; пріоритет покриття очевидний з аудиту 47 untested файлів.

**Зачіпає:** `npm/policy/abie/` (4 пакети), `npm/scripts/lint-conftest.mjs`, `npm/mdc/abie.mdc`
