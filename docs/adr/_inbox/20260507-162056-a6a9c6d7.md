## ADR Перенесення `--avif` з `lint-image` у `check image-avif`

**Контекст:** Скрипт `lint-image` запускався під час `bun run lint` і містив прапорець `--avif`, що породжувало AVIF-файли при кожному lint-прогоні навіть без уведення їх у `.vue`/`.html` — і засмічувало робоче дерево orphan-артефактами.

**Рішення/Процедура/Факт:** Прапорець `--avif` вилучено з `lint-image`; канонічний виклик — `npx @nitra/minify-image --src=. --write` (тільки raster-компресія). AVIF-генерація (`npx @nitra/minify-image --src=. --write --avif`) винесена в окрему команду `npx @nitra/cursor check image-avif`. Після генерації — in-place rewrite посилань у `.vue`/`.html`; `.avif`-файли зберігаються лише якщо принаймні одна заміна реально записана. Orphan-файли (без живих посилань) видаляються.

**Обґрунтування:** `bun run lint` — частий короткий прогін; генерація AVIF коштовна (CPU + диск) і не повинна виконуватись рутинно. `check image-avif` запускається свідомо і за результатом змінює файли — правильне місце для цього side-effect.

**Розглянуті альтернативи:** Не обговорювались явно — тільки вилучення прапорця vs. переміщення.

**Зачіпає:** `package.json` (root + всі workspace-пакети), `npm/scripts/check-image.mjs`, `.cursor/rules/n-image-avif.mdc`, `.cursor/rules/n-image-compress.mdc`, `npm/mdc/image-avif.mdc`, `npm/mdc/image-compress.mdc`.

## ADR Стратегія резолвінгу шляхів зображень у `.vue`/`.html`

**Контекст:** `resolveImagePath` в `check-image.mjs` трактував абсолютні шляхи (`/api-page/1.png`) як `<cwd>/api-page/1.png`, а голі відносні (`assets/img.png`) повертав `null` — через що Vite/Quasar-проєкти (де `/`-шляхи означають `<pkg>/public/<path>`) не отримували жодного rewrite, а лічильник `failedRefs` зростав.

**Рішення/Процедура/Факт:** `resolveImagePath` замінено на `resolveImageCandidates`, яка повертає впорядкований список кандидатів; caller бере перший, поряд з яким існує `.avif`. Порядок для `/x.png`: `<pkg>/public/x.png` → `<pkg>/x.png` → `<cwd>/x.png`. Для голого відносного шляху з хоча б однією `/` (наприклад, `assets/img.png`): відносно файла-джерела + `<pkg>/public/<path>`. Голий шлях без `/` (alias) → порожній список, посилання пропускається без fail. `packageRoot` визначається через `getMonorepoPackageRootDirs()` — найдовший префікс, що містить `sourceAbsPath`.

**Обґрунтування:** Vite і Quasar трактують `/`-шляхи у Vue-шаблонах як посилання на `public/`, а не корінь OS. Кандидатний підхід не ламає legacy mono-репо без `public/` завдяки fallback на `<cwd>`.

**Розглянуті альтернативи:** Єдина резолюція через `<cwd>` (попередня реалізація — неправильна); алернативи не обговорювались, обраний варіант витікає з поведінки фреймворків.

**Зачіпає:** `npm/scripts/check-image.mjs` (функції `resolveImageCandidates`, `checkVueAvifImportsInPackage`), `npm/tests/check-image.test.mjs`.

## ADR Розміщення sha1-інвалідації stale AVIF у `@nitra/minify-image`, а не `@nitra/cursor`

**Контекст:** Після редагування raster-сорсу (`image1.png → нова версія`) відповідний `image1.png.avif` лишався застарілим — CLI не перегенеровував його за наявності файлу на диску. Перше рішення (1.8.195) додало sha1-кеш `.n-image-avif.tsv` і логіку видалення stale `.avif` у `check-image.mjs`.

**Рішення/Процедура/Факт:** Логіку відкочено (1.8.196). Sha1-перевірку зроблено в `@nitra/minify-image@3.3.1`: CLI порівнює sha1 поточного raster-сорсу із записом у вже наявному `.n-minify-image.tsv` і перезаписує `.avif` на mismatch або при відсутності `.avif`. Мінімальна версія `@nitra/minify-image` піднята до **3.3.1** у правилах `image-compress.mdc` та `image-avif.mdc`.

**Обґрунтування:** Sha1 raster-сорсів вже зберігається у `.n-minify-image.tsv` — природне місце для перевірки свіжості AVIF. Розміщення логіки в upstream CLI дає інвалідацію безкоштовно всім консьюмерам `@nitra/minify-image`, без дублювання кеша і без зміни `@nitra/cursor`.

**Розглянуті альтернативи:** Власний кеш `.n-image-avif.tsv` у `@nitra/cursor` (реалізовано і відкочено — дублює сховище й обмежує ефект одним консьюмером).

**Зачіпає:** `npm/mdc/image-compress.mdc`, `npm/mdc/image-avif.mdc`, `.cursor/rules/n-image-compress.mdc`, `.cursor/rules/n-image-avif.mdc`; upstream: `@nitra/minify-image@3.3.1`.

## Knowledge Порядок операцій cleanup: читати оновлені файли, не оригінальні

**Контекст:** `cleanupOrphanAvifs` запускалась до завершення всіх rewrite у `.vue`/`.html`, через що щойно-генеровані `.avif` (ще без посилань у шаблонах) одразу видалялися.

**Факт:** `usedAvifAbs` (множина `.avif`-файлів, на які є активні посилання) заповнюється всередині `checkVueAvifImportsInPackage` через `writeFile` — in-place, по одному файлу. `cleanupOrphanAvifs` викликається лише після того, як `checkVueAvifImports` завершила обхід усіх пакетів. Також: cleanup пропускає весь `packageRoot` для opt-out пакетів (`disable-avif: true`) і ігнорує директорії збірки (`dist`, `build`, `android`, `ios`, `.output`, `.nuxt`, `.cache`) — інакше build/Capacitor-артефакти видалялись би помилково.

**Зачіпає:** `npm/scripts/check-image.mjs` (функції `cleanupOrphanAvifs`, `checkVueAvifImports`, `CLEANUP_EXTRA_IGNORE_DIR_NAMES`).
