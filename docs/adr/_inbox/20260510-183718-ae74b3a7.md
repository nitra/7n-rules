## ADR Впровадження стратегії "Rego-first" для перевірок у `@nitra/cursor`

**Контекст:** Проєкт `@nitra/cursor` мав 24 скрипти `check-*.mjs`, що виконували структурні перевірки маніфестів у JavaScript. Паралельно існував шар `npm/policy/` з rego-полісі, але він був неповним, без тестів, і JS-логіка та rego-логіка поступово розходилися (drift). Потрібно було встановити єдине авторитетне джерело для пер-документних перевірок і усунути дублювання.

**Рішення/Процедура/Факт:**
1. **Алгоритм "Rego-first" закріплено у `.cursor/rules/conftest.mdc`** (alwaysApply: true): пер-документні перевірки (single YAML/JSON, kind/apiVersion, поля, форма масивів) → Rego; cross-file/FS/autofix/pre-YAML text → JS; гібрид → обидва шари.
2. **Path-scoped нагадування** додано до `npm/.claude-template/npm-CLAUDE.md` (регенерується через `npx @nitra/cursor` у `npm/CLAUDE.md`): "Перш ніж писати `check-*.mjs` — оціни, чи задача лягає на rego-полісі".
3. **Phase 6 (abie):** створено 3 нових rego-пакети з `_test.rego` (`base_deployment_preem`, `clean_merged_ignore_branches`, `health_check_policy` — повністю переписано з виправленням drift-бага `spec.config` → `spec.default.config`), додано `_test.rego` до існуючого `http_route_base`. Усього 35 rego-тестів по abie-полісі.
4. **Golden cross-check тести** (`npm/tests/cross-check-rego-abie.test.mjs`, 25 тестів) — перевіряли, що JS-предикати і rego давали однаковий вердикт (drift-детектор). Після Plan B видалено — JS-сторони більше нема.
5. **Utility `runConftestBatch`** (`npm/scripts/utils/run-conftest-batch.mjs`) — генерик-helper для batched-виклику `conftest test` з JSON-output; hard-fail без conftest у PATH.
6. **Plan B (пілот на abie):** JS-orchestrator-функції `ensureAbieBaseDeploymentPreemNodeSelector`, `checkCleanMergedBranch`, `checkHcYamlFiles`, `ensureAbieBaseHttpRouteHostnames` перероблено: тепер вони делегують пер-документну логіку до `runConftestBatch`. Видалено 10+ JS-функцій (`abieBaseHttpRouteHostnamesErrors`, `deploymentDocumentHasAbieBasePreemNodeSelector`, `parseCleanMergedIgnoreBranches`, тощо) — єдине джерело істини тепер у rego. `validateAbieHcYaml` звужено до `validateAbieHcModeline`.
7. **Версії:** 1.8.221 → 1.8.225 (5 релізів за сесію).

**Обґрунтування:** Rego (conftest) read-only за природою, тому ідеально підходить для структурних перевірок одиничного документа. Batched-виклик (`conftest test file1.yaml file2.yaml ... -p <dir> --namespace <ns>`) компілює полісі раз і обробляє всі файли за ~80-150ms незалежно від кількості — прийнятна вартість. JS лишається авторитетним тільки для того, що rego не може: cross-file resolution, readdir, autofix, regex по сирому тексту (до YAML-parse), агрегація між файлами. Дублювання тягнуло за собою drift-баги (знайдено один під час сесії) — rego-first усуває клас проблем.

**Розглянуті альтернативи:**
- **A (JS authoritative + rego-mirror)** — попередній стан; відхилено через неминучий drift.
- **C (тонкі JS-проксі через `opa eval`)** — обговорювалось; відхилено на користь явного batching через `conftest test`.
- **Видалення rego, лишити лише JS** — не розглядалось (rego дає IDE-фідбек через `tsandall.opa`).

**Зачіпає:** `npm/scripts/check-abie.mjs`, `npm/scripts/utils/run-conftest-batch.mjs`, `npm/scripts/lint-conftest.mjs`, `npm/policy/abie/` (4 пакети + `_test.rego`), `npm/tests/check-abie.test.mjs`, `.cursor/rules/conftest.mdc`, `npm/.claude-template/npm-CLAUDE.md`, `npm/CHANGELOG.md`, `npm/package.json`.
