## Knowledge Відсутність `keyPrefix` у Bun native Redis
**Контекст:** Міграція з `ioredis` (де є вбудована опція `keyPrefix`) на Bun native Redis — потрібно зрозуміти, чи є аналогічна опція.

**Рішення/Процедура/Факт:** `RedisOptions` у Bun native Redis (`bun-types/redis.d.ts`) не містить `keyPrefix`. Доступні лише 7 опцій: `connectionTimeout`, `idleTimeout`, `autoReconnect`, `maxRetries`, `enableOfflineQueue`, `tls`, `enableAutoPipelining`. Аналог реалізується вручну через helper-функцію `const k = (key: string) => prefix + key` або тонку обгортку з фабрикою.

**Обґрунтування:** Bun native Redis є мінімалістичним клієнтом без додаткових абстракцій — `keyPrefix` типово є "convenience wrapper" над звичайними операціями, тож ця відповідальність покладена на застосунок.

**Розглянуті альтернативи:** Logical DB (різні БД через `redis://host:port/N`) як груба ізоляція доменів; проте це не еквівалентно `keyPrefix` за семантикою.

**Зачіпає:** Будь-який код, що мігрує з `ioredis`/`node-redis` на `bun:redis`; особливо Lua-скрипти та pub/sub-канали (в `ioredis` `keyPrefix` не застосовується до них — та сама поведінка при ручному підході).
