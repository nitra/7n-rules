## ADR Ігнорування `.claude/worktrees/` у jscpd для всіх проєктів

**Контекст:** При запуску `bunx jscpd .` інструмент детектував фальшиві дублікати — він сканував `.claude/worktrees/`, де Claude Code зберігає тимчасові git worktree-копії поточного репо. Оскільки worktree є дзеркалом робочого дерева, будь-який файл там неминуче збігається з оригіналом.

**Рішення/Процедура/Факт:**
1. `.gitignore` у `cursor/` — додано рядок `.claude/worktrees/`, щоб git (і через нього `gitignore: true` у jscpd) ігнорував папку.
2. `.jscpd.json` у `cursor/` — додано `.claude/worktrees/**` у масив `ignore` як незалежна страховка (на випадок запуску jscpd без git-контексту).
3. `npm/mdc/js-lint.mdc` та `.cursor/rules/n-js-lint.mdc` — версія підвищена 1.17 → **1.18**, додано параграф про `.claude/worktrees/` у секції `.jscpd.json`: обидва підходи (`.gitignore` і `ignore`-масив) тепер задокументовані як норма для всіх проєктів.
4. `npm/package.json` — версія пакета 1.9.5 → **1.9.6**.
5. `npm/CHANGELOG.md` — запис `## [1.9.6]` з описом змін.
6. `.cspell.json` — розширено словник: `worktree`, `worktrees`, `комітних`, `комітні`, `лінтів`, `воркті`, `самозбіги`.

**Обґрунтування:** Worktree-копії є операційним артефактом Claude Code, а не частиною кодової бази; їхнє сканування дає гарантовані хибнопозитивні результати. Подвійне ігнорування (`.gitignore` + `ignore` у `.jscpd.json`) робить поведінку надійною незалежно від середовища запуску. Оскільки правило `js-lint.mdc` — канонічний стандарт для всіх nitra-проєктів, зміна одразу поширюється на весь стек через `@nitra/cursor`.

**Розглянуті альтернативи:** Явний `ignore`-запис у `.jscpd.json` без зміни `.gitignore` (відхилено — worktrees не повинні відстежуватись git у жодному разі; додавання в `.gitignore` є правильним концептуально).

**Зачіпає:** `.gitignore`, `.jscpd.json`, `npm/mdc/js-lint.mdc`, `.cursor/rules/n-js-lint.mdc`, `npm/package.json`, `npm/CHANGELOG.md`, `.cspell.json`
