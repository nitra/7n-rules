## ADR Виключення git worktree-директорій зі сканування jscpd

**Контекст:** Під час запуску `bunx jscpd .` інструмент знаходив хибні дублікати у `.claude/worktrees/` — тимчасових копіях репозиторію, створених Claude Code для ізольованої роботи над задачами. Ці файли є дзеркалом основного репо, тому їх порівняння завжди давало false-positive клони.

**Рішення/Процедура/Факт:**
- Додано `.claude/worktrees/` до `.gitignore` — правильне рішення за семантикою (worktree-копії не є частиною репо)
- Додано `.claude/worktrees/**` до масиву `ignore` у `.jscpd.json` — як страховка при запуску jscpd поза git-контекстом

**Обґрунтування:** `jscpd` підтримує опцію `"gitignore": true` (вже була в конфізі), тому додавання `.claude/worktrees/` до `.gitignore` вирішило б проблему саме по собі. Але явне дублювання в `.jscpd.json` захищає від сценаріїв, де git не ініціалізовано або jscpd запускається з іншої директорії.

**Розглянуті альтернативи:** Не обговорювалися; підхід «обидва файли» було обрано одразу як надійніший.

**Зачіпає:** `.gitignore`, `.jscpd.json`
