## ADR Автоматичне захоплення архітектурних рішень через Stop-хук Claude Code

**Контекст:** Команда хотіла автоматично зберігати нетривіальні рішення, прийняті під час AI-сесій, у вигляді чернеток ADR/Runbook/Knowledge — без ручного опису після кожної сесії.

**Рішення/Процедура/Факт:** Додано Stop-хук (`async: true`, timeout 180s) у `.claude/settings.local.json`, який запускає `.claude/hooks/capture-decisions.sh`. Скрипт: (1) читає JSONL-транскрипт сесії через `jq`, витягаючи `text`, `thinking` та назви `tool_use`-блоків; (2) передає екстракт у `claude -p --model sonnet` з промптом українською; (3) якщо відповідь починається з `^## ` (не NONE) — записує чернетку в `docs/adr/_inbox/<timestamp>-<session>.md`. Рекурсію зупиняє env-var `CAPTURE_DECISIONS_RUNNING=1`: якщо вже встановлено — скрипт виходить одразу.

**Обґрунтування:** Знайдено й виправлено три root-cause баги початкової реалізації: (1) `claude --bare` не читає OAuth-авторизацію (тільки `ANTHROPIC_API_KEY`) → пуста відповідь виглядала як NONE; (2) jq-екстракт пропускав `thinking` і `tool_use` → модель бачила майже порожню сесію і правомірно повертала NONE; (3) grep-фільтр перевіряв `^## \[` (з квадратними дужками), а модель генерує `## ADR Title` без них → всі валідні відповіді відкидались. Також haiku на початковому надто консервативному промпті відповідав NONE навіть на реальних рішеннях — перемкнуто на sonnet. Лінгвістичний баг: апостроф у тексті промпту всередині `$(...heredoc...)` ламав bash-парсинг — замінено на безапострофні конструкції.

**Розглянуті альтернативи:** `asyncRewake` замість `async` (відхилено — потреби у зворотному зв'язку до моделі немає); `--bare` для уникнення рекурсії (не працює з OAuth); haiku-модель (виявилась надто консервативною для задачі класифікації рішень).

**Зачіпає:** `.claude/hooks/capture-decisions.sh`, `.claude/settings.local.json`, `docs/adr/_inbox/` (вихідний каталог чернеток).
