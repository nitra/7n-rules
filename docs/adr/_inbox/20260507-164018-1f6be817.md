## ADR Розщеплення правила `image` на `image-compress` + `image-avif`

**Контекст:** У монорепо з кількома Vue-проєктами (наприклад, адмінка та публічний сайт) стиснення зображень потрібне для всіх, а AVIF-конвертація — лише для внутрішніх застосунків, де браузерна підтримка гарантована. Єдине правило `image` не дозволяло вимкнути лише AVIF для конкретного шляху, залишивши стиснення активним.

**Рішення/Процедура/Факт:** Правило `image` розщеплено на два самостійних:
- **`image-compress`** — валідує `lint-image` скрипт, `.gitignore`, заборонені залежності, легасі-кеш (`npm/scripts/check-image-compress.mjs`, `npm/mdc/image-compress.mdc`, `npm/mdc/image-compress.mdc`).
- **`image-avif`** — AVIF-pipeline: генерація `--avif`, переписування raster-посилань у `.vue`/`.html`, прибирання сиріт (`npm/scripts/check-image-avif.mjs`, `npm/mdc/image-avif.mdc`).

Для opt-out AVIF у конкретному workspace-пакеті: `"@nitra/minify-image": { "disable-avif": true }` у `package.json` цього пакета — `check image-avif` пропускає його цілком.

Автодетект залежностей у `auto-rules.mjs`: `image-compress - [bun]`, `image-avif - [vue, image-compress]` — без `image-compress` `image-avif` не активується.

Автоміграція через `RULE_MIGRATIONS` у `auto-rules.mjs`: при читанні `.n-cursor.json` старий id `image` замінюється на `['image-compress', 'image-avif']` із логом `🔄 Мігровано правило` під час виконання `npx @nitra/cursor`.

Версія: 1.8.194 → 1.8.199. Старі файли видалено: `check-image.mjs`, `mdc/image.mdc`, `.cursor/rules/n-image.mdc`, `tests/check-image.test.mjs`.

**Обґрунтування:** Гранулярне виключення на рівні правила (а не шляху цілком) потрібне через семантичну різницю між стисненням (безпечно скрізь) та AVIF (лише де є повна підтримка браузерів). Роздроблення — єдиний спосіб задовольнити цей use-case без custom-логіки у кожному конкретному репозиторії.

**Розглянуті альтернативи:**
- Розширити поле `ignore` об'єктним записом `{ "path": "apps/site", "rules": ["image"] }` — відхилено як семантично перевантажене.
- Додати окреме поле `disable-rules-paths` у `.n-cursor.json` — відхилено як надлишкове, якщо opt-out можна зробити на рівні `package.json` workspace.
- Використати наявний `"disable-avif": true` у `package.json` без розщеплення — вирішує конкретний use-case, але не дає загального механізму перемикання AVIF глобально через `.n-cursor.json`.
