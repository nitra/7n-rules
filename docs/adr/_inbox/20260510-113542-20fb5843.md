## ADR Документація `@nitra/cursor` через CI4-модель (C4-style) у Markdown

**Контекст:** Проєкт `@nitra/cursor` не мав жодної архітектурної документації. Паралельно було розроблено нове правило `ci4.mdc` (v1.8.221), яке обов'язує кожен проєкт-споживач мати C4-діаграми в Markdown — тож сам `@nitra/cursor` мусить задокументувати себе за власним стандартом (recursive case).

**Рішення/Процедура/Факт:** Ухвалено задокументувати архітектуру `@nitra/cursor` у каталозі `docs/ci4/` з п'ятьма файлами: `README.md` (індекс), `01-context.md` (L1), `02-containers.md` (L2), `03-components.md` (L3), `04-code.md` (L4) та `decisions.md` (двосторонній кросс-реф з ADR). Система позначення — Mermaid C4 у Markdown. Scope обмежено пакетом `npm/` (`@nitra/cursor`); `demo/` — зовнішній. Виділено 6 контейнерів: CLI Rule Sync, CLI Check Runner, CLI Stop-Hook, Capture-Decisions Hook, GitHub Action `setup-bun-deps`, Package Artifact. L4 drill-in охоплює всі контейнери. Кожен компонент матиме посилання на відповідні тести (`scripts/check-*.mjs`) згідно з вимогою `ci4.mdc`. Дизайн-спека збережена у `docs/superpowers/specs/2026-05-10-ci4-model-design.md`.

**Обґрунтування:** Mermaid обрано через нативний рендеринг у GitHub/Cursor без зовнішніх інструментів і відповідність text-first культурі проєкту (markdownlint, cspell). Окремі файли на рівень (замість одного монолітного) дають стабільні URL-якорі (`docs/ci4/02-containers.md#cnt-rule-sync`), на які ADR-файли можуть посилатися без ризику зламатися при переписуванні. Назва `ci4` (замість `c4`) зберігається скрізь для узгодженості з правилом `ci4.mdc`.

**Розглянуті альтернативи:** Один файл `docs/ci4-model.md` — відхилено через нестабільні якорі. Structurizr DSL — відхилено через потребу Docker/CLI залежності. PlantUML — відхилено через відсутність нативного рендерингу в GitHub. "Одна CLI з 4 субкомандами" замість 6 контейнерів — відхилено на користь більш точного C4 розподілу де Capture-Decisions Hook і GH Action — самостійні розгортувані одиниці.

**Зачіпає:** `docs/ci4/` (новий каталог, ще не створений), `docs/adr/_inbox/20260510-112235-20fb5843.md` (bootstrap ADR), `docs/superpowers/specs/2026-05-10-ci4-model-design.md` (дизайн-спека), `npm/mdc/ci4.mdc`, `.cursor/rules/n-ci4.mdc`.
