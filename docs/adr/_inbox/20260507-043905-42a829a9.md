## ADR Нове правило `adr` для впровадження механізму захоплення рішень у проєктах

**Контекст:** У монорепо `@nitra/cursor` вже працює Stop-hook `capture-decisions.sh`, що автоматично витягує ADR/Runbook/Knowledge із транскриптів Claude Code і пише чернетки в `docs/adr/_inbox/`. Цей механізм поки що живе лише в одному проєкті вручну; задача — перетворити його на cursor-правило, яке можна вмикати в будь-якому проєкті через `.n-cursor.json`.

**Рішення/Процедура/Факт:** Заплановано створити чотири артефакти:
1. `npm/mdc/adr.mdc` — опис правила (що таке Stop-hook capture-decisions, які файли потрібні, куди пишуться чернетки);
2. `npm/.claude-template/hooks/capture-decisions.sh` — канонічна копія скрипта в пакеті (single source of truth);
3. `npm/scripts/check-adr.mjs` — перевіряє наявність і коректність `.claude/hooks/capture-decisions.sh`, Stop-hook у `settings.json`/`settings.local.json`, директорії `docs/adr/_inbox/`, запису `.gitignore` для `*.log`;
4. Правило **не** додається до `auto-rules.md` — вмикається вручну через `rules: [..., "adr"]` у `.n-cursor.json`.
Сесія завершилась трьома питаннями до користувача (де реєструвати Stop-hook; чи перевіряти Claude CLI; чи синхронізувати скрипт автоматично через n-fix), тому реальний код ще не написано.

**Обґрунтування:** Механізм уже перевірений у проєкті й дає цінні архітектурні нотатки. Оформлення у вигляді cursor-правила дозволяє впроваджувати його у нові репозиторії одним рядком конфіга замість ручного копіювання скрипта. Ручне вмикання (не `auto-rules`) обрано свідомо: не всі проєкти потребують ADR-inbox, і Claude CLI має бути встановлений у розробника.

**Розглянуті альтернативи:**
- Автоматичне додавання через `auto-rules.md` — відхилено: надто широке охоплення, залежність від Claude CLI.
- Embed скрипта безпосередньо в `.mdc` — не обговорювалось детально; розглядається як резервний варіант, якщо sync через `node_modules` виявиться надто складним.
- Stop-hook у `settings.json` (project-shared) замість `settings.local.json` — відкрите питання, очікується відповідь користувача.

**Зачіпає:** `npm/mdc/adr.mdc` (новий), `npm/.claude-template/hooks/capture-decisions.sh` (новий), `npm/scripts/check-adr.mjs` (новий), `npm/schemas/n-cursor.json` (можливо, розширення переліку допустимих `rules`), `npm/package.json`, `npm/CHANGELOG.md`.
