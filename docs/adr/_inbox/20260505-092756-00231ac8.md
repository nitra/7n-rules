## ADR Правило `n-changelog`: обов'язковий CHANGELOG на рівні PR у Bun monorepo

**Контекст:** У проєктах Bun monorepo не було механізму, який би примусово вимагав оновлення `CHANGELOG.md` перед злиттям PR. Існуюче правило `n-npm-module` покривало лише workspace `npm/`, але решта workspace-ів залишалися без контролю.

**Рішення/Процедура/Факт:**
- Створено `npm/scripts/check-changelog.mjs` — скрипт перевірки, що порівнює поточний стан із базовою гілкою `dev` (через `git merge-base dev HEAD`).
- Логіка: для кожного workspace з кореневого `package.json.workspaces` (+ кореневий пакет, крім `npm/`), у якому є зміни відносно `dev`, перевіряє: (1) `version` підвищено, (2) `CHANGELOG.md` містить запис `## [version] - YYYY-MM-DD`.
- Написано `npm/mdc/changelog.mdc` та `.cursor/rules/n-changelog.mdc` (дзеркальні версії правила).
- Правило додано до `AUTO_RULE_ORDER` в `auto-rules.mjs` та описано в `auto-rules.md` з умовою: активується якщо є кореневий `package.json`.
- Додано тести `npm/tests/check-changelog.test.mjs` (10 сценаріїв: skip без git, skip на гілці `dev`, skip якщо `dev` ref відсутній, workspace без змін, відсутній bump, відсутній запис у CHANGELOG, happy path, тощо).
- Версія пакету підвищена до `1.8.174`, запис у `npm/CHANGELOG.md` додано.
- У `.n-cursor.json` самого репо `changelog` додано до `disable-rules`, бо цей репо використовує `main` замість `dev`.

**Обґрунтування:** Перевірка PR-scoped (а не від `HEAD`) відповідає реальному workflow: розробник робить проміжні коміти без bump-у, а версію та CHANGELOG оновлює один раз перед злиттям. Хардкод бази `dev` обраний через простоту (95% проєктів); `origin/dev` також підтримується як fallback. Приватні workspace-и перевіряються повністю, оскільки changelog є внутрішнім інструментом і `files`-перевірку пропускати немає сенсу для всіх (все одно приватні).

**Розглянуті альтернативи:**
- **Варіант A (порівняння з `HEAD`)** — відкинуто: вимагав би bump у кожному коміті, що суперечить workflow.
- **Пропускати приватні workspace-и** — відкинуто: усі проєкти приватні, changelog для внутрішнього використання однаково важливий.
- **Конфігурована базова гілка** — відкладено: залишено для `.n-cursor.json` у майбутньому.
