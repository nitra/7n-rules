## ADR Компактний npm-модуль: обовʼязковий `files`, заборона `devDependencies`, тести поза тарболом

**Контекст:** Пакет `@nitra/cursor` публікувався без суворих гарантій компактності: `"devDependencies"` могли потрапити до `npm/package.json`, `"files"` — бути відсутніми, а rego-юніт-тести (`*_test.rego`) — включатися в tarball.

**Рішення/Процедура/Факт:**
- `npm-module.mdc` (версія `1.10 → 1.11`) розширено трьома правилами: `"files"` whitelist є обовʼязковим; тести й фікстури мають лежати поза будь-яким виданим шляхом; `npm/package.json` містить лише runtime-залежності — без `devDependencies`.
- `check-npm-module.mjs` отримав `checkPackageCompactness` (перевірка наявності `files` і відсутності `devDependencies`) та `checkNoTestsInPublishedFiles` (glob-резолвер `files` з підтримкою негативних патернів → для кожного файлу: test-каталог, `*.test.*`/`*.spec.*`/`*_test.rego`-суфікс, або AST-імпорт test-фреймворку через oxc-parser).
- До `"files"` у `npm/package.json` додано `"!**/*_test.rego"` — 14 rego-юніт-тестів більше не потрапляють у tarball (підтверджено `npm pack --dry-run`).
- Прибрано `"devDependencies": { "@nitra/cursor": ... }` з `npm/package.json` — пакет уже підключений через `workspace:*` у кореневому `package.json`.
- Написано 14 unit-тестів для `globToRegex`, `findTestFrameworkImport`, `classifyPublishedFileAsTest` у `npm/tests/check-npm-module.test.mjs`.
- Версію підвищено до `1.9.5`, додано запис у `npm/CHANGELOG.md`.

**Обґрунтування:** Мета — максимально компактний модуль: жодних зайвих байтів для користувача пакета, чіткий поділ між тим, що тестується в репо, і тим, що публікується в npm.

**Розглянуті альтернативи:** Детектування тестів за glob-іменами без AST-аналізу — відхилено, бо не вловлює файли з нестандартними іменами, які імпортують test-фреймворки; детектування лише за `_test.rego`-суфіксом — недостатньо для JS-тестів.

**Зачіпає:** `npm/mdc/npm-module.mdc`, `.cursor/rules/n-npm-module.mdc`, `npm/scripts/check-npm-module.mjs`, `npm/package.json`, `npm/CHANGELOG.md`, `npm/tests/check-npm-module.test.mjs`
