## ADR Реорганізація `npm/scripts/` за зразком `npm/policy/`

**Контекст:** `npm/scripts/` зростав як пласка папка: ~28 `check-*.mjs`, кілька `lint-*.mjs` та `run-*.mjs` різних доменів, крос-правильні утиліти (`lint-conftest.mjs`, `auto-rules.mjs`, `sync-claude-config.mjs`, `cli-entry.mjs`) та `utils/`. Навпаки, `npm/policy/` вже має чітку структуру — окремий каталог на кожне правило (`docker/`, `ga/`, `k8s/`, `js_ru/` тощо) зі своїми файлами всередині.

**Рішення/Процедура/Факт:** Реорганізувати `npm/scripts/` так, щоб кожне правило мало власний каталог (назва відповідає policy-slug), куди переносяться всі файли цього домену — `check-{rule}.mjs`, супутні `lint-{rule}.mjs` та `run-{rule}.mjs` (які обслуговують виключно цей домен). Файли, що обслуговують кілька правил або є інфраструктурою CLI (`lint-conftest.mjs`, `auto-rules.mjs`, `sync-claude-config.mjs`, `cli-entry.mjs`, `utils/`), лишаються безпосередньо в `npm/scripts/` (поза каталогами правил).

Виявлена класифікація файлів:
- **Rule-specific** → у каталог правила: `check-ga + lint-ga` → `ga/`, `check-docker + run-docker` → `docker/`, `check-k8s + run-k8s` → `k8s/`, `check-text + run-shellcheck-text + run-v8r` → `text/`, `check-rego + lint-rego` → `rego/`, `check-php + run-php` → `php/`, тощо.
- **Shared / cross-rule** → залишаються в корені `scripts/`: `lint-conftest.mjs` (батч-раннер conftest по всіх policy/*), `auto-rules.mjs`, `sync-claude-config.mjs`, `cli-entry.mjs`, `build-agents-commands.mjs`, `utils/`.

`bin/n-cursor.js` динамічно знаходить `check-*` через `readdir(BUNDLED_SCRIPTS_DIR)` — після реорганізації слід або розширити пошук рекурсивно, або перенести точки входу з урахуванням нових шляхів. Статичні `import` у `n-cursor.js` потребуватимуть оновлення шляхів.

Конвенцію іменування каталогів (snake_case як у `policy/` чи kebab-case як у поточних файлах) не було остаточно узгоджено — brainstorming перервано.

**Обґрунтування:** Симетрія зі структурою `npm/policy/` знижує когнітивне навантаження: розробник, який додає нове правило, одразу знає, куди класти і check-, і lint-/run-файли. Плоска папка вже важко масштабується (28+ файлів без очевидної організації).

**Розглянуті альтернативи:** Зберегти пласку структуру — відхилено як не масштабоване. Перенести лише `check-*` в каталоги, решту лишити в корені — обговорено, але не прийнято (розмиває границі домену).

**Зачіпає:** `npm/scripts/` (усі файли), `npm/bin/n-cursor.js` (статичні `import` і динамічний `readdir` через `BUNDLED_SCRIPTS_DIR`), тести що імпортують `../scripts/check-*.mjs` напряму.
