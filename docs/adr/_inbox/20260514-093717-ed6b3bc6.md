## ADR Скіл `abie-clean`: очищення проєктів від `ru`-середовища

**Контекст:** У проєктах `abie`-екосистеми накопичились артефакти підтримки `ru`-середовища: директорії `ru/`, файли `*-ru`/`values-ru.*`, умовні гілки в GitHub Actions, `runs-on` вирази з `ya`-раннером, Docker-рядки з `BRANCH=ru` та nginx-конфіги з country-роутингом на Росію. Потрібно систематично прибирати ці залишки в будь-якому дочірньому репо.

**Рішення/Процедура/Факт:** Написано скіл `npm/skills/abie-clean/SKILL.md` (version 1.1), який охоплює п'ять зон очищення:
1. **Директорії** — `find . -type d -name "ru" -exec rm -rf {} +`
2. **Файли з суфіксом** — `values-ru.*`, `*.env.prod-ru` та аналогічні
3. **`.github/workflows`** — прибрати `ru` з `branches:`, `ignore_branches:`; спростити тернарні вирази `runs-on` і env-змінних (`NATS_URL`, `NATS_STREAM`, `REGISTRY`) до двогалузевого `dev || ua`
4. **Dockerfile** — замінити умовний `RUN if [ "$BRANCH" = "ru" ]...` на безумовне `RUN bun install && bun vite build ...`
5. **Nginx / інші конфіги** — видалити посилання на `ru`; фінальна перевірка через `git grep -rn 'ru'`

Скіл автоматично підключається до `.n-cursor.json` за правилом `abie-clean - [abie]` (файл `npm/bin/auto-skills.md`).

**Обґрунтування:** Скіл потрібен, щоб механізовано й повторювано очищати будь-який abie-репозиторій без ручного пошуку патернів. Охоплення всіх п'яти зон у одному скілі гарантує, що жоден артефакт `ru` не залишиться після запуску.

**Розглянуті альтернативи:** Не обговорювались; користувач одразу описав перелік зон і попросив доповнити SKILL.md.

**Зачіпає:** `npm/skills/abie-clean/SKILL.md`, `npm/bin/auto-skills.md` (правило автододавання `abie-clean - [abie]`); застосовується до будь-якого дочірнього репо з правилом `abie`.
