## ADR `--avif` видалено з `lint-image`; AVIF-генерація перенесена до `check image`

**Контекст:** Скрипт `lint-image` у кореневому `package.json` містив прапорець `--avif`, через що звичайний `bun run lint` непотрібно генерував `.avif`-двійники для зображень без будь-якого ув'язування із `.vue`/`.html`.

**Рішення/Процедура/Факт:** Прапорець `--avif` заборонено в `lint-image` (перевірка з явним `fail`). Канонічний вигляд: `npx @nitra/minify-image --src=. --write`. Увесь AVIF-pipeline перенесено до `npx @nitra/cursor check image`: (1) `npx @nitra/minify-image --src=. --write --avif` (best-effort, без `--avif` у `lint`), (2) in-place rewrite raster-посилань у `.vue`/`.html` коли AVIF-двійник реально існує на диску, (3) видалення orphan-AVIF. Змінені файли: `.cursor/rules/n-image.mdc` (v1.4→v1.5), `npm/mdc/image.mdc`, `npm/scripts/check-image.mjs`, `npm/tests/check-image.test.mjs`, `npm/package.json` (1.8.184→1.8.196), `npm/CHANGELOG.md`, `package.json` (root).

**Обґрунтування:** `bun run lint` — швидка локальна перевірка; генерація AVIF при кожному lint-і — зайва робота і ризик «зайвих» AVIF без живих посилань. `check image` — явна операція, де відомий весь контекст (workspace-пакети, opt-out, .vue-файли) і можна атомарно виконати generate → rewrite → cleanup.

**Розглянуті альтернативи:** Залишити `--avif` у `lint-image` і повністю відмовитись від rewrite-логіки (відкинуто — AVIF без `.vue`-посилань не потрапляє в production).

**Зачіпає:** `npm/scripts/check-image.mjs`, `npm/tests/check-image.test.mjs`, `npm/mdc/image.mdc`, `.cursor/rules/n-image.mdc`, кореневий `package.json`.

## ADR Виправлення `resolveImagePath` для Vite/Quasar-проєктів

**Контекст:** Реалізація 1.8.193 виконувала 0 rewrite у реальних Quasar/Vite-проєктах: `/api-page/1.png` резолвилось як `<cwd>/api-page/1.png` (не знаходилось), а голі relative-шляхи без `./` (`assets/images/x.png`) повертали `null` і мовчки пропускалися, збільшуючи `failedRefs` і призводячи до orphan-cleanup стосовно AVIF, які мали бути валідними.

**Рішення/Процедура/Факт:** `resolveImagePath` замінено на `resolveImageCandidates`, що повертає впорядкований список кандидатів; перший, чий `.avif` існує, і є резолюцією. Порядок для `/path`: (a) `<pkgRoot>/public<path>`, (b) `<pkgRoot><path>`, (c) `<cwd><path>`. Для голих шляхів хоча б з одним `/` (не alias): відносно файла-джерела + `<pkgRoot>/public/<path>`. Без `/` взагалі (bare module alias) → пропустити без `fail`. Cleanup тепер ігнорує `build`, `android`, `ios`, `.output`, `.nuxt`, `.cache` (AVIF у цих каталогах — артефакти збірки, а не кандидати для видалення). Нові тести: Quasar-style абсолютний `/api-page/...` → `public/`, голий relative у `.html`, opt-out-пакет зберігає свій AVIF. Версія: 1.8.194.

**Обґрунтування:** Vite/Quasar трактують `/` як `<project>/public/`, а не корінь ФС. Без цього виправлення `check image` давав 32 ❌ у реальних проєктах і тут же видаляв тільки що згенеровані AVIF як «orphan».

**Розглянуті альтернативи:** Резолвити лише через `<cwd>` (стара поведінка — не підходить для monorepo з `public/`).

**Зачіпає:** `npm/scripts/check-image.mjs` (функції `resolveImageCandidates`, `cleanupOrphanAvifs`), `npm/tests/check-image.test.mjs`.

## ADR Відкат sha1-кешу AVIF-інвалідації (`deleteStaleAvifs`) на користь патча `@nitra/minify-image`

**Контекст:** Версія 1.8.195 додала `.n-image-avif.tsv` (committed sha1-кеш raster-сорсів) і функції `deleteStaleAvifs` / `refreshAvifSourceHashCache` — щоб при оновленні `image1.png` застарілий `image1.png.avif` видалявся і регенерувався. Але `@nitra/minify-image` вже має власний кеш `.n-minify-image.tsv` зі схожою інформацією.

**Рішення/Процедура/Факт:** Усю sha1-логіку видалено з `check-image.mjs` (відкат до 1.8.194 поведінки). Вирішено реалізувати інвалідацію всередині самого `@nitra/minify-image` — він уже знає sha1 сорсів і може під час `--avif` сам перевіряти свіжість. Відповідно скасовано: `HASH_CACHE_FILENAME`, `checkHashCacheNotIgnored`, `sha1OfFile`, `collectRasterSources`, `deleteStaleAvifs`, `refreshAvifSourceHashCache` і три тести для них. Версія: 1.8.196.

**Обґрунтування:** Дублювати кеш і логіку між двома пакетами — джерело розсинхронізації. Патч в `@nitra/minify-image` дає інвалідацію усім консьюмерам безкоштовно і без додаткового committed-файлу в кожному репо.

**Розглянуті альтернативи:** Залишити `.n-image-avif.tsv` як bridge/fallback до релізу `@nitra/minify-image` (відхилено — плутає відповідальність і додає committed-файл без достатньої причини).

**Зачіпає:** `npm/scripts/check-image.mjs`, `npm/tests/check-image.test.mjs`, `npm/mdc/image.mdc`, `.cursor/rules/n-image.mdc` (v1.6 → v1.5), `npm/package.json` (1.8.195 → 1.8.196).
