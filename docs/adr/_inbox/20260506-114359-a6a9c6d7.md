## ADR Розділення AVIF-генерації між `lint-image` і `check image`

**Контекст:** Скрипт `lint-image` у `package.json` містив прапорець `--avif`, через що команда `bun run lint` (яка входить до агрегованого `lint`) автоматично генерувала AVIF-файли при кожному локальному запуску. Це призводило до небажаної генерації артефактів під час звичайного lint-циклу.

**Рішення/Процедура/Факт:**

- Канонічний `lint-image` тепер: `npx @nitra/minify-image --src=. --write` (без `--avif`)
- AVIF-генерація перенесена виключно в `npx @nitra/cursor check image` (best-effort через `spawnSync`)
- Додано очищення «сиріт»: якщо `.avif`-файл існує, але жодного `.vue`/`.html` не посилається на нього — файл видаляється
- `check-image.mjs` отримав нові функції: `runAvifGeneration()`, `collectOrphanAvifFiles()`, логіку cleanup у `check()`
- `check-image.test.mjs` отримав тести для цих сценаріїв; env-змінна `NITRA_CURSOR_NO_AVIF_RUN` (з `node:process`) дозволяє пропустити `npx`-виклик у тестах
- `n-image.mdc` оновлено до версії `1.5`, синхронізовано до `npm/mdc/image.mdc`
- Кореневий `package.json` виправлено (прибрано `--avif`)
- Версія пакета `@nitra/cursor` підвищена до `1.8.185`, внесено запис у `CHANGELOG.md`

**Обґрунтування:** `bun run lint` запускається часто і повинен бути швидким та детермінованим; генерація AVIF — дорога і мережозалежна операція. Крім того, зберігати `.avif` треба лише тоді, коли є реальний споживач у `.vue`/`.html` — інакше файл стає баластом у git.

**Розглянуті альтернативи:** не обговорювалися
