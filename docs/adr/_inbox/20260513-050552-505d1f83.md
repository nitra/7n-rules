## ADR Компактний npm-пакет: rego-first архітектура перевірок і заборона devDependencies
**Контекст:** У `check-npm-module.mjs` накопичилися перевірки структури одного файлу `npm/package.json`, які за конвенцією проєкту мали б лежати у Rego-полісі. Крім того, `npm/package.json` містив `devDependencies: { "@nitra/cursor": "^1.8.170" }`, хоча він є workspace-членом і `@nitra/cursor` потрапляв у tarball для кінцевих користувачів.
**Рішення/Процедура/Факт:** 1) Правило `npm-module.mdc` (версія `1.11`) отримало три нові вимоги: обов'язковий непорожній whitelist `"files"`, відсутність `"devDependencies"`, тести/fixtures поза опублікованими каталогами (виняток — `*_test.rego` живуть поруч з полісі як conftest-конвенція). 2) Пер-документні deny (`"files"` присутній/непорожній, `"types"` є, немає `"devDependencies"`) перенесено у `npm/policy/npm_module/npm_package_json/npm_package_json.rego`; написано `npm_package_json_test.rego` з 7 позитивними і негативними кейсами. 3) `checkPackageCompactness` видалено з `check-npm-module.mjs`; лишився тільки `checkNoTestsInPublishedFiles` (FS walk + AST-парсинг через oxc-parser — Rego це не вміє). 4) `npm/package.json`: прибрано `devDependencies`; `"!**/*_test.rego"` не додавали (rego-тести повертаємо у tarball). 5) `knip.json`: прибрано застарілі `ignoreDependencies`/`ignoreBinaries` (graphql, vue, @nitra/cursor, vite). 6) `eslint.config.js`: `.claude/worktrees/**` додано до `ignores`. 7) `conftest.mdc` і `npm/.claude-template/npm-CLAUDE.md` отримали чіткіші STOP-блоки: «перед редагуванням `check-*.mjs` — перевір, чи це пер-документна задача для Rego».
**Обґрунтування:** Rego-полісі — авторитетне місце для пер-документних структурних перевірок (conftest.mdc). JS-чек з `if (pkg.devDependencies)` — це дублювання логіки, яка краще виражається через `deny contains msg if { input.devDependencies }`. Поділ зменшує ризик повторення помилки в майбутньому.
**Розглянуті альтернативи:** (а) Залишити обидві перевірки у JS — відхилено, порушує conftest.mdc. (б) Фізично перенести `*_test.rego` за межі `policy/` → `npm/tests/policy/` — відхилено як надмірний rework (conftest шукає `*_test.rego` у тому ж `package`, що й полісі; виграш мінімальний).
**Зачіпає:** `npm/mdc/npm-module.mdc`, `.cursor/rules/n-npm-module.mdc`, `npm/policy/npm_module/npm_package_json/npm_package_json.rego` (новий файл `npm_package_json_test.rego`), `npm/scripts/check-npm-module.mjs`, `npm/package.json`, `knip.json`, `eslint.config.js`, `.cursor/rules/conftest.mdc`, `npm/.claude-template/npm-CLAUDE.md`, `npm/CLAUDE.md`.
