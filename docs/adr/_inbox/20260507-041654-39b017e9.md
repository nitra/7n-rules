## ADR Мінімальна версія Node.js ≥ 26 та використання Temporal API

**Контекст:** Вийшов Node.js 26 з офіційним включенням Temporal API (без прапорів) та іншими нативними вдосконаленнями. Проєкт `n-cursor` не потребує зворотної сумісності з попередніми версіями Node.js.

**Рішення/Процедура/Факт:** Вирішено оновити мінімальну версію Node.js до `>=26` у `engines` полях `package.json` та `.nvmrc`/`.node-version`. Замість legacy `Date` об'єкта — використовувати `Temporal` API (доступний нативно без `--experimental-*` прапорів). Інші оптимізації Node.js 26: нативний `fetch`, `streams`, оновлений `fs`/`path` — використовувати без поліфілів.

**Обґрунтування:** Оскільки сумісність з попередніми версіями не потрібна, можна повністю замінити `new Date()` / `Date.now()` на `Temporal.Now.*`, що дає точніший timezone-aware API без залежностей. Node.js 26 дозволяє прибрати будь-які runtime-поліфіли для `fetch`, `Temporal`, `ReadableStream` тощо.

**Розглянуті альтернативи:** Залишити `>=22` або `>=24` як мінімум — відхилено, бо Temporal API там ще під прапором і потребує поліфілу (`@js-temporal/polyfill`).

**Зачіпає:** `package.json` (root + workspaces `demo`, `npm`), `.nvmrc`, `.node-version`, `.cursor/rules/n-js-run.mdc` (довідкові вимоги до Node-версії), будь-який код що використовує `new Date()`.
