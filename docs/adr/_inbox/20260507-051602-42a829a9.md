## ADR Правило `adr` для автоматичного розгортання механізму capture-decisions

**Контекст:** У проекті `@nitra/cursor` вже існував локальний Stop-хук `.claude/hooks/capture-decisions.sh`, що витягує ADR/Runbook/Knowledge-чернетки з транскриптів Claude Code. Механізм не мав способу тиражуватись у нові проекти через систему правил пакета.

**Рішення/Процедура/Факт:**
- Додано `npm/mdc/adr.mdc` — нове правило із ручним вмиканням через `rules: ["adr"]` у `.n-cursor.json` (навмисно відсутнє в `auto-rules.md`).
- Канонічний скрипт перенесено в `npm/.claude-template/hooks/capture-decisions.sh` з fallback-логікою: якщо `claude` CLI недоступний, використовується `cursor-agent` (`-p --mode ask --output-format text`); моделі конфігуруються через `CAPTURE_DECISIONS_CLAUDE_MODEL` і `CAPTURE_DECISIONS_CURSOR_MODEL`.
- `npm/scripts/sync-claude-config.mjs` оновлено: при `rules` що містить `"adr"` — копіює скрипт у `.claude/hooks/` і додає managed Stop-групу до `.claude/settings.json`; при видаленні `"adr"` зі списку — прибирає managed-групу (але не скрипт).
- ADR Stop-хук перемістився з `.claude/settings.local.json` до `.claude/settings.json` (project-shared), щоб команда бачила його без локальних оверайдів.
- Додано `npm/scripts/check-adr.mjs` із 5 перевірками: байт-збіг скрипта з канонічним, наявність ADR-групи в `settings.json`, відсутність дубля в `.local.json`, покриття `.gitignore` для лог-файлу, наявність LLM CLI (`claude` або `cursor-agent`).
- Тести: `npm/tests/check-adr.test.mjs` (7 кейсів), 5 нових кейсів у `sync-claude-config.test.mjs`. Версію пакета підвищено до `1.8.189`.

**Обґрунтування:** Правило вмикається вручну, оскільки корисність ADR-механізму залежить від робочого процесу команди — автододавання спровокувало б шум у проектах, де команда не веде ADR. Fallback на `cursor-agent` усуває залежність від конкретного CLI; автоматичний sync через `n-cursor` гарантує, що скрипт завжди відповідає канонічній версії пакета. Hook розміщено в project-shared `settings.json`, а не `.local.json`, щоб інфраструктура була видима всій команді без ручного кроку на кожній машині.

**Розглянуті альтернативи:**
- Залишити hook у `.claude/settings.local.json` — відкинуто: новим учасникам команди треба б було вручну копіювати налаштування.
- Додати в `auto-rules.md` для автовмикання — відкинуто: ADR-ведення є усвідомленим вибором, а не технічною вимогою стека.
- Перевіряти лише `claude` CLI без fallback — відкинуто: `cursor-agent` доступний у PATH, вміє звертатись до Claude-моделей, тому варто його підтримати як рівноправну альтернативу.
