## ADR Виправлення lint-порушень: JSDoc, ESLint, jscpd, cspell, markdownlint у npm/scripts

**Контекст:** `bun run lint` (`oxlint` + `eslint` + `jscpd` + `cspell` + `markdownlint`) завершувався з помилками в ~10 скриптах — відсутні JSDoc `@param`/`@returns` описи, статичні регекси всередині функцій, дубльований блок `findAllPackageJsonPaths` у двох файлах, невідомі українські слова у cspell-словнику, заборонені дублі заголовків у CHANGELOG, непрольований Vue SFC парсинг та незаекранована змінна в GitHub Actions.

**Рішення/Процедура/Факт:**

- `oxlint` `require-param` / `require-returns-description`: в усіх `npm/scripts/*.mjs` та `utils/*.mjs` додано описи до кожного `@param` і `@returns` тегу, де вони були відсутні (окремо опрацьовано `check-changelog.mjs`, `sync-claude-config.mjs`, `walkDir.mjs`, `check-vue.mjs`, `check-js-run.mjs`, `check-nginx-default-tpl.mjs`, `bun-sql-scan.mjs` та ін.)
- `e18e/prefer-static-regex`: регекс-літерали перенесено з тіл функцій на рівень модуля як `const` (`N_CURSOR_LINT_GA_RE` у `check-ga.mjs`, `MEGALINTER_CONFIG_NAMES`, `TRAILING_SLASHES_RE` у `walkDir.mjs` та `load-cursor-config.mjs`)
- `sonarjs/no-os-command-from-path`: де виклики `spawn`/`execFileSync` через PATH є легітимними (npx, git у тестах), додано `// eslint-disable-next-line sonarjs/no-os-command-from-path -- <обґрунтування>`
- `sonarjs/prefer-single-boolean-return`: блок `if/return true; return false` у `check-changelog.mjs:workspaceHasChangesAgainstBase` переписано на `return code === 1`
- **Нова утиліта** `npm/scripts/utils/find-package-json-paths.mjs` з `findAllPackageJsonPaths(repoRoot, ignorePaths)` — винесено з `check-js-bun-db.mjs` і `check-js-mssql.mjs`, де код був ідентичним (29 рядків, 130 токенів)
- `eslint.config.js`: додано `vue: ['demo']` до виклику `getConfig(...)`, щоб `vue-eslint-parser` обробляв `demo/src/**/*.vue`; без цього `demo/src/App.vue` та `main.js` давали `Parsing error: Unexpected token <` і `'createApp' is not defined`
- `.jscpd.json`: до `ignore` додано `"npm/.claude-template/**"` — файли шаблону навмисно ідентичні їх копіям у проєктах-споживачах
- `.markdownlint-cli2.jsonc`: додано `"MD024": { "siblings_only": true }` — у `CHANGELOG.md` легітимно мати `### Added` / `### Changed` у кожній версії
- `.cspell.json`: додано ~60 відсутніх українських слів (`Воркспейс`, `бандл`, `бампити`, `регексп` тощо) та технічних термінів (`PACMAN`, `gpgsign`, `Rewake`)
- `.github/workflows/git-ai.yml`: `$GITHUB_PATH` → `"$GITHUB_PATH"` (екранування для `actionlint` SC2086)
- `demo/src/App.vue`: `import welcomeImage from './assets/welcome.png'` → `'./assets/welcome.png.avif'` (вимога правила `image.mdc`)
- Версія `npm/package.json` підвищена `1.8.180 → 1.8.181`, додано запис у `CHANGELOG.md`

**Обґрунтування:** `bun run lint` є gate перед будь-яким merge; всі помилки блокували CI. JSDoc описи вимагає oxlint з активованим `eslint-plugin-jsdoc`; статичні регекси — правило `e18e/prefer-static-regex`, яке запобігає повторній компіляції на кожен виклик; дублювання `findAllPackageJsonPaths` порушувало ліміт jscpd (25 рядків); `.claude-template` є source-of-truth для шаблонів і його ідентичність з копіями є навмисною.

**Розглянуті альтернативи:** Для `sonarjs/slow-regex` спочатку розглядалась заміна `replace(/\/+$/, '')` на `while`-цикл, але у підсумку достатньо було винести регекс у module-scope константу.
