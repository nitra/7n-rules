## ADR Автоматична нормалізація ADR-чернеток у місці замість двоступеневого inbox/promote

**Контекст:** Stop-hook `capture-decisions.sh` накопичує чернетки у `docs/adr/_inbox/` (156 файлів), але формального процесу промоції не існує — файли лежать без обробки. Задокументована в `n-adr.mdc` порада «переносити вручну» на практиці не виконується.

**Рішення/Процедура/Факт:** Відмовитися від концепції окремих папок `_inbox/` / `_promoted/`. Усі файли живуть в одному каталозі (`docs/adr/`). `capture-decisions.sh` пише чернетку з маркером у frontmatter (`status: raw` або коментар `> Ще неочищений`). При досягненні threshold (наприклад 30 необроблених файлів) Stop-hook запускає `promote-decisions.sh`, який:
1. Двофазний LLM-пайплайн: перший виклик — JSON-кластеризація raw-файлів `[{title, sources:[...]}]`; другий — синтез фінального тексту для кожного кластера.
2. **Якщо кластер несе реальне рішення** — LLM перезаписує файл з нормалізованим змістом і прибирає маркер `status: raw`; присвоює `NNNN`-нумерацію.
3. **Якщо кластер тривіальний або дублює вже очищений ADR** — файл видаляється повністю.
4. Recursion guard `ADR_PROMOTE_RUNNING=1` запобігає повторному запуску під час внутрішньої LLM-сесії.

**Обґрунтування:** Єдиний каталог без переміщень — простіша розумова модель: немає «де шукати фінальний ADR»; `git status` після нормалізації показує `modified` (очищено) та `deleted` (відкинуто) — природній review-вікно без окремого `_pending/`. Маркер `status: raw` у frontmatter дозволяє trivially відфільтрувати нероблені файли (`grep -rl 'status: raw' docs/adr/`).

**Розглянуті альтернативи:**
- `_inbox/` → `docs/adr/NNNN-*.md` + сирці у `_inbox/_promoted/` — додаткова папка, складніша схема переміщень, залишає сміттєві сирці.
- Continuous-промоція на кожен Stop — відкинута: рішення ще не стабілізовані під час активної сесії.
- Однофазний LLM-виклик (30 файлів одразу) — ризик упустити теми або склеїти різні рішення; двофазний (кластеризація JSON → синтез кожного кластера) надійніший при дебагінгу.

**Зачіпає:** `npm/.claude-template/hooks/capture-decisions.sh`, новий `npm/.claude-template/hooks/promote-decisions.sh` (планується), `.cursor/rules/n-adr.mdc`, `.n-cursor.json` (конфіг threshold та опційний прапор `"adr-promote"`), структура `docs/adr/`.
