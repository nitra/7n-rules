## ADR Масова міграція перевірок check-*.mjs у Rego-поліси conftest

**Контекст:** Проєкт мав 17+ JavaScript-скриптів `check-*.mjs`, кожен з яких містив логіку перевірки конфігурацій (package.json, workflow YAML, k8s manifests тощо). Вся ця логіка була нетестованою окремо і дублювалась між скриптами. Попередній PoC (ga-поліси) довів, що conftest + Rego — ефективна заміна для декларативних перевірок.

**Рішення/Процедура/Факт:** Портовано перевірки з 12 категорій check-скриптів у 36 Rego-файлів під `npm/policy/`:
- `ga/workflow_common/` — універсальні правила для КОЖНОГО `.github/workflows/*.yml` (concurrency, заборона setup-bun/cache, shell line-continuation, checkout-order)
- `bun/` — `bunfig.toml` (linker=hoisted) та `package.json` (packageManager, lint-скрипт, oxfmt)
- `text/` — `.oxfmtrc.json`, `.cspell.json`, `package.json`, `.markdownlint-cli2.jsonc`
- `style_lint/`, `php/`, `js_lint/`, `js_mssql/`, `js_bun_db/`, `js_run/`, `vue/`, `graphql/`, `image_compress/`, `hasura/`, `adr/`, `capacitor/`, `abie/`, `k8s/`, `npm_module/`

Написано `npm/scripts/lint-conftest.mjs` — єдиний runner, що: читає `rules` з `.n-cursor.json`, пропускає GA-поліси (вони у `lint-ga`), і для кожної активної поліси резолвить target-файли (single або walk) та запускає conftest. Додано `lint-conftest` до кореневого скрипту `lint`. Усунуто дублювання concurrency-правил у 4 наявних GA-полісях (тепер лише у `workflow_common`). Виправлено Rego-трап: `not is_object(missing_field)` повертає `undefined`, а не `true` — замінено на `object.get(input, "field", false) == false`.

**Обґрунтування:** Conftest спроєктований саме для декларативних перевірок структурованих файлів (JSON/YAML) — він читає файл як `input`, Rego-правила виражають інваріанти без побічних ефектів, результати машиночитабельні. JS-скрипти залишаються тонкими оркестраторами лише там, де потрібен AST-скан (oxc-parser для import-аналізу) або FS-обхід з фільтрацією.

**Розглянуті альтернативи:**
- Залишити всю логіку в JS (`check-*.mjs`) — відхилено: немає ізоляції, важко тестувати окремі правила
- Міграція лише чистих JSON/YAML-перевірок, без гібридів — частково реалізовано: гібриди (js-lint, js-run, graphql) портують лише структурну частину, AST-скани залишаються в JS
- Один монолітний Rego-файл замість namespace-ієрархії — відхилено: regal вимагає directory-package-mismatch відповідності, conftest namespace-routing залежить від структури директорій

**Зачіпає:** `npm/policy/` (36 нових .rego файлів), `npm/scripts/lint-conftest.mjs` (новий), `npm/scripts/lint-ga.mjs` (додано workflow_common loop), `package.json` (додано `lint-conftest` до `lint`), `npm/package.json` (версія 1.8.207), `npm/CHANGELOG.md`, `npm/policy/ga/{clean_ga_workflows,clean_merged_branch,git_ai,lint_ga}/*.rego` (видалено дублювання concurrency)
