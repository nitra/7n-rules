## ADR Міграція структурних перевірок GA-workflow із JS до Rego/conftest

**Контекст:** `npm/scripts/check-ga.mjs` (1074 рядки) містив великий блок функцій `validateCleanGaWorkflows*`, `validateCleanMergedBranch*`, `validateLintGaWorkflowStructure`, `validateGitAiWorkflowStructure`, які перевіряли структуру чотирьох канонічних GitHub Actions workflow через JavaScript. Перевірки дублювали один і той самий шаблонний патерн (parse YAML → порівняти рядки/ключі) без декларативного опису правил.

**Рішення/Процедура/Факт:**

- Створено чотири Rego-полісі у `npm/policy/ga/` з namespaced sub-packages (вимога regal): `clean_ga_workflows/`, `clean_merged_branch/`, `lint_ga/`, `git_ai/` — кожна з `package ga.<name>` і Rego v1 синтаксисом (`import rego.v1`, `deny contains msg if { … }`).
- `npm/scripts/lint-ga.mjs` доповнено кроком `runConftestStep()` з масивом `CONFTEST_TARGETS` (всі 4 workflow + `--namespace ga.<name>`); якщо `conftest` відсутній у PATH — soft-skip з `ℹ` і exit 0.
- `check-ga.mjs` скорочено з 1074 → 570 рядків (47%) — видалено всі `validate*`-функції та `checkCanonicalWorkflowsMatchRule`, `checkLintGaWorkflow`, `checkGitAiWorkflow`.
- Додано `.regal/config.yaml` з вимкненим `idiomatic.no-defined-entrypoint` (для conftest-полісі `deny` де-факто entrypoint).
- `npm/package.json` `files` доповнено `"policy"` для включення `.rego` у npm-tarball.
- Версія: `1.8.201 → … → 1.8.205`.

**Обґрунтування:** Декларативні Rego-правила коротші, читабельніші й менш схильні до регресій за сигнатурою порівняно з imperative JS. Conftest — нативний інструмент для структурної валідації YAML, тому `deny`-правила виражають `ga.mdc`-обмеження без helper-ів `getObjKey`/`isExactString`. Fallback гарантує, що відсутність `conftest` не ламає CI без попередньої підготовки середовища.

**Розглянуті альтернативи:**

- Залишити все в JS — відхилено: ~50% файлу — шаблонний boilerplate без декларативної виразності.
- Мігрувати тільки один workflow як PoC — використано як перший крок, але потім повністю мігровано.
- Вбудувати `conftest`-виклик у `check-ga.mjs` замість `lint-ga.mjs` — відхилено: `lint-ga` — правильне місце для lint-pipeline, `check-ga` — окремий structural check.

**Технічна знахідка (Knowledge):** Conftest (OPA 1.15.2) парсить GHA YAML 1.1 boolean ключ `on:` як рядок `"true"`, а не як `"on"` або Rego boolean `true`. Єдиний робочий доступ — `input["true"]`. Рекомендовано зробити alias `gha_on := input["true"]` на початку кожного `.rego` файлу для читабельності.

**Технічна знахідка (Knowledge):** OPA 1.x вимагає Rego v1 синтаксис: `deny contains msg if { … }` замість `deny[msg] { … }`, `allow if { … }` замість `allow { … }`. Додаток `import rego.v1` забезпечує портованість і на OPA 0.x.
