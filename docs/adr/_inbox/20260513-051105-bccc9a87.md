## Knowledge `bun test` не запускає перевірки Rego

**Контекст:** Виникло питання, чи виконує `bun test` також перевірки Rego-правил у репозиторії `n-cursor`.

**Рішення/Процедура/Факт:** `bun test` запускає лише `.test.mjs`-файли у `npm/tests/` — жодного Rego-раннера серед них немає. Rego-тести (`_test.rego`-фікстури) виконуються окремо командою `bun run lint-rego`, яка викликає `conftest verify`. Сценарій `lint-rego.mjs` знаходиться в `npm/scripts/`, а не в `npm/tests/`.

**Обґрунтування:** Коментар у `npm/tests/check-abie.test.mjs` явно фіксує цю межу: «Покриття цих правил тепер забезпечують `_test.rego` фікстури, що виконуються через `bun run lint-rego` (`conftest verify`)». Таким чином, Rego-перевірки свідомо винесені за межі `bun test`.

**Розглянуті альтернативи:** не обговорювалися.

**Зачіпає:** `package.json` (script `test`), `npm/tests/` (Jest/Bun-тести), `npm/scripts/lint-rego.mjs`, `conftest verify`.
