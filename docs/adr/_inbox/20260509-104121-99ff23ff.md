## ADR js-run: mssql- як окремий префікс файлів-підключень у src/conn/

**Контекст:** Правило `js-run` у `@nitra/cursor` об'єднувало MySQL і MSSQL під спільним префіксом `mysql-`, хоча це дві різні СУБД з різними npm-драйверами (`mysql2` vs `mssql`) і різними SQL-діалектами. У реальних монорепо з'являвся файл `mysql-write.js`, що всередині містив `import sql from 'mssql'` — назва файла прямо суперечила його суті, порушуючи головний принцип канону («filename одразу повідомляє, до чого підключаємось»).

**Рішення/Процедура/Факт:**
- `scripts/utils/conn-file-rules.mjs` — оновлено JSDoc та регекс `CONN_FILENAME_RE`: `(?:pg|mysql)` → `(?:pg|mysql|mssql)`.
- `scripts/check-js-run.mjs` — оновлено два JSDoc і два рядки повідомлень: текст помилки тепер перераховує чотири альтернативи (`ql-`, `pg-`, `mysql-`, `mssql-`), pass-message — `ql-/pg-/mysql-/mssql-…`.
- `mdc/js-run.mdc` — розбито пункт «MySQL / MSSQL» на два окремі: `mysql-` та `mssql-`; рядок опису перевірки оновлено до `(pg|mysql|mssql)`.
- Нові unit-тести у `tests/conn-file-rules.test.mjs` (11 тестів) перевіряють прийнятність `mssql-read`, `mssql-write`, `mssql-write-b2b.mts`, відхилення `msql-`, `ms-sql-`, `sqlserver-`, а також коректність `kebabToCamel` для `mssql`-префіксів.
- Додано інтеграційний кейс у `tests/check-js-run-fixture.test.mjs`.
- Версія пакета підвищена `1.8.216` → `1.8.217` (minor-сумісна зміна); запис додано до `CHANGELOG.md`.

**Обґрунтування:** Основний принцип канону — «ім'я файла одразу повідомляє, до якої СУБД підключення». Це порушувалось, коли `mysql-write.js` фактично підключав MS SQL Server. Зміна backward-сумісна: проєкти, що вже використовують `mysql-…` для MSSQL-з'єднань, продовжують проходити валідатор без жодних правок; `mssql-` — рекомендований, але не обов'язковий варіант для нового коду. `kebabToCamel` не потребував змін — він нечутливий до конкретного префіксу і вже коректно дає `mssqlWriteB2b`.

**Розглянуті альтернативи:**
- Компактний варіант `m(y|s)?sql` — відхилено: зловить неіснуючий `msql-` і нечитабельний.
- Alias `mssql-` → `mysql-` (перетворення на рівні валідатора) — відхилено: маскує проблему замість вирішення; назва файла залишалась би `mysql-`.

**Зачіпає:** `npm/scripts/utils/conn-file-rules.mjs` (регекс `CONN_FILENAME_RE`), `npm/scripts/check-js-run.mjs` (JSDoc + повідомлення про помилку + pass-message), `npm/mdc/js-run.mdc` (рядки 137 та 235), `npm/tests/conn-file-rules.test.mjs` (новий файл), `npm/tests/check-js-run-fixture.test.mjs` (новий кейс), `npm/package.json` (version), `npm/CHANGELOG.md`.
