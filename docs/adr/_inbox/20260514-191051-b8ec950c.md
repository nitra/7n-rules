## ADR Замінити ru-переклади на англійські у скілі `abie-clean`

**Контекст:** Скіл `n-abie-clean` описує очистку проєкту від ru-середовища. Секція 5 «Переклади» містила приклади для `@nitra/tf` (flat-формат), але не охоплювала `@nitra/tfm` (multi-locale об'єкт `{ ru: '...' }`). Користувач доповнив скіл прикладом для `@nitra/tfm`, щоб інструкція була повною для обох форматів.

**Рішення/Процедура/Факт:** До `SKILL.md` (`npm/skills/abie-clean/SKILL.md`, рядки 135–190) додано другий приклад трансформації: у `@nitra/tfm` значення ключа `ru` замінюються з кирилиці (росіянізмів) на англійський відповідник; якщо поле `ru` є єдиним перекладом — воно видаляється або перетворюється на `en`. Наявний приклад для `@nitra/tf` (flat `tr`-об'єкт) залишено без змін.

**Обґрунтування:** Проєкт прибирає підтримку ru-локалі повністю. Щоб агент, який виконує `/n-abie-clean`, правильно обробляв обидва API (`tf.bind({ tr })` та `import { tf } from '@nitra/tfm'`), скіл має містити явні «БУЛО → СТАЛО» зразки для кожного формату.

**Розглянуті альтернативи:** не обговорювалися.

**Зачіпає:** `npm/skills/abie-clean/SKILL.md` — секція 5; пакети `@nitra/tfm`, `@nitra/tf`, `@nitra/tfm-node`.
