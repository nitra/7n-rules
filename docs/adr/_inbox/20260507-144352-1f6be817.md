## ADR Розщеплення правила `image` на `image-compress` і `image-avif`

**Контекст:** У монорепо з кількома Vue-проєктами (наприклад, адмінка + публічний сайт) потрібен різний рівень оптимізації зображень: стиснення (compress) актуальне для всіх пакетів, а AVIF-конвертація — лише для адмінки, оскільки AVIF ще не підтримується усіма браузерами і не підходить як основний формат для публічного сайту.

**Рішення/Процедура/Факт:** `check-image.mjs`, `npm/mdc/image.mdc` і `.cursor/rules/n-image.mdc` видалені; замість них створено два самостійні правила — **`image-compress`** і **`image-avif`** — кожне зі своїм `check-*.mjs`, `mdc`-файлом і тестовим файлом. Нова схема залежностей в `auto-rules.mjs`: `image-compress` залежить від `bun`, `image-avif` залежить від `vue` + `image-compress`. `.n-cursor.json`, `CLAUDE.md`, `AGENTS.md` і тести `auto-rules.test.mjs` оновлені; версія пакета піднята до `1.8.197`.

**Обґрунтування:** Монолітне правило `image` не дозволяло вмикати лише компресію без AVIF. Варіант з `package.json`-опцією `disable-avif` вже існував, але вимагав ручного додавання в кожен пакет і не підтримувався schema. Розщеплення на окремі правила дає чистий механізм opt-in/opt-out на рівні `.n-cursor.json`: сайт додає лише `image-compress`, адмінка — обидва.

**Розглянуті альтернативи:**
- Розширення поля `ignore` у `.n-cursor.json` об'єктним записом `{ "path": "apps/site", "rules": ["image"] }` — відкинуто через семантичне перевантаження поля.
- Окреме поле `disable-rules-paths` у `.n-cursor.json` — відкинуто на користь роздроблення правил, яке вирішує проблему чистіше і не потребує нового формату конфігу.
- Наявна package.json-опція `"@nitra/minify-image": { "disable-avif": true }` — існувала, але вимагала ручного налаштування у кожному пакеті без підтримки schema.
