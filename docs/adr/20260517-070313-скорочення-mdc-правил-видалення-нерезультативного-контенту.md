## ADR Скорочення MDC-правил: видалення нерезультативного контенту

**Контекст:** `.mdc` файли у `npm/rules/*/` накопичили значний обсяг пояснювального тексту (описи зовнішніх інструментів, `brew install`, VS Code блоки, `## Перевірка` хвости, дублі, міграційні примітки), який не впливає на AI-директиви і надмірно витрачає контекстне вікно.

**Рішення/Процедура/Факт:** Видалено ~314 рядків з 23 файлів у `npm/rules/*/`. Прибрано 6 стійких патернів: `## Перевірка` секції з командою `npx @nitra/cursor check <id>`, блоки `.vscode/extensions.json` / `.vscode/settings.json`, CLI install-інструкції (`brew`, `apt-get`), описи що робить зовнішній інструмент (hadolint, gitleaks, kubeconform тощо), активаційна мета про `disable-rules`/`auto.md`, дублікати одного й того ж правила. `vue.mdc`, `php.mdc`, `text.mdc` пропущені — потребують окремої перевірки на конфлікти з check-скриптами.

**Обґрунтування:** `.mdc` — виключно AI-контекст: підвантажується у промпт, коли AI редагує файл під `globs:`. Логіка запуску перевірок живе у `rules/<id>/fix/<concern>/check*.mjs`, `rules/<id>/policy/`, `rules/<id>/lint/lint.mjs` — CLI `n-cursor` сканує ці каталоги незалежно від вмісту `.mdc`. `## Перевірка` і VS Code-підказки корисні лише людині, яка читає файл очима — не AI, який його застосовує.

**Розглянуті альтернативи:** Залишити файли без змін; зберегти VS Code блоки як частину канону (відхилено — check-скрипти перевіряють `.vscode/` через окремий концерн); прибрати також borderline-блоки в `adr.mdc` (ENV-таблиця, LLM fallback) — відхилено як ризиковано.

**Зачіпає:** `npm/rules/*/: abie, adr, bun, capacitor, changelog, docker, ga, graphql, hasura, image-avif, image-compress, js-bun-db, js-bun-redis, js-lint, js-mssql, js-run, k8s, nginx-default-tpl, npm-module, ci4/ci4.mdc, rego, style-lint, tauri` — всього 23 `.mdc` файли.
