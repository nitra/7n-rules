---
type: ADR
title: doc-files як правило з тонким скілом
description: Детермінована логіка doc-files переноситься з skill у `npm/rules/doc-files`, а skill лишається тонким агентським workflow поверх правилових команд.
---

**Status:** Accepted
**Date:** 2026-06-14

## Context and Problem Statement

Функціональність doc-files жила в `npm/skills/doc-files/`: агентський workflow і JS-логіка детекції/генерації були в одному місці. Для CI, policy-каналу, lint-агрегатора і стандартних команд потрібне правило `npm/rules/<id>/`, тоді як skill має лишатися інструкцією для агента.

Також після вибору id `doc-files` потрібно було визначити CLI-імена для детектора і генератора.

## Considered Options

* Залишити `doc-files` виключно skill без правила.
* Створити правило `npm/rules/doc-files/` і лишити тонкий skill поверх нього.
* Використати короткі команди `lint-doc` / `fix-doc` замість імен від id правила.
* Використати `lint-doc-files` / `fix-doc-files` як похідні від id правила `doc-files`.

## Decision Outcome

Chosen option: "Правило npm/rules/doc-files/ + тонкий skill і команди lint-doc-files/fix-doc-files", because transcript фіксує явне рішення перейменувати правило на `doc-files` і лишити тонкий skill; імена команд мають виводитися з id правила за конвенцією `scripts.mdc`.

Реалізаційні факти з transcript:

* `git mv npm/skills/doc-files/js → npm/rules/doc-files/js`;
* нові точки входу: `lint-doc-files` і `fix-doc-files`;
* `doc-files <sub>` лишається deprecated alias із warning;
* hook-маркер у `sync-claude-config.mjs` оновлюється на `@nitra/cursor lint-doc-files`;
* skill `npm/skills/doc-files/SKILL.md` делегує на нові команди.

### Consequences

* Good, because детермінований detector отримує policy-канал, `meta.json:lint`, `fix.mjs`, lint-агрегатор і CI-інтеграцію.
* Good, because тонкий skill лишається агентським workflow без дублювання JS-логіки.
* Bad, because старі команди й посилання треба оновити; `doc-files <sub>` може існувати лише як deprecated alias.

## More Information

Ключові файли: `npm/rules/doc-files/meta.json`, `npm/rules/doc-files/fix.mjs`, `npm/rules/doc-files/js/lint.mjs`, `npm/rules/doc-files/lint/lint.mjs`, `npm/skills/doc-files/SKILL.md`, `npm/scripts/sync-claude-config.mjs`, `npm/.claude-template/settings.template.json`. Changeset у transcript: `npm/.changes/260614-0724.md`.

## Update 2026-06-14

Для `lint-doc-files` зафіксовано CI strictness: stale дорівнює `missing ∪ crc-mismatch`, тому CI падає і на відсутню, і на застарілу за CRC документацію. Дефолтний scope команди — changed-vs-origin через `resolveChangedBase()` / `collectChangedFilesSince()`, а повний скан доступний явно через `--full`. Перед увімкненням CI потрібен початковий прогін `fix-doc-files` до зеленого стану.

## Update 2026-06-14

Драфт уточнює, що `doc-files` використовує команди `lint-doc-files` і `fix-doc-files`, а старий `doc-files <sub>` лишається deprecated alias із warning. Для `security` у загальній lint-класифікації використовується `{scope:"per-file", ci:"full"}`: локально можливий per-file режим, але в CI trufflehog запускається повним сканом як defense-in-depth.
