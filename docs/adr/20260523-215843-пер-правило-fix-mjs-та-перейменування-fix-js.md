## ADR пер-правило `fix.mjs` та перейменування `fix/` → `js/`

## Context and Problem Statement
Сесія реалізовувала нову конвенцію entry-point для кожного правила `npm/rules/<id>/`: замість прямого виклику через `discover → runRule` CLI тепер делегує через `rules/<id>/fix.mjs`. Паралельно перейменовано підкаталог `rules/<id>/fix/<concern>/` → `rules/<id>/js/<concern>/`, щоб naming відображав технологію (JS), а не функцію.

## Considered Options
* Per-rule `fix.mjs` entry-point + rename `fix/` → `js/`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Per-rule `fix.mjs` entry-point + rename `fix/` → `js/`", because:
1. Пряма спрощена точка входу `bun rules/<id>/fix.mjs` — без необхідності знати CLI або `discover-checkable-rules`;
2. `js/` conveying "технологія", а `policy/` — "rego-технологія" — узгоджена convention за типом, а не функцією;
3. CLI `check`-команда спрощується: `listRuleIds + dynamic import + mod.run({walkCache})` замість `discoverCheckableRules + runRule`.

### Consequences
* Good, because transcript фіксує очікувану користь: `bun rules/abie/fix.mjs` + `npx @nitra/cursor check abie` — обидва запускаються без змін у CLI.
* Good, because transcript фіксує: 104 нових тестових кейси, 0 регресій.
* Bad, because transcript не містить підтверджених негативних наслідків.
