# Правило efes з автодетектом за полем repository у package.json

**Status:** Accepted
**Date:** 2026-05-18

## Context and Problem Statement

До системи правил `@nitra/cursor` потрібно додати правило `efes` для проєктів Efes Cloud. Аналогічний механізм автодетекту вже існує для `abie` — за маркером у полі `repository` у кореневому `package.json`.

## Considered Options

* Автодетект за полем `repository` у кореневому `package.json` (за аналогією з `abie` / `ABIE_REPOSITORY_URL_MARKER`)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Автодетект за полем `repository`", because у transcript явно задана умова «якщо в `package.json` в `"repository"` починається з `https://github.com/efes-cloud`»; реалізація наслідує наявний патерн `ABIE_REPOSITORY_URL_MARKER`.

### Consequences

* Good, because правило вмикається автоматично без ручної конфігурації для всіх репозиторіїв під `github.com/efes-cloud`, аналогічно до `abie`.
* Good, because transcript фіксує: 27 pass, 0 fail (`bun test scripts/auto-rules.test.mjs`) після додавання тест-кейсу детекту.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

Нові файли: `npm/rules/efes/efes.mdc`, `npm/rules/efes/auto.md`.

Змінені файли: `npm/scripts/auto-rules.mjs` — додано `EFES_REPOSITORY_URL_MARKER = 'https://github.com/efes-cloud/'`, змінну `isEfes`, `'efes'` у масиві правил і в масиві `enabled`-записів; `npm/scripts/auto-rules.test.mjs` — додано `'efes'` до `ALL_RULES` і тест-кейс детекту.

Умова активації (з `auto.md`): поле `repository` у кореневому `package.json` містить рядок `https://github.com/efes-cloud/`.

Версія пакету: `1.13.41` → `1.13.42`.
