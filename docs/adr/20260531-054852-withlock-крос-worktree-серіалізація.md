## ADR withLock крос-worktree серіалізація

## Context and Problem Statement
`withLock` зберігав стан локу (lock-директорія + dedup-кеш) у `node_modules/.cache/n-cursor/<key>/` поточного CWD. Оскільки `node_modules` у кожного git-worktree свій, лок між різними linked-worktree не серіалізувався, що дозволяло N паралельних прогонів eslint/Stryker — порушення правила «один важкий прогін за раз».

## Considered Options
* Перенести дефолтний `cacheDir` під `git-common-dir` (один каталог на всі worktree)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Перенести `cacheDir` під `git-common-dir`", because єдиний `<git-common-dir>/n-cursor/<key>/` однаковий для головного checkout і всіх linked-worktree, тому `mkdirSync`-lock стає крос-worktree мʼютексом без зміни семантики.

### Consequences
* Good, because transcript фіксує очікувану користь: один `mkdirSync`-атомарний lock серіалізує важкі CLI-команди між усіма worktree машини.
* Bad, because transcript не містить підтверджених негативних наслідків.

## ADR `meta.json` замість `auto.md` + поле `worktree` для скілів

## Context and Problem Statement
Кожен скіл у `npm/skills/<id>/` мав `auto.md` — однорядковий файл із умовою автоактивації. Формат не дозволяв додати структуровані поля (зокрема `worktree`-прапорець), а агент, виконуючи скіл, не отримував інформації про необхідність worktree — її не було де зберігати.

## Considered Options
* `meta.json` — JSON-файл з полями `auto` і `worktree`
* `meta.yaml` / `skill.yaml` — YAML-формат
* Markdown-файл з YAML frontmatter (`meta.md`)

## Decision Outcome
Chosen option: "`meta.json`", because JSON-схема в `npm/schemas/` вже практикується в репо; `auto-skills.mjs` є суто програмним парсером; JSON валідується без додаткових залежностей.

### Consequences
* Good, because transcript фіксує очікувану користь: структурований формат дозволяє декларувати `worktree: boolean` поруч із `auto`-умовою; JSON-схема з'явиться в `npm/schemas/skill-meta.json`.
* Bad, because transcript не містить підтверджених негативних наслідків.
