## Про #6 — lint pre-existing vs introduced

### Суть проблеми

Зараз `flow verify` запускає lint на **всьому файлі**, хоча ти змінив лише кілька рядків. Якщо у файлі вже є старі порушення — гейт падає, навіть якщо ти їх **не вносив**.

### Конкретні приклади з цієї сесії

**Задача #7 (review.mjs).** Я додав 3 рядки промпта — і `flow verify` впав на `review.test.mjs` з цими знахідками:
```
review.test.mjs:48:33: error eslint(require-await): Async function has no `await`
review.test.mjs:51:33: error eslint(require-await): Async function has no `await`
review.test.mjs:13:20: error eslint(no-empty-function): Unexpected empty function `noop`
```
Ці порушення були **до мого коміту** — але гейт не знав цього. Довелося виправляти чужий lint-борг перед тим, як можна було продовжити.

### Як виглядало б з #6

**Зараз:**
```
❌ gate: lint
review.test.mjs:13: error no-empty-function  ← pre-existing, не ти вніс
review.test.mjs:48: error require-await      ← pre-existing
```

**Варіант A (label-only):**
```
⚠️ gate: lint (1 introduced, 2 pre-existing)
review.test.mjs:13: warning [pre-existing] no-empty-function
review.test.mjs:48: warning [pre-existing] require-await
review.test.mjs:57: error   [introduced]  no-unused-vars  ← твій рядок
❌ блокує (будь-яке знайдено)
```

**Варіант B (label + relax):**
```
⚠️  gate: lint
review.test.mjs:13: info [pre-existing] no-empty-function   ← не блокує
review.test.mjs:48: info [pre-existing] require-await       ← не блокує
review.test.mjs:57: error [introduced]  no-unused-vars      ← блокує
```

### Технічна реалізація

Щоб відрізнити introduced від pre-existing, треба:
1. `oxlint --format json` (замість `stdio: 'inherit'` — це зміна `lint.mjs:28`)
2. Розпарсити `git diff HEAD` → рядки, які ти додав
3. finding.line ∈ доданих рядків → `introduced`; інакше → `pre-existing`

### Головне питання

**B усуває реальне тертя** (ти торкнувся файлу → чужий борг тебе блокує). A — просто кращий вивід, але не вирішує блокера.

Тобі як зручніше: **A** (наочніше, але все ще блокує), **B** (не блокує на чужому), чи щось між ними (наприклад, B але зберігати `--strict` для файлів, де сам створив рядок)?
