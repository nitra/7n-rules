## Context and Problem Statement
`n-cursor lint-ci` існував як чистий аліас для `runLint({ full: true, readOnly: true })` — жоден реальний workflow, root `package.json` або скіл його не викликав (лише doc-рядки, `CHANGELOG.md` і рядок `default`-помилки). Поверхня CLI утримувала зайву команду без окремого callera.

## Considered Options

## Decision Outcome

## ADR Розподіл ролей doc-files: детект у `lint`, генерація окремо у `fix-doc-files`

## Context and Problem Statement
Команди `lint-doc-files`, `fix-doc-files` і deprecated-аліас `doc-files <sub>` утворювали три точки входу для документації файлів. Виникло питання: чи є дублювання між lint-кроком doc-files і `fix-doc-files`, і чи не стоїть об'єднати їх в один флаг.

## Considered Options
* Зробити `lint-doc-files` / lint-крок аліасом-надмножиною `fix-doc-files` — єдина команда для виявлення й генерації.
* Додати `--fix` до `n-cursor lint`, щоб lint-крок doc-files генерував доки, коли `!readOnly`.
* Залишити чіткий поділ: `lint` (і `lint-doc-files`) = детект-only/exit 1; `fix-doc-files` = генерація через локальну LLM.
* Зробити lint-крок таким, що генерує, якщо `!readOnly` і модель доступна.

## Decision Outcome
Chosen option: "Залишити чіткий поділ: lint-крок detect-only, `fix-doc-files` — генерація", because lint-крок `rules/doc-files/js/lint.mjs` не мутує файлів і не потребує моделі (детермінований/дешевий); генерація (`runDocFilesGenCli`) вимагає `isModelAvailable()` — її провал у CI або без omlx дав би непередбачувану exit-семантику. Рядки `reportStale` делегують генерацію: `→ перегенеруй: npx @nitra/cursor fix-doc-files`.

### Consequences
* Good, because transcript фіксує очікувану користь: lint-цикл залишається детермінованим і CI-безпечним; генерація залишається opt-in без сюрпризів для середовищ без локальної LLM.
* Bad, because асиметрія контракту: doc-files `lint()` ігнорує третій аргумент `{ readOnly }`, хоча оркестратор `orchestrate.mjs:74` передає `{ readOnly }` усім правилам; питання про `lint --fix` як проміжний варіант залишено відкритим наприкінці сесії.

## More Information
`npm/rules/doc-files/js/lint.mjs`: `export function lint(files, cwd)` — без третього аргументу, лише `reportStale()`. Оркестратор: `npm/rules/lint/js/orchestrate.mjs:74` — `mod.lint(changed, cwd, { readOnly })`. `runDocFilesGenCli`: `npm/rules/doc-files/js/docgen-files-batch.mjs:29` — `await isModelAvailable()` як hard guard. Deprecated `doc-files <sub>` підтверджено мертвим (0 живих callerів); видалення заплановано як наступний крок (breaking → major).
