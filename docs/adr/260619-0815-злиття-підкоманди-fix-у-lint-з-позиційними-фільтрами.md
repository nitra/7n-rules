## ADR Злиття підкоманди `fix` у `lint` з позиційними фільтрами

## Context and Problem Statement
CLI `@nitra/cursor` мав окрему підкоманду `fix <rule>`, яка запускала виправлення конформності для конкретного правила. Наявність двох окремих команд (`lint` і `fix`) з перекриваючою функціональністю ускладнювала API bin-точки входу.

## Considered Options
* Зберегти окремі підкоманди `fix` і `lint`
* Прибрати `fix` і поглинути її функціональність у `lint` через позиційні аргументи-фільтри

## Decision Outcome
Chosen option: "Прибрати `fix` і поглинути її функціональність у `lint` через позиційні аргументи-фільтри", because коментар у `npm/bin/n-cursor.js:1540` явно фіксує: «Позиційні (не-флаг) аргументи — фільтр правил конформності (напр. `lint changelog`): прогнати лише конформність цих правил, без лінтер-скану (мапить колишній `fix <rule>`)».

### Consequences
* Good, because transcript фіксує очікувану користь: єдина підкоманда `lint` покриває всі сценарії — delta-скан, full-скан, CI-режим (`--read-only`), і точковий fix конкретного правила через позиційний аргумент.
* Bad, because колишні виклики `fix <rule>` потребують міграції на `lint <rule>`; transcript не містить підтверджених негативних наслідків окрім зламу зворотної сумісності.
