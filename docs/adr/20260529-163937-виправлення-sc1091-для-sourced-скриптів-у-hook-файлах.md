## ADR Виправлення SC1091 для sourced-скриптів у hook-файлах

## Context and Problem Statement
`shellcheck` завершувався з помилкою (exit 1) на SC1091 при перевірці `.claude/hooks/capture-decisions.sh` та `.claude/hooks/normalize-decisions.sh` (і їх копій у `npm/.claude-template/hooks/`). Директива `# shellcheck source=` вказувала на шлях `npm/.claude-template/hooks/lib/tooling-only.sh` — абсолютний відносно кореня репо — але без прапорця `-x` `shellcheck` не міг відстежити `source` і видавав SC1091.

## Considered Options
* Змінити `# shellcheck source=` на відносний шлях `lib/tooling-only.sh` і додати `disable=SC1091` в той самий рядок директиви
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Відносний `shellcheck source` + `disable=SC1091`", because тест у `/tmp` підтвердив: форма `# shellcheck source=lib/tooling-only.sh disable=SC1091` (без прапорця `-x`) дає `exit: 0`, і саме такий відносний шлях вже використовувала тестова фікстура `sync-claude-config.test.mjs`.

### Consequences
* Good, because `shellcheck` тепер завершується з `exit: 0` для всіх чотирьох файлів без додаткових прапорців CLI.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- `.claude/hooks/capture-decisions.sh` (рядок 39)
- `.claude/hooks/normalize-decisions.sh` (рядок 44)
- `npm/.claude-template/hooks/capture-decisions.sh` (рядок 39)
