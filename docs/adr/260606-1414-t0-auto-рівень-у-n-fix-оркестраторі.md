## ADR T0-auto рівень у n-fix оркестраторі

## Context and Problem Statement

## ADR gemma3:4b: вимога --no-tools для pi CLI

## Context and Problem Statement
`pi` CLI завжди надсилає tools API формат навіть у режимі `--mode text`. Модель `gemma3:4b` (Ollama) відхиляє такі запити з помилкою `400 registry.ollama.ai/library/gemma3:4b does not support tools`, що унеможливлює її використання як tool-free воркера без додаткового прапора.

## Considered Options
* Флаг `--no-tools` у виклику `pi` для `gemma3:4b`
* Використовувати `gemma4:4b` (приймає tools API формат, ігнорує у `--mode text`)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Флаг `--no-tools` із умовою `model === 'gemma3:4b'`", because `gemma3:4b` менша (3.3 GB проти 5.3 GB) і підходить для M2 8GB; `--no-tools` вимикає надсилання tools API, що підтверджено прямим тестом (`{"ok":true}` за кілька секунд).

### Consequences
* Good, because transcript фіксує очікувану користь: `gemma3:4b --no-tools` повернув коректний результат у прямому тесті з 3 entries.
* Bad, because warm inference навіть із `--no-tools` займає 78–90s для реальних промптів із 9 entries; модель галюцинує точні рядки → практично непридатна для config-JSON задач.

## More Information
- `benchmarks/tool-free/run.mjs`: `MODEL_NO_TOOLS = MODEL === 'gemma3:4b'`; виклик: `pi -p "..." --model gemma3:4b --no-session --no-tools --mode text`
- Cold-start після eviction `gemma4:4b`: ~230s (3:50 total у прямому тесті)
- Prewarm timeout у harness: `180_000ms`
