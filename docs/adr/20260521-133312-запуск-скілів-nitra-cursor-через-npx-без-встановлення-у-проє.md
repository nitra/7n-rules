## ADR Запуск скілів `@nitra/cursor` через `npx` без встановлення у проєкт

## Context and Problem Statement
Скіли пакета `@nitra/cursor` (`npm/skills/*/SKILL.md`) раніше були доступні лише через slash-команди всередині Cursor IDE після синхронізації правил. Потрібен спосіб запускати ті самі скіли з будь-якого репозиторію без `devDependencies` і без синку — виключно через `npx`.

## Considered Options
* Окремий пакет `@nitra/skills` з власним репозиторієм і публікацією
* Додати CLI скілів безпосередньо до існуючого пакета `@nitra/cursor`

## Decision Outcome
Chosen option: "Додати CLI скілів до `@nitra/cursor`", because скіли вже зберігаються в `npm/skills/*/SKILL.md` того самого пакета; окремий пакет потребував би дублювання файлів і окремого release-cycle.

### Consequences
* Good, because `npx @nitra/cursor skill list` / `skill prompt` / `skill claude` доступні одразу після публікації `1.13.70` без нових залежностей.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Нові файли: `npm/scripts/skills-cli.mjs` (логіка list/prompt/claude/cursor), `npm/bin/n-skills.js` (окремий bin), `npm/scripts/skills-cli.test.mjs`.
Зміни: `npm/bin/n-cursor.js` — підкоманда `skill`; `npm/package.json` — `"n-skills": "./bin/n-skills.js"` у `bin`, версія `1.13.70`.

## ADR Скорочений синтаксис `skill <id>` без підкоманди `prompt`

## Context and Problem Statement
Після першої реалізації команда `npx @nitra/cursor skill prompt lint "task"` виглядала надмірно багатослівно для найчастішого сценарію — генерації промпту.

## Considered Options
* Підтримувати лише повний синтаксис `skill prompt <id> "task"`
* Додати скорочення `skill <id> "task"` як alias до `prompt`

## Decision Outcome
Chosen option: "Додати скорочення `skill <id> "task"`", because автор явно запросив коротший UX і асистент одразу реалізував fallback: якщо `command` не збігається з `list`/`prompt`/`claude`/`cursor`, воно трактується як `skillId`.

### Consequences
* Good, because transcript фіксує очікувану користь: `npx @nitra/cursor skill lint "task"` — найкоротший шлях до промпту.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Логіка fallback у `npm/scripts/skills-cli.mjs`: якщо `command` не є зарезервованим словом, він вважається `skillId`, `skillName` стає першим `taskPart`. Тест `'скорочення skill <id> без prompt'` доданий у `npm/scripts/skills-cli.test.mjs`.

## ADR Нормалізація префікса `n-` у назвах скілів

## Context and Problem Statement
Slash-команди Cursor мають формат `/n-lint`, `/n-fix` тощо, тоді як директорії скілів у пакеті називаються `lint`, `fix`. Потрібна однозначна відповідність між іменем з CLI і директорією на диску.

## Considered Options
* Вимагати вказувати точне ім'я директорії (`lint`, `fix`)
* Приймати обидва варіанти (`n-lint` → `lint` через strip префікса)

## Decision Outcome
Chosen option: "Приймати обидва варіанти з нормалізацією `n-` → ``", because асистент реалізував `normalizeSkillId` у `skills-cli.mjs`, і в документації явно зазначено: «Id скілу — ім'я каталогу або з префіксом `n-`».

### Consequences
* Good, because transcript фіксує очікувану користь: відповідність між `/n-lint` у Cursor і `npx … skill n-lint` у термінані.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Функція `normalizeSkillId` у `npm/scripts/skills-cli.mjs`. Список скілів команди `list` виводить канонічні імена без префікса.

## ADR Контекст CWD у промпті скілу

## Context and Problem Statement
Скіл запускається з довільного репозиторію через `npx`; без інформації про проєкт інструкція SKILL.md позбавлена контексту.

## Considered Options
* Передавати лише вміст `SKILL.md` і завдання
* Доповнювати промпт файлами `package.json`, `tsconfig.json`, `.n-cursor.json` з `process.cwd()`

## Decision Outcome
Chosen option: "Доповнювати промпт файлами з CWD", because асистент реалізував `buildSkillPrompt` з читанням цих трьох файлів і явно задокументував їх у відповіді: «контекст CWD: `package.json`, `tsconfig.json`, `.n-cursor.json` (якщо є)».

### Consequences
* Good, because transcript фіксує очікувану користь: агент отримує мінімально достатній контекст проєкту без ручного копіювання.
* Neutral, because transcript не містить підтвердження наслідку щодо розміру промпту при великих `package.json`.

## More Information
Функція `buildSkillPrompt` у `npm/scripts/skills-cli.mjs`; файли включаються лише якщо існують (`existsSync`).
