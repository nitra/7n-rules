## OKF frontmatter — зведена таблиця

| Поле | Статус | Тип | Опис |
|------|--------|-----|------|
| `type` | **REQUIRED** | string | Вид концепту (вільний рядок, напр. `ADR`, `Playbook`, `Runbook`) |
| `title` | Recommended | string | Human-readable назва |
| `description` | Recommended | string | Одне речення-резюме |
| `resource` | Recommended | URI | Посилання на зовнішній asset (для ADR — не потрібне) |
| `tags` | Recommended | list | Короткі теги для фільтрації |
| `timestamp` | Recommended | ISO 8601 | Дата останньої зміни |
| будь-які інші | Allowed | any | Продюсери можуть додавати довільні ключі |

## Conformance (§9)

Бандл відповідає OKF v0.1 якщо:
1. Кожен `.md` файл має **parseable YAML frontmatter**
2. Кожен frontmatter має **непорожнє поле `type`**
3. `index.md` і `log.md` відповідають своїм секціям

## Мінімальна зміна для наших ADR

Зараз після нормалізації — **нуль frontmatter**. Для OKF-сумісності потрібно лише:

```yaml
---
type: ADR
title: <з H1 heading>
session: 17ad250b-...     # наш custom field — дозволено
---
```

`description` — опціонально (можна брати першу фразу Context section).
`resource` — не потрібен (ADR — абстрактна ідея, не фізичний asset).
