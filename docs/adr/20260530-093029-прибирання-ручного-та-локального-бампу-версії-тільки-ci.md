## ADR Прибирання ручного та локального бампу версії — тільки CI

## Context and Problem Statement
У проекті паралельні PR конфліктують через те, що `version` у `package.json` може бумпитись як вручну розробниками, так і потенційно автоматично локально. Потрібно зробити CI єдиним місцем, де відбувається бамп версії, щоб уникнути merge-конфліктів.

## Considered Options
* Залишити legacy-шлях: ручний bump `version` + прямий запис у `CHANGELOG.md` приймається перевіркою `consistency.mjs` як альтернатива change-файлам.
* Зробити CI єдиним джерелом бампу: прибрати legacy-шлях з `consistency.mjs`, change-файли (`.changes/*.md`) стають єдиним workflow — `release.mjs` агрегує їх у бамп + CHANGELOG лише в CI.

## Decision Outcome
Chosen option: "Зробити CI єдиним джерелом бампу", because користувач явно зазначив, що бамп версії має відбуватися лише в CI, щоб уникнути конфліктів при паралельному мержі.

### Consequences
* Good, because transcript фіксує очікувану користь: усуваються merge-конфлікти по `package.json` при паралельних PR.
* Bad, because transcript не містить підтверджених негативних наслідків.
