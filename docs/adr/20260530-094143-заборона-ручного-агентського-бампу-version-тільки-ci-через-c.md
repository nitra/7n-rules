## ADR Заборона ручного/агентського бампу `version` — тільки CI через change-файли

## Context and Problem Statement

Субагенти (під час виконання `n-fix`, `n-coverage-fix` тощо), читаючи правило `n-changelog.mdc`, самостійно вирішували підняти `version` у `package.json`. Оскільки правило містило legacy-альтернативу («ручний bump приймається як альтернатива change-файлу»), перевірка `consistency.mjs` зеленила такий bump — паралельні гілки конфліктували на полі `version`. Версійний bump має виконуватися **тільки** в CI через `release.mjs`, що агрегує `.changes/*.md`.

## Considered Options

* **Підхід 1 — тільки документація:** прибрати legacy-рядок з `.mdc`, нічого не міняти у `consistency.mjs`. Агент, що все одно бампне, пройде перевірку непоміченим.
* **Підхід 2 — документація + активний fail перевірки:** прибрати legacy-рядок і змінити `consistency.mjs` так, щоб ручний bump давав fail із повідомленням «відкоти version і поклади change-файл».
* **Підхід 3 — окремий write-time guard:** додати окремий хук, що забороняє зміну `version` поза release-комітом. Визнаний надлишковим, бо pre-commit hook `npm-changelog` вже гонить `check changelog`.

## Decision Outcome

Chosen option: "Підхід 2 — документація + активний fail перевірки", because він усуває і спокусу (агент більше не читає «можна вручну»), і тихе проходження бампу: наявний pre-commit hook `npm-changelog` автоматично блокує коміт при червоній перевірці. Правило однорідне для **всіх** гілок — ручний bump заборонений і на `main` (уточнення користувача).

### Consequences

* Good, because transcript фіксує очікувану користь: конфлікти між паралельними гілками на полі `version` стають неможливими — єдиний артефакт у feature-гілці є `.changes/*.md`.
* Good, because існуючий pre-commit hook `npm-changelog` (вже виконує `check changelog`) автоматично блокує коміт без додаткового механізму.
* Bad, because потрібно оновити тести у `npm/rules/changelog/js/tests/consistency/tests/check.test.mjs` — кейси, що очікували pass на ручний bump, тепер мають очікувати fail.

## More Information

Змінювані артефакти:
- `npm/rules/changelog/js/consistency.mjs` — прибрати гілки `Vbase !== Vcurrent → pass`; додати fail-повідомлення з інструкцією `n-cursor change …`
- `npm/rules/changelog/changelog.mdc` — видалити legacy-рядок, STOP-блок зробити категоричним; bump version `'3.0' → '3.1'`
