## ADR Файлова структура вузлів задач за думка.MD

## Context and Problem Statement
Для системи `n-cursor graph` потрібен механізм зберігання вузлів задач (DAG-вузлів), щоб кожен вузол мав чітко визначений стан, бюджет і критерії завершення. У проєкті існував документ `docs/думка.MD` із описом рекурсивного складеного ОАГ, але жодна директорія `tasks/` ще не була ініціалізована.

## Considered Options
* Файлова структура `tasks/<node-name>/task.md` за специфікацією `docs/думка.MD`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Файлова структура `tasks/<node-name>/task.md` за специфікацією `docs/думка.MD`", because специфікація вже існувала в репо і задавала точний формат: YAML-frontmatter (`created_at`, `budget_sec`, опціонально `parent`, `deps`) + markdown-секції `## Task`, `## Done when`, `## Inputs`.

### Consequences
* Good, because стан вузла визначається виключно наявністю файлів: лише `task.md` → `waiting`; наявність `run_*.md` → `running`; наявність `outputs_*.md` → `resolved` — без окремої БД або реєстру.
* Bad, because transcript не містить підтверджених негативних наслідків.
