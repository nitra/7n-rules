# Виправлення блокувальних тестів як передумова запуску Stryker coverage

**Status:** Accepted
**Date:** 2026-05-29

## Context and Problem Statement

Під час виконання `/n-coverage-fix` команда `bun run coverage` падала на двох незалежних дефектах у тестовому шарі, що унеможливлювало запуск Stryker (помилка "There were failed tests in the initial test run"). Кожен дефект вимагав окремого точкового виправлення.

## Considered Options

- Виправити кожен блокувальний дефект безпосередньо в коді
- Запускати Stryker напряму (`npx @stryker-mutator/core run`) в обхід `bun run coverage`
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Виправити кожен блокувальний дефект", because обидва дефекти були реальними помилками в коді, а `bun run coverage` є канонічним entry-point згідно зі скілом `n-coverage-fix`.

### Consequences

- Good, because після виправлень `bun run coverage` пройшла інтеграційні перевірки і Stryker зміг запуститися з incremental report.
- Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

**Дефект 1 — ізоляція тесту `auto-rules.test.mjs`:**
Файл: `npm/scripts/tests/auto-rules.test.mjs`, рядок 282.
Виправлення: `root: process.cwd()` → `root: dir` (тимчасова директорія `withTmpDir` callback).
Reference-патерн: сусідній тест на L300 того ж файлу вже використовує `root: dir`.
Перевірка: `bunx vitest run scripts/tests/auto-rules.test.mjs -t "disable-rules vue"` → 1 passed.

**Дефект 2 — рівень заголовку версії у `npm/CHANGELOG.md`:**
Файл: `npm/CHANGELOG.md`, рядок 6.
Виправлення: `# [1.30.0]` → `## [1.30.0]` (Keep a Changelog вимагає H2 для версійних секцій; `#` залишається тільки для рядка `# Changelog`).
Перевірка: `npx @nitra/cursor check changelog` → `✅ npm/CHANGELOG.md: перша секція [1.30.0] збігається з npm/package.json`.

**Рішення 3 — тести для вцілілих мутантів `opts.fix` gate (L189, L211):**
Джерело мутантів: `npm/rules/test/coverage/coverage.mjs`, рядки L189 (`if (opts.fix)`) та L211 (`runCoverageSteps({ fix: false })`).
Рішення: додати окремі `describe`-блоки з прямими викликами `runCoverageSteps({fix:true})` та `runCoverageSteps({fix:false})` у `npm/rules/test/coverage/tests/coverage.test.mjs`.
Результат: suite зросла з 54 до 57 passed.
Перевірка: `cd npm && bunx vitest run rules/test/coverage/tests/coverage.test.mjs` → 57 passed.

## Update 2026-05-29

### Bump версії `npm/package.json` до `1.30.1` після зміни тестів

Після додавання тестів у `coverage.test.mjs` інтеграційний check вимагає узгодженості `package.json#version` і першої секції `CHANGELOG.md`.

Рішення: бампнути до `1.30.1` і додати секцію `## [1.30.1] - 2026-05-29` у `npm/CHANGELOG.md` перед `## [1.30.0]`.
Файли: `npm/package.json`, `npm/CHANGELOG.md`.
Перевірка: `bun run coverage` → `✅ npm/CHANGELOG.md: перша секція [1.30.1] збігається з npm/package.json`.

Додатково: Stryker sandbox при першому запуску не мав `.n-cursor.json`, що тимчасово заважало інкрементальному прогону; вирішилося повторним запуском після очищення `.tmp/`.

## Update 2026-05-29

**Ізоляція тесту auto-rules (root: dir):** `npm/scripts/tests/auto-rules.test.mjs:282` передавав `root: process.cwd()` замість `root: dir` у `detectAutoRules` — тест залежав від реального дерева cursor і блокував Stryker dry-run. Виправлення за аналогією з L300. Верифікація: `bunx vitest run scripts/tests/auto-rules.test.mjs -t "disable-rules vue"` → 1 passed.

**opts.fix gate (L189, L211):** 4 вцілілих мутанти (`if(true)`, `if(false)`, `{}`, `fix:true`) у `coverage.mjs` усунені тестами у `coverage.test.mjs`. Техніки: spy на `withLock.mock.calls` для L189; `.toString()` inspection для L211 (regex: `FIX_TOKEN_RE`, `FALSE_TOKEN_RE`, `FIX_TRUE_RE`). Bad: якщо мінімайзер видалить токен `fix`, `.toString()`-тест може хибно провалитись.

**CHANGELOG heading level:** `npm/CHANGELOG.md` рядок 6 містив `# [1.30.0]` замість `## [1.30.0]` — падіння integration-тесту в `tests/integration-repo-checks.test.mjs`. Виправлення відповідно до `n-changelog.mdc`.

## Update 2026-05-29

**Ручна перевірка мутантів при зависанні Stryker:** Stryker завис після 54+ хв (incremental cache недійсна: `0 of 25053 reused`; PID 90242, 91321 з Wed09AM — 3000+ хв CPU). Рішення: `kill 90242 91321`, вручну застосувати мутант → `bunx vitest run -t "<name>"` → відновити × 4. Усі 4 вбиті за ~10 хв. Bad: COVERAGE.md не оновився — 93.62% (132/141) до наступного `bun run coverage`. Commit: `d42384b`.
