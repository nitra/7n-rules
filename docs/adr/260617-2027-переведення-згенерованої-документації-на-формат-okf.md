## ADR Переведення згенерованої документації на формат OKF

## Context and Problem Statement
Проєкт використовує `/n-docgen` (skill `doc-files`) для генерації поведінкових markdown-файлів у `docs/`. Після ознайомлення зі специфікацією [Open Knowledge Format](https://cloud.google.com/blog/products/data-analytics/how-the-open-knowledge-format-can-improve-data-sharing) виникла потреба узгодити формат згенерованих файлів з OKF-стандартом (YAML frontmatter + standard markdown links + структурована директорія концептів).

## Considered Options
* Оновити шаблон генерації `doc-files` для OKF-сумісності (YAML frontmatter `type`, `title`, `tags`, стандартні посилання)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Оновити шаблон генерації `doc-files` для OKF-сумісності", because користувач явно заявив намір ("я хочу проапдейтити файли які генерує doc-files під сумісність з цим стандартом") після прочитання офіційного блог-посту Google Cloud про OKF.

### Consequences
* Good, because OKF використовує "just markdown / just files / just YAML" — формат залишається читабельним у будь-якому редакторі та git-репо без SDK.
* Bad, because користувач сам поставив питання про втрату сумісності з Marksman LSP; transcript не містить підтвердженої відповіді — сесія завершилася до вирішення цього питання.

## More Information
