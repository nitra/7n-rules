title: Міграція doc-files зі скіла у правило зі збереженням тонкого скіла
description: Логіку doc-files перенесено у правило, щоб отримати policy-канал і зберегти агентний workflow через тонкий скіл.
---

**Status:** Accepted

**Date:** 2026-06-14

## Context and Problem Statement

Логіка `doc-files` жила у `npm/skills/doc-files/js/`. Скіл не має слотів для policy-каналу, зокрема GA-workflow і `package.json`-скриптів через Rego, а ці механізми потрібні для CI-енфорсу застарілості документації. Підкоманда `lint-doc` лишалася окремою командою без повного rule-level policy enforcement.

## Considered Options

- Перейменувати правило на `doc-files` і лишити тонкий скіл.
- Зробити `doc-files` виключно скілом і прибрати правило `doc`.

## Decision Outcome

Chosen option: "Перейменувати правило на `doc-files` і лишити тонкий скіл", because лише правило має policy-канал для GA-workflow і `package.json`-скриптів через Rego, а тонкий скіл зберігає агентний workflow.

### Consequences

- Good, because правило отримує повний набір слотів: `meta.json`, `.mdc`-канон, `fix.mjs`, `lint/lint.mjs` і `js/lint.mjs`.
- Good, because тонкий скіл може делегувати у rule-level реалізацію без дублювання основної логіки.
- Bad, because `git mv npm/skills/doc-files/js → npm/rules/doc-files/js` вимагав оновлення re-export у `npm/skills/doc-aggregate/js/docgen-ignore.mjs`.
- Neutral, because rego-policy і GA-workflow для `doc-files` свідомо відкладені через незавершене рішення щодо layout у consolidation-спеці.

## More Information

Змінені файли з transcript: `npm/rules/doc-files/meta.json`, `npm/rules/doc-files/fix.mjs`, `npm/rules/doc-files/lint/lint.mjs`, `npm/rules/doc-files/js/lint.mjs`, `npm/rules/doc-files/doc-files.mdc`, `npm/skills/doc-aggregate/js/docgen-ignore.mjs`.

Додаткові зафіксовані факти:

- `npm/tests/fix-mjs-contract.test.mjs` оновив лічильник правил з 35 до 36.
- Для CLI прийнято top-level команди `lint-doc-files` і `fix-doc-files`, а `doc-files <sub>` лишено як deprecated-аліас.
- `settings.template.json` і `sync-claude-config.mjs` використовують hook-маркер `@nitra/cursor lint-doc-files`.
- Пер-файловий timing `fix-doc-files` інструментовано через wrapper навколо `callLlm` у `npm/rules/doc-files/js/docgen-gen.mjs`.
- `N_LOCAL_MIN_MODEL` прийнято як стандартний tier-env для локального docgen замість docgen-specific `N_CURSOR_DOCGEN_MODEL`.
