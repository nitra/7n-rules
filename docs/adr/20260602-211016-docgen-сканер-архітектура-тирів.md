# docgen: архітектура сканера та тришарова ієрархія документації

**Status:** Accepted
**Date:** 2026-06-02

## Context and Problem Statement

Скіл `n-docgen` потребував рішень щодо: обходу файлової системи без LLM/мережі; організації ієрархії doc-файлів; визначення меж модулів; порядку виконання тирів; уникнення рекурсивного сканування згенерованих `docs/`.

## Considered Options

- Синхронний `readdirSync` без LLM/network для сканера
- LLM або мережеві запити для обходу файлової системи
- Тришарова ієрархія: Tier 1 (file-level) → Tier 2 (module-summary) → Tier 3 (domain docs)
- Межа модуля = наявність `package.json`
- Суворо послідовні тири; батчі по 5 агентів паралельно в Tier 1/2; Tier 3 — один агент
- Інші варіанти для ієрархії, меж модулів, порядку тирів у transcript не обговорювалися.

## Decision Outcome

Chosen option: "Синхронний `readdirSync` + тришарова ієрархія + межа за `package.json` + послідовні тири з батчами по 5", because сканер повинен бути передбачуваним без зовнішніх сервісів; Tier 2 читає результати Tier 1, Tier 3 — результати Tier 2, тому міжтирна паралелізація дасть неповні вхідні дані.

Додаткові рішення: `docs/` в `IGNORED_DIRS` (уникнення рекурсії); сканер вказує `exists`, скіл вирішує skip/overwrite; `meta.json` без поля `auto` (opt-in, бо `"ніколи"` невалідне для `skill_meta.mjs`); суфікс `-domain` для Tier 3 slug при колізії; `Array#toSorted()` замість `.sort()` (відповідність `unicorn/no-array-sort`).

### Consequences

- Good, because сканер детермінований і відтворюваний; Tier 2/3 гарантовано агрегують повні вхідні дані.
- Bad, because суворий порядок Tier 1 → 2 → 3 унеможливлює повну паралелізацію.
- Bad, because файли навмисно розміщені в `docs/` не будуть задокументовані.
- Neutral, because transcript не містить підтвердження щодо нестандартних пакетів без `package.json`.

## More Information

Файл: `npm/skills/docgen/js/docgen-scan.mjs` (`findModuleRoots`, `IGNORED_DIRS` включає `docs`). Команди: `docgen scan`, `docgen modules` у `npm/bin/n-cursor.js`. Tier 1: `<dir>/docs/<stem>.md`; Tier 2: `<module_root>/docs/ARCHITECTURE.md`; Tier 3: `docs/<domain>.md` або `docs/<domain>-domain.md`. Три розташування скіла: `.cursor/skills/n-docgen/SKILL.md` (canonical), `npm/skills/docgen/SKILL.md` (published, byte-identical), `.pi/skills/n-docgen/SKILL.md` (delegation stub). `meta.json`: `{ "worktree": false }` без `auto`.
