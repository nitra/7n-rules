# Скіл abie-clean: очищення проєктів від ru-артефактів

**Status:** Accepted
**Date:** 2026-05-14

## Контекст

У проєктах `abie`-екосистеми накопичилися артефакти підтримки `ru`-середовища: директорії `ru/`, файли `*-ru` та `values-ru.*`, умовні гілки в GitHub Actions, вирази `runs-on` з `ya`-раннером, Docker-рядки з `BRANCH=ru` та nginx-конфіги з country-роутингом на Росію. Потрібен систематичний спосіб прибирати ці залишки у будь-якому дочірньому репозиторії.

## Рішення/Процедура/Факт

Створено скіл `npm/skills/abie-clean/SKILL.md` (version 1.1), який охоплює п'ять зон очищення:

1. **Директорії** — `find . -type d -name "ru" -exec rm -rf {} +`
2. **Файли з суфіксом** — `values-ru.*`, `*.env.prod-ru` та аналогічні.
3. **`.github/workflows`** — прибрати `ru` з `branches:` і `ignore_branches:`; спростити тернарні вирази `runs-on` та env-змінних (`NATS_URL`, `NATS_STREAM`, `REGISTRY`) до двогалузевого `dev || ua`.
4. **Dockerfile** — замінити умовний `RUN if [ "$BRANCH" = "ru" ]...` на безумовне `RUN bun install && bun vite build ...`.
5. **Nginx / інші конфіги** — видалити посилання на `ru`; фінальна перевірка через `git grep -rn 'ru'`.

Скіл автоматично підключається до `.n-cursor.json` за правилом `abie-clean - [abie]` у файлі `npm/bin/auto-skills.md`.

## Обґрунтування

Скіл механізує повторюване очищення будь-якого abie-репозиторію без ручного пошуку патернів. Охоплення всіх п'яти зон в одному скілі гарантує, що жоден артефакт `ru` не залишиться після запуску.

## Розглянуті альтернативи

Не розглядалися; перелік зон очищення визначено однозначно.

## Зачіпає

`npm/skills/abie-clean/SKILL.md`, `npm/bin/auto-skills.md` (правило авто-додавання `abie-clean - [abie]`); застосовується до будь-якого дочірнього репозиторію з правилом `abie`.

## Update 2026-05-14

Секцію 5 «Переклади» скілу `npm/skills/abie-clean/SKILL.md` доповнено прикладом для формату `@nitra/tfm` (multi-locale об'єкт `{ ru: '...' }`): значення ключа `ru` замінюються на англійський відповідник; якщо `ru` є єдиним перекладом — воно видаляється або перетворюється на `en`. Наявний приклад для `@nitra/tf` (flat `tr`-об'єкт) залишено без змін. Зачіпає: `npm/skills/abie-clean/SKILL.md` (секція 5), пакети `@nitra/tfm`, `@nitra/tf`, `@nitra/tfm-node`.

## Update 2026-05-14

### Переклади у `@nitra/tf` / `@nitra/tfm` / `@nitra/tfm-node`

Переклади є частиною ru-очистки і підпадають під скіл `abie-clean`. Два паттерни:

- **`@nitra/tf`** (webpack): замінювати значення у словнику перекладів — `tr: { 'Привіт': 'Привет' }` → `tr: { 'Привіт': 'Hello' }`.
- **`@nitra/tfm`** / **`@nitra/tfm-node`**: замінювати значення ключа `ru:` у мультимовному словнику — `{ Так: { ru: 'Да' } }` → `{ Так: { ru: 'Yes' } }`. Якщо поруч із `ru:` вже присутній англійський запис — видаляти `ru:`-запис повністю, не замінювати.
