## ADR concern-based JS-концерни + per-policy `target.json` у `npm/rules/`

**Контекст:** Після фаз 1-4 реорганізації пакету правила живуть у `npm/rules/<id>/`, але JS-логіка і Rego-полісі мали різні структури: `js/check.mjs` — монофайл, полісі — підкаталоги. CLI (`discoverCheckScripts`) не бачив правило без `js/check.mjs`, тому pure-Rego правила без JS-обгортки взагалі не запускались.

**Рішення/Процедура/Факт:**
- Нова файлова структура: `js/<name>/check.mjs` (або `check-<concern>.mjs`) дзеркалить `policy/<name>/`; один `<name>` = один «концерн»; `utils/` на рівні правила для спільного кешу.
- `policy/<name>/target.json` декларує таргет: `{ files: { single: "..." } | { walkGlob: "..." }, required?, missingMessage? }` — піддіректорія стає самодостатньою без центрального `TARGETS`.
- `js/<name>/applies/check.mjs` (опційно) експортує `applies()` — rule-level gate без окремого `rule.json`.
- CLI (`discover-checkable-rules.mjs`, `resolve-target-files.mjs`, `run-rule.mjs`) знаходить обидві форми (нову + legacy-fallback `js/check.mjs`), резолвить файли через `picomatch` із shared walk-кешем, запускає applies → JS-концерни → policy-концерни.
- Пілот: `rules/rego/` мігровано першим — `js/check.mjs` видалено, замість нього `js/applies/check.mjs` + три `target.json`; bump 1.10.0 → 1.11.0.
- `conftest.combine` вилучено зі схеми до першого реального юзкейса (YAGNI); `walkGlob.required` заборонено у JSON Schema.
- ADR `20260515-123025-concern-target-design.md` + spec `2026-05-15-npm-rules-concern-and-target-design.md` зафіксовано.

**Обґрунтування:** Legacy `lint-conftest.mjs:TARGETS` — єдиний глобальний реєстр таргетів — порушував cohesion: додаючи правило, треба було редагувати два файли. Декларативний `target.json` поряд із `.rego` робить каталог правила самодостатнім. Симетрія `js/<name>/` ↔ `policy/<name>/` дозволяє гібридні концерни (JS обчислює список файлів, Rego валідує вміст) без дублювання. `picomatch` обрано через нуль транзитивних залежностей і компіляцію glob у RegExp (швидкий matcher для батчів).

**Розглянуті альтернативи:** `rule.json` per-rule зі списком targets (відкинуто: locality гірша, ніж per-policy); name-convention без маніфесту (відкинуто: конфлікт `single` vs `walkGlob` для одного й того ж `<name>`); `conftest test .` feed-everything (відкинуто: не покриває канонічні безкіндові файли типу `.cspell.json`; O(rules × tree) обходів); глобальний кеш-singleton (відкинуто: ламає ізоляцію тестів; замінено на `Map` у scope `runChecks`).

**Зачіпає:** `npm/bin/n-cursor.js` (`discoverCheckScripts`, `runChecks`); нові утиліти `npm/scripts/utils/{discover-checkable-rules,resolve-target-files,run-rule}.mjs`; `npm/schemas/target.json`; `npm/rules/rego/js/` (видалено `check.mjs`, додано `applies/check.mjs`); `npm/rules/rego/policy/*/target.json`; `npm/package.json` (picomatch + версія 1.11.0); `npm/CHANGELOG.md`.
