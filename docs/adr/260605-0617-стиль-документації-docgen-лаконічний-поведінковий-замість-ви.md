## ADR Стиль документації docgen — лаконічний поведінковий замість вичерпного реалізаційного

## Context and Problem Statement
Скіл `docgen` генерував документацію з розділами `## Залежності` (деталі `node:fs`, `node:fs/promises`, `node:path`), `## Функції` з таблицями сигнатур і `Rebuild Test`-переліками. Такий формат описує реалізацію, а не призначення: перераховані внутрішні назви (`isAbieRuleEnabled`) і stdlib-виклики не несуть бізнес-цінності та старіють разом із кодом.

## Considered Options
* Вичерпний стиль: усі залежності, сигнатури, Rebuild Test, таблиці типів — «максимум деталей»
* Лаконічний поведінковий стиль: Огляд, Поведінка, Публічний API (лише за нетривіальної поверхні), Де використовується, Гарантії поведінки — без stdlib, без внутрішніх назв, без сигнатурних блоків

## Decision Outcome
Chosen option: "Лаконічний поведінковий стиль", because користувач схвалив запропонований зразок `enabled.md` — de-факто задокументував opt-in, покроковий алгоритм, fail-safe, read-only без жодного рядка stdlib. Ціль — «зрозуміти файл без читання реалізації» (`Behavior Test`), а не відтворити сигнатури.

### Consequences
* Good, because transcript фіксує очікувану користь: документ самодостатній, не старіє при рефакторингу внутрішніх хелперів, субагент на `kustomization-patches.mjs` (3 нетривіальних exports) все одно коректно додав `## Публічний API`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Зміни внесено в три копії скіла: `npm/skills/docgen/SKILL.md`, `.cursor/skills/n-docgen/SKILL.md` (обидві робочі), `.pi/skills/n-docgen/SKILL.md` (стаб, опис). У Tier-1 промпті: секція `## Публічний API` — лише за нетривіальної зовнішньої поверхні; для leaf-предикатів — пропускати. Замість `## Помилки` — `## Гарантії поведінки` (інваріанти: read-only, не кидає виняток, fail-safe-дефолти). Еталонний файл: `npm/rules/abie/lib/docs/enabled.md`. 15 файлів `npm/rules/abie` перегенеровано в новому стилі (3 батчі по 5 субагентів).

## ADR Вибір локальної Ollama-моделі для Tier 1 docgen на 8 GB M2

## Context and Problem Statement
Tier 1 скіла `docgen` генерує документацію для 1042+ файлів через Claude-субагенти (~26 M токенів, ~$300–570 за прогін). Для зниження вартості розглянуто роутинг Tier 1 виключно на локальну Ollama; при цьому 8 GB unified memory (Apple M2) задає жорстку межу: модель + KV-cache + ОС мають вміститися в ~5–5.5 GB.

## Considered Options
* `gemma4:e4b` (9.6 GB, вже встановлена)
* `gemma3:4b` (~3.3 GB)
* `qwen2.5-coder:3b` (~2.4 GB)
* `qwen2.5:7b` (~5.1 GB)
* `qwen2.5-coder:7b` (~5.1 GB)
* `llama3.1:8b` (~5.7 GB)
* `llama3.2:3b` (~2.9 GB)
* `gemma2:2b` (~3.0 GB)
* `phi3.5` (~4.0 GB)
* `qwen3:4b` (~3.6 GB)

## Decision Outcome
Chosen option: "gemma3:4b як основний двигун; qwen2.5-coder:3b як альтернатива для швидкості", because обидва повністю вміщаються в GPU (100% GPU за `ollama ps`): `gemma3:4b` — 4.2 GB, ~14 tok/s, найкраща якість української та точність (opt-in, алгоритм, fail-safe); `qwen2.5-coder:3b` — 2.4 GB, 36 tok/s (×2.6), прийнятна якість із незначними англіцизмами. Усі 7–8B-моделі (qwen2.5:7b, qwen2.5-coder:7b, llama3.1:8b) потрапили в CPU-офлоад і відсіяні. `gemma4:e4b` (9.6 GB > RAM) дав 0.4 tok/s через постійний своп. `qwen3:4b` ігнорує `/no_think` і генерує російською — непридатний без додаткового тюнінгу.

### Consequences
* Good, because transcript фіксує очікувану користь: `gemma3:4b` дає wall ~76 с на малому файлі (vs >10 хв таймауту у `gemma4:e4b`), Tier 1 оцінюється в ~22–38 год послідовно при $0 хмарних токенів.
* Bad, because на одній машині паралелізм (`concurrency = 1`) — другий одночасний запит ділить ту саму GPU-пропускну здатність; батчі по 5 зі скіла зникають, час = `файлів × час_на_файл`. Крім того, числа tok/s у марафонному прогоні забруднені контенцією (циклічне load/unload 25 GB через 8 GB), тому реальний швидкісний замір — лише з окремого чистого старту.

## More Information
Бенчмарк запущено скриптом `/tmp/docgen-bench/run.py`; вихід — `/tmp/docgen-bench/results.jsonl`; доки кожної моделі — `/tmp/docgen-bench/out/<model>.md`. Еталон якості — `npm/rules/abie/lib/docs/enabled.md` (секції Огляд / Поведінка / Де використовується / Гарантії поведінки; без stdlib, без сигнатур). Модель завантажується через `ollama pull`; вивантаження між замірами — `ollama stop`. Рекомендована архітектура гібрида: Tier 1 → локальна Ollama (послідовно, `--max-bytes`, quality-gate, strip-fences) → Claude-хвіст (файли, що провалили gate або завеликі) → Claude Tier 2/3 (~2.6 M токенів). Промпт потрібно винести в `npm/skills/docgen/js/docgen-prompt.mjs` як спільне джерело для обох двигунів.
