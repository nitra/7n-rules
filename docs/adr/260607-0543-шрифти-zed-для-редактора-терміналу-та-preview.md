---

**Status:** Accepted
**Date:** 2026-06-07

## Context and Problem Statement

Користувач має астигматизм і шукав у Zed моноширинний шрифт, де літери не виглядають вузькими чи зщільненими. Поточний `Menlo` не підтримує проміжні ваги між Regular і Bold, тому зміна `buffer_font_weight` не давала видимого ефекту. Також потрібно було відокремити шрифт терміналу від редактора та з'ясувати, чи можна окремо зменшити шрифт markdown preview.

## Considered Options

- Залишити `Menlo` і змінювати `buffer_font_weight`.
- Використати `Atkinson Hyperlegible Mono` для редактора.
- Використати `JetBrains Mono` або `JetBrains Mono Thin` для терміналу.
- Змінити `markdown.preview.fontSize` у Cursor settings.
- Змінити `ui_font_size` у Zed.
- Змінити `buffer_font_size` у Zed.
- Використати CSS injection для markdown preview.
- Використати тимчасовий zoom `Cmd -` у фокусі preview-панелі.

## Decision Outcome

Chosen option: "Atkinson Hyperlegible Mono для редактора, JetBrains Mono Thin для терміналу, `ui_font_size: 21` без ізольованої зміни preview", because `Menlo` не давав видимого контролю ваги, Atkinson Hyperlegible Mono розроблений для кращої читабельності при порушеннях зору, JetBrains Mono Thin дає тонший шрифт у терміналі, а transcript підтвердив відсутність окремого persistent-налаштування тільки для markdown preview у Zed.

### Consequences

- Good, because Atkinson Hyperlegible Mono має ширші літероформи та кращу розрізнюваність схожих символів, що прямо відповідає проблемі астигматизму.
- Good, because явний `terminal.font_family` відокремлює термінал від `buffer_font_family` редактора.
- Good, because повернення `ui_font_size: 21` зберігає зручний розмір sidebar, tabs, chat і панелей.
- Bad, because markdown preview і changes panel залишаються прив'язаними до загального `ui_font_size`; ізольованого налаштування preview через `settings.json` transcript не підтвердив.
- Bad, because `Cmd -` у preview є лише тимчасовим workaround і скидається при закритті вкладки.
- Neutral, because transcript не містить підтвердження інших негативних наслідків від встановлення шрифтів через Homebrew Cask.

## More Information

Файл налаштувань:
- `/Users/vitaliytv/.config/zed/settings.json`

Фінальні релевантні налаштування з transcript:
```json
"buffer_font_family": "Atkinson Hyperlegible Mono",
"buffer_font_weight": 400,
"ui_font_size": 21,
"terminal": {
  "font_size": 17.0,
  "font_family": "JetBrains Mono Thin"
}
```

Команда встановлення шрифтів:
```sh
brew install --cask font-atkinson-hyperlegible-mono font-jetbrains-mono
```

Transcript facts:
- `markdown.preview.fontSize` у Cursor settings не впливає на preview у Zed і був помилковою спробою.
- `ui_font_size` тестувався зі значеннями `14` і `16`, але зміна впливала на весь UI, не лише на markdown preview.
- `buffer_font_size` керує редактором, але transcript не підтвердив його вплив на markdown preview.
- CSS injection для markdown preview через `settings.json` у Zed не зафіксований як підтримуваний варіант.
- Markdown preview у Zed рендериться окремо і не має підтвердженого isolated persistent font-size setting у transcript.

## Update 2026-06-07

- Додатково зафіксовано спробу зменшити вагу UI у Zed через `"ui_font_weight": 300` для sidebar та інших UI-елементів.
- Transcript не містить підтвердження фінального користувацького відгуку після застосування `ui_font_weight: 300`.
- `terminal.font_weight: 400` у Zed не дав видимого ефекту; робочим варіантом став явний `terminal.font_family: "JetBrains Mono Thin"`.
- Для markdown preview перевірені значення `ui_font_size`: `14`, `16`, `21`; фінально повернуто `21`, бо менші значення погіршували колонки 1–2.
