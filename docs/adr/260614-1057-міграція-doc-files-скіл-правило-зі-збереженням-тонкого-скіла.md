## ADR Міграція doc-files: скіл → правило зі збереженням тонкого скіла

## Context and Problem Statement
Логіка `doc-files` жила у `npm/skills/doc-files/js/`. Скіл не має слотів для policy-каналу (GA-workflow, `package.json`-скрипт через rego) — ці механізми доступні лише правилу. `lint-doc` лишався підкомандою без CI-енфорсу через policy.

## Considered Options
* Перейменувати правило на `doc-files`, лишити тонкий скіл (варіант 1)
* Зробити `doc-files` виключно скілом — прибрати правило `doc` (варіант 2)

## Decision Outcome
Chosen option: "Перейменувати правило на `doc-files`, лишити тонкий скіл", because лише правило несе policy-канал (GA-workflow, `package.json`-скрипт через rego), без якого неможливий CI-енфорс застарілості документації; тонкий скіл зберігає агентний workflow.

### Consequences
* Good, because правило дає повний набір слотів: `meta.json`, `.mdc`-канон, `fix.mjs` (policy через `runStandardRule`), `lint/lint.mjs` (CLI-детектор), `js/lint.mjs` (адаптер агрегатора).
* Bad, because `git mv npm/skills/doc-files/js → npm/rules/doc-files/js` вимагав оновлення re-export у `npm/skills/doc-aggregate/js/docgen-ignore.mjs` і виявив прихований баг (стороннє `*/` у JSDoc `js/lint.mjs` — виправлено).

## More Information
Змінені файли: `npm/rules/doc-files/meta.json`, `npm/rules/doc-files/fix.mjs`, `npm/rules/doc-files/lint/lint.mjs`, `npm/rules/doc-files/js/lint.mjs`, `npm/rules/doc-files/doc-files.mdc`, `npm/skills/doc-aggregate/js/docgen-ignore.mjs`. ## ADR N_LOCAL_MIN_MODEL як стандартний тир-env замість N_CURSOR_DOCGEN_MODEL

## Context and Problem Statement
Для запуску `fix-doc-files` на локальній машині (omlx) потрібно вказати модель. Спершу використали `N_CURSOR_DOCGEN_MODEL` (docgen-специфічний env). Пакет уже має тир-систему (`npm/lib/models.mjs`, `resolveModel(tier)`) з єдиним universal env `N_LOCAL_MIN_MODEL`, яку `generateDoc()` читає через `N_CURSOR_DOCGEN_MODEL ?? resolveModel('min') ?? …`.

## Considered Options
* `N_LOCAL_MIN_MODEL` — стандартний тир-env пакета (honors `resolveModel('min')`)
* `N_CURSOR_DOCGEN_MODEL` — docgen-специфічний env, не впливає на інші скіли

## Decision Outcome
Chosen option: "`N_LOCAL_MIN_MODEL`", because це вже стандарт пакета; всі скіли, що використовують `resolveModel('min')`, підхоплять одне налаштування; `N_CURSOR_DOCGEN_MODEL` не потрібен — він лише override для docgen, тоді як `N_LOCAL_MIN_MODEL` є канонічним "локальний мінімальний тир".

### Consequences
* Good, because transcript фіксує очікувану користь: `env -i` тест підтвердив, що при `N_LOCAL_MIN_MODEL=omlx/gemma-4-e2b-it-4bit` і відсутньому `N_CURSOR_DOCGEN_MODEL` генерація виконується успішно (`✓ OK: 1 score=100`); інші скіли на `min`-тирі теж отримують default.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
`~/.zshenv`: `export N_LOCAL_MIN_MODEL=omlx/gemma-4-e2b-it-4bit`. Причина моделі e2b (а не e4b-OptiQ): 16GB RAM, omlx memory ceiling 12GB — `gemma-4-e4b-it-OptiQ-4bit` потребує 13.07GB (exit 18), `gemma-4-e2b-it-4bit` влазить. `DEFAULT_OMLX_MODEL` у пакеті застарілий (`mlx-community--gemma-4-e2b-it-4bit` vs `gemma-4-e2b-it-4bit` на сервері) — без env `callLlm` падає з "Model not found".
