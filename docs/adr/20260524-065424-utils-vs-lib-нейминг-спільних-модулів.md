# Конвенція `utils/` vs `lib/` для спільних JS-модулів

**Status:** Accepted
**Date:** 2026-05-24

## Context and Problem Statement

Кілька JS-файлів мають спільну логіку, яку виносять в окремий каталог. Відсутність явного розподілу між `utils/` і `lib/` призводила до неоднозначного розміщення та ускладнювала навігацію.

## Considered Options

- `utils/` для всього спільного коду
- `lib/` для всього спільного коду
- Розподіл: `utils/` для generic helpers, `lib/` для domain-специфічних підсистем
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Розподіл: `utils/` для generic helpers, `lib/` для domain-специфічних підсистем", because правило одного погляду: якби цей файл міг бути в lodash/date-fns — це `utils/`; якщо він прив'язаний до домену або має side effects — це `lib/`.

Конвенція:

- **`utils/`** — низькорівневі, чисті, generic helpers без бізнес-логіки і без залежностей від домену: `formatDate()`, `chunk(array, size)`, `retry(fn, opts)`, `sleep(ms)`
- **`lib/`** — внутрішні модулі/підсистеми, що знають про домен, мають власний state або side effects: `lib/gmail-client.js`, `lib/circuit-breaker.js`, `lib/skill-loader.js`

### Consequences

- Good, because `fix js-lint` проходить без зауважень після додавання секції до `js-lint.mdc`
- Bad, because transcript не містить підтверджених негативних наслідків

## More Information

- Зміни: `npm/rules/js-lint/js-lint.mdc` (v1.23 → v1.24, нова секція «Структура спільних модулів»), `.cursor/rules/n-js-lint.mdc` (дзеркальне оновлення)

## Update 2026-05-24

**Аудит поточного стану (10 каталогів `npm/rules/*/utils/`):** 9 з 10 порушують «npm-пакет-тест» правила `n-js-lint.mdc` і мають бути перейменовані на `lib/`:

- `abie/utils` → `lib/` (k8s-tree, kustomization-patches, overlay-paths — domain-state + cross-rule залежності)
- `changelog/utils` → `lib/` (package-manifest.mjs — парсер під конкретний концерн)
- `docker/utils` → `lib/` (docker-hadolint.mjs запускає зовнішній бінар/контейнер)
- `graphql/utils` → `lib/` (GraphQL-AST-сканер)
- `js-bun-db/utils` → `lib/` (bun-sql-scan)
- `js-lint/utils` → `lib/` (ребілд oxlint-конфігу)
- `js-mssql/utils` → `lib/` (domain-сканер MSSQL)
- `js-run/utils` → `lib/` (5 сканерів: bunyan→pino, conn-strings, promise/setTimeout, env)
