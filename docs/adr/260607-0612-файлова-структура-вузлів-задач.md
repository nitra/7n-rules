**Status:** Accepted
**Date:** 2026-06-07

## Context and Problem Statement

Для системи `n-cursor graph` потрібен механізм зберігання DAG-вузлів задач, щоб кожен вузол мав визначений стан, бюджет і критерії завершення. У репозиторії вже існував документ `docs/думка.MD` з описом рекурсивного складеного ОАГ, але директорія `tasks/` ще не була матеріалізована.

## Considered Options

- Файлова структура `tasks/<node-name>/task.md` за специфікацією `docs/думка.MD`.
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Файлова структура `tasks/<node-name>/task.md` за специфікацією `docs/думка.MD`", because специфікація вже існувала в репозиторії і задавала формат вузла: YAML frontmatter з метаданими та markdown-секції для опису задачі, критеріїв завершення і вхідних даних.

### Consequences

- Good, because стан вузла визначається наявністю файлів без окремої бази даних або централізованого реєстру.
- Good, because `## Done when` дає явні критерії для подальшої перевірки або аудиту.
- Bad, because transcript не містить підтвердження негативних наслідків.
- Neutral, because transcript не містить підтвердження остаточного рішення щодо назви окремого UI-проєкту для перегляду графа.

## More Information

Створені вузли у рамках сесії:

- `tasks/ui-task-view/task.md` — UI-перегляд графу `n-cursor graph`, budget 3600 секунд.
- `tasks/coverage-skill-test/task.md` — перевірка скілу `n-coverage-fix` після міграції `claude-agent-sdk` → `pi`, budget 1800 секунд.
- `tasks/skills-orchestrator-migration/task.md` — продовження міграції `npm/skills/` на JS-оркестратор, budget 7200 секунд.

Формат `task.md`: YAML frontmatter (`created_at`, `budget_sec`, опційно `parent`, `deps`) і секції `## Task`, `## Done when`, `## Inputs`.

Запуск вузла: `n-cursor graph run tasks/<name>`.

Контекст: `docs/думка.MD`, `npm/lib/models.*`, незакомічені зміни в `npm/skills/docgen-gen.mjs`, `llm-worker.mjs`, `subagent-runner.mjs`.
