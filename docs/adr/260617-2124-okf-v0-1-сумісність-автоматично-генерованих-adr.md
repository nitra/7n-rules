## ADR OKF v0.1 сумісність автоматично-генерованих ADR

## Context and Problem Statement
Автоматично нормалізовані ADR не мали YAML frontmatter у "Clean"-стані, тоді як OKF (Open Knowledge Format) §9 вимагає наявності frontmatter з непорожнім полем `type` у кожному `.md` файлі. Це робило `docs/adr/*.md` невалідними відносно OKF v0.1 SPEC.

## Considered Options
* Додати OKF-сумісний YAML frontmatter (`type: ADR`, `title`, `tags`, `timestamp`) у `rewrite`-операції normalize-pipeline
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Додати OKF-сумісний YAML frontmatter у rewrite-операції", because це єдиний спосіб відповідати вимозі OKF §9 без зміни структури тіла файлу (MADR-секції залишаються без змін).

### Consequences
* Good, because transcript фіксує очікувану користь: кожен `.md` файл у `docs/adr/` тепер відповідає OKF v0.1 §9 Conformance вимогам.
* Bad, because це відхід від MADR v4.0.0 minimal принципу "без frontmatter у clean-файлах" — хоча структурного конфлікту між OKF frontmatter та MADR тілом немає.
