# Системне виправлення yannh URL для груп *.k8s.io у check.mjs

**Status:** Accepted
**Date:** 2026-05-18

## Context and Problem Statement

У `expectedSchemaUrlForTypedManifest` (файл `npm/rules/k8s/fix/manifests/check.mjs`) URL схем для груп типу `networking.k8s.io` формувався як `<kind>-<group-з-крапками-замінені-на-дефіси>-<version>.json` — наприклад, `networkpolicy-networking-k8s-io-v1.json`. Репозиторій yannh/kubernetes-json-schema зберігає ці файли під скороченою назвою (лише перший сегмент групи до першої крапки), тому такі URL повертають HTTP 404. Початково зроблений точковий виняток лише для `NetworkPolicy` не вирішував проблему для Ingress, Role, ClusterRole та інших груп.

## Considered Options

* Виправити лише `NetworkPolicy` — захардкодований виняток для `networking.k8s.io/v1`
* Виправити системно — брати лише перший сегмент `group` (до першої крапки) для всіх груп із `YANNH_GROUPS`

## Decision Outcome

Chosen option: "Виправити системно", because HTTP-перевірка підтвердила однаковий патерн для всіх груп: `ingress-networking-k8s-io-v1.json` → 404, `ingress-networking-v1.json` → 200; `role-rbac-authorization-k8s-io-v1.json` → 404, `role-rbac-v1.json` → 200. Загальне правило `group.slice(0, firstDot)` усуває клас помилок одразу.

### Consequences

* Good, because всі 218 тестів у `npm/rules/k8s/` проходять після заміни; залишків зламаних URL у кодовій базі не знайдено.
* Good, because системне рішення усуває необхідність додавати по одному виняткові при кожному новому ресурсі з `*.k8s.io`-групою.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

Правило трансформації: `group.indexOf('.') === -1 ? group : group.slice(0, group.indexOf('.'))` перед підстановкою у шаблон `<kind>-<groupPart>-<version>.json`.

HTTP-верифікація через `curl -sI` до `https://raw.githubusercontent.com/yannh/kubernetes-json-schema/master/v1.33.9-standalone-strict/`. Перевірено пари: `networkpolicy-networking-v1.json` → 200 / `networkpolicy-networking-k8s-io-v1.json` → 404; `ingress-networking-v1.json` → 200 / `ingress-networking-k8s-io-v1.json` → 404; `role-rbac-v1.json` → 200 / `role-rbac-authorization-k8s-io-v1.json` → 404.

Змінені файли: `npm/rules/k8s/fix/manifests/check.mjs`, `npm/rules/k8s/k8s.mdc` (`1.34` → `1.35`), `.cursor/rules/n-k8s.mdc` (`1.34` → `1.35`), `npm/package.json` (`1.13.40` → `1.13.41`), `npm/CHANGELOG.md`.

Тести: `bun test npm/rules/k8s/fix/manifests/check-schema.test.mjs` — 180 pass; `bun test npm/rules/k8s/` — 218 pass.
