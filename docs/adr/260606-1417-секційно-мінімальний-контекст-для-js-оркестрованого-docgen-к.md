## ADR Секційно-мінімальний контекст для JS-оркестрованого docgen-конвеєра

## Context and Problem Statement
Оркестрований конвеєр v1 надсилав повний код файлу в кожну з 4 секційних LLM-підзапитів (`Огляд`, `Поведінка`, `API`, `Гарантії`), що давало 4× інгест і час у 2–5 разів більший за one-shot режим при тій самій якості (~86%).

## Considered Options
* v1: спільний повний код-префікс на всі секції (через KV-cache надію)
* v2: секційно-мінімальний контекст — код тільки в `Поведінку`, решта секцій отримують крихітний факт-лист

## Decision Outcome
Chosen option: "v2 — секційно-мінімальний контекст", because KV-cache між stateless `/api/chat`-запитами ненадійний на 8 GB, а `Огляд` / `API` / `Гарантії` не потребують повного коду — лише header, список експортів і markers.

### Consequences
* Good, because час overlay знизився 310→77 с (×4), k8s 141→55 с (швидше one-shot); оркестрація перестала програвати в часі.
* Bad, because Gemma 3n E4B (`gemma4:4b`) повертає порожній рядок для секцій без коду — orchestrated-режим для неї фундаментально непрацездатний; `gemma4:4b` залишений лише для one-shot.

## More Information
Реалізація: `npm/skills/docgen/js/docgen-prompts.mjs` → `sectionMessages(facts, src)` (код лише в `behavior`-елементі масиву); `npm/skills/docgen/js/docgen-gen.mjs` → `generateOrchestrated`. Коміт `17cfca32`. Бенч: 3 файли × 2 режими, `gemma3:4b`, 8 GB Apple M2.

## ADR Детермінований Stage-2 лінт сигнатур (0 токенів)

## Context and Problem Statement
Після переходу на секційно-мінімальний контекст секція `Поведінка` єдина бачить повний код, тому модель echo-їть JS-сигнатури (`isUaKustomizationPath(rel)…`) і внутрішні імена (`readAndParseYamlDocs`) у прозовий текст.

## Considered Options
* Додатковий LLM-промпт для очищення сигнатур
* Детермінований regex-зріз після генерації (0 токенів)

## Decision Outcome
Chosen option: "Детермінований regex-зріз (Stage 2)", because дефекти — формат, не зміст, тому їх найкраще прибирати механічно без додаткових токенів.

### Consequences
* Good, because transcript фіксує очікувану користь: `isUaKustomizationPath(rel)` → `isUaKustomizationPath` без опису; витік `readAndParseYamlDocs` усунуто.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

## ADR Детермінований scoreDoc + Tier-2 хмарний fallback

## Context and Problem Statement
Локальна модель `gemma3:4b` давала 75% на складних файлах (`npm/rules/ga`) — зокрема витік внутрішніх імен у `Гарантіях` і галюцинований кеш. Необхідний автоматичний механізм виявлення низькоякісного виходу і ескалації до хмарного Claude.

## Considered Options
* Детермінований скоринг проти фактів (0 токенів) + хмарний fallback якщо score < порогу
* LLM-суддя (окремий промпт до тієї ж моделі) для самооцінки
* Embedding-косинус між виходом і reference-зразками

## Decision Outcome
Chosen option: "Детермінований scoreDoc + Tier-2 Claude fallback", because Stage 0 вже містить `facts` (маркери, `internalSymbols`, `markers.caches`) — перевірка виходу проти них не коштує токенів і є надійнішою за LLM-самооцінку.

### Consequences
* Good, because transcript фіксує очікувану користь: `no-overview` (-25), `short-behavior` (-20), `cache-hallucination` (-20), `internal-name:X` (-10 кожен) — детектовані без LLM; `fix.mjs` score=100 після фіксу regex.
* Bad, because cache-детектор мав хибне спрацювання на коректне "Не використовує кешування" (regex `/кеш/i` — занадто широкий); виправлено на `cacheHit = /кешу[єє]/i.test() && !/не\s+кешу|без\s+кешу/i.test()`.

## More Information
`parseSections(md)`, `scoreDoc(md, facts)`, `claudeOneShot(facts, src, model)` у `npm/skills/docgen/js/docgen-gen.mjs`. Tier 2 активується лише якщо `score < 70 && env.ANTHROPIC_API_KEY`. Використовує `@anthropic-ai/sdk` (вже є в `bun.lock` як `^0.100.1`). `STYLE` та `oneShotPromptText` експортовані з `npm/skills/docgen/js/docgen-prompts.mjs`. Коміт `98d8b45d`.
