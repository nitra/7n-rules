## ADR Авто-встановлення зовнішніх CLI-інструментів (`ensureTool`)

## Context and Problem Statement
Скрипти пакета `@nitra/cursor` залежать від зовнішніх CLI-інструментів (`conftest`, `hk`, `shellcheck`, `hadolint`, `opa`, `regal`, `kubeconform`, `kubescape`, `actionlint`, `dotenv-linter`). До цього рішення кожен call-site самостійно шукав бінарник через `resolveCmd`, а при невдачі — або замовчував, або друкував ad-hoc текстову підказку зі встановлення, що призводило до неодноманітної поведінки і «тихих» збоїв.

## Considered Options
* Єдиний seam `ensureTool(toolId)` з авто-встановленням через пакетний менеджер (brew/scoop) або GitHub Release binary (Linux / fallback)
* Залишити ad-hoc `resolveCmd` + текстові install-hints у кожному call-site (поточний стан)
* GitHub Release binary для всіх платформ (єдина пінована версія)

## Decision Outcome
Chosen option: "Єдиний seam `ensureTool(toolId)` з авто-встановленням через пакетний менеджер або GitHub Release binary", because це усуває дублювання install-логіки, дає однакову hard-fail поведінку в усіх сьютах і відповідає наявній логіці `lock-cache-dir.mjs`/`with-lock.mjs`.

**Per-platform matrix (зафіксована в `npm/scripts/lib/ensure-tool.mjs` та `scripts.mdc`):**

| OS | Метод |
|---|---|
| macOS | `brew install <formula>` |
| Windows | `scoop install <app>` (winget не має ні `hk`, ні `conftest`; Scoop Main має обидва) → fallback GitHub Release binary |
| Linux | GitHub Release binary (download + `chmod +x`) |

**hk:** `ensureTool('hk')` (бінарник) + `ensureHkInstall(hkBin)` у потоці `n-fix`; `hk install` пропускається в CI (`process.env.CI`).
**Режим невдачі:** скрізь hard-fail (exit non-zero / `throw`).
**Opt-out:** `N_CURSOR_NO_AUTO_INSTALL=1` → skip install, одразу hard-fail (використовується в тестах).
**Версії:** latest, без перевірки (brew/scoop дають latest; Linux GitHub Release — `fetchLatestVersion` через GitHub API).

### Consequences
* Good, because transcript фіксує очікувану користь: `withBinRemovedFromPath` тепер автоматично виставляє `N_CURSOR_NO_AUTO_INSTALL=1`, тож негативні тести не тягнуть реальний brew/curl.
* Good, because `docker-hadolint`: docker-run fallback прибрано — hadolint тепер виключно нативний бінарник через `ensureTool`, що спрощує контракт `lintDockerfileWithHadolint`.
* Bad, because winget не має `hk` і `conftest` (підтверджено через `github.com/microsoft/winget-pkgs`), тому Windows-шлях через Scoop не є ідеально безшовним: якщо Scoop відсутній — CLI падає на GitHub Release binary з додатковим кроком.
