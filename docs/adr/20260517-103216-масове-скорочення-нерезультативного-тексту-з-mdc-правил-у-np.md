## ADR Масове скорочення нерезультативного тексту з .mdc-правил у npm/rules/

**Контекст:** Файли правил у `npm/rules/` містили значну кількість тексту, що не впливає на поведінку LLM або автоматичних перевірок: інструкції зі встановлення (`brew install`, `apt install`), блоки `.vscode` без директив-полісі, секції «## Перевірка» з описом ручного запуску, пояснення зовнішніх інструментів (`hadolint`, `gitleaks`, `kubeconform`), активаційні мета-секції та міграційний контекст для старих версій.

**Рішення/Процедура/Факт:** Проведено аудит 26 файлів через паралельний аналіз. Застосовано очищення до 23 файлів, прибрано ~314 рядків. Категорії видаленого: секції «## Перевірка» (18 файлів), блоки `.vscode` без полісі (8 файлів), `brew`/`apt` інструкції, описи зовнішніх інструментів, активаційні мета-секції (`abie`, `adr`), дублікати (`bun.mdc`). Залишено без змін: `vue`, `php`, `text` (ризик суперечностей), ENV-таблиця `adr` (borderline). Збережено: канонічні snippets, version pins, behavioral directives, `.vscode`-конфіги де є реальна полісі.

**Обґрунтування:** Нерезультативний текст збільшує контекстне вікно без користі для LLM і скриптів перевірок, ускладнює підтримку і маскує канонічні директиви серед пояснювального шуму.

**Розглянуті альтернативи:** Видалення лише `.vscode`-блоків (часткове — відхилено); залишити без змін (відхилено); включити `vue`/`php`/`text` (відкладено через ризик суперечностей).

**Зачіпає:** `npm/rules/**/*.mdc` — 23 файли; пілот — `security/security.mdc` (108 → 73 рядки); `adr.mdc`, `bun.mdc`, `k8s.mdc`, `docker.mdc` та ін.

## ADR Архітектура `template/` каталогів як єдиного джерела канону у правилах npm/rules/

**Контекст:** Канонічні фрагменти і повні файли-зразки (merge-шаблони, `.gitleaks.toml`, workflow `.yml`) зберігалися виключно у fenced-блоках `.mdc`, унеможливлюючи їх пряме використання скриптами `check-*.mjs` і `conftest.rego` без дублювання.

**Рішення/Процедура/Факт:** Введено `template/` на рівні концерну (`policy/<concern>/template/` або `fix/<concern>/template/`) з native-syntax файлами та суфіксами: `.snippet.<ext>` (канонічний фрагмент або повний файл), `.deny.<ext>` (наявність = порушення), `.contains.<ext>` (відсутність = порушення). Conftest отримує шаблони через `--data <tempfile>` як `data.<ns>.template.*`. Введено loader-утиліту і `template-checks` утиліту; `runConftestBatch` розширено опціональним `templateData`. У `.mdc` — посилання на `template/`, а не дубль. Spec у `docs/superpowers/specs/2026-05-17-template-dir-design.md` (layout, семантика, data-path, loader, 6-фазовий план міграції, класифікація концернів, ризики).

**Обґрунтування:** Rule-level `template/` (один каталог на правило) відхилено через collision: `k8s` — 4 концерни на `*.yaml`, `npm-module` — 2 концерни на `package.json`. Concern-level усуває неоднозначність і тримає шаблон поряд зі check-скриптом. Native-syntax + суфікси обрано замість DSL-маніфесту — природніший для читання і редагування.

**Розглянуті альтернативи:** Rule-level `template/` (відхилено — collision); єдиний `check.json` DSL з ключами `required`/`forbidden`/`contains` (Підхід 2 — відхилено як зайвий шар); збереження snippets тільки в `.mdc` (першопричина проблеми — відхилено).

**Зачіпає:** `npm/rules/*/policy/*/template/`, `npm/rules/*/fix/*/template/`; `check-*.mjs`; `*.rego`; `runConftestBatch`; `.mdc` всіх правил з фрагментами; `docs/superpowers/specs/2026-05-17-template-dir-design.md`.
