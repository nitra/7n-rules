## ADR Concern-based JS + декларативний target.json для правил @nitra/cursor
**Контекст:** Кожне правило мало один монолітний `js/check.mjs` (наприклад, `abie` — 1153 рядки) і дублювало файловий targeting для rego-полісі: списки файлів жили одночасно в JS-викликах `runConftestBatch` і в hardcoded TARGETS-таблиці `lint-conftest.mjs` (~280 рядків). Масштаб рефакторингу або розширення будь-якого правила вимагав синхронізувати три місця.

**Рішення/Процедура/Факт:** Введено декларативні `target.json` поруч із кожним `<concern>.rego` — маніфест описує, які файли фідити в conftest (`{ "files": { "single" | "walkGlob" } }`). CLI сам резолвить файли і викликає `runConftestBatch`; JS-код більше не дублює виклики. Кожне правило розпиляне на `js/<concern>/check.mjs` замість плаского `js/check.mjs`. Applies-гейт через опційний `applies()` export у `js/applies/check.mjs`. `lint-conftest.mjs` переписано з hardcoded TARGETS на auto-discovery через `discoverCheckableRules` + читання `target.json`. Legacy-fallback для flat `js/check.mjs` вилучений після повної міграції всіх 25 вшитих правил. Попутно виправлено: kebab→snake namespace у `run-rule.mjs`, picomatch array-negation у `resolve-target-files.mjs`, subdir-preference у discovery (уникнення double-run), `runSync()` хардкодив `mdc/` замість `rules/` після phase 1-4 rename (кинав «Не знайдено каталог правил пакету» при `npx @nitra/cursor`).

**Обґрунтування:** Одна одиниця відповідальності = один concern-каталог: rego, JS або hybrid. Pure-rego правила можливі без жодного `.mjs`. Shared walk-cache у `resolve-target-files.mjs` — один обхід дерева на унікальний `ignorePaths`-сигнатур. Module-level кеш у `abie/utils/k8s-tree.mjs` — `findK8sYamlFiles` + `collectDeploymentDirs` обчислюються один раз між концернами одного прогону.

**Розглянуті альтернативи:** Залишити монолітний `js/check.mjs` з розширенням TARGETS-таблиці — відхилено (масштабування O(n) дублювання); окремий concerns-registry JSON — відхилено (target.json вже є поруч із rego і є природним джерелом правди).

**Зачіпає:** `npm/scripts/utils/discover-checkable-rules.mjs`, `npm/scripts/utils/run-rule.mjs`, `npm/scripts/utils/resolve-target-files.mjs`, `npm/scripts/lint-conftest.mjs`, `npm/bin/n-cursor.js`, `npm/schemas/target.json`, 25 каталогів `npm/rules/<id>/js/<concern>/`, ~50 нових `npm/rules/<id>/policy/<concern>/target.json`.
