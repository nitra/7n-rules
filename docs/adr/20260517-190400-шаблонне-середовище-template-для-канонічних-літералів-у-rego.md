## ADR Шаблонне середовище (template/) для канонічних літералів у rego-policy

## Context and Problem Statement
У `npm/rules/*/policy/` rego-файли містили канонічні значення (cron-вирази, workflow-назви, поля `package.json`) як inline-літерали. При зміні канону потрібно було редагувати і `.rego`, і `.mdc`, і тести — три місця одночасно.

## Considered Options
* Залишити канон inline у `.rego`
* Виносити канон у `template/<файл>.snippet.*` та передавати через `conftest --data` як `data.template.snippet`

## Decision Outcome
Chosen option: "template/-driven canon", because всі три компоненти (`.rego`, тести, `.mdc`-lLink) тягнуть канон з одного джерела, і `findMissingMdcRefs` перевіряє, що жоден template-файл не залишається без посилання в `.mdc`.

### Consequences
* Good, because transcript фіксує очікувану користь: зміна канону в одному `template/*.snippet.*` автоматично підхоплюється rego, тестами (через `template_data`) і документацією (через `@`-лінк у `.mdc`).
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Перевіряється: `bun -e 'import { findMissingMdcRefs } from "./npm/scripts/utils/check-mdc-template-refs.mjs"; console.log(await findMissingMdcRefs(...))'`

## Context and Problem Statement

## Considered Options
* Generic walker (рекурсивний subset-of, один спільний rego для всіх workflow)
* Per-concern field-by-field (як є: кожен концерн — окремий rego з вибірковими deny-правилами)

## Decision Outcome
Chosen option: "Per-concern field-by-field (як є)", because користувач явно обрав цей варіант; він зберігає семантичні deny-повідомлення і не потребує нового generic-механізму.

### Consequences
* Good, because transcript фіксує очікувану користь: opa test 354/354 без нових абстракцій; deny-повідомлення залишаються специфічними для кожного поля.
* Bad, because transcript не містить підтверджених негативних наслідків.

## ADR Regal line-length підвищено до 220 символів

## Considered Options
* Лишити дефолтний ліміт 120 і переносити рядки вручну
* Підвищити ліміт у `.regal/config.yaml` до значення, що покриває максимальну довжину після `opa fmt`

## Decision Outcome
