## ADR Структурний скіп tooling-only сесій в ADR-хуках і канонічний `ignorePaths` для `docs/adr/**`

## Context and Problem Statement
Після кожної сесії, де змінювалися лише tooling-файли (`.cspell.json`, `docs/adr/*.md`, `CHANGELOG.md`, `package.json#version`), ADR-хуки `capture-decisions.sh` та `normalize-decisions.sh` генерували нові чернетки та передавали їх до LLM. Це спричиняло нескінченний цикл «tooling-зміна → новий ADR → cspell-помилка у ADR → нова tooling-сесія → новий ADR». Крім того, `docs/adr/**` не входив до канонічного `ignorePaths` у `.cspell.json`, через що машинно-генеровані MADR-документи перевірялися cspell, що само по собі провокувало наступний цикл.

## Considered Options
* Виключно промптова інструкція `OUTPUT NONE ONLY IF the session is genuinely trivial` (наявне рішення)
* Детермінований pre-LLM bash-фільтр `is_tooling_only_change` в обох хуках + `docs/adr/**` у канонічному `ignorePaths`

## Decision Outcome
Chosen option: "Детермінований pre-LLM bash-фільтр + розширення `ignorePaths`", because промптова інструкція виявилася недостатньою — агент у багатьох випадках повертав ADR для tooling-сесій попри `OUTPUT NONE ONLY IF`; детермінований bash-фільтр гарантує skip до виклику LLM.

### Consequences
* Good, because transcript фіксує очікувану користь: tooling-only сесії (`.cspell.json`, `docs/adr/*.md`, `CHANGELOG.md`, `package.json#version`) більше не генерують ADR-чернеток і не кличуть LLM.
* Good, because `docs/adr/**` у канонічному `ignorePaths` (через `npm/rules/text/policy/cspell/template/.cspell.json.snippet.json`) автоматично поширюється на споживацькі репо через `npx @nitra/cursor fix text`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- `npm/rules/text/policy/cspell/template/.cspell.json.snippet.json` — додано `"docs/adr/**"` після `"**/k8s/**/*.yaml"`
- `npm/rules/text/policy/cspell/cspell_test.rego` — оновлені фікстури `template_data`/`valid_cfg` та новий тест `test_deny_missing_docs_adr`; opa PASS: 44/44

- `npm/.claude-template/hooks/normalize-decisions.sh` — ті самі функції + per-draft delete-фільтр; ENV-перемикач `ADR_NORMALIZE_SKIP_TOOLING_ONLY` (default `1`)

## ADR Виправлення `isRunAsCli` — передача `import.meta.url` як аргумент

## Context and Problem Statement
`isRunAsCli()` у `scripts/cli-entry.mjs` та `scripts/lib/run-rule-cli.mjs` перевіряла `import.meta.url` файла, де функція визначена, а не файла-caller'а. `import.meta` є лексично прив'язаним — у helper-функції він завжди вказує на власний файл helper'а. Через це ~40 callsites у `rules/<id>/fix.mjs`, `lint/*.mjs`, `bin/rename-yaml-extensions.mjs` завжди отримували `false` і виходили без output та без дії.

## Considered Options
* Параметр `isRunAsCli(import.meta.url)` з дублюванням impl у двох файлах
* Консолідація в одну реалізацію у `scripts/cli-entry.mjs` + параметр `isRunAsCli(metaUrl)` + re-export з `scripts/lib/run-rule-cli.mjs`

## Decision Outcome
Chosen option: "Консолідація в одну реалізацію + параметр", because user явно обрав цей варіант; re-export зберігає існуючі import-шляхи у callers без масового рефакторингу.

### Consequences
* Good, because transcript фіксує очікувану користь: `node rules/text/fix.mjs` після фіксу виводить `🔍 fix text — перевірка правила` і повний звіт замість мовчазного виходу 0.
* Good, because `realpathSync` нормалізує symlink-шляхи (macOS `/tmp` ↔ `/private/tmp`, pnpm content-addressable links) — перевірено тестом у `cli-entry.test.mjs`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- `npm/scripts/cli-entry.mjs` — `isRunAsCli(metaUrl)` з `realpathSync` для нормалізації
- `npm/scripts/lib/run-rule-cli.mjs` — дублікат impl видалений, замість нього `export { isRunAsCli } from '../cli-entry.mjs'`
- 40 callsites оновлено через `sed -i '' 's/if (isRunAsCli())/if (isRunAsCli(import.meta.url))/g'`
- Нові тести: `npm/scripts/tests/fixtures/cli-entry-as-cli.mjs` (fixture), 3 нові кейси у `npm/scripts/tests/cli-entry.test.mjs`
- `npm/package.json` версія `1.18.2`; `npm/CHANGELOG.md` секція `[1.18.2]`
- Bun test: 1044 pass, 0 fail після фіксу
