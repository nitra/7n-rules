```
Крок 0 → Крок 1 → Крок 2 → Крок 3 → Крок 4 → Крок 5
```

Читає кореневий `package.json#scripts`. Якщо є `"coverage"` → `bun run coverage`; якщо є `"test"` → `bun run test`. Інакше fallback: `n-cursor coverage` / `bun test`. Решта кроків використовує ці змінні — не хардкод.

**Крок 3 — Agent на кожен файл**

**Крок 4 — `bun test`** (або test-команда з Кроку 0)
Якщо тести падають — повертає Agent виправляти.

**Крок 5 — re-run coverage + рішення**
