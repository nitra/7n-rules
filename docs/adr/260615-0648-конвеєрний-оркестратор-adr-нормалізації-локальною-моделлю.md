**Status:** Accepted
**Date:** 2026-06-15

## Context and Problem Statement

Поточний `normalize-decisions.sh` використовував single-shot підхід: один великий prompt з усіма чернетками й clean-файлами мав повернути одну JSON-відповідь з усіма операціями. На cloud-моделях це працювало, але локальна 4B-модель `omlx/gemma-4-e4b-it` не витримувала 32k-токенний prompt: transcript фіксує `omlx curl exit 18` або зависання понад 5 хвилин.

## Considered Options

- Single-shot перенос: той самий prompt і той самий JSON, але через `omlx`.
- Конвеєрний підхід: JS оркеструє глобальний стан, а LLM відповідає лише на вузькі verifiable-питання.

## Decision Outcome

Chosen option: "Конвеєрний підхід з інверсією керування", because `naive-local` впав на infra-рівні, а hardened `new-local` завершив батч локально без хмарних викликів, зі структурно валідними MADR і детермінованим покриттям no-decision випадків.

### Consequences

- Good, because конвеєр виконується з `cloudCalls: 0` і не передає чернетки зовнішнім LLM-сервісам.
- Good, because глобальні інваріанти — coverage, slug uniqueness, union-find кластери й MADR structural lint — гарантує JS, а не модель.
- Good, because hardened варіант додав `isNoDecision` gate і 3 голоси з порогом 0.6 для draft-draft edge-judge.
- Bad, because wall-clock час близько 17 хвилин на 10 драфтів не кращий за cloud-gold близько 14 хвилин через серійні локальні inference-виклики.
- Neutral, because transcript не містить підтвердження production-коміту цих змін; зміни лишилися в worktree `main-adr-exp` не закоміченими.

## More Information

- Реалізація у worktree `main-adr-exp`: `npm/scripts/lib/adr/normalize-pipeline.mjs`, `npm/scripts/lib/adr/normalize-cli.mjs`.
- CLI subcommand: `adr-normalize-local` у `npm/bin/n-cursor.js`.
- Hook integration: `npm/.claude-template/hooks/normalize-decisions.sh`, local-backend активується за відсутності `claude`/`cursor-agent` або `ADR_NORMALIZE_BACKEND=local`.
- Експериментальні файли: `experiments/adr-exp/RESULTS.md`, `experiments/adr-exp/out/*/summary.json`.
- Конвеєрні стадії: lexical top-K retrieval, binary edge-judge, union-find clustering, kind-judge, gen-MADR, merge-additions, deterministic validation gate.
- Модель: `omlx/gemma-4-e4b-it-OptiQ-4bit`, env `N_LOCAL_MIN_MODEL`.
- Transcript фіксує 45 зелених тестів, з них 13 нових юніт-тестів детермінованих функцій у `npm/scripts/lib/adr/tests/normalize-pipeline.test.mjs`.

## Update 2026-06-14

Перед конвеєрною реалізацією було зафіксовано, що поточний `.claude/hooks/normalize-decisions.sh` є bash-hook на 498 рядків і працює через хмарний LLM. Референсним патерном для нового оркестратора визначено `npm/scripts/lib/fix/orchestrator.mjs`, а модельна інфраструктура вже мала маршрут `npm/lib/models.mjs`: префікс `omlx/<model>` іде в `npm/lib/omlx.mjs` як локальний HTTP-виклик, решта — у хмару. Супутній ручний сценарій описано в `.cursor/skills/n-adr-normalize/SKILL.md`.

## Update 2026-06-15

Експеримент порівняв три arms:

- `naive-local`: той самий single-shot prompt на `omlx/gemma-4-e4b-it-OptiQ-4bit`; впав на infra-рівні через `omlx curl exit 18` або зависання.
- `cloud-gold`: той самий single-shot prompt на sonnet; завершився, але потребував хмари й близько 811 секунд.
- `new-local`: per-draft конвеєр із вузькими LLM-мікрозадачами; завершив 10 чернеток локально з `each-once=true`, `madrInvalid=0`, `cloudCalls=0`, `escalations=0`.

Додатково зафіксовано інверсію керування: JS тримає union-find кластери, coverage-гейт «кожен файл рівно один раз», унікалізацію slug-ів і MADR structural lint, а LLM відповідає лише на вузькі задачі `edge-judge`, `kind-judge`, `gen-MADR` і `merge-additions`. Виявлена регресія kind-judge для no-decision draft стала підставою для детермінованого гейту `не обрано → delete`.

## Update 2026-06-15

- Зафіксовано A/B-результат локального pipeline backend проти cloud-gold на 10 фікстурах: 7/7 MADR валідних, повне покриття, 0 cloud-викликів і 0 infra-крахів.
- Реалізація включає `npm/scripts/lib/adr/normalize-pipeline.mjs`, `npm/scripts/lib/adr/normalize-cli.mjs`, `npm/scripts/lib/adr/tests/normalize-pipeline.test.mjs`, backend-селектор `ADR_NORMALIZE_BACKEND=local|claude|cursor` у `npm/.claude-template/hooks/normalize-decisions.sh` і subcommand `adr-normalize-local` у `npm/bin/n-cursor.js`.
- Додаткові hardening-факти з transcript: детермінований `isNoDecision` gate для `delete` без LLM та консервативний edge-judge із 3 голосами й confidence ≥ 0.6.
- Зафіксований trade-off: локальний серійний прогін мав wall-clock близько 17 хв на 37 `omlx`-викликів, але зберіг приватність і не потребував cloud-токенів.
