# Заміна синхронного Stop hook на PostToolUse з інкрементальною маршрутизацією

**Status:** Accepted
**Date:** 2026-05-25

## Context and Problem Statement

Синхронний `Stop` hook запускав повний `n-fix` (15–20 правил послідовно) на кожному turn-і. Кумулятивна затримка перевищувала timeout 60s на великих репо і вбивала хук навіть при коректному коді. Потрібен підхід, що прогоняє лише правила, релевантні для зміненого файлу.

## Considered Options

- Лишити Stop-hook, підняти timeout до 600s
- `PostToolUse` з грубим фільтром по типу інструмента, але повний `fix`
- `PostToolUse` з точковою маршрутизацією за `tool_input.file_path` через routing-таблицю picomatch
- `PostToolUse` + Stop-hook як backstop із timeout 600s

## Decision Outcome

Chosen option: "PostToolUse з точковою маршрутизацією без Stop-hook backstop", because user явно обрав цей варіант; після правки одного `.mjs` крутиться лише `js-lint`, а не 18 правил; Stop-hook видалено повністю, дисципліна довіряється `PostToolUse` + ручному `/n-fix` перед PR.

### Consequences

- Good, because після правки одного `.mjs` запускається лише `js-lint`, а не 18 правил.
- Good, because legacy cleanup у `sync-claude-config.mjs` автоматично прибирає старі Stop-entries зі `.claude/settings.json` консьюмерів.
- Bad, because якщо агент модифікував файли через `Bash`-команди (а не `Edit/Write`), `PostToolUse` не спрацьовує і порушення не буде виявлено автоматично.

## More Information

- `npm/scripts/post-tool-use-fix.mjs` — `routeFilePathToRules(filePath)` + `runPostToolUseFixCli`; routing через `picomatch`
- Routing-таблиця: `*.{mjs,js,cjs,ts,tsx,jsx}` → `js-lint`; `*.vue` → `js-lint,style-lint,vue`; `*.{css,scss,sass}` → `style-lint`; `**/k8s/**/*.{yaml,yml}` → `k8s`; `*.rego` → `rego`; `Dockerfile*` → `docker`; `.github/workflows/*.{yml,yaml}` → `ga`; `**/*.md` (поза `docs/adr/`) → `text`; `**/package.json` → `npm-module,bun`
- `npm/.claude-template/settings.template.json` — `hooks.PostToolUse`, matcher `Edit|Write|MultiEdit`, timeout 300
- `npm/scripts/sync-claude-config.mjs` — `MANAGED_HOOK_COMMAND_MARKER = '@nitra/cursor post-tool-use-fix'` + `LEGACY_STOP_HOOK_COMMAND_MARKER`
- `adr` правило свідомо виключено з маршрутизації (залишено в async `normalize-decisions.sh`)
- Тести router: 21 test у `post-tool-use-fix.test.mjs`, всі pass
- CLI subcommand `stop-hook` видалено відразу без deprecation-периоду; консьюмери, що явно викликали його, отримають `unknown command`
- Версія пакета: `1.20.0 → 1.21.0` (minor, BREAKING)
