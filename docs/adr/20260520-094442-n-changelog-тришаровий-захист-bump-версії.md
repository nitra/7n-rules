# Тришаровий захист від пропуску bump версії після змін у `npm/`

**Status:** Accepted
**Date:** 2026-05-20

## Context and Problem Statement

Після редагування `npm/skills/lint/SKILL.md` асистент завершував відповідь без підвищення `version` у `npm/package.json` і без нового запису в `npm/CHANGELOG.md`, попри правило `n-changelog.mdc` з `alwaysApply: true`. Одна секція «чеклист» наприкінці довгого тексту правила виявилася ненадійною: увага LLM після `npx @nitra/cursor` переключалася на завершення задачі замість закриття релізу пакета.

## Considered Options

* Оновити лише чеклист у `changelog.mdc` (статус-кво — показав себе ненадійним)
* STOP-блок на початку `changelog.mdc` + перехресне нагадування в `scripts.mdc` + pre-commit хук `hk.pkl`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "STOP-блок на початку `changelog.mdc` + перехресне нагадування в `scripts.mdc` + pre-commit хук `hk.pkl`", because один абзац «перед завершенням задачі» легко губиться в контексті LLM; три незалежні шари (STOP на початку правила, cross-STOP у пов'язаному правилі, програмний стоп-кран при commit) спрацьовують незалежно один від одного.

### Consequences

* Good, because STOP на початку `changelog.mdc` перехоплює LLM раніше в контексті — до завершення задачі.
* Good, because `scripts.mdc` підсилює нагадування саме тоді, коли агент редагує файли у `npm/rules/`, `npm/skills/`, `npm/scripts/`.
* Good, because pre-commit крок у `hk.pkl` не дає зробити commit без bump незалежно від поведінки LLM.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

* `npm/rules/changelog/changelog.mdc` оновлено до версії `2.5`: STOP-блок додано на початку тексту.
* `.cursor/rules/scripts.mdc`: доданий перехресний STOP після правок у `npm/rules/`, `npm/skills/`, `npm/scripts/`.
* `hk.pkl`: доданий pre-commit крок `check changelog` при змінах у `npm/**`.
* Bump: `npm/package.json` `1.13.62` → `1.13.63`; запис у `npm/CHANGELOG.md` `## [1.13.63] - 2026-05-20`.
* Верифікація після правок: `bun ./npm/bin/n-cursor.js check changelog`.
