## План: scope `coverage` gate тільки на змінені файли

### Принцип

`base_commit` з `.flow.json` → CLI-флаг `--base` → провайдер звужує vitest + Stryker до змінених файлів. Покриває обидва сценарії: закомічені кроки (autonomous) і брудне дерево (verify вручну).

### Крок 1 — `npm/scripts/lib/changed-files.mjs`
Додати `collectChangedFilesSince(base, cwd)`:
- `git diff ${base}..HEAD --name-only` — закомічені зміни відносно base
- `git diff HEAD --name-only` — uncommitted working-tree
- `git ls-files --others --exclude-standard` — нові untracked
- Union усіх трьох → дедупліковано
- Якщо `base` null/undefined — fallback до `collectChangedFiles(cwd)` (поточна поведінка)

### Крок 2 — `npm/bin/n-cursor.js` (~1528)
Парсити `--base <sha>` у `case 'coverage':` → пробрасувати у `runCoverageCli({ fix, base })`.

### Крок 3 — `npm/rules/test/coverage/coverage.mjs`
- `runCoverageCli` і `runCoverageSteps` приймають `base?`
- Якщо `base` є: обчислити `changedFiles = collectChangedFilesSince(base, cwd)`
- Якщо `changedFiles.length === 0`: log + `return 0` (pass, не fail — нічого не змінилося)
- Передавати `changedFiles` у `provider.collect(cwd, { changedFiles })`

### Крок 4 — `npm/rules/js-lint/coverage/coverage.mjs`

- **Stryker**: `--mutate <files>` — перекриває `mutate` з конфіга. Якщо після фільтрації source-files немає → skip Stryker (return `{caught:0, total:0, survived:[]}`)
