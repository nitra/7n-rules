# CI4 ↔ ADR — централізований cross-ref

Таблиця посилань між елементами CI4-моделі і ADR-чернетками з [`docs/adr/_inbox/`](../adr/) (а в майбутньому — кураторськими ADR з `docs/adr/`). Доповнює inline-секції `## Related decisions` у файлах рівнів — використовуйте цей файл як індекс для глобального пошуку.

## Як читати таблицю

- **Element ID** — стабільний якорь з префіксом `ctx-/cnt-/cmp-/code-` (див. [`README.md`](README.md#конвенція-якорів)).
- **Level** — рівень CI4, де елемент описаний.
- **ADR** — повний шлях до ADR-чернетки.
- **Дата** — дата ADR (з frontmatter `captured` або з імені файлу).
- **Резюме** — 1-2 речення про те, що ADR говорить про цей елемент.

## Як додавати запис при кураторстві ADR

1. Перенести ADR з `docs/adr/_inbox/` у `docs/adr/<номер>-<slug>.md` (за чинною конвенцією — формат фінальних ADR ще не зафіксований; використовуйте найкраще доступне на час кураторства).
2. У тілі ADR додати рядок:

   ```markdown
   Related CI4: [<element-name>](../ci4/0X-<level>.md#<anchor-id>)
   ```

3. У цьому файлі (`decisions.md`) додати рядок у відповідну таблицю.
4. У `## Related decisions` секцію файлу рівня (`01..04-*.md`) додати посилання, якщо ADR суттєво стосується конкретного елемента.

Дублювання навмисне — пункт 3 є індексом для пошуку, пункт 4 — контекстом при читанні рівня.
