## Огляд

`DEFAULT_CACHE_DIR` задає стандартне місце для кешу нотаток про міграції, а `migrationCacheKey` — стабільний ключ для пошуку запису в межах одного прогону. `readMigrationCache` і `writeMigrationCache` працюють із цим кешем fail-safe: помилки перехоплюються, назовні не кидаються, а за частини збоїв повертається порожнє значення на кшталт `null`. `withKnownMigrationNotes` додає вже відомі нотатки про міграцію до поточного контексту, щоб не дублювати повторне визначення однакових даних у межах того самого запуску.

## Поведінка

- `DEFAULT_CACHE_DIR` — спільний каталог кешу міграцій на цій машині, щоб повторні оновлення одного й того ж пакета між репо не починали дослідження з нуля.
- `migrationCacheKey` — формує стабільний безпечний ключ кешу для конкретної пари пакет/діапазон версій.
- `readMigrationCache` — читає кешований запис міграції для пари версій; за відсутнього або пошкодженого кешу мовчки повертає `null`.
- `writeMigrationCache` — зберігає результат уже проаналізованої міграції в кеш для подальших прогонів на цій машині.
- `withKnownMigrationNotes` — додає до промпта підсумок уже відомої міграції з кешу, щоб пропустити повторне дослідження і перейти до перевірки використання в поточному проєкті.

## Публічний API

- DEFAULT_CACHE_DIR — спільний каталог кешу міграцій для всієї машини, щоб різні репозиторії не дублювали однакові LLM-перевірки версій.
- migrationCacheKey — перетворює `pkg`, `from` і `to` на стабільне безпечне імʼя файла для спільного кешу між репозиторіями.
- readMigrationCache — повертає збережений результат міграції для тієї самої пари версій, якщо він уже є на цій машині; пошкоджений або відсутній запис дає `null`.
- writeMigrationCache — записує результат уже виконаного LLM-дослідження міграції, щоб наступні репозиторії з тим самим оновленням могли його повторно використати.
- withKnownMigrationNotes — додає до prompt короткий підсумок відомої міграції з кешу й переводить runner одразу до пошуку використання в поточному проєкті.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
- Кешує результати в межах одного прогону.
