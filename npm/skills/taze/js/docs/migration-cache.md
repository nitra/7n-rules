## Огляд

`DEFAULT_CACHE_DIR`, `migrationCacheKey`, `readMigrationCache`, `writeMigrationCache`, `withKnownMigrationNotes` працюють з кешем міграцій у межах одного прогону. Функції поводяться fail-safe: вони не кидають винятків назовні, а за окремих помилок повертають порожнє значення, наприклад `null`, щоб не зупиняти обробку.

## Поведінка

- `DEFAULT_CACHE_DIR` — задає спільний каталог кешу міграцій для цієї машини, щоб повторно використовувати вже проаналізовані оновлення між різними repo/worktree.
- `migrationCacheKey` — перетворює пару версій пакета на стабільний ключ для файла кешу, придатний для повторного використання між різними репозиторіями.
- `readMigrationCache` — читає кеш міграції для конкретного оновлення і, якщо запису немає або він пошкоджений, безпомилково повертає `null`.
- `writeMigrationCache` — зберігає результат аналізу міграції в кеш, щоб наступні запуски на цій машині не повторювали той самий пошук з нуля.
- `withKnownMigrationNotes` — додає до промпта підсумок уже відомої міграції з кешу, щоб одразу перейти до перевірки застосовності в поточному проєкті.

## Публічний API

- DEFAULT_CACHE_DIR — спільний каталог кешу міграцій для всієї машини, щоб повторно не досліджувати ті самі версійні переходи в різних репо.
- migrationCacheKey — перетворює пару версій і пакет у стабільну безпечну назву файлу для спільного кешу між worktree й репо.
- readMigrationCache — підбирає готовий запис про міграцію для тієї самої пари версій, а зіпсований або відсутній кеш тихо ігнорує.
- writeMigrationCache — зберігає результат уже виконаного LLM-дослідження, щоб наступні запуски з тим самим bump-ом не починали з нуля.
- withKnownMigrationNotes — додає до prompt підсумок уже відомої міграції й пропускає стартове дослідження changelog/diff, коли кеш дає готову підказку.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
- Кешує результати в межах одного прогону.
