## Огляд

`package.json`

Файл читає версії з `Cargo.toml`, виділяє специфікації залежностей, порівнює дві версії маніфесту та збирає diff по змінах версій. Для монорепо проходить по списку `Cargo.toml` і агрегує результати для всіх знайдених маніфестів. Усі помилки перехоплює fail-safe: не кидає винятків назовні й за окремих збоїв повертає порожнє значення замість помилки.

## Поведінка

- parseCargoVersion — виділяє версійну частину Cargo-специфікатора й повертає її як нормалізоване числове ядро; для не-версійних значень повертає null.
- extractCargoVersionSpec — дістає версійний specifier із запису залежності Cargo.toml; для path/git-залежностей без версії повертає null.
- diffCargoToml — порівнює два розпарсені Cargo.toml і визначає зміни залежностей, розділяючи їх на major та minor/patch.
- collectCargoDiff — проходить по списку Cargo.toml у монорепо, зіставляє кожен файл із його backup-файлом і збирає агрегований diff; якщо один із файлів відсутній або невалідний, пропускає його без падіння.

## Публічний API

- Конфіги, на які спирається код: `package.json` — джерело списку залежностей і меж для порівняння змін.
- `parseCargoVersion` — розбирає Cargo-версійний specifier і заповнює відсутні частини нулями.
- `extractCargoVersionSpec` — дістає версію залежності з запису `Cargo.toml`, як з короткого рядка, так і з inline-таблиці.
- `diffCargoToml` — порівнює два розібрані `Cargo.toml` і повертає зміни по залежностях.
- `collectCargoDiff` — збирає diff для всіх `Cargo.toml` у монорепо, зіставляючи кожен маніфест із його backup-версією.

Changelog: не перевіряв — змін у файлах не вносив.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
