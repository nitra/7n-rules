## Огляд

Модуль запускає taze-орchestrator, підключає доступних provider-ів, формує звіт і координує один прогін у межах поточного запуску. Його роль — дати єдину точку входу для запуску, де `callRunner`, `loadPluginTazeProviders`, `formatReport` і `runTazeOrchestrator` працюють узгоджено, а результати можна повторно використовувати в межах прогону. Обробка помилок fail-safe: винятки не виходять назовні, а стан прогону кешується на час виконання.

## Поведінка

runTazeOrchestrator запускає спільний цикл для всіх доступних екосистем, збирає провайдерів через loadPluginTazeProviders або бере їх із інжектів, далі для кожної екосистеми зберігає проміжний стан, виконує оновлення пакетів, ізолює виклики для кожного major-стрибка через callRunner і не дає збою одного пакета зупинити решту прогону. Дані для роботи беруться з конфігурації проєкту та сигналів репозиторію, зокрема .n-rules.json і package.json, а результатом стає структурований звіт разом зі зведенням по екосистемах. У межах одного запуску зберігається кеш, щоб повторно використовувати вже підняті дані та не дублювати одні й ті самі перевірки.

loadPluginTazeProviders підтягує плагінні провайдери екосистем, відсіює биті або невалідні модулі й повертає лише ті, що придатні до роботи, не валячи весь прогін через проблему в одному плагіні. Цей список стає джерелом для runTazeOrchestrator, тож плагіни визначають, які екосистеми взагалі потрапляють у цикл обробки.

callRunner виконує один ізольований прогін обраного runner-а для окремого пакета або пакета-міграції, повертаючи не лише успіх, а й текстовий результат або помилку для включення в підсумок. Його вихід напряму живить проміжні кроки runTazeOrchestrator і визначає, чи запис піде в успішні зміни, чи в fail-safe-обробку без падіння всього процесу.

formatReport збирає фінальний markdown-звіт із накопичених результатів усіх екосистем, зберігаючи детермінований порядок і однакове представлення для успішних, пропущених і проблемних записів. Цей звіт стає фінальним артефактом runTazeOrchestrator і не залежить від окремого LLM-виклику для формування самої форми підсумку.

## Публічний API

- callRunner — Диспетчер одного ітеративного виклику на обраний раннер. `pi` — вбудований
pi-агент (`@7n/llm-lib/agent-skill`; текст перехоплюється через `deps.out`,
бо `runAgentSkill` не повертає його напряму — лише стрімить у stdout).
`cursor`/`codex` — napi-міст ACP (`@7n/llm-lib/acp`; текст — прямий return,
idle-timeout і видимість прогресу вже вбудовані в сам міст). Тір `avg`
(рішення И/Ф3, задача T7) — паритет з pi-гілкою: `runAcpAgent` резолвить
tier→env/args/post-session-config у Rust, а не бере модель з персонального
CLI-конфіга на машині.
- loadPluginTazeProviders — Завантажує EcosystemProvider-и з активних плагінів проєкту: `.n-rules.json`
(або автодетект за файловими сигналами — `pyproject.toml` → lang-python) →
`resolvePlugins` (плагін доставляється автоматично при першому запуску) → handler-модулі
extension-point `taze` → default-експорт кожного валідується
`assertEcosystemProvider`. Битий плагін (нема модуля/невалідна форма) —
warning і пропуск, не провал прогону.
- formatReport — Компонує підсумковий звіт (крок 8 SKILL.md) детерміновано з результатів
ітерацій — без окремого LLM-виклику для самого звіту. Усі екосистеми
(включно з npm/bun — плагін `@7n/rules-lang-js`, фаза 5a) — рівноправні
секції; екосистема без manifests — тиша.
- runTazeOrchestrator — Оркеструє taze: чистий цикл по EcosystemProvider-ах (кроки 1-3/7/8 —
детерміновано в провайдері, кроки 4-6 — по одному ізольованому виклику
`callRunner` на кожен major-запис) замість одного величезного непрозорого
ходу на весь монорепо. Ядро — двигун без мовної специфіки (фаза 5a spec):
ВСІ екосистеми, включно з npm/bun (`@7n/rules-lang-js`), приходять з
плагінів (extension-point `taze`). Падіння одного пакета/однієї екосистеми
не втрачає прогрес по інших.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- Кешує результати в межах одного прогону.
