## Огляд

Цей файл виконує детерміноване створення change-файлів для воркспейсів, у яких виявлено релевантні зміни, але відсутній відповідний change-файл. Він автоматично зчитує опис змін з останнього коміту Git для кожного проблемного воркспейсу та генерує change-файл із рівнем збільшення `patch` в розділі `Changed`. Це реалізує T0-autofix для забезпечення узгодженості.

## Поведінка

Поведінка:

1. Аналізує вихідний текст на наявність маркера, що вказує на наявність релевантних змін без відповідного change-файлу.
2. Визначає всі воркспейси, які мають цю проблему, використовуючи `patterns`.
3. Для кожного виявленого воркспейсу автоматично отримує опис змін на основі сутності останнього коміту Git.
4. Створює change-файл для кожного воркспейсу, встановлюючи рівень збільшення (bump) як `patch` та розділ як `Changed`.
5. Помилки під час створення change-файлу обробляються локально, і процес не генерує винятків.
6. `patterns` містить визначення для виконання описаної логіки.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
