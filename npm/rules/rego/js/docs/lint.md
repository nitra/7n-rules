## Огляд

Файл забезпечує послідовний лінт Rego-полісі, розташованих у `npm/rules/`, використовуючи інструменти `opa check --strict`, `regal lint` та, за наявності, `conftest verify`. Ця послідовність перевіряє компіляцію полісів з типами за допомогою `opa check --strict`, що виявляє помилки, такі як незадекларовані змінні чи неоднозначні правила, які можуть бути проігноровані `regal`. Далі виконується статичний лінт Rego за допомогою `regal lint` (https://docs.styra.com/regal), що перевіряє відповідність Rego-полісів стандарту `rego.v1` та ідіоматичність. Опційно, якщо `conftest` доступний у PATH (https://www.conftest.dev/install/), виконуються юніт-тести політик за допомогою `conftest verify`. Усі інструменти рекурсивно знаходять файли `.rego` у цільових директоріях, ігноруючи інші розширення, і їхня робота забезпечується механізмом `ensureTool` для гарантованого доступу до інструментів. Процес лінту базується на конфігурації `target.json`.

## Поведінка

runLintRegoSteps виконує послідовний лінт Rego-полісі, використовуючи `opa check --strict`, `regal lint` та опціонально `conftest verify` для цілей у `npm/rules/`.
runLintRego серіалізує запуск лінту Rego-полісі через механізм дедуплікації.
lint делегує виконання лінту Rego-полісі.

## Публічний API

runLintRegoSteps — Виконує перевірку Rego (`opa check --strict`) та лінтинг (`regal lint`) для визначених цілей. Зупиняє виконання з помилкою, якщо попередні перевірки не пройшли.
runLintRego — Виконує лінтинг Rego у публічному режимі, забезпечуючи унікальність результатів на основі стану Git-дерева.
lint — Передає команду на лінтинг Rego до функції `runLintRego`.
