## Огляд

Виконує `conftest verify` (read-only detector) для верифікації Rego-правил. Механізм контролює відповідність правил на рівні всього каталогу `npm/rules`, оскільки це повний запуск, який не входить до delta-плану і спрацьовує лише через `n-rules lint --full` або цільовий `n-rules lint rego`.

## Поведінка

1. Визначається директорія, яка є коренем проєкту.
2. Перевіряється наявність цільових папок, які відповідають `npm/rules`.
3. Якщо цільові папки знайдені та інструмент `conftest` доступний, виконується команда `conftest verify` для усіх цільових папок.
4. Якщо `conftest verify` повертає ненульовий код, реєструється вимога на виправлення з деталями помилки.
5. Якщо команда успішна або папки не знайдено або `conftest` відсутній, повертається фінальний звіт про перевірку.

## Публічний API

lint — перевіряє на відповідність статичних аналізів (rego/conftest\_verify).
