## Огляд

Модуль перевіряє задане правило у бібліотечному режимі, застосовуючи логіку від `applies` до `mdc-refs`. При запуску як окрема програма, він завантажує конфігурацію та виконує вибіркове завантаження (вайтлістинг). Результат перевірки (0 або 1) визначає код виходу процесу.

## Поведінка

1. Викликається функція `run` для запуску перевірки правила.
2. Перевірка виконується у бібліотечному режимі, застосовуючи логіку: `applies` $\rightarrow$ `JS-concerns` $\rightarrow$ `policy` $\rightarrow$ `mdc-refs`.
3. Якщо скрипт виконується як окрема програма (standalone), запускається повна оркестрація правила, що включає завантаження конфігурації, вайтлістинг та підсумок.
4. Результат виконання (0 для успіху, 1 для порушення) встановлюється як код виходу процесу.

## Публічний API

run — виконує послідовність дій: застосовує правило, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
