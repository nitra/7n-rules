Модуль виконує визначене правило, перевіряючи його застосовність та застосовуючи політику до посилань MDC. При запуску як окрема утиліта, він виконує повний цикл, включаючи завантаження конфігурації.

## Поведінка

1. Викликається функція `run` для запуску правила.
2. Правило виконує перевірку, застосовуючи логіку: `applies` $\rightarrow$ `JS-concerns` $\rightarrow$ `policy` $\rightarrow$ `mdc-refs`.
3. Якщо скрипт виконується як окрема утиліта (standalone), запускається повний цикл виконання правила, що включає завантаження конфігурації, вайтлістинг та підсумок.

## Публічний API

run — виконує послідовність дій: застосовує правила, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
