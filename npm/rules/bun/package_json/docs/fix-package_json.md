## Огляд

Файл `T0-autofix` забезпечує відповідність політикам `bun/package_json`, видаляючи з `package.json` верхнерівневі поля, заборонені конфігурацією в `template/package.json.deny.json`, а також скрипти, що відповідають шаблонам `'lint'` або `'lint-*'`, відповідно до вимог `bun.mdc`.

## Поведінка

Поведінка:
1. Ініціалізація перевірки шляхом аналізу змін, що викликали попередження з причиною 'policy-deny' та містять файл.
2. Зчитування конфігурації заборонених полів з `package.json.deny.json` в каталозі template.
3. Ітерація по унікальних файлах, що потребують виправлення.
4. Для кожного файлу:
    а. Зчитування вмісту файлу `package.json`.
    б. Видалення з об'єкта `package.json` всіх верхнерівневих полів, які зазначені в конфігурації заборонених полів.
    в. Видалення всіх записів у об'єкті `scripts`, імена яких починаються з 'lint' (включаючи `scripts.lint` та `scripts.lint-*`).
    г. Запис модифікованого об'єкта назад у файл `package.json` із відступом у 2 пробіли.
    ґ. Фіксація модифікації файлу.
5. Повернення списку змінених файлів та повідомлення, що описує видалені ключі.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
