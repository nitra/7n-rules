Цей модуль виконує визначене правило, запускаючи перевірку на основі даних, кешованих у межах поточного прогону. Публічна функція `run` ініціює повний цикл виконання, що включає завантаження конфігурації та формування підсумку.

## Поведінка

1. Викликається функція `run` для запуску перевірки правила.
2. Перевірка виконується, використовуючи контекст прогону, який може містити кешовані дані.
3. Якщо скрипт запускається як окрема утиліта командного рядка, виконується повний цикл запуску правила, включаючи завантаження конфігурації та підсумок.

## Публічний API

run — виконує послідовність кроків: застосування правил, обробку JS-зацікавлень, перевірку політик та посилання на MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
