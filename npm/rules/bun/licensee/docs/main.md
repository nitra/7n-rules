## Огляд

Встановлює конфігурацію ліцензій, спираючись на файл .licensee.json, якщо він відсутній. Виконує перевірку відповідності ліцензій усіх npm-залежностей проєкту встановленим вимогам, що визначаються в конфігурації. У випадку виявлення порушень, система повідомляє про них, підтверджуючи чисту відповідність або фіксуючи розбіжності. Маркери повідомлень згадуються у Поведінці: (bun.mdc).

## Поведінка

Поведінка:

1. Викликається `lint` для запуску перевірки ліцензій npm-залежностей.
2. Якщо файл конфігурації `.licensee.json` відсутній:
   a. Якщо режим читання лише (`readOnly`) увімкнено, повідомляється, що необхідно запустити `npx @7n/rules fix bun` локально для генерації (bun.mdc).
   b. Якщо режим читання лише вимкнено, створюється `.licensee.json` з дефолтним allowlist (`blueOak: bronze`) та повідомляється про створення конфігурації.
3. Визначається команда `bun`. Якщо `bun` не знайдена у системному шляху, повідомляється про це (bun.mdc).
4. Виконується команда `bun x licensee --production --quiet` у корені проєкту.
5. Якщо команда `licensee` завершується успішно (статус 0), повідомляється, що ліцензії відповідають вимогам.
6. Якщо команда `licensee` завершується з помилкою (не 0), повідомляється про порушення ліцензій (код) (bun.mdc).

## Публічний API

lint — аналізує код на відповідність заданим стилістичним нормам у контексті `bun`.
licensee — перевіряє відповідність ліцензій усіх використаних npm-залежностей файлу `.licensee.json`.
