Модуль реалізує логіку для виконання перевірок. При запуску окремо він забезпечує повний цикл перевірки, а при виклику як бібліотека виконує лише виконання правила. Результат роботи визначається за допомогою публічної функції `run`, яка повертає код, що вказує на відсутність (0) або наявність (1) виявлених порушень.

## Поведінка

1. Якщо скрипт виконується як окрема програма, він виконує повний цикл перевірки, включаючи завантаження конфігурацій, списки дозволених елементів та підсумки.
2. Якщо скрипт викликається як частина бібліотеки, він виконує функцію `run` для запуску правила.
3. Функція `run` запускає стандартне правило, використовуючи контекст прогону, що може включати кеш.
4. Функція `run` повертає код: 0, якщо порушень немає, або 1, якщо порушення виявлено.

## Публічний API

run — Запускає послідовність: застосування правил, перевірку JS-питань, відповідність політиці та посилання на MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
