Модуль виконує перевірки відповідно до визначених правил. При запуску як окремий інструмент, він завантажує конфігурацію та білий список. Під час виконання перевірок у контексті прогону, модуль використовує кешування в межах цього прогону. При запуску як окремий інструмент, він надає підсумок та повертає результат у вигляді коду виходу.

## Поведінка

1. Викликається функція `run` для запуску перевірки правила.
2. Перевірка виконується у режимі бібліотеки, використовуючи контекст прогону.
3. Якщо скрипт запускається як окремий інструмент командного рядка, виконується повний цикл запуску правила, включаючи завантаження конфігурації, білий список та підсумок.
4. Результат виконання повертається як код виходу процесу.

## Публічний API

run — виконує послідовність дій: застосовує правила, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
