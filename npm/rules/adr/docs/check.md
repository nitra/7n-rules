Модуль виконує логіку перевірки, ініційовану через публічну функцію `run`. У контексті прогону він запускає перевірку, використовуючи кешовані дані. При виклику як окрема утиліта, він виконує повну оркестрацію, завантажуючи конфігурацію та список дозволених елементів. Результат перевірки визначає код виходу процесу.

## Поведінка

1. Викликається функція `run` для запуску перевірки правила.
2. Перевірка виконується у режимі бібліотеки, використовуючи контекст прогону.
3. Якщо скрипт виконується як окрема утиліта командного рядка, запускається повна оркестрація правила, що включає завантаження конфігурації, список дозволених елементів та підсумок.
4. Результат виконання (0 для успіху, 1 для порушення) встановлюється як код виходу процесу.

## Публічний API

run — виконує послідовність дій: застосовує правило, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
