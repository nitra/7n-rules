## Огляд

Публічна функція `lint` перевіряє код на наявність потенційно небезпечних витоків даних, використовуючи `trufflehog` з ігноруванням файлів, зазначених у `.trufflehog-exclude`. Функція ідентифікує як підтверджені, так і невідомі секрети. Успішне виконання (код 0) означає відсутність порушень; код помилки повідомляє про виявлені викриття.

## Поведінка

1. Викликається `lint` для виконання сканування всієї директорії репозиторію на наявність секретних даних.
2. Сканування здійснюється за допомогою інструменту `trufflehog` із виключенням файлів, зазначених у `.trufflehog-exclude`.
3. Виконуються перевірки секретів, які були підтверджені або невідомі.
4. Процес завершується з кодом, який вказує на відсутність секретів (0) або наявність порушень (інший код).

## Публічний API

lint — проводить перевірку якості коду на поверхні.
security: trufflehog — сканує весь репозиторій на наявність секретів.
filesystem — здійснює сканування всієї файлової системи репозиторію.
