## Огляд

Управляє життєвим циклом перевірки. Ініціює валідацію у режимі бібліотеки, використовуючи контекст поточного прогону, що визначає застосування кешування. Якщо код виконується як основна програма, він ініціює повний цикл перевірки, включаючи завантаження конфігурації та відображення підсумків, викликаючи публічну функцію `run`.

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується у режимі бібліотеки, використовуючи контекст прогону.
3. Якщо скрипт виконується як окрема програма, запускається повний процес перевірки, включаючи завантаження конфігурації та підсумки.

## Публічний API

run — виконує послідовність дій: застосовує правило, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
