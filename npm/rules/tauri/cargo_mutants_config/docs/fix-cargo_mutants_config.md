## Огляд

Файл відповідає за детерміноване формування або доповнення файлів `.cargo/mutants.toml` у директорії `src-tauri`. Він ініціалізує або канонізує Tauri-ключі в цих файлах, забезпечуючи коректність конфігурації. Логіка формування була перенесена сюди з попереднього detector-а, який лише реєстрував дефекти (`mutants-config-missing` / `mutants-keys-missing`). Механізм реалізовано через єдину поверхню лінту (`structured violations (test/apply)`). При роботі з цільовими каталогами здійснюється повторне сканування `src-tauri` з `ctx.cwd`, а статус кожного файлу перевіряється для забезпечення ідемпотентності (вже завершені стани ігноруються).

## Поведінка

1. `patterns` визначає логіку автоматичного виправлення конфігурацій `tauri/cargo_mutants_config`.
2. Якщо виявлено відсутність конфігурації або необхідних ключів у файлах `<ws>/src-tauri/.cargo/mutants.toml`, ініціюється процес виправлення.
3. Система сканує каталоги `src-tauri` з поточного робочого каталогу.
4. Для кожного знайденого каталогу перевіряється наявність цільового файлу `.cargo/mutants.toml`.
5. Якщо файл відсутній, він створюється з базовою конфігурацією.
6. Якщо файл існує, перевіряються відсутні ключі у конфігурації.
7. Якщо ключі відсутні, файл доповнюється за допомогою канонічних Tauri-ключів.
8. Результати змін у файлах фіксуються у відповіді системи.
