## Огляд

Модуль виконує валідацію. При запуску як окрема утиліта він здійснює повний цикл, включаючи завантаження конфігурації та підсумок. При імпорті він виконує послідовну перевірку, використовуючи наданий контекст.

## Поведінка

1. Якщо скрипт виконується як окрема утиліта командного рядка, він виконує повний цикл перевірки, включаючи завантаження конфігурації та підсумок, і завершує роботу з відповідним кодом виходу.
2. Якщо скрипт імпортується в інший процес, викликається функція `run` для запуску перевірки. Ця функція виконує перевірку, застосовуючи логіку: `applies` $\rightarrow$ `JS-concerns` $\rightarrow$ `policy` $\rightarrow$ `mdc-refs`, використовуючи наданий контекст прогону.

## Публічний API

run — виконує послідовність етапів: застосування правил, обробку JS-задач, перевірку політик та посилання на MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
