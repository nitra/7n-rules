## Огляд

Перевіряє відповідність конфігурації `lint-docker.yml` встановленим політикам, використовуючи публічну функцію `lint`. Цей процес перевіряє, чи відповідає конфігурація Docker вимогам проекту. Під час аналізу ігноруються каталоги `.github` та `.git`. Функція є read-only: не вносить жодних змін до файлової системи чи бази даних. Повертається список знайдених порушень.

## Поведінка

1. Ініціює перевірку конфігурації файлу `lint-docker.yml` для визначення відповідності політичним вимогам.
2. У процесі перевірки ігнорує директорії `.github` та `.git`.
3. Повертає результат перевірки, який містить список знайдених порушень.

## Публічний API

lint — Виявляє, чи відповідає код політиці виявлення проблем (політиці-концерну).

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Свідомо пропускає шляхи: `.github`, `.git`.
