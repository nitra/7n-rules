Модуль керує послідовністю виконання перевірок: застосування логіки $\rightarrow$ перевірку JS-зацікавленостей $\rightarrow$ політику $\rightarrow$ посилання MDC. При запуску як окрема програма модуль завантажує конфігурацію, застосовує вайтлістинг та виводить детальний звіт про результати перевірки.

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується, застосовуючи логіку: `applies` $\rightarrow$ `JS-concerns` $\rightarrow$ `policy` $\rightarrow$ `mdc-refs`.
3. Якщо скрипт виконується як окрема програма, запускається повна оркестрація перевірки, включаючи завантаження конфігурації, вайтлістинг та підсумок.

## Публічний API

run — виконує послідовність дій: застосовує правила, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
