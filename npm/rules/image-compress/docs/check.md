Модуль виконує визначене правило, використовуючи контекст прогону з кешуванням результатів. При запуску як інструмент командного рядка, він завантажує конфігурацію, перевіряє білий список та надає підсумок. Модуль є лише для читання (не змінює файлову систему чи бази даних).

## Поведінка

1. Викликається функція `run` для запуску правила.
2. Правило виконується, використовуючи контекст прогону, що включає кешування.
3. Якщо скрипт запускається як окремий інструмент командного рядка, виконується повний цикл запуску правила, включаючи завантаження конфігурації, перевірку білого списку та підсумок.

## Публічний API

run — виконує послідовність дій: застосовує правило, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
