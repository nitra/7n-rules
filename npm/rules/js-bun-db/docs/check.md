Модуль виконує правила у бібліотечному режимі, використовуючи контекст поточного прогону. Він запускає повний цикл правил, включаючи завантаження конфігурації та фільтрацію за білими списками, якщо скрипт викликається як окремий інструмент. Результат виконання повертається як код виходу процесу.

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується у бібліотечному режимі, використовуючи контекст прогону.
3. Якщо скрипт запущено як окремий інструмент командного рядка, виконується повний цикл запуску правила.
4. Повний цикл включає завантаження конфігурації, застосування білих списків та підбиття підсумків.
5. Результат виконання повертається як код виходу процесу.

## Публічний API

run — виконує послідовність перевірок: застосовує правила, аналізує JavaScript-залежності, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
