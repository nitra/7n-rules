Модуль валідує JS-контекст відповідно до політики `mdc-refs`. При запуску як окрема програма він завантажує конфігурацію, застосовує білий список та генерує звіт про валідацію.

## Поведінка

1. Викликається функція `run` для запуску правила. Це виконує перевірку, застосовуючи політику `mdc-refs` до JS-контексту.
2. Якщо скрипт виконується як окрема програма (standalone), запускається оркестрація правила. Це включає завантаження конфігурації, застосування білого списку та підсумок.

## Публічний API

run — виконує послідовність дій: застосовує правило, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
