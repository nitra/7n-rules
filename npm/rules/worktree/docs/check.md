## Огляд

Модуль виконує визначене правило. Він запускає правило, використовуючи контекст прогону, який може містити дані, кешовані у межах цього прогону. При запуску як окрема утиліта, він ініціалізує виконання, завантажує конфігурацію та перевіряє білий список.

## Поведінка

1. Викликається функція `run` для запуску правила.
2. Правило виконується, використовуючи контекст прогону, що може містити кеш.
3. Якщо скрипт запускається як окрема утиліта командного рядка, виконується повний цикл запуску правила, включаючи завантаження конфігурації, перевірку білого списку та підсумок.

## Публічний API

run — виконує послідовність кроків: застосування правил, перевірку JS-зацікавленостей, відповідність політиці та посилання на MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
