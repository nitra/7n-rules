## Огляд

Цей файл перевіряє на відповідність внутрішнім URL-адресам конфігураційні файли середовища (такі як `dev.env` або `ua.env`) у репозиторії. Функція `lint` реєструє повідомлення про успішну валідацію або виявлені порушення, використовуючи механізм перехоплення помилок (`fail-safe`) замість викидання винятків. Поведінка відповідає вимогам маркерів повідомлень, зазначених у (abie.mdc).

## Поведінка

Поведінка
1. Швидкість аналізу: функція `lint` аналізує файли конфігурації середовища (`dev.env`, `ua.env`), шукаючи порушення внутрішніх URL.
2. Пошук файлів: сканує репозиторій для пошуку файлів конфігурації середовища, ігноруючи шляхи, визначені конфігурацією Cursor.
3. Обробка відсутності файлів: якщо файли конфігурації середовища не знайдено, перевірка DNS-адрес середовища пропускається, і реєструється повідомлення (abie.mdc).
4. Зчитування конфігурацій: для кожного знайденого файлу намагається прочитати його вміст. У разі невдачі читання реєструється помилка, і обробка цього файлу припиняється.
5. Валідація URL: вміст файлу валідується для перевірки відповідності внутрішніх URL визначенню середовища, виходячи з імені файлу.
6. Реєстрація результатів: якщо жодних помилок не виявлено у файлі, реєструється повідомлення про успішну відповідність (abie.mdc). У разі виявлення порушень, для кожного порушення реєструється помилка (abie.mdc).
7. Обробка винятків: функція не кидає винятків назовні; усі внутрішні помилки перехоплюються та реєструються як помилки.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
