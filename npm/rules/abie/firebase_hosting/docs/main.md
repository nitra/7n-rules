## Огляд

Файл сканує каталоги першого рівня репозиторію для виявлення конфігурацій Firebase Hosting, спираючись на `firebase.json`. При скануванні свідомо ігнорує каталоги `.git`, `node_modules` та `.firebase`. Код повідомляє про порушення, якщо виявляє заборонений контент. Поведінка системи підтримує маркування повідомленнями (abie.mdc).

## Поведінка

Поведінка:

1. `main`: Створює репортер для фіксації результатів перевірки.
2. `main`: Читає вміст каталогу, вказаного як корінь репозиторію. Якщо зчитування неможливе, фіксує помилку та завершує роботу.
3. `main`: Виявляє всі підкаталоги першого рівня, ігноруючи каталоги `.git` та `node_modules`.
4. `main`: Для кожного такого каталогу перевіряє наявність конфігураційних файлів Firebase Hosting (`.firebaserc`, `firebase.json`) або директорії `.firebase`.
5. `main`: Якщо виявлено заборонений файл або директорію, фіксує це як порушення, згадуючи маркер ``та конфіг`firebase.json`.
6. `main`: Якщо в каталогах першого рівня не знайдено артефактів Firebase Hosting, фіксує успіх з маркером ``.
7. `main`: Завершує роботу з кодом, що відображає статус перевірки.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- Свідомо пропускає шляхи: `.git`, `node_modules`, `.firebase`.
