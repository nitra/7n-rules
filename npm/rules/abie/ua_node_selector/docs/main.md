## Огляд

Огляд

Аналізує відповідність конфігураційних файлів встановленим вимогам. Визначає необхідність застосування патча для конфігурації та перевіряє відповідні файли на наявність необхідних елементів, фіксуючи статус відповідності. Повідомляє про відповідність або необхідність змін (abie.mdc).

## Поведінка

1. Викликається `main` для початку перевірки.
2. Збираються шляхи до YAML-файлів Kubernetes, ігноруючи визначені конфігураційні шляхи.
3. Визначаються директорії з Deployment.
4. Якщо Deployment у дереві Kubernetes відсутній, повідомляється про успішну відповідність вимогам, оскільки patch `nodeSelector` (ua) не потрібен.
5. Ідентифікуються всі YAML-файли, які належать до конфігурації `ua`.
6. Якщо таких YAML-файлів відсутні, повідомляється про необхідність додати патч `ua/kustomization.yaml` з патчем на Deployment у полі `/spec/template/spec/nodeSelector` та `preem: false` (abie.mdc).
7. Для кожного ідентифікованого YAML-файлу `ua` перевіряється:
   а. Чи застосовується патч `nodeSelector` (ua), перевіряючи наявність Deployment у дереві Kubernetes цього пакета (abie). Якщо ні, фіксується успішна відповідність, оскільки патч не застосовується.
   б. Чи вдалося прочитати вміст файлу. У разі невдачі, фіксується помилка.
   в. Чи містить вміст файлу необхідний патч для цілі Deployment: шлях `/spec/template/spec/nodeSelector` та `preem: false` (abie.mdc). Якщо ні, фіксується необхідність застосувати зазначений патч (abie.mdc).
8. За результатами ітерацій повертається код завершення перевірки.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
