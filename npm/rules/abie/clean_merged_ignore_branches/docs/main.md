## Огляд

Цей модуль виконує аналіз YAML-конфігурацій для забезпечення коректності структури файлу. Він реалізує функціонал `lint` для валідації конфігураційних даних, пов'язаних із процесами злиття гілок. Аналіз здійснюється виключно для вказаного файлу, ігноруючи системні директорії, такі як `.github` та `.git`.

## Поведінка

1. Викликається функція `lint` для аналізу конфігурацій файлу `.github/workflows/clean-merged-branch.yml`.
2. Система перевіряє лише цей специфічний файл, свідомо пропускаючи директорії `.github` та `.git`.
3. Результат аналізу повертається як уніфікований вихід з переліком виявлених порушень.

## Публічний API

lint — виявляє, чи відповідність коду політичним вимогам, визначеним у конфігурації.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Свідомо пропускає шляхи: `.github`, `.git`.
