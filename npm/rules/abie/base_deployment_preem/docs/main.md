## Огляд

Виконує перевірку конфігурацій, пов'язаних з розгортанням, шляхом виклику публічної функції `lint`. Оцінка фокусується на конфігураціях у каталогах, але свідомо ігнорує будь-які шляхи, що містять `base/`.

## Поведінка

1. Викликається функція `lint` для оцінки конфігурацій, що стосуються розгортання.
2. Перевіряються файли у каталогах, що містять Kubernetes-конфігурації, зокрема `.yaml` та `.yml` файли, які знаходяться в директорії `k8s/base/`.
3. Перевірка виключає файл `kustomization.yaml` з каталогу `**/k8s/**/base/**/`.

## Публічний API

lint — виявляє відповідність політики concern-а, згенерованої codegen-обгорткою.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Свідомо пропускає шляхи: `base/`.
