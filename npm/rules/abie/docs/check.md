Модуль ініціює виконання перевірки правил. У бібліотечному режимі він запускає перевірку, використовуючи контекст поточного прогону. Як окремий інструмент командного рядка, він запускає повну оркестрацію правил.

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується у бібліотечному режимі, використовуючи контекст прогону.
3. Якщо код виконується як окремий інструмент командного рядка, запускається оркестрація для повного виконання правила.

## Публічний API

run — виконує послідовність дій: застосовує правила, перевіряє JS-занепокоєння, застосовує політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
