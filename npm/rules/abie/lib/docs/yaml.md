## Огляд

Overview
Файл надає спільні хелпери для перевірки документів у форматі YAML. Допомагає парсингу документів з опційним `modeline`, видалення BOM, розділення вмісту на рядки та перевірку наявності певних директив.

## Поведінка

Поведінка: Функції працюють у послідовності з читання файлу до парсингу YAML-документів. `readFile` зчитує вміст у вигляді рядка. `stripBom` видаляє Byte Order Mark на початку рядка. Потім `body` розділяється на рядки за допомогою `LINE_SPLIT_RE`. Перевіряється наявність `modeline` за допомогою `MODELINE_RE` у першому рядку. Якщо модель існує, решта рядків використовується для парсингу через `parseAllDocuments`. Якщо парсинг YAML викликає помилку, використовується `failFn` для зафіксованого повідомлення, а функція повертає `null`.

## Публічний API

- MODELINE_RE — Розпізнає modeline `yaml-language-server` з `$schema=` у першому рядку файлу; захоплює URL схеми.
- LINE_SPLIT_RE — Поділ вмісту на рядки незалежно від стилю переносу (LF чи CRLF).
- stripBom — Прибирає BOM на початку файлу.
- isDeploymentDoc — Чи YAML-документ — це `kind: Deployment`.
- silentFail — No-op fail-handler для функцій, що мовчки повертають null/[] при помилці парсингу.
- readAndParseYamlDocs — Зчитує і парсить YAML-документи з файлу. BOM і modeline (перший рядок `$schema`)
автоматично прибираються перед `parseAllDocuments`. При помилці читання/парсингу
викликає `failFn` і повертає `null`.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
