# hc-yaml.mjs

## Огляд

Модуль `hc-yaml.mjs` — це **вузькоспеціалізований валідатор modeline** для YAML-файлів `hc.yaml` (Health Check) у контексті правила **abie** (`abie.mdc`). Файл відповідає за єдину перевірку: чи **перший рядок** YAML-документа є валідним [yaml-language-server](https://github.com/redhat-developer/yaml-language-server) modeline-коментарем, що вказує саме на схему `HealthCheckPolicy` із публічного каталогу CRDs.

Контекст і розподіл відповідальностей:

- **Цей модуль** перевіряє лише наявність і коректність modeline-рядка `# yaml-language-server: $schema=...` із очікуваним URL.
- **Per-document структурна валідація** самого ресурсу `HealthCheckPolicy` (відповідність схемі полів, обов'язкові ключі тощо) живе у **OPA/Rego-політиці** `policy/health_check_policy/health_check_policy.rego` — її CLI прогонить через згенерований `target.json` (див. ADR/`n-rego.mdc`).
- Перевірка свідомо «м'яка» по структурі YAML: вона не парсить весь документ, а працює з сирим текстом, бо її єдина мета — гарантувати IDE-підказки через `$schema`.

Модуль написаний у стилі **ESM (`.mjs`)**, без побічних ефектів, без I/O, з чистими функціями і константами — придатний для прямого імпорту перевірками вищого рівня (наприклад, чек-скриптами `npm/rules/abie/checks/`).

## Експорти / API

Файл має **два публічні експорти**:

| Експорт                  | Тип                            | Призначення                                                                                  |
| ------------------------ | ------------------------------ | -------------------------------------------------------------------------------------------- |
| `ABIE_HC_SCHEMA_URL`     | `string` (константа)           | Очікуваний URL `$schema` для `hc.yaml`. Єдине джерело істини для перевірок і повідомлень.   |
| `validateAbieHcModeline` | `(raw: string, relPath: string) => string \| null` | Перевіряє перший рядок `hc.yaml` на правильний modeline; повертає текст помилки або `null`. |

### `ABIE_HC_SCHEMA_URL`

```js
export const ABIE_HC_SCHEMA_URL = 'https://datreeio.github.io/CRDs-catalog/networking.gke.io/healthcheckpolicy_v1.json'
```

- **Тип:** `string` (рядкова константа, immutable за домовленістю `const`).
- **Значення:** URL JSON-схеми ресурсу `HealthCheckPolicy v1` у каталозі CRDs від проєкту **datreeio** (group `networking.gke.io`, kind `HealthCheckPolicy`, версія `v1`).
- **Семантика:** саме цей URL **повинен** з'являтися у modeline-рядку `# yaml-language-server: $schema=...` усіх `hc.yaml` файлів, які підпадають під правило `abie.mdc`.
- **Чому константа експортується:** щоб тести та інші перевірки могли:
  - звіряти повідомлення про помилки з очікуваним URL,
  - використовувати єдине джерело істини без копіювання рядка.

## Функції

### `validateAbieHcModeline(raw, relPath)`

#### Сигнатура

```js
export function validateAbieHcModeline(raw, relPath)
```

JSDoc-сигнатура:

```text
@param {string} raw      — повний вміст YAML-файла (як прочитано з disk).
@param {string} relPath  — відносний шлях до файла (використовується лише
                            у тексті повідомлень про помилки).
@returns {string | null} — текст помилки (рядок з префіксом relPath) або
                            null, якщо modeline валідний.
```

#### Параметри

| Параметр  | Тип      | Опис                                                                                                                                                                          |
| --------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `raw`     | `string` | Сирий вміст файла. Може містити BOM на початку — він прибирається `stripBom`. Очікується, що це звичайний YAML-текст, можливо порожній.                                       |
| `relPath` | `string` | Шлях до файла відносно кореня репозиторію/перевірки. Використовується **лише** у формуванні повідомлень про помилки (для дружнього виводу). Не використовується для I/O.      |

#### Повертає

- **`string`** — текст помилки, якщо modeline невалідний. Усі повідомлення:
  - починаються з `${relPath}:` (щоб користувач бачив, у якому файлі проблема),
  - закінчуються маркером `(abie.mdc)` (вказує на правило, яке порушено).
- **`null`** — якщо все гаразд: перший рядок є коректним modeline і URL збігається з `ABIE_HC_SCHEMA_URL`.

#### Алгоритм / гілки виконання

Функція є **чистою** (детермінована, без I/O, без мутацій вхідних даних). Її логіка послідовна:

1. **BOM-strip.** `stripBom(raw)` прибирає UTF-8 BOM (`﻿`) з початку, якщо він є. Результат — у локальну змінну `body`.
2. **Розбиття на рядки.** `body.split(LINE_SPLIT_RE)` ділить вміст за регексом `/\r?\n/u` — це коректно обробляє і Unix-, і Windows-переноси рядків.
3. **Перевірка першого рядка на «порожньо».**
   - Гілка `lines.length === 0 || lines[0].trim() === ''` → повертається помилка:
     ```text
     ${relPath}: перший рядок порожній — потрібен # yaml-language-server: $schema=… (abie.mdc)
     ```
   - Технічна примітка: `String.prototype.split` з ненульовим джерелом завжди повертає масив із принаймні одним елементом, тож умова `lines.length === 0` — захисний інваріант для випадку, якщо реалізація платформи відхилиться (TS-friendly та fault-tolerant код).
4. **Парсинг modeline.** `lines[0].match(MODELINE_RE)` намагається витягнути URL `$schema` із першого рядка.
   - `MODELINE_RE` визначено у `yaml.mjs`:
     ```js
     /^#\s*yaml-language-server:\s*\$schema=(\S+)\s*$/
     ```
   - Якщо `match` повернув `null` → повертається помилка:
     ```text
     ${relPath}: перший рядок має бути modeline $schema (abie.mdc)
     ```
5. **Перевірка URL.** Захоплена група `m[1]` порівнюється з `ABIE_HC_SCHEMA_URL`.
   - Якщо не дорівнює → повертається багаторядкова помилка з очікуваним URL у відступі:
     ```text
     ${relPath}: $schema має бути
          <ABIE_HC_SCHEMA_URL>
          (abie.mdc)
     ```
6. **Успіх.** Якщо жодна з гілок-помилок не спрацювала — повертається `null`.

#### Side effects

- **Немає.** Функція не виконує I/O, не змінює аргументи, не пише у глобальний стан, не кидає винятків (у штатних сценаріях). Уся «реактивність» — повернення рядка або `null`.
- Не валідуючий ні YAML-структуру документа, ні існування файла — лише перший рядок як текст.

#### Приклади використання

Валідний випадок:

```js
import { validateAbieHcModeline, ABIE_HC_SCHEMA_URL } from './hc-yaml.mjs'

const raw = `# yaml-language-server: $schema=${ABIE_HC_SCHEMA_URL}
apiVersion: networking.gke.io/v1
kind: HealthCheckPolicy
`
validateAbieHcModeline(raw, 'k8s/hc.yaml')
// → null
```

Помилка — порожній перший рядок:

```js
validateAbieHcModeline('\napiVersion: ...\n', 'k8s/hc.yaml')
// → 'k8s/hc.yaml: перший рядок порожній — потрібен # yaml-language-server: $schema=… (abie.mdc)'
```

Помилка — modeline відсутній:

```js
validateAbieHcModeline('apiVersion: networking.gke.io/v1\n', 'k8s/hc.yaml')
// → 'k8s/hc.yaml: перший рядок має бути modeline $schema (abie.mdc)'
```

Помилка — невірний URL:

```js
const raw = '# yaml-language-server: $schema=https://example.com/other.json\n'
validateAbieHcModeline(raw, 'k8s/hc.yaml')
// → 'k8s/hc.yaml: $schema має бути\n     https://datreeio.github.io/CRDs-catalog/networking.gke.io/healthcheckpolicy_v1.json\n     (abie.mdc)'
```

## Залежності

### Внутрішні (relative imports)

Імпорт із сусіднього файла:

```js
import { LINE_SPLIT_RE, MODELINE_RE, stripBom } from './yaml.mjs'
```

- **`LINE_SPLIT_RE`** — `RegExp = /\r?\n/u`. Розділювач рядків для cross-platform поділу тексту.
- **`MODELINE_RE`** — `RegExp = /^#\s*yaml-language-server:\s*\$schema=(\S+)\s*$/`. Регулярний вираз для розпізнавання modeline-коментаря з захопленням URL у групу 1.
- **`stripBom(s: string): string`** — прибирає UTF-8 BOM (`﻿`) на початку рядка, якщо він є; інакше повертає рядок без змін.

Ці три імпорти **спільно використовуються** в усіх abie-валідаторах modeline у цьому пакеті, забезпечуючи однорідну поведінку (немає дублювання regex/BOM-логіки).

### Зовнішні (npm)

- **Відсутні.** Модуль не імпортує жодного npm-пакета. Уся робота — на стандартних можливостях JS (`String#split`, `String#match`, `String#trim`).
- Жодних викликів `node:*` модулів (немає `fs`, `path` тощо) — це гарантує те, що функція легко тестується pure-функціонально, без mock-ів файлової системи.

### Глобалі та середовище

- Не використовуються (`process.env`, `globalThis` тощо не залучені).

## Потік виконання / Використання

### Місце у CI4 / у правилах abie

Модуль є **частиною інфраструктури правил abie** і використовується чек-скриптами, що валідують YAML-файли Kubernetes-конфігурації (`hc.yaml` — Health Check Policy). Типова інтеграція:

1. Чек-скрипт читає файл `hc.yaml` із диска (наприклад, через `readFile` із `node:fs/promises`).
2. Скрипт викликає `validateAbieHcModeline(raw, relPath)`.
3. Якщо повернуто рядок — додає його у звіт-помилок (наприклад, через `failFn`/масив `errors`).
4. Окремо, на наступному рівні, **`policy/health_check_policy/health_check_policy.rego`** через CLI (зчитує `target.json`) виконує структурну валідацію самого `HealthCheckPolicy`-ресурсу.

Таким чином, ця функція — це **«перший фільтр»**: дешевий, текстовий, без YAML-парсингу, що гарантує IDE-підказки і не дозволяє файлу потрапити далі без правильної прив'язки до схеми.

### Послідовність обробки `raw`

```
raw (string)
  │
  ├─ stripBom ──► body
  │
  ├─ body.split(LINE_SPLIT_RE) ──► lines[]
  │
  ├─ lines.length === 0 || lines[0].trim() === '' ?
  │     └─ так ──► повернути помилку «перший рядок порожній»
  │
  ├─ lines[0].match(MODELINE_RE) ──► m
  │     └─ null ──► повернути помилку «має бути modeline $schema»
  │
  ├─ m[1] !== ABIE_HC_SCHEMA_URL ?
  │     └─ так ──► повернути помилку «$schema має бути <URL>»
  │
  └─ повернути null  (success)
```

### Контракти та інваріанти

- **Чистота.** Функція детермінована: однаковий `raw` → однаковий результат (за умови незмінного `relPath`).
- **Безпека:** не виконує жодних небезпечних операцій (eval, fs, network), тому її можна викликати з будь-якого контексту, включно з лінтерами та pre-commit hook-ами.
- **i18n повідомлень.** Усі тексти помилок українською — відповідає внутрішнім вимогам проєкту до user-facing повідомлень у правилах.
- **Маркер `(abie.mdc)`** наприкінці кожного повідомлення дозволяє грепати помилки по правилу і вести юзера у відповідне правило-документ.

### Тестованість

Модуль навмисно тонкий і детермінований, що робить його ідеальним для покриття unit-тестами:

- 4 гілки виконання → 4 негативних + 1 позитивний тест-кейс.
- Не потребує mock-ів файлової системи: достатньо подавати рядки у `raw`.
- BOM-кейс перевіряється передачею рядка з префіксом `'﻿'`.

## Rebuild Test

Контрольні точки, за якими файл можна відновити з нуля без втрати функціональності:

1. **ESM-модуль** (`.mjs`), що валідує modeline `hc.yaml` у контексті правила `abie.mdc`.
2. **Імпортує** з `./yaml.mjs` три символи: `LINE_SPLIT_RE`, `MODELINE_RE`, `stripBom`.
3. **Експортує константу `ABIE_HC_SCHEMA_URL`** зі значенням:
   `https://datreeio.github.io/CRDs-catalog/networking.gke.io/healthcheckpolicy_v1.json`.
4. **Експортує функцію `validateAbieHcModeline(raw, relPath)`** з контрактом:
   - повертає `string` (текст помилки) або `null`;
   - усі повідомлення префіксовані `${relPath}:` і завершені маркером `(abie.mdc)`;
   - порядок перевірок: (a) порожній/відсутній перший рядок, (b) рядок не відповідає `MODELINE_RE`, (c) URL не дорівнює `ABIE_HC_SCHEMA_URL`.
5. **Жодних залежностей** від `node:*` модулів чи npm-пакетів — лише локальний імпорт із `./yaml.mjs`.
6. **Жодних побічних ефектів** і I/O.
7. Документація у JSDoc вказує, що структурна (per-document) валідація живе в Rego-політиці `policy/health_check_policy/health_check_policy.rego`.
