# enabled.mjs

## Огляд

Модуль `enabled.mjs` — це **rule-level applies-гейт** для правила `abie` у системі `n-cursor`. Його єдина відповідальність — визначити, чи увімкнено правило `abie` у конфігураційному файлі репозиторію `.n-cursor.json` (поле `rules`).

Контекст використання:

- Файл імпортується з `js/applies.mjs` правила `abie` і експортується звідти як функція `applies()`.
- Якщо `applies()` повертає `false`, CLI `n-cursor` **повністю пропускає** всі концерни правила `abie` — включно з `lint`, `fix` і `policy`. Це класичний механізм opt-in: правило не діє, поки користувач явно не додасть його ім'я у масив `rules` файлу `.n-cursor.json`.
- Файл `.n-cursor.json` має лежати у корені репозиторію (`root`), який передається параметром.

Шаблон роботи: прочитати JSON-конфіг, дістати масив `rules`, нечутливо до регістру/пробілів перевірити наявність токена `'abie'`. Будь-яка помилка (відсутній файл, нечитаний файл, невалідний JSON, відсутнє/неправильного типу поле `rules`) трактується як «правило вимкнено» (`false`) — без кидання винятку.

## Експорти / API

| Експорт                  | Тип                                 | Призначення                                                                  |
| ------------------------ | ----------------------------------- | ---------------------------------------------------------------------------- |
| `isAbieRuleEnabled(root)` | `async (string) => Promise<boolean>` | Перевіряє, чи містить `.n-cursor.json:rules` рядок `abie`. Named export.     |

Інших експортів (default, констант) модуль **не має**. Константа `CONFIG_FILE = '.n-cursor.json'` визначена на рівні модуля, але не експортується.

## Функції

### `isAbieRuleEnabled(root)`

Асинхронна функція-предикат: чи увімкнено правило `abie` у конфігу.

**Сигнатура:**

```js
async function isAbieRuleEnabled(root: string): Promise<boolean>
```

**Параметри:**

- `root` (`string`, обов'язковий) — абсолютний (або відносний до `cwd` процесу) шлях до кореня репозиторію. До нього приєднується ім'я `.n-cursor.json` через `path.join`.

**Повертає:**

- `Promise<boolean>`:
  - `true` — якщо файл `.n-cursor.json` існує, є валідним JSON, поле `rules` — масив, і хоча б один його елемент після приведення `String(r).trim().toLowerCase()` дорівнює рядку `'abie'`.
  - `false` — у всіх інших випадках (див. розділ «Потік виконання»).

**Side effects:**

- **Файлова система (read-only):** синхронний виклик `existsSync` і асинхронний `readFile` для шляху `<root>/.n-cursor.json`. Жодних записів, мутацій чи мережевих звернень.
- **Винятків не кидає:** усі очікувані помилки (відсутній файл, помилка читання, невалідний JSON) перехоплюються `try/catch` і конвертуються у `false`. Якщо `existsSync` або `path.join` несподівано кинуть синхронний виняток (наприклад, через нелегальні аргументи на рівні Node.js), він прокинеться нагору — але це поза нормальним сценарієм використання.
- **Нечутливість до регістру/пробілів:** елементи масиву `rules` нормалізуються через `String(r).trim().toLowerCase()`, тож `'Abie'`, `' abie '`, `'ABIE'` трактуються як увімкнене правило.
- **Толерантність до типу елементів:** через `String(r)` нечислові/нерядкові елементи (числа, об'єкти) теж приводяться до рядка — об'єкт стане `'[object object]'` і не співпаде з `'abie'`, тож не зламає логіку.

## Залежності

**Стандартна бібліотека Node.js:**

- `node:fs` — використовується синхронний `existsSync` для швидкого short-circuit на відсутньому файлі (щоб уникнути виклику `readFile`, який кинув би `ENOENT`).
- `node:fs/promises` — асинхронний `readFile(path, 'utf8')` для читання вмісту конфігу як UTF-8 рядка.
- `node:path` — `join` для побудови шляху `<root>/.n-cursor.json` крос-платформенно.

**Зовнішні пакети:** немає.

**Внутрішні модулі проєкту:** немає прямих імпортів. Файл імпортується іншими модулями правила `abie` (зокрема `npm/rules/abie/js/applies.mjs`) — тобто він є **листковим** (leaf) у графі залежностей правила.

**Очікуваний формат `.n-cursor.json`:**

```json
{
  "rules": ["abie", "інше-правило"]
}
```

— поле `rules` має бути масивом рядків з іменами правил, які користувач увімкнув для проєкту.

## Потік виконання / Використання

### Покроковий потік `isAbieRuleEnabled(root)`

1. Сформувати повний шлях `p = join(root, '.n-cursor.json')`.
2. Якщо файл за шляхом `p` **не існує** (`existsSync(p) === false`) — повернути `false` і вийти.
3. Спробувати асинхронно прочитати вміст файлу як UTF-8 (`readFile(p, 'utf8')`):
   - Якщо `readFile` кинув виняток (наприклад, EACCES, EISDIR) — повернути `false`.
4. Спробувати розпарсити прочитаний рядок як JSON (`JSON.parse(raw)`):
   - Якщо `JSON.parse` кинув `SyntaxError` (невалідний JSON) — повернути `false`.
5. Отримати поле `cfg?.rules` (з optional chaining — безпечно, якщо `cfg` сам по собі `null` або не-об'єкт).
6. Якщо `rules` **не масив** (`Array.isArray(rules) === false`) — повернути `false`.
7. Пройти масивом `rules` через `Array.prototype.some`:
   - Для кожного `r` обчислити `String(r).trim().toLowerCase()`.
   - Якщо результат строго дорівнює `'abie'` — повернути `true`.
   - Інакше — `false`.

### Сценарії використання

**1. Виклик з `applies.mjs` правила abie:**

```js
// npm/rules/abie/js/applies.mjs (концептуальний приклад)
import { isAbieRuleEnabled } from '../lib/enabled.mjs'

export async function applies(root) {
  return isAbieRuleEnabled(root)
}
```

CLI `n-cursor` під час сканування правил викликає `applies(cwd)` для кожного правила; для `abie` ця функція делегує `isAbieRuleEnabled`. Якщо результат `false` — CLI не запускає ані `lint`, ані `fix`, ані `policy` правила `abie`.

**2. Включення правила користувачем:**

Користувач у корені свого репо створює/редагує `.n-cursor.json`:

```json
{
  "rules": ["abie"]
}
```

— після цього `isAbieRuleEnabled(process.cwd())` поверне `true`, і правило почне діяти.

**3. Вимкнення правила:**

Достатньо прибрати рядок `'abie'` з масиву `rules`, видалити поле `rules`, видалити сам файл `.n-cursor.json` або зробити його невалідним JSON — у всіх випадках функція поверне `false`.

### Поведінкові гарантії (rebuild test — інваріанти, які можна відновити з коду)

- `isAbieRuleEnabled('/немає/такого/шляху') === false` — відсутній файл → `false`, без винятків.
- `isAbieRuleEnabled` з невалідним JSON у файлі → `false`.
- `isAbieRuleEnabled` з `{"rules": "abie"}` (рядок, не масив) → `false` (бо `Array.isArray` хибне).
- `isAbieRuleEnabled` з `{"rules": []}` → `false` (порожній масив, `some` повертає `false`).
- `isAbieRuleEnabled` з `{"rules": ["abie"]}` → `true`.
- `isAbieRuleEnabled` з `{"rules": [" ABIE "]}` → `true` (нормалізація `trim().toLowerCase()`).
- `isAbieRuleEnabled` з `{"rules": ["other"]}` → `false`.
- `isAbieRuleEnabled` з `{"rules": ["other", "abie"]}` → `true`.
- `isAbieRuleEnabled` з `{}` (немає поля `rules`) → `false`.
- Функція **ніколи** не модифікує файл і не пише на диск.
