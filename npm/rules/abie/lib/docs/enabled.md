# enabled.mjs

## Огляд

Гейт увімкнення правила `abie`: вирішує, чи правило взагалі діє у цьому репозиторії.
Підключається до applies-механізму правила — якщо гейт каже «вимкнено», CLI `n-cursor`
повністю пропускає всі концерни `abie` (`lint`, `fix`, `policy`). Семантика — opt-in:
правило мовчить, поки користувач явно не ввімкне його у конфізі.

## Поведінка

Користувач вмикає правило, додавши його ім'я у масив `rules` файлу `.n-cursor.json`
в корені репозиторію:

```json
{ "rules": ["abie"] }
```

Перевірка наявності **нечутлива до регістру й пробілів**: `Abie`, ` ABIE `, `abie` —
усі вмикають правило. Щоб вимкнути — прибрати ім'я з `rules` (або видалити поле / файл).

Покроково:

1. Знайти `.n-cursor.json` у переданому корені; якщо файлу немає — **вимкнено**.
2. Прочитати й розпарсити як JSON; помилка читання чи невалідний JSON — **вимкнено**.
3. Дістати поле `rules`; якщо це не масив — **вимкнено**.
4. Якщо хоч один елемент масиву (після нормалізації регістру/пробілів) дорівнює імені
   правила — **увімкнено**, інакше — **вимкнено**.

## Де використовується

Підключається до applies-механізму правила `abie` як рішення «чи запускати правило».
Викликається CLI `n-cursor` під час сканування правил перед `lint`/`fix`/`policy`.

## Гарантії поведінки

- Будь-яка помилка чи неоднозначність трактується як «вимкнено» — функція **ніколи не
  кидає виняток** назовні (fail-safe у `false`).
- Операція **read-only**: файл лише читається, жодних записів на диск чи мережевих звернень.
- Нечислові/нерядкові елементи `rules` (числа, об'єкти) безпечно ігноруються, а не ламають
  перевірку.
