# applies.mjs

## Огляд

Файл `npm/rules/abie/js/applies.mjs` — це **applies-гейт** правила `abie` у системі правил `n-cursor`. Модуль виконує дві задачі:

1. Експортує функцію `applies(cwd)`, яка визначає на **rule-level**, чи застосовне правило `abie` до конкретного репозиторію. Рішення береться з поля `rules` у файлі `.n-cursor.json`, що читає утиліта `isAbieRuleEnabled`.
2. Експортує функцію `check()`, яка є тонким діагностичним концерном: вона лише друкує `context-pass`-повідомлення про те, що правило `abie` увімкнено. Реальну роботу перевірок виконують інші концерни (JS- й policy-перевірки) цього самого правила.

Поведінка інтеграції з CLI така: якщо `applies()` повертає `false`, CLI `n-cursor` **пропускає всі концерни** правила `abie` (як JS-, так і policy-перевірки). Якщо повертає `true`, CLI продовжує виконувати концерни правила, а `check()` додає лише позитивний context-репорт про факт активації.

## Експорти / API

| Символ | Тип | Призначення |
| --- | --- | --- |
| `applies` | `function(cwd?: string) => Promise<boolean>` | Перевіряє, чи увімкнене правило `abie` для заданого `cwd`. Делегує рішення в `isAbieRuleEnabled`. |
| `check` | `function() => number` | Друкує `context-pass` про факт активації правила `abie` і повертає exit-код (`0` — OK, `1` — порушення). |

Інших експортів (default-export, констант, класів) у файлі `applies.mjs` немає.

## Функції

### `applies(cwd = process.cwd())`

- Сигнатура: `applies(cwd?: string): Promise<boolean>`.
- Параметри:
  - `cwd` *(необов’язковий, `string`)* — корінь репозиторію, який треба перевіряти. Якщо не передано, використовується `process.cwd()` (поточна робоча директорія Node-процесу).
- Повертає: `Promise<boolean>` — результат виклику `isAbieRuleEnabled(cwd)`.
- Семантика:
  - `true` — правило `abie` увімкнено в `.n-cursor.json` репозиторію `cwd`. CLI повинен виконати концерни правила.
  - `false` — правило `abie` вимкнено або не сконфігуровано. CLI має пропустити всі концерни цього правила (як JS, так і policy).
- Реалізація: одно-рядковий проксі-виклик `return isAbieRuleEnabled(cwd)`; жодних додаткових обчислень, кешу чи побічних ефектів усередині `applies` немає — вся логіка інкапсульована в `isAbieRuleEnabled` (модуль `../lib/enabled.mjs`).

### `check()`

- Сигнатура: `check(): number`.
- Параметри: відсутні.
- Повертає: `number` — exit-код, отриманий від `reporter.getExitCode()`. За JSDoc-контрактом: `0` — OK, `1` — порушення.
- Поведінка:
  1. Створює репортер виклик `createCheckReporter()` (фабрика з `../../../scripts/lib/check-reporter.mjs`).
  2. Викликає `reporter.pass('Правило abie увімкнено — виконуємо перевірки')` — це **context-pass** повідомлення (інформаційне, не fail).
  3. Повертає підсумковий exit-код через `reporter.getExitCode()`.
- Призначення: `check()` цього файлу — лише діагностичний індикатор у звіті, що правило `abie` активне; реальні перевірки виконують інші концерни правила (інші файли в `npm/rules/abie/`).

## Залежності

Імпорти, оголошені в `applies.mjs`:

- `createCheckReporter` з `../../../scripts/lib/check-reporter.mjs` — фабрика репортера, що збирає `pass`/`fail`/`context`-повідомлення концерну та формує підсумковий exit-код через `getExitCode()`.
- `isAbieRuleEnabled` з `../lib/enabled.mjs` — функція, що читає `.n-cursor.json` за шляхом `cwd` і повертає булеве значення увімкненості правила `abie` (на основі поля `rules`).

Зовнішніх npm-пакетів (поза монорепо) `applies.mjs` напряму не імпортує. Глобальний `process.cwd()` використовується як default для параметра `cwd`.

## Потік виконання / Використання

Типовий сценарій інтеграції з CLI `n-cursor` для правила `abie`:

1. CLI обходить активні правила і для кожного викликає його `applies(cwd)`.
2. Для правила `abie` викликається `applies()` із цього файлу:
   - Делегує в `isAbieRuleEnabled(cwd)`, який читає `.n-cursor.json` у `cwd` і дивиться поле `rules`.
   - Повертає `Promise<boolean>`.
3. Якщо `applies()` повернув `false` — CLI **пропускає всі концерни** правила `abie` (і JS, і policy). Жодних подальших дій по цьому правилу не виконується.
4. Якщо `applies()` повернув `true` — CLI запускає концерни правила. Серед них:
   - `check()` із цього файлу друкує `context-pass`: «Правило abie увімкнено — виконуємо перевірки», і повертає `0` за відсутності зафіксованих порушень у репортері.
   - Інші концерни правила `abie` виконують реальні JS- та policy-перевірки.
5. Exit-код від `check()` агрегується з кодами інших концернів за стандартним конвейером CLI.

Приклад прямого використання у Node-середовищі:

```js
import { applies, check } from './applies.mjs'

const enabled = await applies('/path/to/repo')
if (enabled) {
  const code = check()
  // code === 0 — context-pass, правило активне
}
```

Ключові інваріанти:

- `applies()` — єдина точка прийняття рішення про активність правила `abie` на rule-level; CLI довіряє її результату беззастережно.
- `check()` ніколи не виконує реальних перевірок коду — це навмисно тонкий діагностичний концерн, який лише підтверджує активацію правила в репорті.
