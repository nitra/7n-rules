## Огляд

Цей файл виконує комплексну перевірку конфігурацій, збираючи та фільтруючи файли `ua/kustomization.yaml`. Для кожного відповідного файлу він перевіряє конфігурацію з огляду на необхідність та результати застосування патчу `HTTPRoute`, генеруючи та валідуючи комбінований текст патчу `NginxRun`. Результати перевірки відображаються для кожного об'єкта.

## Поведінка

1. `main` запускає процес перевірки.
2. Збираються шляхи до YAML-файлів Kubernetes, ігноруючи конфігураційні шляхи.
3. Фільтруються знайдені YAML-файли для ідентифікації тих, що стосуються `ua/kustomization.yaml`.
4. Для кожного відповідного шляху, якщо не застосовується патч HTTPRoute для `ua` (наприклад, відсутній `vite.config` у пакеті), фіксується успіх.
5. Якщо застосування патча необхідне, аналізуються посилання на спільні бекенди в контексті Kubernetes для відповідного пакета.
6. У разі виявлення базових помилок під час аналізу, відповідний шлях фіксується з помилкою, і перевірка для нього припиняється.
7. Якщо аналіз безпосередньо не видає помилок, зчитується вміст відповідного YAML-файлу.
8. З вмісту зчитуваного файлу генерується комбінований текст патчу NginxRun.
9. Цей патч валідується для відповідності вимогам `ua`, використовуючи інформацію про посилання та базові помилки.
10. Якщо валідація успішна, фіксується успіх. В іншому випадку, фіксується помилка з результатами валідації.
11. Усі результати перевірок збираються та повертається фінальний код виходу.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- Кешує результати в межах одного прогону.
