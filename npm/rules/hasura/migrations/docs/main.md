## Огляд

Файл перевіряє, чи відповідає структура директорії міграцій Hasura встановленим критеріям: присутність лише файлів `up.sql` та відсутність файлів з іменем `down.sql`.

Викликається `main` для перевірки директорії міграцій Hasura. Якщо директорія `hasura/migrations` відсутня, перевірка вважається успішно виконаною, оскільки для неї не потрібна перевірка наявності `down.sql` (hasura.mdc). Якщо директорія присутня, виконується рекурсивний пошук у ній. Будь-які файли з іменем `down.sql` знаходяться і реєструються як порушення (hasura.mdc), оскільки в директорії міграцій має бути присутній лише `up.sql` (hasura.mdc). Якщо `down.sql` не знайдено, перевірка вважається успішно виконаною (hasura.mdc).

## Поведінка

Поведінка:

1. Викликається `main` для перевірки директорії міграцій Hasura.
2. Якщо директорія `hasura/migrations` відсутня, перевірка вважається успішно виконаною, оскільки для неї не потрібна перевірка наявності `down.sql` (hasura.mdc).
3. Якщо директорія присутня, виконується рекурсивний пошук у ній.
4. Будь-які файли з іменем `down.sql` знаходяться ідентифіковані та зареєстровані як порушення (hasura.mdc), оскільки в директорії міграції має бути присутній лише `up.sql` (hasura.mdc).
5. Якщо `down.sql` не знайдено, перевірка вважається успішно виконаною (hasura.mdc).

## Публічний API

main — перевіряє відсутність файлів `down.sql` у директорії міграцій Hasura, гарантуючи, що міграції містять лише `up.sql`.
