Модуль виконує перевірку, застосовуючи послідовність логік: `застосування`, `JS-concerns`, `policy` та `mdc-refs`. При запуску як окрема утиліта, він завантажує конфігурацію та формує підсумки.

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується, застосовуючи логіку: `applies` $\rightarrow$ `JS-concerns` $\rightarrow$ `policy` $\rightarrow$ `mdc-refs`.
3. Якщо код виконується як окрема утиліта (CLI), запускається повна оркестрація правил, що включає завантаження конфігурації, вайтлістинг та підсумки.

## Публічний API

run — виконує послідовність дій: застосовує правила, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
