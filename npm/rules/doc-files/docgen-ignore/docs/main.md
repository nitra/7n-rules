## Огляд

Визначає, які шляхи слід ігнорувати під час збору документації. Це досягається через фіксований набір каталогів, які виключені за замовчуванням (зокрема, `.git` та `node_modules`). Функціонал надає публічні експозиції `DOCGEN_IGNORE_GLOBS` та `isDocgenIgnored`, що дозволяють контролювати цей список.

## Поведінка

Поведінка
DOCGEN_IGNORE_GLOBS надає фіксований список шляхів, які слід ігнорувати при генерації документації, включаючи такі каталоги як `node_modules` та `.git`.
isDocgenIgnored визначає, чи має відносний шлях бути пропущеним при зборі документації, враховуючи конфігурацію ігнорування та підтримуючи перевірку для каталогів.

## Публічний API

DOCGEN_IGNORE_GLOBS — Список шаблонів файлів та папок, які ігноруються інструментом для генерації документації.
isDocgenIgnored — Визначає, чи певний шлях слід пропустити при генерації документації.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
- Свідомо пропускає шляхи: `.git`, `node_modules`.
