## Огляд

Огляд
Файл реалізує CLI-обгортку канонічного детектора застарілості файлової документації з `doc-files.mdc`. Детектор визначає актуальність файлових доків, не використовуючи виклики LLM.

Поведінка
runLintDocFilesSteps: Виконує повний або точковий детект файлової документації. При виявленні застарілих або відсутніх доків завершує роботу з кодом 1.
runLintDocFilesCli: Служить публічним інтерфейсом CLI. Залежно від прапорців виконує повний або точковий детект. Може звузити пошук лише до відсутніх документів (`--missing-only`), надавати JSON-лістинг кандидатів (`--json`) або делегувати перевірку через хук (`--hook`/`--git`/`--degraded`), що призводить до виходу з кодом 2 або 0 відповідно до протоколу.
main: Виконує детермінований пошук та повідомлення про застарілі/сирітські доки згідно з логікою `doc-files.mdc`, звітуючи про незакриті елементи, якщо така ситуація виникає.

## Поведінка

runLintDocFilesSteps проводить повний або точковий детект файлової документації; виходить із кодом 1, якщо знайдено застарілі або відсутні доки.
runLintDocFilesCli слугує публічною CLI-формою, делегуючи виконання повного/точкового детекту або сканування у залежності від отриманих прапорців.
main виконує швидкий детермінований ремонт застарілих/сирітських доків (оновлення CRC, видалення сирітських доків), а потім звітує про ті, що залишилися незакритими.

## Публічний API

runLintDocFilesSteps — Виконує повне або точкове виявлення з документів. Повертає 1, якщо документи застарілі, або 0, якщо всі актуальні.
runLintDocFilesCli — Публічна командна лінійка `lint-doc-files`. Запускається через хук або Git. Без `--json` виконує повне або точкове виявлення через `runStandardLint` для ключа `lint-doc-files`; з `--json` лише сканує.
main — Основний вхідний пункт для JS-колективу. Спочатку швидко виправляє застарілі/сиронні документи, а потім повідомляє про лише ті порушення, що не були усунуті.
