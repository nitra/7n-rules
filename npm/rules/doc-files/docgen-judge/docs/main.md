## Огляд

Огляд: Цей файл визначає інтерфейс для механізму семантичного судження якості згенерованої технічної документації за допомогою мовної моделі. Він конфігурує параметри (`JUDGE_MODEL`, `JUDGE_ENABLED`, `JUDGE_CONFIDENCE`) та надає read-only функції для ініціації судження та інтерпретації його результатів.

Поведінка:
JUDGE_MODEL — конфігурує LLM, яка використовується для семантичного судження якості документації.
JUDGE_ENABLED — вказує, чи активовано семантичний гейт для перевірки точності документації.
JUDGE_CONFIDENCE — встановлює мінімальний поріг впевненості, необхідний для визначення документації неточним (degraded).
parseDocVerdict — аналізує та валідує структуру судження (вердикт, впевненість, причина) з сирого тексту відповіді моделі.
judgeDoc — виконує семантичне судження згенерованої документації проти вмісту вихідного файлу через LLM.
judgeFailsDoc — визначає, чи вважати документацію неякісною, якщо вердикт 'inaccurate' перевищує заданий поріг впевненості.

## Поведінка

JUDGE_MODEL — визначає модель, яка використовується для семантичного судження якості документації.
JUDGE_ENABLED — вказує, чи активовано семантичний гейт для перевірки точності документації.
JUDGE_CONFIDENCE — встановлює мінімальний рівень впевненості для визначення документації неточним (degraded).
parseDocVerdict — витягує та валідує структуру судження (вердикт, впевненість, причина) із сирого тексту відповіді моделі.
judgeDoc — виконує семантичне судження згенерованої документації, порівнюючи її з вмістом вихідного файлу за допомогою потужної LLM.
judgeFailsDoc — визначає, чи слід вважати документацію неякісною, якщо вердикт був 'inaccurate' і його впевненість перевищує встановлений поріг.

## Публічний API

JUDGE_MODEL — Використовується модель-суддя із наймінімальнішим хмарним рівнем.
JUDGE_ENABLED — Гейт для перевірки роботи судді вмикається автоматично, якщо вказана модель `N_CLOUD_MIN_MODEL`.
JUDGE_CONFIDENCE — Мінімальний рівень впевненості, необхідний, щоб суддя визнав документ як знижений у якості (degraded), якщо вирок є неточним.
parseDocVerdict — Отримує та перевіряє структуру JSON із висновком (verdict) з відповіді LLM.
judgeDoc — Оцінює згенерований документ потужною моделлю, порівнюючи його з оригінальним джерелом.
judgeFailsDoc — Визначає, чи має вирок від судді статус зниженого документа (degraded) на підставі неточності (`inaccurate`) та високої впевненості.
