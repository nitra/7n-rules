## Огляд

Виявлення та агрегація структурних елементів для генерації документації. Модуль надає публічний інтерфейс через функції `extractAnchors`, `anchorTokens` та `anchorsToPrompt`. Він ідентифікує та обробляє необхідні метадані: витягує посилання на **URLs** (наприклад, https://h/), маркування повідомлень (згідно з rule.mdc) та конфігураційні файли (settings.local.json, .local.json, capacitor.config.json, .config.json). Операції з цими даними є read-only; модуль може здійснювати мережеві запити. Зібрані дані готуються до формування промптів для великих мовних моделей.

## Поведінка

extractAnchors витягує з коду анкори, такі як URL-адреси, імена констант-рядків, маркери помилок та посилання на конфігураційні файли.
anchorTokens повертає плоский список усіх токенів (URL-адрес, назв констант, маркерів помилок та конфігурацій), які мають бути присутні в документації.
anchorsToPrompt форматує витягнуті анкори в компактний текстовий блок, придатний для включення до системного промпта для LLM.
