## Огляд

Модуль перевіряє документацію на наявність застарілих описів, використовуючи конфігурацію з meta.json. Він збирає та звітує про застарілі описи. При виникненні помилок модуль перехоплює їх (fail-safe) і не кидає винятків назовні. За певних помилок він повертає порожнє значення замість винятку.

## Поведінка

1. Викликається `lint` для перевірки стану документації.
2. Якщо не в режимі читання та не в режимі автоматичного виправлення, відбувається збір застарілих описів.
3. Якщо застарілих описів немає, повертається 0.
4. Якщо застарілих описів є, і режим читання або режим автоматичного виправлення вимкнено, викликається `reportStale`, який друкує список застарілих файлів на stderr та повертає 1.
5. Якщо застарілих описів є, і режим автоматичного виправлення увімкнено, друкується повідомлення про спробу авто-фіксу.
6. Викликається `runGenerationBatch` для спроби автоматичного виправлення застарілих описів.
7. Після спроби виправлення, знову збираються застарілі описи.
8. Викликається `reportStale` з новим списком застарілих описів, і повертається відповідний exit-код (0 або 1).

## Публічний API

lint — виконує агрегацію для файлів документації, використовуючи LLM для виправлень.
llmFix — генерує контент за бажанням, якщо у `meta.json` вказано `llmFix:true`; інакше лише виявляє проблеми.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
