## Огляд

Модуль сканує репозиторій, щоб виявити описи, які стали застарілими або є сирітськими, порівнюючи їх із метаданими, визначеними в meta.json.

## Поведінка

1. Збирає список застарілих (відсутніх або з невідповідністю CRC) описів, або сканує весь репозиторій, якщо не вказано конкретні файли.
2. Визначає список джерел-кандидатів, які потребують перевірки, на основі змінених файлів або повного сканування.
3. Якщо вказано режим лише для детекції (readOnly) або відключено автоматичний фікс (llmFix), виконується перевірка:
    а. Якщо знайдено застарілі описи, виводиться повідомлення про їх застарілість у stderr.
    б. Якщо знайдено сирітські описи (доки, для яких видалено джерело), виводиться повідомлення про них у stderr.
    в. Повертається код виходу 1, якщо знайдено будь-які проблеми.
4. Якщо увімкнено автоматичний фікс (llmFix):
    а. Для знайдених застарілих описів ініціюється пакетна генерація (з використанням LLM, якщо доступно).
    б. Виконується очищення сирітських описів.
    в. Повторно збирається список застарілих та сирітських описів.
    г. Якщо після фіксу знайдено проблеми, виводиться повідомлення про них у stderr.
    д. Повертається код виходу 1, якщо знайдено проблеми.

## Публічний API

lint — виконує агрегацію для файлів документації з метою виявлення помилок за допомогою LLM.
llmFix — генерує виправлення для документації, якщо в конфігурації `meta.json` встановлено `llmFix:true`.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
