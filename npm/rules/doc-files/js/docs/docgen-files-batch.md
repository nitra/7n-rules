Я готовий до роботи. Будь ласка, надайте мені файл, який потрібно задокументувати.

## Поведінка

Поведінка:
selectTargets відбирає цілі для генерації документації — або всі, якщо встановлено прапор перезапису, або застарілі/деградовані документи, які ще не були перезапущені.
purgeOrphanedDocs видаляє документи, для яких більше немає відповідного вихідного файлу, а також оновлює індекси в директоріях документації.
runDocFilesGenCli ініціює генерацію документації: спочатку прибирає сирітських документів, а потім послідовно генерує документи для вибраних цілей, керуючи процесом через механізм обриву при системних збоях.
runGenerationBatch виконує послідовний прогін генерації документації для заданого набору цілей, застосовуючи механізми обриву при повторюваних системних збоях і звітністю про результати.
runDocFilesStampCli оновлює метадані (frontmatter) вже існуючих документів, заповнюючи їх актуальним хешем вихідного файлу, без використання LLM, а після цього оновлює індекси.

## Публічний API

selectTargets — визначає, які документи підлягають генерації:

- default — вибирає застарілі або "погіршені" документи, які ще не були доретраховані з оновленим CRC.
- `--overwrite` — вибирає всі документи.

Degraded-документ після невдалої спроби доретраю стає "повторним" (`retried: true`) і не змінюється, поки не зміниться джерело. При зміні джерела, він стає застарілим, і лічильник скидається. Процес завершується навіть при помилках.

purgeOrphanedDocs — видаляє документи, які посилаються на відсутні вихідні файли, та синхронізує індексний файл `index.md`. Якщо в папці `docs` залишається лише `index.md` або порожньо, ця папка очищується.

runDocFilesGenCli — виконує генерацію документації для застарілих або відсутніх документів через інструмент командного рядка `doc-files gen`.

runGenerationBatch — виконує основний цикл генерації: перевіряє стан локального бекенду, послідовно обробляє цілі (`targets`) за допомогою `generateOne` з захистом від послідовних збоїв. Завершується звітністю. Це ядро використовується як інструментом CLI, так і для перевірки лише змінених файлів.

runDocFilesStampCli — встановлює або оновлює метадані (`source` + `crc`) у існуючих документах без залучення LLM. Це робиться для міграції документів, які ще не мають CRC, зберігаючи при цьому дані про модель та якість із метаданих.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
