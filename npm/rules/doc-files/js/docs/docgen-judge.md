## Огляд

Огляд
Цей файл керує процесом валідації якості згенерованого документа, використовуючи модель-арбітра. Він визначає, чи активна перевірка (`JUDGE_ENABLED`), та встановлює критерії для винесення вердикту згенерованого документа через `judgeDoc` або `judgeFailsDoc`, порівнюючи його з вихідним вмістом.

## Поведінка

JUDGE_MODEL — вказує модель-суддю, яку використовує для оцінки якості документації.
JUDGE_ENABLED — визначає, чи активовано семантичний гейт для перевірки документації.
JUDGE_CONFIDENCE — встановлює мінімальний поріг впевненості для відзначення документації як деградованої через неточність.
parseDocVerdict — витягує та валідує семантичний вердикт із текстової відповіді моделі-судді.
judgeDoc — оцінює згенеровану документацію, порівнюючи її з вмістом вихідного файлу, і повертає вердикт судді.
judgeFailsDoc — визначає, чи має документ бути відзначений як деградований, на основі вердикту та порогового рівня впевненості.

## Публічний API

JUDGE_MODEL — використовує хмарну модель `N_CLOUD_MIN_MODEL` як суддю.
JUDGE_ENABLED — автоматично вмикає гейт, якщо обрано `N_CLOUD_MIN_MODEL`.
JUDGE_CONFIDENCE — визначає мінімальний рівень впевненості, щоб оцінити документ як знижений через неточність.
parseDocVerdict — витягує та перевіряє JSON з рішенням (verdict) із відповіді LLM.
judgeDoc — оцінює згенерований документ потужною моделлю порівняно з оригіналом.
judgeFailsDoc — визначає, чи слід вважати документ зниженим (degraded) на підставі рішення `inaccurate` з високою впевненістю.
