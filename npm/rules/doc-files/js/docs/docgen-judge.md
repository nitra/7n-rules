# docgen-judge.mjs

## Огляд

Модуль визначає статус механізму судження через `JUDGE_ENABLED` та встановлює модель судді за допомогою `JUDGE_MODEL`. Він встановлює поріг впевненості через `JUDGE_CONFIDENCE`. Модуль виконує судження над документацією за допомогою `judgeDoc` та визначає, чи документація є неприйнятною за допомогою `judgeFailsDoc`. Результати судді вилучаються та валідуються за допомогою `parseDocVerdict`.

## Поведінка

JUDGE_ENABLED — Вмикає семантичний verdict-гейт лише за явним налаштуванням середовища.
JUDGE_MODEL — Визначає модель, яку використовує суддя для оцінки документації.
JUDGE_CONFIDENCE — Встановлює мінімальний рівень впевненості, необхідний для позначення документації як деградованої.
parseDocVerdict — Витягує та валідує JSON-об'єкт з відповіді судді, перевіряючи його відповідність схемі.
judgeDoc — Судить згенеровану документацію сильною моделлю, порівнюючи її з вмістом вихідного файлу.
judgeFailsDoc — Визначає, чи слід вважати документацію деградованою, якщо verdict судді є 'inaccurate' і його впевненість перевищує встановлений поріг.

## Публічний API

JUDGE_ENABLED — Увімкнення функціоналу судді лише при встановленні змінної середовища `N_CURSOR_DOCGEN_JUDGE=1`.
JUDGE_MODEL — Вибір моделі-судді, де значення змінної середовища має пріоритет над вибором моделі за замовчуванням (`min`).
JUDGE_CONFIDENCE — Мінімальний рівень впевненості, необхідний для позначення документа як зниженої якості (`degraded`) за вердиктом `inaccurate`.
parseDocVerdict — Видобуває та перевіряє структуру вердикту у форматі JSON із сирого виводу великої мовної моделі.
judgeDoc — Оцінює згенерований документ за допомогою потужної моделі, порівнюючи його з вихідним джерелом.
judgeFailsDoc — Визначає, чи повинен документ бути позначений як зниженої якості, якщо вердикт є `inaccurate` і відповідає мінімальній впевненості.
