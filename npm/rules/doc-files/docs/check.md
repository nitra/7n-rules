Модуль перевіряє відповідність документації встановленим правилам, використовуючи конфігурації, описані у package.json. Він порівнює логіку JavaScript, політику та посилання на MDC з фактичним вмістом. Мета — виявити структурні та контентні порушення, такі як відсутність або застарілі документи.

## Поведінка

1. Викликається `run` для запуску перевірки.
2. Перевірка застосовує правила: конфігурації, логіку JS, політику та посилання на MDC.
3. Структурні вимоги (наявність `workflow lint-doc-files.yml` та скрипт у `package.json`) задовольняються.
4. Контентні порушення (відсутні або застарілі документи) виявляються.
5. Якщо код виконується як окремий інструмент, запускається процес, що відповідає за виконання правила.

## Публічний API

run — виконує послідовність перевірок: застосовує правила для файлів документації, перевіряє відповідність JS-зацікавленостей та політикам, а також перевіряє посилання на MDC.
