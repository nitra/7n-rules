## Огляд

Цей модуль реалізує спеціалізований fix-worker для правила `doc-files`, який замінює автономний pi-agent контрольованим пайплайном. Він парсить stale-файли з `violation-output` та передає їх у `runGenerationBatch` через публічну функцію `runDocFilesFixWorker`. Після генерації якість кожного файлу оцінюється: файли з $\text{score} < \text{QUALITY\_THRESHOLD}$ або $\text{null}$ повертають `{ applied: false }` та видаляються, щоб наступний рунг інтерпретував їх як `missing`. Виняток становить останній рунг (`isAvg=true`), який зберігає результати навіть при низькій якості, відмічаючи їх з `tier:cloud-avg`. Контракт цього worker-seam (`orchestrator.mjs`) гарантує повернення об'єкта `{ applied, touchedFiles, error, rollback }`.

## Поведінка

Поведінка:

1.  Викликати runDocFilesFixWorker, який парсить вивід обмеження, щоб визначити списки файлів, які потребують генерації документації.
2.  Якщо списки файлів відсутні, операція завершується з невдачею.
3.  Викликати runDocFilesFixWorker для передачі визначених цілей у пакетну генерацію.
4.  Після генерації перевірити якість кожного згенерованого файлу.
5.  Якщо якість файлу не відповідає встановленому порогові значенню або не визначена, і не встановлений прапор `isAvg`, то видалити цей файл.
6.  Якщо якість файлу відповідає порогові значенню, зберегти файл.
7.  Якщо встановлено прапор `isAvg`, то зберігати всі згенеровані файли, незалежно від їхньої якості.
8.  У разі будь-якої помилки, відновити стан файлової системи за допомогою механізму відкату.
