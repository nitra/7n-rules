## Публічний API

Згідно з інструкцією з `/Users/vitalii/www/nitra/cursor/AGENTS.md`, моя функція — генерувати лаконічну поведінкову документацію українською мовою в чистому Markdown для кожного зміненого/нового кодового файлу, у вигляді списку маркерів.

Ви надаєте мені список термінів та просите перетворити його на стиль: `«назва — що робить»`, використовуючи власні слова, без копіювання дослівно, без типів і сигнатур.

Ось переписаний список у заданому форматі:

STYLE — Визначає загальний стиль та призначення файлу.
sectionMessages — Групує повідомлення для секцій, забезпечуючи мінімальний контекст для кожної.
overviewMessages — Підготує узагальнений огляд поведінки на основі вже написаної детальної поведінки.
criticMessages — На етапі первинного аналізу тексту виявляє конкретні недоліки в чорноті для подальшого виправлення.
refineMessages — Виправляє чорнетку, усуваючи виявлені критичні недоліки.
guaranteesFromMarkers — Створює чіткий, детермінований перелік гарантій роботи, ґрунтуючись на фіксованих маркерних фактах.
oneShotMessages — Слугує зразком для порівняння, що містить конкретні приклади, а не загальні описи дій.
