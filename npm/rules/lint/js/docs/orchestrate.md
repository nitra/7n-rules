Модуль забезпечує виконання лінтування коду. Він завантажує правила лінтування з конфігурацій `.n-cursor.json` та `meta.json`. Публічна функція `selectLintRules` вибирає ці правила. Функція `runLint` виконує лінтування у режимі перевірки або режимі виправлення.

## Поведінка

selectLintRules вибирає і алфавітно сортує ідентифікатори правил для лінтування на основі їхньої видимості в конфігурації та визначеного обсягу (пер-файл або повний).
runLint запускає процес лінтування, виконуючи або перевірку (read-only), або виправлення (fix) залежно від наданих параметрів, з можливістю обмеження обсягу до конкретних правил.

## Публічний API

selectLintRules — обирає ідентифікатори правил для контексту, у алфавітному порядку.
runLint — ініціює процес лінтування.
  full — сканує весь репозиторій, порівнюючи його з базовою версією.
  readOnly — лише виявляє проблеми без внесення змін.
  rules — виконує повне сканування лише для вказаного набору правил.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
