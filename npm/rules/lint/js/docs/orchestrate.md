## Огляд

Модуль керує процесом лінтування коду. Він визначає активні правила лінтування, спираючись на конфігурації `meta.json` та `.n-cursor.json`. За допомогою публічних функцій, він може вибрати ці правила (`selectLintRules`) або запустити перевірку коду (`runLint`), виконуючи перевірку відповідно до обраних правил.

## Поведінка

selectLintRules вибирає і алфавітно сортує ідентифікатори правил для лінтування на основі їхньої конфігурації та активності в `.n-cursor.json`, враховуючи, чи потрібен повний прогін.
runLint запускає процес лінтування: виконує перевірку для змінених файлів (у режимі дельти) або для всього репозиторію (у режимі `--full`), застосовуючи правила з `meta.json` та `.n-cursor.json`, а також виконує конформність та форматування у режимі виправлення.

## Публічний API

selectLintRules — обирає ідентифікатори правил для контексту, у алфавітному порядку.
runLint — ініціює процес лінтування.
  full — сканує весь репозиторій (якщо `true`) або лише зміни порівняно з початковим станом (якщо `false`, за замовчуванням).
  readOnly — лише виявляє проблеми без їх виправлення (якщо `true`) або виконує виправлення (якщо `false`, за замовчуванням).
  rules — при обмеженому обсязі (scope) виконує повне сканування лише обраних правил (лінтер та відповідність, по всьому репозиторію).

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
