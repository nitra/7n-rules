## Огляд

Модуль керує процесом лінтування коду. Він вибирає активні правила лінтування, спираючись на конфігурацію в meta.json та .n-cursor.json. Після вибору правил, модуль ініціює запуск лінтування за допомогою runLint.

## Поведінка

selectLintRules вибирає і алфавітно сортує ідентифікатори правил для лінтування на основі їхньої конфігурації та активності в `.n-cursor.json`, з можливістю обмеження лише правилами, що застосовуються до окремих файлів.
runLint запускає повний процес лінтування, який може включати перевірку змінених файлів відносно базової точки, виконання лінтера для активних правил, а в режимі `--full` — аналіз конформності та форматування в режимі виправлення.

## Публічний API

selectLintRules — вибирає ідентифікатори правил для контексту в алфавітному порядку.
runLint — ініціює процес лінтування.
  full — виконує повний аналіз репозиторію, порівнюючи його з початковим станом.
  readOnly — лише виявляє проблеми без внесення змін.
  rules — виконує повний прогін лише для вказаних правил у заданому обсязі.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
