## Огляд

T0-autofix для `image-avif/avif_generation` керує автоматичним оптимізаційним процесом для зображень. Він генерує `.avif` двійники за допомогою команди `npx @nitra/minify-image --avif`, а потім замінює растрові посилання у файлах `.vue`/`.html` на посилання на `<path>.avif`. Логіка, яка раніше лише звітувала про виявлені проблеми у `detector-і`, тепер тут відповідає за мутації. Процес забезпечує ідемпотентність: після генерації `.avif` повторне сканування (`scanAvif`) гарантує, що згенеровані файли стають видимими, а відсутні посилання — заміненими. Досягається це через структуру порушень: `test/apply`.

## Поведінка

1. Перевіряє, чи існують порушення, пов'язані з відсутніми, що вимагають переписування або "сирітськими" зображеннями у контексті AVIF.
2. Якщо порушення присутні, запускає процес генерації AVIF-двійників за допомогою зовнішнього інструменту.
3. Після генерації виконує повторне сканування, щоб ідентифікувати необхідні дії.
4. Для кожного знайденого файлу, що потребує переписування, замінює растрове посилання у відповідному файлі на посилання на `.avif`.
5. Для кожного "сирітського" файлу `.avif` видаляє його.
6. Повертає список змінених файлів та повідомлення про виконані дії.
