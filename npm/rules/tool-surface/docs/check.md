## Огляд

Модуль виконує перевірки в межах одного прогону, використовуючи кеш, що формується під час цього прогону. При виклику публічної функції `run` він завантажує конфігурацію та перелік дозволених елементів для формування підсумку.

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується з використанням контексту прогону, що може містити кеш.
3. Якщо скрипт запускається як окрема утиліта (standalone), виконується оркестрація правил, яка включає завантаження конфігурації, перелік дозволених елементів та підсумок.

## Публічний API

run — виконує послідовність дій: застосовує правила, перевіряє JS-занепокоєння, застосовує політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
