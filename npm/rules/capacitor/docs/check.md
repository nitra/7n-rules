Модуль виконує перевірку, використовуючи контекст прогону, який може містити кешовані дані, що зберігаються лише у межах поточного прогону. При запуску як окрема утиліта, публічна функція `run` ініціює повну оркестрацію правила, включаючи завантаження конфігурації, білого списку та підсумку. Перевірка не змінює стан файлової системи чи бази даних.

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується з використанням контексту прогону, що може містити кеш.
3. Якщо скрипт виконується як окрема утиліта, запускається повна оркестрація правила, що включає завантаження конфігурації, білий список та підсумок.

## Публічний API

run — виконує послідовність кроків: застосування правил, обробку JS-зацікавлень, перевірку політик та посилання на MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
