## Огляд

Оскільки ви просите переписати секцію "overview" (Огляд), прибравши перелічені issues та дотримуючись усіх вимог технічного письменника, я зосереджуся на конкретизації механізму ізоляції та доданіму необхідний маркер повідомлення згідно з `test.mdc`.

Test coverage $\text{test.mdc}$

Цей файл відповідає за збір та виконання всіх тестових сценаріїв у проєкті. Для забезпечення ізоляції кожного тестового завдання, процес `forks` створює окремі середовища виконання. Файл збирає тестові файли згідно з конвенціями, ігноруючи директорії залежностей та збірки. Після виконання тестів вимірюється покриття коду за допомогою провайдера `v8`, що генерує звіти у форматах `lcov` та `text-summary`.

## Поведінка

Поведінка:

1. Збирає тестові файли з усіх директорій, що відповідають конвенціям (`**/*.test.{js,mjs}` та `tests/**/*.test.{js,mjs}`).
2. Ігнорує директорії з встановленими залежностями (`**/node_modules/**`), згенеровані файли (`**/dist/**`) та результати збірки Stryker (`**/reports/stryker/**`).
3. Виконує тести у середовищі `node`.
4. Ізолює тестові процеси за допомогою `forks`, що забезпечує відокремлення кожного тестового завдання.
5. Вимірює покриття коду, використовуючи провайдер `v8` та створюючи звіти у форматах `lcov` та `text-summary`.
