## Огляд

Надайте код або файл, який потрібно документувати, щоб я міг створити секцію "Огляд" відповідно до вимог.

## Поведінка

1. Викликається `main` для ініціалізації перевірки.
2. Визначається, чи увімкнено перевірку для JavaScript. Якщо ні, перевірка завершується.
3. Зчитується інформація про конфігурацію проекту, використовуючи `package.json` як джерело прав.
4. Визначається список усіх кореневих каталогів, що містять JavaScript.
5. Перевіряються наявність канонічних файлів конфігурації (`stryker.config.baseline.mjs`, `stryker.config.vue.baseline.mjs`, `stryker-vue-macros-ignorer.mjs`, `vitest.config.baseline.js`). Якщо хоча б один відсутній, перевірка завершується.
6. Для кожного кореневого JavaScript-каталогу виконується наступне:
   а. Визначається, чи містить каталог файли Vue.
   б. Базовий конфігураційний файл Stryker копіюється до цільового місця, замінюючи рядок з назвою конфігу на фактичну назву конфігу, знайденого в каталозі.
   в. Якщо каталог Vue, виконується поглиблене доповнення конфігурації Stryker для додавання плагінів, необхідних для обробки Vue-макросів, до існуючого файлу, з гарантуванням збереження форматування.
   г. Додатковий конфігураційний файл для Vue створюється/оновлюється.
   д. Базовий конфігураційний файл Vitest створюється/оновлюється.
7. Додаються до файлу `.gitignore` шаблони для ігнорування артефактів тестування Stryker та покриття.
8. Повертається кінцевий код виходу перевірки.
9. Код свідомо не перевіряє шляхи в каталозі `node_modules`.
10. Усі внутрішні помилки перехоплюються, і перевірка завершується з відповідним статусом, що запобігає вильоту.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- Свідомо пропускає шляхи: `node_modules`.
