## Огляд

Детектор concern-а `vitest-api-conventions`: перевіряє, що жоден тестовий файл проєкту
не викликає `expect(...).toBe(...)` з об'єктним чи масивним літералом як першим аргументом.
Така перевірка завжди хибна незалежно від вмісту (reference equality на новоствореному
значенні) — канонічна заміна для об'єктів/масивів — `toEqual` (deep equality).

## Поведінка

Обходить дерево проєкту (пропускаючи `node_modules` і шляхи з cursor-ignore) і збирає всі
файли з іменем `*.test.mjs`/`*.test.js`. У кожному файлі шукає виклики `.toBe(`, чий перший
аргумент — саме об'єктний (`{...}`) чи масивний (`[...]`) літерал: сканує парність дужок
з урахуванням рядкових і template-літералів усередині, щоб не збитись на дужки в рядках.
Виклик рахується порушенням лише якщо одразу після закриваючої дужки літерала (з пропуском
пробілів) іде дужка, що закриває сам виклик, `)` — тобто ланцюжок на кшталт
`.toBe([...].join('\n'))` не матчиться, бо результат виклику — рядок-примітив, а не
посилання на масив/обʼєкт.

Для кожного знайденого випадку формує порушення з файлом (відносний шлях від cwd) і
номером рядка.

## Публічний API

`lint(ctx)` — читає всі тестові файли проєкту (`ctx.cwd`), перевіряє їх на заборонений
патерн `toBe` з об'єктним/масивним літералом і повертає список порушень
(`{ file, message, reason }` на кожен знайдений виклик), або порожній результат, якщо
жоден тестовий файл не порушує конвенцію (test.mdc, vitest-api-conventions).

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД) — лише читає файли й повертає порушення.
- Сканує лише `*.test.mjs`/`*.test.js`; інші файли (в т.ч. `node_modules`) ігноруються.
