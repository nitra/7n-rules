## Огляд

Огляд
Файл гарантує відповідність усіх тестових файлів встановленому розташуванню. Він сканує репозиторій, щоб підтвердити, що всі файли з розширенням `*.test.mjs` знаходяться у директорії `tests`.

Поведінка
Запускає `main` для перевірки коректності розташування всіх тестових файлів (`*.test.mjs`). Сканує репозиторій, ігноруючи директорії, визначені у конфігурації. Для кожного тестового файлу перевіряє його наявність у каталозі `tests`. Якщо тест виявлено поза директорією `tests`, фіксується його повний відносний шлях. У випадку відсутності порушень, інформується, що всі тестові файли розташовані коректно (test.mdc). Якщо виявлено порушення, для кожного файлу, що не відповідає правилам (test.mdc), повідомляється його шлях та необхідність переміщення до відповідної піддиректорії `tests`.

## Поведінка

1. Запускає `main` для перевірки розташування всіх тестових файлів з розширенням `*.test.mjs`.
2. Сканує репозиторій, ігноруючи директорії, вказані в конфігурації.
3. Для кожного знайденого тестового файлу перевіряє, чи він знаходиться в каталозі з назвою `tests`.
4. Якщо тест знайдено поза директорією `tests`, фіксує його повний відносний шлях.
5. Якщо порушень немає, повідомляє, що всі тестові файли розташовані коректно (test.mdc).
6. Якщо порушення виявлено, для кожного файлу, що порушує правила (test.mdc), повідомляє шлях та вказує, що тест має бути переміщено до відповідної піддиректорії `tests`.

## Публічний API

- main — Встановлює, чи знаходяться файли, призначені для тестування, у відповідному каталозі `tests/` (test.mdc).
