## Огляд

Реалізує логіку виконання правила. Викликає публічну функцію `run`, використовуючи контекст прогону, який може містити дані, кешовані у межах поточного прогону. При запуску як окрема програма, ініціалізує систему, завантажує конфігурацію та перевіряє білий список, після чого завершує роботу з відповідним кодом виходу.

## Поведінка

1. Викликається функція `run` для запуску правила.
2. Правило виконується, використовуючи контекст прогону, який може містити кеш.
3. Якщо скрипт виконується як окрема програма (standalone), запускається оркестрація правила, що включає завантаження конфігурації, перевірку білого списку та підсумок.
4. Завершення роботи скрипта відбувається з кодом виходу, визначеним під час оркестрації.

## Публічний API

run — виконує послідовність дій: застосовує правила, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
