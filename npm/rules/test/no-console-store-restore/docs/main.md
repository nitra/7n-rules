## Огляд

Перевіряє відповідність тестової логіки стандарту, забобонячи пряме присвоєння методів об'єкта `console` у JS-тестах. Забезпечує використання рекомендованого механізму мокування (`vi.spyOn.mockReturnValue`) для збереження цілісності тестового середовища.

Поведінка:
main запускає перевірку вказаного кореня репозиторію.
Збирається список усіх файлів, що відповідають шаблону JS-тестів.
Для кожного знайденого тестового файлу зчитується його вміст.
Перевіряється наявність прямого присвоєння методів об'єкта `console` (наприклад, `console.log = ...`).
Якщо порушень не знайдено, виводиться успішне повідомлення згідно з (test.mdc).
У разі виявлення порушення (пряме присвоєння), фіксується місце інциденту (файл та рядок) і генерується повідомлення про помилку, яке рекомендує замінити присвоєння на `vi.spyOn.mockReturnValue`, відповідно до (test.mdc, no-console-store-restore).
Виводиться відповідний код завершення.

## Поведінка

Поведінка:

1. main викликається для запуску перевірки вказаного кореня репозиторію.
2. Збирається список усіх файлів, що відповідають шаблону JS-тестів.
3. Для кожного знайденого тестового файлу зчитується його вміст.
4. Перевіряється, чи містить вміст пряме присвоєння методів об'єкта `console` (наприклад, `console.log = ...`).
5. Якщо жодного такого присвоєння не знайдено у жодному тестовому файлі, система повідомляє про успіх згідно з (test.mdc).
6. Якщо знайдено порушення (пряме присвоєння), система фіксує місце порушення (файл та рядок) і повідомляє про помилку, рекомендуючи замість цього використовувати `vi.spyOn.mockReturnValue` згідно з (test.mdc, no-console-store-restore).
7. Виводиться відповідний код завершення.

## Публічний API

main — перевіряє, що файли тесту не перевизначають консольні методи через пряме присвоєння, вимагаючи використання `vi.spyOn` для мокування.
