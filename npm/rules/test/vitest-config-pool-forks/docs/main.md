## Огляд

Огляд:
Перевіряє конфігураційний файл Vitest для валідації наявності ізоляції тестів через режим `forks`. Це забезпечує відсутність гонок даних при паралельному виконанні тестів.

Поведінка:

1. Викликається публічна функція `main` для аналізу конфігураційного файлу Vitest.
2. Перевірка пропущена, якщо файли `vitest.config.mjs` або `vitest.config.js` відсутні.
3. При знаходженні файлу його вміст зчитується.
4. Перевірка успішна, якщо вміст файлу містить шаблон `pool: 'forks'` (test.mdc).
5. Перевірка провалена, якщо вміст файлу не містить шаблон `pool: 'forks'` (test.mdc), оскільки це обов'язковий захист від гонок даних у паралельних тестах.

## Поведінка
