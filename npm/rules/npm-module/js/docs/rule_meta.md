## Огляд

Модуль перевіряє відповідність набору правил, визначених у каталогах, конфігурації, що описується у meta.json. Він валідує наявність файлу `auto.md` та коректність метаданих для кожного правила. Також він перевіряє коректність полів `auto` та `lint` у метаданих, забезпечуючи відповідність предикатів та типів експортованих функцій з файлу `main.mjs` вимогам, встановленим у meta.json.

## Поведінка

1. Викликається функція check.
2. Створюється репортер для фіксації результатів валідації.
3. Визначається каталог, де розташовані правила.
4. Якщо каталог правил відсутній, фіксується успіх, оскільки немає правил для перевірки.
5. Для кожного каталогу правила у визначеному місці виконується перевірка правила.
6. Перевірка правила включає:
    а. Перевірку на наявність файлу `auto.md` у каталозі правила; якщо він є, фіксується помилка.
    б. Зчитування метаданих правила з `meta.json`. Якщо файл відсутній або невалідний, фіксується помилка.
    в. Перевірку поля `auto` у метаданих. Якщо поле присутнє, перевіряється його валідність:
        i. Якщо поле нерозпізнане, фіксується помилка.
        ii. Якщо використовується `predicate`, перевіряється, чи існує цей предикат у визначених правилах. Якщо ні, фіксується помилка.
    г. Перевірку поля `lint` у метаданих. Якщо поле присутнє, перевіряється його валідність:
        i. Якщо поле нерозпізнане, фіксується помилка.
        ii. Зчитується вміст `main.mjs` у каталозі правила.
        iii. Перевіряється, чи експортує `main.mjs` функцію `lint` відповідно до значення поля `lint`. Якщо ні, фіксується помилка.
7. Якщо всі перевірки для правила пройшли успішно, фіксується успіх.
8. Після обробки всіх правил повертається код виходу репортера.

## Публічний API

check — перевіряє відповідність усіх файлів `npm/rules/<id>/meta.json` встановленим вимогам.
