Модуль виконує набір правил у контексті прогону, використовуючи дані, кешовані протягом поточного прогону. Публічна функція `run` запускає логіку, яка завантажує конфігурацію та застосовує визначений набір правил. Результат роботи визначає код виходу процесу: 0 при успіху або 1 при виявленні порушень.

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується з використанням контексту прогону, що може включати кеш.
3. Якщо скрипт виконується як окрема утиліта командного рядка, запускається оркестрація правил.
4. Оркестрація правил виконує завантаження конфігурації, застосовує список дозволених правил та підсумовує результати.
5. Результат роботи визначає код виходу процесу: 0 — успіх, 1 — виявлено порушення.

## Публічний API

run — виконує послідовність дій: застосовує правила, аналізує JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
