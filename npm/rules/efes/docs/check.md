Модуль виконує перевірки на відповідність заданим правилам. Він запускає перевірку у бібліотечному режимі, використовуючи кешовані дані, або ініціює повний запуск правила, якщо викликається як окрема утиліта. Повний запуск включає завантаження конфігурації, застосування білих списків для фільтрації, та підбиття підсумків, результат якого визначає код виходу.

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується у бібліотечному режимі, використовуючи контекст прогону для доступу до кешованих даних.
3. Якщо скрипт виконується як окрема утиліта командного рядка, ініціюється повний запуск правила.
4. Повний запуск правила включає завантаження конфігурації, застосування білих списків та підбиття підсумків.
5. Результат виконання визначає код виходу процесу.

## Публічний API

run — виконує послідовність дій: застосовує правила, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
