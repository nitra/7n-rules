Модуль визначає унікальні корені каталогів із іменем `k8s` шляхом пошуку файлів `*.yaml`. Якщо такі файли відсутні, виконання завершується з кодом 0 без запуску зовнішніх утиліт. Для знайдених коренів виконується перевірка маніфестів. `kubeconform` валідує маніфести проти OpenAPI-схем Kubernetes (https://github.com/yannh/kubeconform#readme), при цьому його версія узгоджується з конфігурацією. `kubescape` сканує маніфести на відповідність стандартам (NSA, MITRE, CIS тощо), використовуючи файл `.kubescape-exceptions.json` для винятків, і орієнтується на цільову лінію релізу, що відповідає конфігурації `kubeconform`.

## Поведінка

pathHasK8sSegment визначає, чи містить шлях сегмент директорії `k8s` відносно кореня репозиторію, і ігнорує шляхи в `.github/`.
k8sRootFromFile знаходить абсолютний шлях до каталогу `…/k8s`, піднявшись від абсолютного шляху до YAML-файлу.
findK8sRoots знаходить унікальні абсолютні шляхи до каталогів `k8s`, скануючи файли `*.yaml` у корені репозиторію, ігноруючи шляхи, визначені конфігурацією.
buildKubescapeExceptionsArgs створює аргументи для `kubescape`, якщо в корені репозиторію існує файл `.kubescape-exceptions.json`.
findKustomizationDirs знаходить абсолютні шляхи до каталогів, що містять білдабельний `kustomization.yaml` у межах каталогу `…/k8s`, виключаючи компоненти.
runLintK8s виконує повний лінтинг Kubernetes-маніфестів, запускаючи `kubeconform` для перевірки схем та `kubescape` для сканування конфігурації для всіх знайдених каталогів `k8s`.

## Публічний API

pathHasK8sSegment — визначає, чи містить шлях сегмент `k8s` у структурі проєкту.
k8sRootFromFile — знаходить найближчий до файлу каталог `k8s`, що містить маніфест.
findK8sRoots — збирає список усіх унікальних коренів `k8s` у поточному робочому каталозі, якщо знайдено файли YAML.
buildKubescapeExceptionsArgs — формує список файлів винятків для інструменту kubescape, якщо існує конфігураційний файл `.kubescape-exceptions.json`.
findKustomizationDirs — знаходить каталоги, що містять файл `kustomization.yaml` та слугують точками входу для Kustomize.
runLintK8s — виконує перевірку Kubernetes-маніфестів, використовуючи механізм блокування та дедуплікації на основі стану Git.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- Свідомо пропускає шляхи: `.github`, `.git`.
