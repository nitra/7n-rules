## Огляд

Функціонал здійснює поверхневий linting для файлів Kubernetes (`.yaml`/`.yml`) з каталогу `k8s/` за допомогою `kubeconform` для валідації схеми. Доступний у двох режимах: приймає список конкретних файлів через `ctx.files`, або сканує корені `k8s/` для повного аналізу. Валідація застосовується до кожного маніфеста індивідуально, ігноруючи крос-файловий стан. Цей детектор працює лише з читанням даних (read-only); він не виконує жодних записів у файлову систему чи базу даних.

## Поведінка

Поведінка:
1. Якщо контекст лінту не містить переліку файлів, система шукає кореневі директорії Kubernetes-маніфестів у поточному робочому каталозі, ігноруючи конфігураційні шляхи.
2. Якщо контекст лінту надає список файлів, система обробляє лише ті файли, які мають розширення `.yaml` або `.yml`.
3. Система запускає валідацію всіх визначених маніфестів за допомогою інструменту `kubeconform`.
4. Якщо `kubeconform` виявляє невалідні маніфести, спрацьовує порушення відповідно до правила (k8s.mdc).
5. Система повертає результат лінтування.

## Публічний API

lint — знаходить помилки у конфігураціях Kubernetes за допомогою kubeconform.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
