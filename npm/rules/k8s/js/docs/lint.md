## Огляд

Цей файл знаходить унікальні корені каталогів з іменем `k8s` у репозиторії, виявляючи їх за шляхами файлів з розширенням `*.yaml`. При відсутності таких файлів, процес завершується з кодом виходу 0 без виклику зовнішніх інструментів. Для знайдених коренів виконується перевірка маніфестів: `kubeconform` валідує YAML-маніфести проти OpenAPI-схем Kubernetes (https://github.com/yannh/kubeconform#readme), використовуючи узгоджену версію Kubernetes. `kubescape` сканує маніфести на відповідність стандартам (NSA, MITRE, CIS тощо), враховуючи винятки, визначені у файлі `.kubescape-exceptions.json` (https://github.com/kubescape/kubescape#readme).

## Поведінка

pathHasK8sSegment визначає, чи містить шлях сегмент директорії `k8s` відносно кореня репозиторію, і ігнорує шляхи, що належать до `.github/`.
k8sRootFromFile знаходить абсолютний шлях до каталогу `…/k8s`, піднявшись від абсолютного шляху до YAML-файлу.
findK8sRoots знаходить унікальні абсолютні шляхи до каталогів `k8s` за наявності файлів `*.yaml` під деревом кореня, ігноруючи шляхи, що належать до `.github/`.
buildKubescapeExceptionsArgs будує аргументи `--exceptions <file>` для kubescape, якщо існує файл `.kubescape-exceptions.json` у корені репозиторію.
findKustomizationDirs знаходить абсолютні шляхи до каталогів, що містять білдабельний `kustomization.yaml` у межах каталогу `…/k8s`, виключаючи компоненти (`kind: Component`).
runLintK8s виконує повний цикл перевірки Kubernetes-маніфестів: запускає kubeconform для всіх знайдених каталогів `k8s`, а потім запускає kubescape для сканування, використовуючи Kustomize для збірки маніфестів, якщо це можливо.
lint делегує виконання повного циклу перевірки Kubernetes-маніфестів функції `runLintK8s`.

## Публічний API

pathHasK8sSegment — Визначає, чи містить шлях сегмент директорії `k8s`.

k8sRootFromFile — Знаходить каталог `…/k8s`, що містить маніфест, рухаючись вгору від заданого файлу.

findK8sRoots — Збирає список унікальних коренів `k8s`, якщо в поточній директорії знайдено файли `*.yaml`.

buildKubescapeExceptionsArgs — Формує аргументи для інструменту kubescape, вказуючи файл `.kubescape-exceptions.json`, якщо він присутній у корені проєкту.

findKustomizationDirs — Виявляє каталоги, які слугують точками входу для Kustomize, оскільки містять `kustomization.yaml` з відповідним вмістом.

runLintK8s — Виконує перевірку Kubernetes-маніфестів, використовуючи механізм блокування та дедуплікації на основі стану Git-дерева.

lint — Ініціює процес перевірки Kubernetes-маніфестів через інтерфейс командного рядка.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- Свідомо пропускає шляхи: `.github`, `.git`.
