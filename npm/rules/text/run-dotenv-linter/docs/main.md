## Огляд

Модуль забезпечує валідацію файлів `.env*` для конфігурації середовища. Викликаючи `runDotenvLinter`, здійснюється два етапи: спочатку автоматичне усунення помилок (`fix`) за допомогою `dotenv-linter`, який рекурсивно сканує файли, ігноруючи `node_modules` та `.envrc`, а також `.bak`-файли. Після виправлення виконується фінальна перевірка (`check`), щоб підтвердити відповідність стандарту. Доступні деталі проєкту можна знайти за посиланням https://git.io/JLbXn. Якщо `dotenv-linter` не встановлений, система інформує про необхідність його встановлення.

## Поведінка

Поведінка:

1. Визначається шлях до інструменту `dotenv-linter`. Якщо інструмент відсутній у системному шляху, виводиться інформація про його встановлення, зокрема посилання https://git.io/JLbXn, та повертається код помилки.
2. Визначаються шляхи, які виключаються з перевірки, зокрема `node_modules` та `.envrc`.
3. Якщо виклик не виконується в режимі `readOnly`, ініціюється перший прогін `dotenv-linter` у режимі `fix` для автоматичного виправлення виявлених проблем у файлах `.env*`.
4. Після потенційного авто-фіксу ініціюється другий прогін `dotenv-linter` у режимі `check`.
5. Якщо `checkRun` повертає нульовий код виходу, процес вважається успішним.
6. У разі помилки виконання або виявлення залишкових порушень у `checkRun`, виводиться відповідне повідомлення, а функція `runDotenvLinter` повертає код помилки.

## Публічний API

runDotenvLinter — автоматично виправляє помилки у конфігурації середовища та завершує перевірку. https://git.io/JLbXn

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Свідомо пропускає шляхи: `node_modules`.
