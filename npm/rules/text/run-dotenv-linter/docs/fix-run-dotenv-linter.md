## Огляд

Файл задає T0-autofix для `text/run-dotenv-linter`, перенесений із `text/check/fix-check.mjs` за `docs/specs/2026-07-02-text-check-per-file-split-design.md §1`. Публічна `patterns` вмикає сценарій, у якому достатньо факту, що `dotenv-linter` щось знайшов: виправлення делегується `runDotenvLinter` / `dotenv-linter fix`, без per-violation даних і без власного запису у ФС. Файл існує, щоб lint-система могла безпечно підключати autofix для `.env*`, свідомо пропускати `node_modules`, перехоплювати помилки fail-safe і для певних помилок повертати порожнє значення замість винятку.

## Поведінка

1. `patterns` оголошує T0-autofix для порушень `text/run-dotenv-linter`, щоб автоматично застосовувати виправлення `.env*` файлів через зовнішній dotenv-linter.

2. `patterns` спрацьовує лише тоді, коли серед порушень є факт знаходження проблеми dotenv-linter; деталі окремих порушень для виправлення не потрібні.

3. `patterns` перед виправленням знаходить `.env*` файли в межах репозиторію, включно з файлами, які можуть бути git-ignored.

4. `patterns` свідомо пропускає шляхи всередині `node_modules`, а також службові або резервні env-файли, які не мають оброблятися як звичайні dotenv-файли.

5. Якщо env-файлів немає або обхід файлів недоступний, `patterns` завершує роботу без змін і повертає порожній список змінених файлів.

6. `patterns` фіксує початковий вміст знайдених env-файлів, запускає автоматичне виправлення та порівнює результат, щоб повідомити лише про реально змінені файли.

7. Для кожного реально зміненого файлу `patterns` реєструє запис у контексті виконання, щоб lint-система коректно врахувала внесені autofix-зміни.

8. Якщо виправлення змінили файли, `patterns` повертає список змінених файлів і коротке повідомлення з їх кількістю; якщо змін немає — повертає порожній результат.

9. Помилки читання окремих файлів обробляються fail-safe: замість винятку використовується порожнє значення для порівняння, щоб autofix не переривав увесь lint-процес.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
- Свідомо пропускає шляхи: `node_modules`.
