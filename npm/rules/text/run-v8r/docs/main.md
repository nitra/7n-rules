## Огляд

Цей скрипт реалізує пакетну перевірку синтаксису та відповідності схем для файлів у форматах `json`, `json5`, `yaml`, `yml` та `toml` за один виклик `lint-text`, замість чотирьох окремих викликів `v8r`. Для забезпечення валідації використовується каталог схем `@nitra/cursor` (`v8r-catalog.json` з каталогу `schemas` пакета), який передається в `v8r` як прапор `-c`. Якщо не вказані користувачем `glob` шаблони, скрипт автоматично сканує типові шаблони для `.json`, `.json5`, `.yml`, `.yaml`, `.toml` у дереві проєкту. Важливо: один процес `v8r` завершується кодом 98, якщо хоча б один із наданих `glob` не знаходить файлів, що запобігає перевірці інших розширень. При коді виходу 0 або 98 вивід `v8r` прихований; інакше, вивід друкується, і процес завершується з кодом першого невдалого `v8r`.

## Поведінка

Поведінка:
DEFAULT_V8R_GLOBS — Визначає стандартні шаблони для пошуку файлів у форматах json, json5, yml, yaml, toml.
V8R_CATALOG_PATH — Містить абсолютний шлях до каталогу схем `@nitra/cursor` (`v8r-catalog.json`), що використовується для перевірок v8r.
runV8rWithGlobs — Послідовно запускає перевірку формату v8r для кожного наданого або стандартного набору шаблонних виразів, використовуючи каталог схем `@nitra/cursor`.

## Публічний API

- DEFAULT_V8R_GLOBS — Шаблонні патерни для виявлення файлів у форматах, підтримуваних v8r.
- V8R_CATALOG_PATH — Місцезнаходження каталогу з каталогом схем v8r-catalog.json.
- runV8rWithGlobs — Ініціює серію запуску команди v8r з використанням наданих шаблонів.
