## Огляд

Цей інструмент є CLI-обгорткою, що використовується як підкоманда `lint-text` з `bin/n-cursor.js`. Він послідовно виконує низку інструментів лінтингу тексту, керуючись конфігурацією з `meta.json`. Обгортка автоматично встановлює необхідні бінарники (`shellcheck`, `dotenv-linter`) для забезпечення стабільної роботи в різних середовищах. Процес перевіряє контент у чіткій послідовності: правопис (`cspell`), скрипти (`shellcheck`), файли оточення (`dotenv-linter`), Markdown-документи (`markdownlint-cli2`) та схеми конфігурацій (`v8r`). При виявленні першого ненульового коду в ланцюжку, він повертається як код виходу, припиняючи подальші перевірки.

## Поведінка

runLintTextCli виконує повний ланцюжок перевірок тексту (cspell, shellcheck, dotenv-linter, markdownlint, v8r), автоматично встановлюючи необхідні інструменти, і повертає код виходу першого кроку, що не пройшов.
lint виконує повний ланцюжок перевірок тексту, делегуючи виконання `runLintTextCli`.

## Публічний API

runLintTextCli — Серіалізує для запуску лінтера тексту, використовуючи блокування та усуваючи дублікати на основі стану Git-дерева.
lint — Координує запуск лінтера тексту, викликаючи `runLintTextCli`.
