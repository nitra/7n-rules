## Огляд

CLI-обгортка над канонічним `lint-text` (text.mdc). Вона забезпечує послідовну перевірку текстових ресурсів репозиторію. Перед виконанням перевірок, `ensureTool` автоматично встановлює необхідні бінарники (`shellcheck`, `dotenv-linter`) через `brew/scoop/GitHub Release per-platform`. Функція `lint` делегує виклик `runLintTextCli` (який викликається через `n-cursor lint text`). Ланцюжок перевірок включає: правопис (`cspell` з `@nitra/cspell-dict`), синтаксис скриптів (`shellcheck` з авто-фіксом), конфігураційні файли (`dotenv-linter` з авто-фіксом), форматування Markdown (`markdownlint-cli2`) та валідацію схем (`v8r`). При отриманні першого ненульового коду з ланцюжка, він повертається як код виходу, припиняючи подальші кроки. Конфігурації, на які спирається код: meta.json.

## Поведінка

runLintTextCli виконує повний ланцюжок перевірок тексту: правопис (cspell), синтаксис скриптів (shellcheck), конфігураційні файли (.env*), форматування Markdown та валідацію схем (v8r).
lint делегує виклик до runLintTextCli, забезпечуючи аналіз всього репозиторію.

## Публічний API

runLintTextCli — Серіалізує дані для лінтування тексту, використовуючи блокування та усуваючи дублікати на основі стану Git-дерева.
lint — Координує процес лінтування тексту, викликаючи `runLintTextCli`.
