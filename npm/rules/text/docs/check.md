## Огляд

Модуль ініціює перевірку через публічну функцію `run`, яка виконується у контексті прогону з кешуванням результатів у межах цього прогону. При запуску як окремий інструмент, він ініціює повну оркестрацію, включаючи завантаження конфігурації та підсумок. Результат роботи повертається через код виходу процесу.

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується у бібліотечному режимі, використовуючи контекст прогону.
3. Якщо код виконується як окремий інструмент командного рядка, запускається повна оркестрація правила.
4. Оркестрація включає завантаження конфігурації, перевірку білого списку та підсумок.
5. Результат виконання повертається як код виходу процесу.

## Публічний API

run — виконує послідовність дій: застосовує правила, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
