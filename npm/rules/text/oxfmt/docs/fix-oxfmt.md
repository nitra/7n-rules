## Огляд

T0-autofix для `text/oxfmt`: детерміноване форматування через `oxfmt --write` саме тих файлів,
які детектор позначив як неформатовані (`data.kind === 'oxfmt-unformatted'`). Формат визначає
`.oxfmtrc.json`. Запис незворотний (поза rollback); коректність підтверджує повторна перевірка.

## Поведінка

- Збирає унікальні файли з відповідних порушень і застосовує `oxfmt --write` на них.
- До списку змінених потрапляють лише файли, чий вміст фактично змінився (порівняння до/після).

## Публічний API

- `patterns` — масив T0-патернів (`id` / `test` / `apply`), що його споживає центральний fix-pipeline.
  Єдиний патерн `text-oxfmt-write` спрацьовує за наявності `oxfmt-unformatted` порушень із файлом.

## Гарантії поведінки

- Якщо oxfmt відсутній у PATH — no-op (без змін).
- Записуються лише фактично змінені файли; кожен реєструється через `recordWrite`.
