## Огляд

Read-only детектор форматування `text/oxfmt`: за допомогою oxfmt у режимі переліку відмінностей
знаходить файли, які не відповідають канонічному формату проєкту. oxfmt — мандатний форматер
(Prettier заборонено), конфіг — `.oxfmtrc.json` (включно з `ignorePatterns`). Сам детектор нічого
не змінює; автоформатування виконує окремий T0 (`oxfmt --write`).

## Поведінка

- Обмежується безконфліктною formatter-доменою: `.js/.mjs/.cjs/.jsx/.ts/.tsx/.mts/.cts/.vue/.css/.scss`
  (типи на кшталт md/json/yaml, де є інше тулінг, не зачіпає).
- У повному режимі перевіряє весь проєкт; у дельта-режимі — лише передані змінені файли цих типів.
- На кожен неформатований файл повертає одне порушення (`reason: oxfmt-unformatted`) з прив'язкою до файлу.

## Гарантії поведінки

- Якщо oxfmt відсутній у PATH — перевірка тихо пропускається (не порушення), як для інших зовнішніх тулів.
- Детектор не мутує репозиторій; формат застосовує T0-фіксер.
- Поважає `.oxfmtrc.json` (правила формату й `ignorePatterns`).
