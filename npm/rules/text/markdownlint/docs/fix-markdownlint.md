## Огляд

Файл описує T0-autofix для `text/markdownlint`: `patterns` позначає Markdown-вміст, де markdownlint щось знайшов, а виправлення виконується через `markdownlint-cli2 --fix`. Він існує, щоб fail-safe запускати автоматичне виправлення markdownlint без винятків назовні; read-only детектор не змінює дані, а застосування повторно аналізує вміст, тому деталі окремих порушень тут не потрібні.

Логіку перенесено з `text/check/fix-check.mjs` відповідно до `docs/specs/2026-07-02-text-check-per-file-split-design.md §1`.

## Поведінка

1. `patterns` надає T0-autofix для перевірки `text/markdownlint`, який спрацьовує лише коли попередня діагностика зафіксувала проблему markdownlint.

2. `patterns` повторно аналізує Markdown-файли проєкту перед виправленням, бо для цього сценарію достатньо факту наявності порушення без прив’язки до конкретного рядка чи колонки.

3. `patterns` застосовує стандартне автоматичне форматування markdownlint до відстежуваних Markdown-файлів, щоб усунути типові порушення стилю без ручного редагування.

4. `patterns` порівнює стан файлів до і після виправлення та повідомляє лише про реально змінені файли.

5. `patterns` працює fail-safe: якщо список файлів недоступний або змін немає, повертає порожній результат без винятку назовні.

6. `patterns` приглушує службовий вивід інструмента виправлення, щоб результат містив тільки корисну інформацію про змінені файли.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
