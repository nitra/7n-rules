## Огляд

Використовує `php-cs-fixer` для валідації стилю PHP-коду. Визначає файли для перевірки на основі наданого списку або для всіх файлів у робочому каталозі. Процес є read-only: він не вносить жодних змін до файлової системи чи бази даних. Для коректної роботи опирається на конфігурації, визначені у `composer.json`. Перевірка виконується за допомогою `php-cs-fixer fix --dry-run --diff`, вимагаючи наявності відповідних конфігурацій та виконуваного файлу `php-cs-fixer` у `vendor/bin`.

## Поведінка

Поведінка
1. Перевіряє наявності `composer.json` у корені робочого каталогу. Якщо відсутній, перевірка пропускається.
2. Визначає цілі для перевірки: або всі файли, що закінчуються на `.php`, якщо не вказані конкретні файли, або наданий список файлів.
3. Шукає виконуваний файл `php-cs-fixer` у `vendor/bin` кореневого каталогу. Якщо не знайдено, перевірка пропускається.
4. Запускає `php-cs-fixer` у режимі сухого запуску з відмінностями для всіх визначених цілей.
5. Якщо команда завершується з кодом, відмінним від нуля, фіксується порушення, що включає вивід команди.
6. В іншому випадку, результати перевірки повертаються як успішні.
