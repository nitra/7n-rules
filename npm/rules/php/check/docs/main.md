## Огляд

Виконує linting-поверхню для PHP-коду проєкту. Це забезпечує перевірку коду відповідно до визначених стандартів, включаючи `composer.json` як основну базу для конфігурацій. Процес включає виконання `composer audit`, а також запуск інструментів PHPStan, Psalm, PHP-CS-Fixer та PHPCS. Для коректності роботи відстежуються шляхи, виключаючи `.git` та `node_modules`. Процес може використовувати кешування даних протягом одного прогону.

## Поведінка

Поведінка:
getPhpcsCodePaths сканує вказаний корінь репозиторію та визначає потенційні директорії для перевірки за допомогою phpcs.
runPhpLintSteps виконує послідовний збірний аудит проєкту на відповідність стандартам PHP, включаючи перевірку за допомогою composer audit, PHP-CS-Fixer, phpcs, PHPStan та Psalm. При цьому свідомо ігнорує шляхи до `.git` та `node_modules`.
lint ініціює комплексний лінт-прогін для PHP-коду у вказаному корені, використовуючи результати виконання runPhpLintSteps.
