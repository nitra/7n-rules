## Огляд

Програма перевіряє наявність ключових конфігураційних файлів проєкту, зокрема `composer.json` та `package.json`, згідно з вимогами (php.mdc). Вона забезпечує відповідність проєкту заданим стандартам, ігноруючи каталоги `.github` та `.git`.

## Поведінка

1. Викликається функція main.
2. Перевіряється наявність файлу composer.json у корені проекту. У разі відсутності, фіксується відхилення з посиланням на (php.mdc).
3. Перевіряється наявність файлу package.json у корені проекту. У разі відсутності, фіксується відхилення з посиланням на (php.mdc).
4. Перевіряється наявність файлу .github/workflows/lint-php.yml. У разі відсутності, фіксується відхилення з необхідністю створити його згідно з (php.mdc).
5. Функція повертає код виходу, що відображає загальний стан перевірок.
6. Перевірка не враховує директорії .github та .git.

## Публічний API

main — перевіряє, чи відповідає проєкт критеріям, встановленим у правилі php.mdc.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Свідомо пропускає шляхи: `.github`, `.git`.
