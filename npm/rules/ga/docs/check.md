Модуль реалізує логіку застосування політик, визначених у конфігурації, для перевірки відповідності коду. При виклику через інтерфейс командного рядка (CLI) модуль виконує оркестрацію, завантажує конфігурацію та перевіряє білий список, надаючи фінальний звіт. При виконанні основної функції він запускає перевірку правил, застосовуючи визначені політики.

## Поведінка

1. Викликається функція `run` для запуску перевірки правила. Це виконує застосування політики, пов'язаної з JS-занепокоєннями та посиланнями MDC.
2. Якщо скрипт виконується як окрема програма (standalone), запускається оркестрація CLI для виконання правила. Це включає завантаження конфігурації, перевірку білого списку та підсумок.

## Публічний API

run — виконує послідовність дій: застосовує правила, перевіряє JS-занепокоєння, застосовує політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
