## Огляд

Перевіряє наявність файлу `pyproject.toml` у корені для ініціалізації процесу лінтування Python. Якщо файл відсутній, процес завершується з кодом 0. Якщо файл присутній, перевіряється наявність `uv` (єдиного пакет-менеджера, без Poetry). Якщо `uv` знайдено, виконуються обов'язкові кроки: `uv lock --check` (перевірка актуальності lock-файлу) та `uv sync --frozen` (строге зібрання середовища з `uv.lock`), згідно з документацією https://docs.astral.sh/uv/. Опційні лінтери (`ruff`, `mypy`) запускаються лише у випадку їх доступності через `uv run`, причому `ruff` виконує виправлення (`auto-fix`) та форматування.

## Поведінка

runLintPythonSteps виконує основні кроки лінтування Python, перевіряючи актуальність lock-файлу та синхронізуючи середовище за допомогою `uv` (https://docs.astral.sh/uv/). Запускає лінтери (`ruff`, `mypy`) у режимі виправлення, якщо вони доступні.
runLintPython серіалізує виклик `runLintPythonSteps` через механізм `runStandardLint` для забезпечення дедуплікації на основі стану Git-дерева.
lint є публічним адаптером, який делегує роботу `runLintPython`, дозволяючи вказати режим без мутацій (`readOnly`).

## Публічний API

runLintPythonSteps — Виконує внутрішні етапи перевірки коду Python без блокування.
runLintPython — Виконує публічну команду перевірки коду Python, гарантуючи унікальність виконання на основі стану репозиторію.
lint — Центральний механізм, що керує запуском перевірки коду Python, викликаючи `runLintPython`.

## Гарантії поведінки
