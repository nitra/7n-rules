## Огляд

Виконує логіку визначеного правила. У режимі бібліотеки запускає правило, використовуючи дані, кешовані в контексті прогону. У режимі окремої програми ініціює оркестрацію для виконання правила.

## Поведінка

1. Викликається функція `run` для запуску логіки правила.
2. Логіка правила виконується в режимі бібліотеки, використовуючи контекст прогону для доступу до кешованих даних.
3. Якщо скрипт виконується як окрема програма, запускається оркестрація CLI для виконання правила.

## Публічний API

run — Запускає послідовність перевірок: застосування правил, JS-зацікавленостей, політик та посилань MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
