## Огляд

Огляд
Файл реалізує lint-поверхню для Python, що об'єднує перевірки за допомогою інструментів `uv`, `ruff`, `mypy` та перевірку ліцензій пакетів (`pip-licenses`). Він забезпечує валідацію коду Python відповідно до стандартів.

Поведінка
runLintPythonSteps — Виконує послідовність перевірок якості коду Python, включаючи валідацію середовища та виконання інструментів (`uv`, `ruff`, `mypy`) і перевірку ліцензій пакетів. (python.mdc)
runLintPython — Запускає основну логіку перевірки якості коду Python, обробляючи помилки у режимі fail-safe, не викликаючи зовнішніх винятків. (python.mdc)
lint — Ініціює виконання перевірок якості коду Python, чим може керувати режим "read-only" (не пише у ФС/БД). (python.mdc)

## Поведінка

runLintPythonSteps — Виконує послідовність перевірок якості коду Python, включаючи валідацію середовища та виконання інструментів типу `uv`, `ruff`, `mypy` та перевірку ліцензій пакетів.
runLintPython — Запускає основну логіку перевірки якості коду Python.
lint — Ініціює виконання перевірок якості коду Python у режимі, що залежить від булевого прапорця `readOnly`.

## Публічний API

runLintPythonSteps — Виконує послідовність кроків для линтингу в середовищі Python.
runLintPython — Запускає линтинг для коду Python.
lint — Перевіряє поверхню коду Python на відповідність стилістичним нормам.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
