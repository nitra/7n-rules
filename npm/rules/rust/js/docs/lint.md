Цей файл керує процесом перевірки коду Rust у вказаному корені. Він перевіряє наявність інструментів Rust, а потім застосовує форматування та виправлення коду відповідно до режиму роботи. На завершення він ініціює перевірку, враховуючи попередження для визначення загального статусу якості коду.

## Поведінка

1. Викликається `lint` для запуску оркестрації перевірки коду Rust.
2. Перевіряється наявність файлу `Cargo.toml` у вказаному корені.
3. Якщо `Cargo.toml` відсутній, перевірка Rust пропускається.
4. Визначається шлях до інструменту `cargo`.
5. Якщо `cargo` не знайдено, перевірка Rust завершується з помилкою.
6. Запускається форматування коду Rust.
7. Якщо режим `readOnly` увімкнено, форматування виконується в режимі перевірки (`--check`).
8. Якщо режим `readOnly` вимкнено, форматування коду Rust виконується з метою виправлення.
9. Якщо форматування не вдалося, перевірка завершується з помилкою.
10. Якщо режим `readOnly` вимкнено, запускається виправлення коду за допомогою `clippy` (`--fix`).
11. Якщо виправлення не вдалося, перевірка завершується з помилкою.
12. Запускається фінальна перевірка коду за допомогою `clippy` з увімкненими попередженнями.
13. Результат виконання перевірки повертається як код виходу.

## Публічний API

lint — фіксує точку входу для оркестрації `n-cursor lint rust`.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
