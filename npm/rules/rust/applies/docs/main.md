## Огляд

Функціональність визначає, чи підходить даний каталог для застосування правил. Публічна функція `applies` здійснює цей аналіз, тоді як функція `main` ініціює процес повної перевірки, звітуючи про результати застосування правил до каталогу. Сканування каталогу ігнорує системні та залежностні каталоги, такі як `.git` та `node_modules`, а також конфігураційний файл `Cargo.toml`.

## Поведінка

Поведінка:
applies визначає, чи є правило застосовним до поточного каталогу, перевіряючи наявність файлу Cargo.toml або його присутності у підкаталогах, і свідомо ігнорує каталоги .git, node_modules та .next.
main виконує логіку перевірки, ініціалізує репортер та інформує про застосування правил Rust, повертаючи код завершення.

## Публічний API

Будь ласка, надайте код або список елементів коду, які потрібно задокументувати.

- Свідомо пропускає шляхи: `.git`, `node_modules`.
