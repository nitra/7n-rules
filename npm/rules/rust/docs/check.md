## Огляд

Модуль ініціює виконання процесу перевірки через публічну функцію `run`. Він використовує дані, кешовані в межах поточного прогону. При запуску як окрема утиліта, він забезпечує повний цикл перевірки, включаючи завантаження конфігурації та підсумків. Модуль є лише для читання даних (не записує в ФС/БД).

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується, використовуючи контекст прогону, що може містити кешовані дані.
3. Якщо скрипт виконується як окрема утиліта командного рядка, запускається повний процес перевірки, включаючи завантаження конфігурації та підсумки.

## Публічний API

run — виконує послідовність дій: застосовує правила, аналізує JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
