## Огляд

Визначає логіку виконання перевірки. Викликає перевірку у режимі бібліотеки, якщо викликається з іншого контексту, або ініціює повну оркестрацію правила, якщо запускається як окремий інструмент командного рядка. Оркестрація включає завантаження конфігурації, перевірку білого списку та формування підсумку.

## Поведінка

1. Викликається функція `run` для запуску перевірки.
2. Перевірка виконується у режимі бібліотеки, використовуючи контекст прогону.
3. Якщо код виконується як окремий інструмент командного рядка, запускається повна оркестрація правила.
4. Оркестрація включає завантаження конфігурації, перевірку білого списку та підсумок.
5. Завершення роботи інструмента командного рядка відбувається з кодом виходу, визначеним під час оркестрації.

## Публічний API

run — виконує послідовність дій: застосовує правила, обробляє JS-занепокоєння, перевіряє політику та посилання MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
