Модуль виконує логіку, ініційовану викликом `run`. Він застосовує правила, обробляє JS-запити та здійснює посилання MDC. Дані, що читаються, є read-only. Кешування відбувається в межах одного прогону. При запуску як інструмент командного рядка виконується повний цикл обробки.

## Поведінка

1. Викликається функція run для запуску правила.
2. Запуск правила виконує послідовність: застосування, обробка JS-запитань, політика, посилання MDC.
3. Якщо скрипт виконується як самостійний інструмент командного рядка, виконується повний запуск правила, включаючи завантаження конфігурації, вайтлістинг та підсумок.

## Публічний API

run — Виконує послідовність перевірок: застосування правил $\to$ аналіз JS-зацікавленостей $\to$ застосування політик $\to$ перевірка посилань MDC.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
