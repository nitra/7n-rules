## Огляд

Скрипт, що запускається після install через `node npm/scripts/smoke-check-imports.mjs`, динамічно імпортує кожен `.mjs`-файл пакету, щоб зловити зламані named exports, які проявляються лише на етапі завантаження модуля. Він працює як post-install smoke-check перед release/publish, свідомо пропускає `node_modules`, є fail-safe і не кидає винятків назовні.

## Поведінка

1. Після install скрипт обходить модулі пакету й відбирає лише `.mjs`-файли з бібліотечних частин, придатні для імпорту без запуску CLI-побічних ефектів.
2. Свідомо пропускає артефакти й технічні зони на кшталт `node_modules`, щоб перевірка залишалась про реальний код пакету, а не про стороннє чи згенероване.
3. Динамічно імпортує кожен відібраний модуль окремо, щоб виявити регресії залежностей, які проявляються лише на рівні завантаження модуля.
4. Фіксує всі помилки імпорту й продовжує перевірку далі, щоб один збій не приховав інші проблемні модулі.
5. Після завершення показує підсумок: скільки модулів імпортувались успішно, а скільки ні.
6. Якщо є проблемні модулі, виводить їх список із короткою причиною і завершується з неуспішним статусом; якщо ні — підтверджує, що імпорт проходить чисто.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- Свідомо пропускає шляхи: `node_modules`.
