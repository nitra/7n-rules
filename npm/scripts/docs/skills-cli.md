## Огляд

Цей модуль забезпечує керування функціоналом скілів пакета `@7n/rules`. Він каталогізує доступні скіли на основі наявності файлу `SKILL.md` у відповідному пакеті. Для виконання скілу збирається контекст, що включає інструкції скілу та інформацію з конфігураційних файлів проєкту: `package.json`, `tsconfig.json`, `.n-rules.json`. Система дозволяє або вивести список доступних скілів (`npx @7n/rules skill list`), або ініціювати виконання обраного скілу (наприклад, `npx @7n/rules skill pi taze`), з можливістю передачі додаткових аргументів. Пріоритетним механізмом виконання є інтеграція з вбудованим pi-агентом; поряд з ним підтримуються рівноцінні зовнішні CLI-раннери (`cursor`, `codex`, `claude`) через ACP (`@7n/llm-lib/acp`).

## Поведінка

- `normalizeSkillId` знімає префікс `n-` з імені скілу для приведення його до стандартного ID.
- `listSkillIds` отримує відсортований список ID скілів, які мають файл `SKILL.md` у вказаній директорії скілів.
- `buildSkillPrompt` збирає комплексний промпт для виконання скілу, об'єднуючи інструкцію скілу з конфігураційними файлами проєкту, такими як `package.json`, `tsconfig.json` та `.n-rules.json`.
- `resolveBundledPackageRoot` визначає абсолютний шлях до кореня пакету `@7n/rules`, використовуючи інформацію про поточний модуль.
- `runSkillsCli` виконує логіку командного інтерфейсу для керування скілами: може вивести список доступних скілів, зібрати промпт для скілу або ініціювати його виконання через вбудований pi-агент чи один із зовнішніх ACP-раннерів (`cursor`, `codex`, `claude`).

## Публічний API

- normalizeSkillId — знімає префікс `n-` з імені скілу, приводячи його до id каталогу в пакеті.
- listSkillIds — повертає відсортований список id скілів, що мають `SKILL.md`.
- buildSkillPrompt — збирає промпт виконання: інструкція скілу + контекст проєкту (`package.json`, `tsconfig.json`, `.n-rules.json`); кидає, якщо скіл невідомий.
- resolveBundledPackageRoot — абсолютний шлях до кореня встановленого пакета `@7n/rules`.
- runSkillsCli — асинхронний entrypoint підкоманди `skill`: `list`, друк промпта на stdout, або виконання через `pi` (рекомендовано) чи `cursor`/`codex`/`claude` (зовнішні CLI через ACP). Повертає exit-код.

## Гарантії поведінки

- Сам модуль лише читає файли й збирає промпт; **виконання** делегується агенту: `pi` (вбудований, мутує дерево, запускає bash) або зовнішнім `cursor`/`codex`/`claude` через ACP (`@7n/llm-lib/acp`) — промпт іде у `session/prompt` по stdio/JSON-RPC, не через argv/stdin-текст.
- Тира моделі для `pi`-runner береться з `main.json.tier` скіла (дефолт `max`).
- `cursor`/`codex`/`claude` — рівноцінні зовнішні раннери, без deprecated-статусу; дозвіл на tool call ACP-агент отримує напряму від клієнта (`runAcpAgent`), без потреби в allowlist-обходах.
