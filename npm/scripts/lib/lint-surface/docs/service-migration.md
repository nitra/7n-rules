## Огляд

Спільні хелпери T0-автоміграторів сервіс-канону (ADR 260718-0835) для CI-плагінів: розбір `n-rules`-команд токенами, визначення релевантних доменів сервісу тими самими glob-ами, що `ci plan`, і фабрика T0-патерну з fail-open обходом файлів-порушників. Винесено в ядро, щоб мігратори ci-azure (`fix-service_deploy_pipeline`) і ci-github (`fix-service_deploy_workflow`) не дублювали логіку й не дрейфували.

## Поведінка

Поведінка:
parseNRulesCmd розбирає повний текст команди кроку на токени для визначення домену сервісу та шляху, якщо команда містить маркер `n-rules lint` або `n-rules ci plan`.
relevantDomains визначає список ідентифікаторів правил (rule-id), які є активними для заданого сервісу, якщо його розташування відповідає умовам у застосованих правила.
createMigrationFixPattern створює патерн для автоматичного виправлення, який обходить файли, що мають порушення, та застосовує мігратор для кожного такого файлу, ігноруючи помилки окремих файлів.

## Публічний API

parseNRulesCmd — розбиває команди `n-rules lint …` або `n-rules ci plan …` на окремі частини для обробки.
relevantDomains — визначає, які домени сервісу активні, ґрунтуючись на тому, які файли у піддеревах відповідають визначеним правилам.
createMigrationFixPattern — створює шаблон виправлення для міграції, проходячи по файлах із порушеннями певного правила та застосовуючи до них функцію `migrateFile`, при цьому пропускаючи помилки окремих файлів для ручного виправлення.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
