## Огляд

Єдиний рендерер уніфікованої поверхні лінтингу. Він агрегує та форматує вихідні дані (`LintResult`), отримані від детекторів. Це забезпечує однаковий візуальний вигляд повідомлень про порушення, незалежно від того, який консерн їх виявив, та є централізованою точкою для відображення звітів. Також він може генерувати список технічної інформації, якщо вона була надана.

## Поведінка

renderViolations збирає порушення, групує їх за ID правила та зосередження, а потім формує згрупований текстовий звіт про кожен випадок.
renderDiagnostics формує текстовий список технічної інформації про діагностики, якщо вона надана.

## Публічний API

renderViolations — відображає порушення, згруповані за типом проблеми, повертає вміст для подальшого відображення.
renderDiagnostics — відображає технічні деталі, якщо встановлено режим детального виведення.
