Цей модуль виявляє паттерни для автоматичного виправлення (autofix) шляхом сканування директорій `npm/rules/{rule}/js/fix-*.mjs` та `npm/rules/{rule}/policy/{concern}/fix-*.mjs` для всіх правил проєкту. Він динамічно імпортує кожен відповідний файл і збирає масиви `patterns` одноразово під час завантаження модуля через top-level await. Модуль не виконує операцій з файловою системою чи БД; він працює в режимі лише для читання та гарантує стійкість роботи шляхом перехоплення помилок, повертаючи `null` замість винятку.

## Поведінка

1. Перевіряє існування каталогу `rulesDir`. Якщо він відсутній, повертає порожній масив.
2. Виконує пошук всіх файлів з шаблонами `fix-*.mjs` у підкаталогах `js/` та `policy/*/` усередині `rulesDir`.
3. Для кожного знайденого файлу виконує динамічний імпорт.
4. Якщо імпорт успішний, витягує масив `patterns` з імпортованого модуля та додає його до загального списку. У випадку помилки імпорту фіксує цю помилку, але продовжує роботу.
5. Забезпечує унікальність згенерованих паттернів на основі їхніх `id`, відкидаючи повторювані записи.
6. Повертає відфільтрований, унікальний масив `T0Pattern` за допомогою `discoverT0Patterns`.

## Публічний API

discoverT0Patterns — збирає всі T0-паттерни з файлів `fix-*.mjs` у всій директорії правил.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
