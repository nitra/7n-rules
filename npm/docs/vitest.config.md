## Огляд

Файл задає кореневе vitest-середовище для `@7n/rules`, щоб тестове оточення було ізольоване від локальної машини й службових trace-джерел. Він ігнорує `node_modules`, а також sandbox-копії Stryker і git-worktree чекаутів, щоб прогін тестів не захоплював зайві дерева проєкту. Також він відсікає вплив git trace2 і LLM-трейсу під час запусків, де це може спотворити результати.

## Поведінка

1. Вмикає тестове середовище для кореня пакета `@7n/rules`, щоб локальні налаштування машини не впливали на результат прогону.
2. Підхоплює тести з двох основних розкладок: поруч із кодом і в top-level `tests/`.
3. Ігнорує службові копії та дублікати тестового дерева, зокрема `node_modules`, `dist`, `reports/stryker`, `.worktrees` і `.claude/worktrees`, щоб не запускати ті самі сценарії повторно й не підхоплювати sandbox-чекаути.
4. Вимикає git Trace2 для тестових процесів, щоб локальний AI/git-трейс не створював затримки й таймаути під час git-важких перевірок.
5. Перенаправляє LLM wire-trace у тимчасову директорію, щоб тестові прогони не засмічували робочий trace-файл користувача.
6. Дає тестам запас часу на повільніші git-операції та локальну I/O-латентність, не маскуючи справжні зависання.
7. Запускає тести в окремих процесах, щоб ізоляція між файлами зберігалася навіть у разі випадкового спільного стану.
8. Збирає coverage у форматах `lcov` і `text-summary` для подальшого аналізу результатів.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Свідомо пропускає шляхи: `node_modules`.
