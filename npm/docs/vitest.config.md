## Огляд

Файл відповідає за налаштування середовища для виконання тестів. Він ідентифікує та обробляє файли з розширенням .test.{js,mjs} у корені та у піддиректоріях tests/. Для забезпечення ізоляції тестів використовується механізм фокусування процесів ('forks'). У процесі виконання збирається покриття коду у форматах lcov та text-summary. Середовище налаштовано так, щоб ігнорувати такі шляхи як .git та node_modules.

## Поведінка

Поведінка:

1. Визначає обсяг тестів, що підлягають валідації, охоплюючи файли з розширенням .test.{js,mjs} як у кореневій директорії, так і у піддиректоріях tests/.
2. Ігнорує каталоги node_modules та dist при виконанні тестів.
3. Виключає каталоги, що містять звіти Stryker, для запобігання фейлам, спричинених запуском поза кореневим репозиторієм.
4. Налаштовує середовище виконання тестів на 'node'.
5. Примусово вимикає переслідування подій Git через встановлення змінної середовища GIT_TRACE2_EVENT в значення '0', щоб запобігти блокуванням I/O під час паралельних Git-операцій.
6. Відводить глобальний LLM wire-trace у тимчасовий файл через змінну середовища N_LLM_TRACE_PATH: тести fix-pipeline ганяють реальний startChain/writeTrace з @7n/llm-lib/trace, і без override кожен прогін дописував би фейкові chain-записи у справжній ~/.n-rules/llm-trace.jsonl, засмічуючи аналітику myllm і chains-report. Тест із власним trace-файлом перевизначає змінну локально.
7. Встановлює максимальний час виконання тесту (testTimeout) у 20000 мілісекунд для забезпечення запасу часу при виконанні складних тестів, залежних від I/O.
7. Використовує pool 'forks' для ізоляції процесів між тестовими файлами, як додатковий механізм безпеки.
8. Збирає покриття коду за допомогою провайдера v8 та виводить звіти у форматах lcov та text-summary.
