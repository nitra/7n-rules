## Огляд

Overview
Файл конфігує мутаційне тестування Stryker для пакета `@7n/rules`, використовуючи `vitest-runner` з покриттям `perTest` та інкрементальним кешем у звітах. Файл забезпечує перевірку на вплив мутації на production-код, включаючи виключення певних директорій та файлів з покриття.

## Поведінка

1. Запуск тестування використовує тестовий раннер vitest
2. Аналіз покриття здійснюється через perTest для вимірювання впливу мутації на тести
3. Ізоляція мутантів відбувається в пам'яті через AST-patching
4. Репортери генерують JSON та clear-text звіти
5. Зберігання результатів між запусками забезпечує відновлення стану після збою
6. Спеціально визначені шляхи виключаються з покриття
   - Виключаються файли з директорії tests
   - Виключаються файли з директорії __fixtures__
   - Виключаються файли з директорії fixtures
   - Виключаються файли з директорії data
   - Виключаються файли з директорії template
   - Виключаються файли з директорії templates
   - Виключаються файли з директорії rules/test/js/data/stryker_config/stryker-vue-macros-ignorer.mjs
7. Покриття застосовується до скриптів, правил, бінарних файлів
8. Інтеграційні тести для бінарних файлів покриваються через subprocess
