## Огляд

Цей файл налаштовує Vitest для локального прогону тестів плагіна `ci-azure`. Він забезпечує, що тести виконуються лише для файлів у директорії `rules`, імітуючи середовище виконання з тим самим env-каноном, що й кореневий пакет `@7n/rules`. Логіка прогону тестів свідомо ігнорує шляхи в `node_modules`. Цей конфіг не містить власних операцій запису в файлову систему чи базу даних.

## Поведінка

Використовується конфігураційний файл Vitest для локального запуску тестів правил плагіна `ci-azure`.
1. Визначається, що тести повинні включати лише файли, які знаходяться у директорії `rules`, а саме у папки з розширенням `.test.{js,mjs}`.
2. Ігноруються всі файли та папки всередині директорії `node_modules`.
3. Тести виконуються в середовищі `node`.
4. Налаштовується специфічне середовище (`env`), яке встановлює `GIT_TRACE2_EVENT` у значення `'0'` та визначає шлях для трасування LLM (`N_LLM_TRACE_PATH`) у тимчасовій директорії операційної системи.
5. Встановлюється тайм-аут для тестів на рівні 20 секунд.
6. Тестування відбувається з використанням пулу `forks` для ізоляції процесів.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Свідомо пропускає шляхи: `node_modules`.
