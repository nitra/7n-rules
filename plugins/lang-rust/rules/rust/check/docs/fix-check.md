## Огляд

`patterns` застосовує автоматичні виправлення для перевірок проєкту й звертається до мережі, щоб отримати актуальний результат. Він працює fail-safe: не кидає винятків назовні та за певних помилок повертає порожнє значення замість збою.

## Поведінка

1. `patterns` визначає два автоматичні сценарії виправлення для Rust-перевірок: один прибирає форматувальні порушення, другий відновлює відсутній `deny.toml`, щоб проєкт міг проходити `cargo deny`.
2. Для форматування сценарій спочатку перевіряє, чи доступний `cargo`; якщо ні — нічого не змінює. Далі бере всі відстежувані `*.rs` файли з репозиторію, запускає `cargo fmt --all` і повертає лише ті файли, які реально змінилися.
3. Для `deny.toml` сценарій спочатку намагається використати `cargo deny init`, якщо доступний `cargo-deny`. Коли генерація успішна, він фіксує створений файл як результат.
4. Якщо `cargo-deny` або `cargo` недоступні, сценарій усе одно створює детермінований мінімальний `deny.toml`, щоб не залишати перевірку без виправлення. У цьому шаблоні дозволений registry відповідає `https://github.com/rust-lang/crates.io-index`.
5. Обидва сценарії працюють fail-safe: не викидають винятки назовні, а в разі проблем повертають порожній результат або локальне виправлення, щоб не зупиняти весь ланцюг перевірок.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
