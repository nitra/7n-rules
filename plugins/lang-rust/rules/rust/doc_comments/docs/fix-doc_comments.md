## Огляд

`patterns` переводить суміжні `//`-блоки в rustdoc лише для двох випадків: `///` над `pub`-елементом і `//!` для header-блоку на початку файлу. Текст автора зберігається дослівно, а всі порушення без суміжного коментаря лишаються для LLM-ladder-у. Після обробки повертає список змінених файлів у вигляді шляхів до цих файлів.

## Поведінка

`patterns` зберігає T0-правило для автоматичного підвищення суміжних `//`-блоків до rustdoc і запускає його лише тоді, коли серед порушень є придатні до механічної заміни; далі вона групує такі порушення за файлом, читає кожен файл один раз, застосовує `promoteBlock` до всіх зібраних діапазонів у межах цього файлу, записує змінений вміст назад і повідомляє про touched-файли. `promoteBlock` працює як спільний механізм переписування для цих блоків: він зберігає оригінальний текст рядків, змінюючи лише префікс коментаря на відповідний doc-варіант залежно від типу блоку, тож `patterns` не вигадує новий зміст, а лише переносить уже наявний текст у форму rustdoc. Порушення без суміжного коментаря лишаються поза цим потоком і передаються далі на LLM-ladder.

## Публічний API

- promoteBlock — Підвищує `//`-рядки діапазону до `///` або `//!`.
- patterns — повертає перелік відомих шаблонів правил, щоб їх можна було знайти й вибрати для перевірки або генерації документації
