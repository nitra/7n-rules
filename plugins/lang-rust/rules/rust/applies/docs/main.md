## Огляд

Визначає, чи застосовувати публічні перевірки `applies` і `lint` до поточного кореня, щоб правила спрацьовували лише тоді, коли для цього є підстава.

Поводження обмежене видимим результатом перевірки: секція не описує внутрішній пошук чи приховані правила прийняття рішення.

## Поведінка

`applies` визначає, чи взагалі запускати правила Rust у поточному корені: спершу перевіряється наявність `Cargo.toml` у корені, а якщо його там немає — шукається `Cargo.toml` у дереві проєкту з урахуванням пропущених директорій. Це дає змогу підключати правила і для монорепозиторіїв, де Rust-проєкт лежить не в корені.

`lint` спирається на вже підтверджену застосовність і повертає лише контекстний результат без змін у файловій системі. Його результат повідомляє, що `Cargo.toml` знайдено і правила Rust застосовано, тобто цей файл працює як легкий узгоджувальний крок між виявленням Rust-дерева та подальшими перевірками.

## Публічний API

- applies — перевіряє, чи підходить вхідне значення під набір правил, і використовує цей висновок далі в обробці
- lint — знаходить невідповідності у вхідних даних і повертає їх як перелік проблем для виправлення

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
