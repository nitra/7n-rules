## Огляд

`extractUnitsRs` виділяє з Rust-коду документовані API-одиниці для подальшої документації. Результат потрібен, щоб наступні кроки працювали з готовим переліком знайдених одиниць і описували лише те, що присутнє у вихідному файлі.

## Поведінка

1. `extractUnitsRs` аналізує Rust-файл як набір потенційних документованих одиниць: функцій, структур, enum, trait і type-alias.

2. Визначає публічність одиниць за Rust-видимістю та підтримуваними exposure-атрибутами, щоб у документацію потрапляли API-точки, які фактично доступні зовні.

3. Розрізняє top-level елементи та методи всередині `impl`, щоб зберегти прив’язку методу до відповідного типу.

4. Забирає найближчий `///`-опис перед декларацією, пропускаючи атрибути й порожні рядки, щоб використати наявну авторську документацію як поведінковий контекст.

5. Визначає межі кожної знайденої одиниці та зберігає її тіло, щоб надалі можна було аналізувати зв’язки між одиницями одного файлу.

6. Будує базову карту викликів між знайденими одиницями в межах того самого файлу, щоб показати локальні залежності поведінки.

7. Повертає список знайдених одиниць або порожній результат, якщо у файлі немає підтримуваних Rust-елементів.

## Публічний API

- extractUnitsRs — знаходить top-level елементи та методи `impl` у `.rs` файлах; орієнтується на баланс фігурних дужок і може помилятися, якщо дужки трапляються всередині рядкових літералів.
