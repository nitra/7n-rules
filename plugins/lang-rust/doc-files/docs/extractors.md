## Огляд

Публічна функція `extractFactsRust` збирає структурований факт-лист про `.rs`-файл, прив’язаний до `relPath` і мови `rs`, для генерації доки. Працює read-only над переданим рядком джерела (без ФС/мережі/кешу).

## Поведінка

1. `extractFactsRust` формує факт-лист для `.rs`-файлу з прив’язкою до `relPath` і мови `rs`.
2. Збирає верхньорівневий module doc як короткий `header`, якщо файл починається з Rust-коментарів для модуля.
3. Визначає публічні exports: враховує звичайні `pub`-елементи та `fn`, які стають видимими через exposure-атрибути.
4. Для кожного export фіксує назву, вид елемента і короткий опис з `///`-коментарів, якщо вони є.
5. Окремо виділяє приватні `fn` як `localSymbols`; якщо їхня назва збігається з export, вони не дублюються.
6. Класифікує imports на `stdlib` та `external`; `internal` свідомо повертає порожнім.
7. Обчислює маркери поведінки: чи файл read-only, чи є обробка помилок, чи повернення пов’язане з `Result` або `Option`, чи є ознаки мережевої роботи, чи є кешування.
8. Для визначення помилок враховує як прості сигнали, так і `match`-гілки з `Err`.
9. Не виконує записів у ФС чи БД.
10. Перевага надається стабільному кешуванню в межах одного прогону.

## Публічний API

- `extractFactsRust` — збирає факт-лист для `.rs` файлу.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
