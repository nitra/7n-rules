## Огляд

Файл надає `extractFactsRust` для формування фактів про Rust-код, щоб документаційний пайплайн міг спиратися на видиму поведінку без повторного аналізу джерела. Власних операцій запису у файлову систему чи базу даних тут немає. Повторні звернення в межах одного прогону обслуговуються з кешу.

## Поведінка

1. `extractFactsRust` формує факт-лист для Rust-файлу, щоб документаційний пайплайн міг описувати модуль за його спостережуваною поведінкою.
2. Витягує верхній module-level опис, публічно доступні елементи, згруповані залежності та ознаки обробки помилок через `match` з гілкою `Err`.
3. Враховує функції, які стають публічно доступними не лише через явний `pub`, а й через exposure-атрибути.
4. Повертає структуровані факти без маркера непідтримуваного файлу, щоб наступні кроки могли генерувати документацію без повторного аналізу Rust-коду.
5. Використовує кешування в межах одного прогону, щоб повторні звернення до тих самих даних не дублювали обчислення.

## Публічний API

- extractFactsRust — Витягує факт-лист для `.rs` файлу.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Кешує результати в межах одного прогону.
