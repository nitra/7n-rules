## Огляд

Ізолює тестову конфігурацію плагіна від решти репозиторію, щоб перевірки цього плагіна не змішувалися з іншими сценаріями. Свідомо пропускає шляхи `node_modules` і не виконує записів у ФС чи БД.

## Поведінка

1. Запускає тестову конфігурацію лише для плагіна, щоб ізолювати перевірки від решти репозиторію.
2. Формує середовище виконання так, щоб тести працювали стабільно в Node-режимі та не конфліктували з інструментами трасування Git і LLM.
3. Пропускає `node_modules`, щоб не витрачати ресурси на сторонні залежності.
4. Виконує тести у forked-процесах, щоб зменшити взаємний вплив між кейсами.
5. Обмежує час одного прогону, щоб завислі сценарії не блокували перевірку надовго.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Свідомо пропускає шляхи: `node_modules`.
