## Огляд

Налаштовує Vitest для тестів плагіна `lang-rust`: запускає include лише тестів плагіна `lang-rust` у Node-середовищі та свідомо пропускає `node_modules`. Файл існує, щоб закріпити `env`-канон ядра для цього плагіна й не змішувати його перевірки з іншими частинами проєкту.

## Поведінка

1. Обмежує запуск Vitest тестами плагіна lang-rust, щоб перевірки охоплювали лише релевантні сценарії цього плагіна.
2. Свідомо пропускає `node_modules`, щоб зовнішні залежності не потрапляли в тестовий набір.
3. Виконує тести в Node-середовищі з окремими процесами, щоб ізолювати виконання та наблизити поведінку до основного тестового канону репозиторію.
4. Встановлює стабільні змінні середовища для тестів, щоб уникати стороннього git trace2-впливу та спрямовувати LLM trace у тимчасове місце.
5. Обмежує тривалість тесту, щоб завислі або надто довгі сценарії завершувались контрольовано.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Свідомо пропускає шляхи: `node_modules`.
