## Огляд

Огляд

Секція описує поведінку документа, що спирається на конфіги `mutation.json` і `incremental.json`.

Вона має бути лаконічною, без додаткових припущень і без зайвих деталей поза межами цих джерел.

## Поведінка

1. Запускає mutation testing через `vitest` і використовує `vitest.config.mjs` як джерело конфігурації для тестового прогону.
2. Оптимізує прогін так, щоб перевіряти лише тести, які реально зачіпають змінений код, і тим самим зменшує вартість кожного циклу перевірки.
3. Зберігає службові результати в `reports/stryker/mutation.json`, щоб їх можна було використовувати для аналізу якості тестового покриття.
4. Підтримує відновлення попереднього стану прогонів через `reports/stryker/incremental.json`, щоб не втрачати вже оброблені результати між запусками.
5. Працює без окремого sandbox для копіювання залежностей; це прибирає зайву інфраструктурну складність під час запуску в цьому проєкті.
