## Огляд

Файл готує дані lcov із `cargo llvm-cov` для coverage-провайдера Rust: окремо зводить загальні підсумки по звіту та розкладає покриття по файлах. Це потрібно, щоб інтерфейс покриття міг показувати сумарні значення й список файлів у спільному форматі, де абсолютні `SF`-шляхи узгоджуються на боці викликача.

## Поведінка

parseLcovTotals спершу проходить по lcov-даних і збирає загальні підсумки покриття для всього звіту: рядки та функції рахуються окремо, а результат лишається придатним для подальшого показу в coverage-інтерфейсі без прив’язки до конкретного файла.

parseLcovPerFile працює з тим самим lcov-потоком, але формує розбивку по файлах: для кожного запису бере шлях із звіту як є, обчислює відсоток покриття рядків і повертає готові рядкові підсумки для відображення або сортування. Абсолютні SF-шляхи не приводяться тут до кореня крейта — це очікується на боці викликача, щоб обидва результати можна було узгодити в одному звіті.

## Публічний API

- parseLcovTotals — Агрегує LF/LH/FNF/FNH по всіх записах lcov.
- parseLcovPerFile — Per-file рядкове покриття з lcov (`SF:`/`LF:`/`LH:`; шляхи — як у файлі).
