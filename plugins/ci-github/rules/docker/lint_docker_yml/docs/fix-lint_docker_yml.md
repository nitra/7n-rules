## Огляд

Файл задає patterns для приведення CI-workflow docker-лінту в консюмер-репозиторіях до канонічного шаблону концерну. Він існує, щоб уніфікувати перевірку docker-домену без ручного узгодження шаблону.

## Поведінка

1. `patterns` визначає єдиний детермінований сценарій виправлення для CI-workflow лінту docker-домену в консюмер-репо.
2. Сценарій забезпечує наявність канонічного workflow для docker-лінту: якщо файл відсутній, він має бути створений зі стандартного шаблону концерну.
3. Якщо workflow уже існує, сценарій приводить його до канонічного стану через пріоритет значень шаблону, зберігаючи сторонні ключі й коментарі поза шаблонною частиною.
4. Поведінка потрібна, щоб усі консюмер-репозиторії мали однаковий базовий CI-контроль docker-домену без ручного копіювання шаблону.

## Публічний API

- patterns — Один детермінований патерн: deep-merge канонічного snippet-а концерну в
`.github/workflows/lint-docker.yml` консюмер-репо — відсутні ключі додаються,
канонічні значення мають пріоритет, коментарі й ключі поза шаблоном не
чіпаються; якщо файлу немає — створюється зі snippet-а.

## Гарантії поведінки

- Сценарій зберігає коментарі й ключі поза шаблонною частиною наявного workflow.
