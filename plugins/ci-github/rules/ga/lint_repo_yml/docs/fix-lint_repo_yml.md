## Огляд

Збігає шляхами `.github/workflows/lint-repo.yml` і повертає `patterns`, щоб репо-рівневий CI lint-workflow у консюмер-репо відповідав канонічному шаблону концерну.

Якщо файл вже існує, оновлює лише канонічний workflow-snippet; якщо ні — формує його з шаблону для `lint-repo.yml`.

Свідомо не зачіпає `.github` і `.git`, щоб не виходити за межі репо-лінту.

## Поведінка

1. `patterns` задає один детермінований шаблонний фікс для приведення `.github/workflows/lint-repo.yml` у консюмер-репо до канонічного вигляду концерну.
2. Якщо цільовий workflow-файл уже існує, у нього вносяться лише зміни в межах канонічного snippet-а: відсутні ключі додаються, а канонічні значення мають пріоритет.
3. Якщо цільового workflow-файлу немає, він створюється на основі канонічного snippet-а.
4. Ключі й коментарі поза шаблоном не зачіпаються, тож локальні доповнення поза канонічною частиною зберігаються.
5. Під час роботи свідомо обходяться шляхи `.github` і `.git`, щоб не втручатися в службові каталоги.

## Публічний API

- patterns — Один детермінований патерн: deep-merge канонічного snippet-а концерну в
`.github/workflows/lint-repo.yml` консюмер-репо — відсутні ключі додаються,
канонічні значення мають пріоритет, коментарі й ключі поза шаблоном не
чіпаються; якщо файлу немає — створюється зі snippet-а.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Свідомо пропускає шляхи: `.github`, `.git`.
