## Огляд

`patterns` задає спільні налаштування VS Code для консюмер-репо відповідно до канонічного шаблону концерну `vscode_settings`. Це потрібно, щоб однаковий базовий набір налаштувань застосовувався узгоджено в усіх споживачах цього шаблону.

## Поведінка

1. `patterns` задає один канонічний сценарій вирівнювання спільних VS Code-налаштувань у консюмер-репо з корпоративним шаблоном для `settings.json`.
2. Якщо цільовий `settings.json` відсутній, він створюється на основі канонічного шаблону.
3. Якщо `settings.json` уже існує, з нього зберігаються всі значення поза шаблоном, а канонічні налаштування концерну мають пріоритет і доповнюють або оновлюють спільну конфігурацію.
4. Це гарантує, що репозиторії отримують однакову базову поведінку VS Code без втрати локальних несуперечливих налаштувань.

## Публічний API

- patterns — Один детермінований патерн: deep-merge канонічного snippet-а концерну в
`.vscode/settings.json` консюмер-репо — відсутні ключі додаються, канонічні
значення мають пріоритет, ключі поза шаблоном не чіпаються; якщо файлу
немає — створюється зі snippet-а.

## Гарантії поведінки
