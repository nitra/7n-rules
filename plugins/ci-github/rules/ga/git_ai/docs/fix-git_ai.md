## Огляд

Файл описує `patterns` для T0-фіксу концерну `git_ai`: приводить workflow інтеграції git-ai у консюмер-репозиторії до канонічного GitHub Actions шаблону концерну.

Це потрібно, щоб workflow git-ai відповідав очікуваному шаблону інтеграції та не обробляв шляхи `.github` і `.git`.

## Поведінка

1. `patterns` визначає єдиний автоматичний фікс для концерну `git_ai`, щоб workflow інтеграції git-ai у консюмер-репозиторії відповідав канонічному шаблону.

2. Фікс підтримує наявний workflow актуальним: додає відсутні обов’язкові частини, пріоритезує канонічні значення концерну та зберігає сторонні налаштування, які не конфліктують із шаблоном.

3. Якщо workflow інтеграції git-ai відсутній, `patterns` забезпечує його створення з канонічного шаблону, щоб репозиторій одразу отримав очікувану GitHub Actions інтеграцію.

4. Під час застосування фікса свідомо не обробляються службові шляхи `.github` і `.git`, щоб не зачіпати інфраструктурні області поза цільовим workflow.

## Публічний API

- patterns — Один детермінований патерн: deep-merge канонічного snippet-а концерну в
`.github/workflows/git-ai.yml` консюмер-репо — відсутні ключі додаються,
канонічні значення мають пріоритет, коментарі й ключі поза шаблоном не
чіпаються; якщо файлу немає — створюється зі snippet-а.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Свідомо пропускає шляхи: `.github`, `.git`.
