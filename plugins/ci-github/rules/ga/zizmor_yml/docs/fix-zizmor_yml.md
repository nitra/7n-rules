## Огляд

`patterns` приводить `.github/zizmor.yml` до канонічного шаблону концерну для `zizmor`, щоб консюмер-репо залишалося узгодженим зі стандартизованим конфігом.

## Поведінка

1. `patterns` визначає один детермінований шаблон для вирівнювання `.github/zizmor.yml` із канонічним концерновим конфігом.
2. Якщо цільовий файл відсутній, `patterns` ініціалізує його з канонічного snippet-а.
3. Якщо файл уже існує, `patterns` додає відсутні ключі та зберігає канонічні значення як пріоритетні.
4. `patterns` не чіпає коментарі й інші ключі поза шаблоном.

## Публічний API

- patterns — Один детермінований патерн: deep-merge канонічного snippet-а концерну в
`.github/zizmor.yml` консюмер-репо — відсутні ключі додаються, канонічні
значення мають пріоритет, коментарі й ключі поза шаблоном не чіпаються;
якщо файлу немає — створюється зі snippet-а.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
