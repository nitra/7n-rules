## Огляд

Файл тримає канонічний сценарій оновлення CI workflow для лінту k8s-домену в consumer-репо, щоб `.github/workflows/lint-k8s.yml` приводився до шаблону концерну. Він існує, щоб звіряти workflow з канонічним snippet-ом концерну й застосовувати стандартний набір змін для цього сценарію, не виходячи за межі `.github` та `.git`.

## Поведінка

1. `patterns` задає єдиний канонічний сценарій виправлення для CI workflow лінту k8s-домену.
2. Під час застосування сценарію шаблон концерну зливається з `.github/workflows/lint-k8s.yml`: відсутні елементи додаються, а канонічні значення мають пріоритет.
3. Якщо цільового файлу немає, `patterns` ініціює його створення з канонічного snippet-а.
4. `patterns` навмисно не втручається в позатимплейтні ключі та коментарі, а також свідомо пропускає `.github` і `.git` як межі обробки.

## Публічний API

- patterns — Один детермінований патерн: deep-merge канонічного snippet-а концерну в
`.github/workflows/lint-k8s.yml` консюмер-репо — відсутні ключі додаються,
канонічні значення мають пріоритет, коментарі й ключі поза шаблоном не
чіпаються; якщо файлу немає — створюється зі snippet-а.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Свідомо пропускає шляхи: `.github`, `.git`.
