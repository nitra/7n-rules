## Огляд

Локальний Vitest-конфіг для плагіна ci-github: запускає лише його rule-тести в межах цього плагіна, з тим самим env-каноном, що й кореневий конфіг. Свідомо пропускає `node_modules`, щоб не підхоплювати сторонні файли поза робочою областю плагіна.

## Поведінка

1. Забезпечує локальний прогін лише тестів правил плагіна ci-github.
2. Працює з тим самим середовищем запуску, що й кореневий Vitest-конфіг, щоб результати були узгоджені з основним пайплайном.
3. Обмежує область пошуку тестів файлами плагіна; `node_modules` свідомо пропускає.
4. Виконує тести в Node-середовищі з достатнім часом на виконання для повільніших сценаріїв.
5. Готує окремий trace-файл у тимчасовій теки, щоб підтримати діагностику запуску без впливу на робочу теку.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Свідомо пропускає шляхи: `node_modules`.
