## Огляд

Файл піднімає суцільні блоки `//`-коментарів, які стоять впритул над export або на початку файла, до JSDoc `/** … */` без зміни авторського тексту. Це потрібно, щоб T0-autofix механічно готував такі коментарі до подальшої документації, а порушення без суміжного коментаря лишав для `LLM-ladder` / `default-worker`.

## Поведінка

`patterns` задає T0-автофікс для випадків, де суцільний блок `//`-коментарів можна без втрати змісту підняти до JSDoc. Під час перевірки він відбирає лише ті порушення, які позначені як придатні до такого перетворення, групує їх за файлом і застосовує зміни так, щоб обробка кількох блоків у межах одного файла не ламала позиції наступних замін. Після цього файл читається, відповідні фрагменти замінюються результатом `promoteLineBlock`, і лише реально змінені файли записуються назад; у результаті повертається список зачеплених файлів і коротке повідомлення про кількість перетворень. Якщо в межах одного файла є дубльовані вказівки на той самий блок, вони зводяться до однієї дії; якщо блок не стоїть на початку рядка, він лишається без змін, щоб не зачепити довільний інший код.

`promoteLineBlock` виконує саме текстове перетворення: бере суцільний блок `//`-рядків і робить із нього JSDoc, зберігаючи початковий відступ та дослівний авторський текст. Для одного рядка виходить компактний однорядковий doc-коментар, для кількох — звичайний багаторядковий формат. Це гарантує, що T0 не додає нового змісту, а лише переводить уже наявний коментар у формат, придатний для подальшої документації; випадки без суміжного блоку лишаються поза цим автофіксом.

## Публічний API

- promoteLineBlock — Перетворює блок `//`-рядків на JSDoc, зберігаючи відступ першого рядка.
- patterns — набір умов для відбору випадків, які можна автоматично підняти до JSDoc.
