## Огляд

Файл підтримує T0-прохід для `js/doc_comments`: він знаходить лише придатні суцільні блоки `//`-коментарів, що стоять безпосередньо над експортом або на початку файла, і перетворює їх на JSDoc-коментар без зміни авторського тексту. Це потрібно, щоб механічно переводити в doc-коментарі лише очевидні суцільні блоки `//`, а порушення без суміжного коментаря не обробляти на цьому рівні — їх допише `default-worker`.

## Поведінка

`patterns` задає T0-прохід для файлів із порушеннями `js/doc_comments`: він відбирає лише ті випадки, де є придатний до підвищення суцільний блок `//`-коментарів, групує їх за файлом, читає вміст, застосовує зміни з кінця файла до початку й записує результат назад лише тоді, коли текст справді змінився. Під час цього один і той самий блок може бути згаданий кілька разів, тому перед заміною він дедуплікується; якщо коментар не стоїть на початку рядка, він лишається без змін і переходить далі по ланцюжку перевірок.

`promoteLineBlock` працює як спільне правило перетворення для цього прохідного сценарію: бере дослівний текст коментаря блоку, зберігає відступ і переводить його в JSDoc-форму без зміни змісту автора. Саме цей результат `patterns` підставляє у файл як фінальний канонічний варіант, тож уся взаємодія зводиться до одного потоку: знайти придатні блоки, нормалізувати їх у doc-коментарі, оновити файл і зафіксувати, які файли були змінені.

## Публічний API

- promoteLineBlock — Перетворює блок `//`-рядків на JSDoc, зберігаючи відступ першого рядка.
- patterns — повертає перелік правил для перевірки, що шлях до параметра не веде на застарілий або неправильний шаблон.
