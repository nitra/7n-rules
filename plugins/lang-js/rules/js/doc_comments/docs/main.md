## Огляд

Перевіряє, чи файл узагалі є кандидатом на doc-коментарі, і лише для таких випадків запускає `checkFileDocComments`; `node_modules` свідомо пропускає. `isDocCommentTarget`, `promotableLineBlockBefore` і `promotableHeaderBlock` визначають, що саме може бути підняте до doc-рівня перед застосуванням перевірок, а `lint` зводить це в один прохід для контролю поведінки документації.

## Поведінка

lint спочатку відбирає лише ті файли, що підпадають під правила для doc-коментарів через isDocCommentTarget, і для решти нічого не перевіряє; шляхами з node_modules свідомо не займається.

Після відбору lint передає вміст кожного файла до checkFileDocComments, а той поєднує дві перевірки: наявність header-JSDoc на початку файла та наявність JSDoc над кожним експортом, який дає collectExports. Для пропущених експортувань, де декларація існує окремо від export-специфікатора, вимога коментаря прив’язується до самої декларації.

У разі відсутності header-JSDoc checkFileDocComments може врахувати придатний суцільний верхній блок `//` як кандидат на підвищення до header-JSDoc; аналогічно, для окремих експортів придатний суцільний блок `//` безпосередньо над декларацією може бути кандидатом на підвищення до JSDoc. Проміжний стан при цьому не зберігається: усі висновки формуються з поточного тексту файла та списку коментарів парсера.

promotableLineBlockBefore і promotableHeaderBlock використовуються як правила визначення таких кандидатів: перший шукає щільний блок `//` прямо над декларацією, другий — щільний `//`-блок на початку файла до будь-якого коду. Обидва підтримують той самий підхід до інтерпретації вихідного тексту: між коментарями всередині блока і між блоком та ціллю допускаються лише пробіли й переводи рядка.

Результатом роботи lint є список порушень для кожного перевіреного файла, сформований із висновків checkFileDocComments і прив’язаний до relPosix-шляху файлу.

## Публічний API

- isDocCommentTarget — Чи підпадає файл під вимогу doc-коментарів.
- promotableLineBlockBefore — Суцільний блок `//`-коментарів, що стоїть ВПРИТУЛ над позицією `pos`
(між блоком і декларацією — лише пробіли/переводи рядка; всередині блоку —
теж). Це кандидат на T0-підвищення до JSDoc.
- promotableHeaderBlock — Провідний суцільний `//`-блок на самому початку файлу (перед будь-яким кодом)
— кандидат на T0-підвищення до header-JSDoc.
- checkFileDocComments — Перевіряє один файл: header + JSDoc над кожним експортом.
- lint — Detector js/doc_comments: per-file (дельта) або повний обхід.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Свідомо пропускає шляхи: `node_modules`.
