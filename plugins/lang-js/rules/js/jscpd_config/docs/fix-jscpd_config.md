## Огляд

`patterns` визначає узгоджені з правилами проєкту зміни для `.jscpd.json` і слугує централізованим орієнтиром для того, що саме має бути виправлено в цьому конфігу.

## Поведінка

1. `patterns` формує перелік правил для уніфікації шаблонного конфіга проєкту.
2. Кожне правило орієнтується на конфіг `.jscpd.json`, щоб тримати його в узгодженому, очікуваному форматі.
3. `patterns` потрібен для централізованого опису того, що саме має бути виправлено в `.jscpd.json`, без ручного втручання в інші файли чи артефакти.
4. `patterns` не змінює файлову систему або базу даних і працює лише як опис набору правок для цього конфіга.
