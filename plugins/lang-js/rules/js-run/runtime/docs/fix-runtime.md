Процес T0-autofix автоматично створює стандартизований `jsconfig.json` у backend-пакетах, що знаходяться в каталозі `src/`, якщо такий конфігураційний файл відсутній. Для визначення канонічного вигляду цього файлу використовується шаблон, визначений у `policy/jsconfig/template/jsconfig.json.snippet.json`.

Конфіги, на які спирається код: jsconfig.json, jsconfig.json.snippet.json

## Поведінка

1. Ініціалізується контент канонічного файлу `jsconfig.json` шляхом читання з `jsconfig.json.snippet.json`.
2. Протягується масив `patterns` для ідентифікації робочих простірців, яким бракує файлу `jsconfig.json`.
3. Для кожного шаблону в `patterns` перевіряється наявність підходящого вихідного стану.
4. Якщо шаблон спрацьовує, здійснюється пошук усіх відповідних робочих простірців у контексті.
5. Для кожного знайденого робочого простірця, який містить каталог `src/` та не має `jsconfig.json`, відбувається запис канонічного контенту у файл `jsconfig.json`.
6. У результаті, що створюються файли `jsconfig.json` у потрібних пакетах фіксується та повертається відповідний статус.
