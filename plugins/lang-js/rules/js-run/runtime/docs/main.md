## Огляд

`lint` перевіряє workspace-пакети на відповідність контракту js-run.mdc (jsconfig, `#conn/`-alias, заборонені імпорти, `process.env`/`checkEnv`, `setTimeout`, `Temporal`, OTEL configmap) і допомагає підтримувати узгодженість між кодом і супровідними файлами. Шлях `base/` свідомо пропускається, тож перевірка зосереджується на решті пакета. Процес працює fail-safe: помилки перехоплюються, назовні не кидаються, а в окремих збоях повертається порожнє значення замість винятку.

## Поведінка

`lint` відокремлює перевірки для backend-пакетів від frontend-пакетів: якщо в пакеті є `src/`, він очікує наявність валідного `jsconfig.json` і далі проганяє набір правил для вихідного коду та метаданих пакета; пакети, які виглядають як frontend через `vite` у `devDependencies`, у backend-гілку не потрапляють. Дані для перевірок беруться з `package.json` і зі зібраних source-файлів пакета, а результати кожної перевірки зливаються в єдиний потік повідомлень про порушення або успіх. Під час обходу свідомо пропускається `base/`, а правило для `#conn/` додатково вимагає узгодженості між файлами, назвами експортів і декларацією alias у `package.json`. Перевірки імпортів, використання середовищних змінних, небажаного `setTimeout`-очікування, `Temporal` та OTEL configmap виконуються як незалежні сигнали в межах одного пакета; помилки перехоплюються, і за окремих збоїв замість винятку повертається порожній результат.

`lint` запускає цей потік лише для workspace-пакетів, не для кореня репозиторію, і збирає підсумок у форматі lint-result. Він не виконує власних записів у ФС чи БД, а лише координує перевірки на основі наявних файлів і конфігів проєкту.

## Публічний API

- lint — Перевіряє відповідність проєкту правилам js-run.mdc лише для workspace-пакетів (не корінь репо).

Код спирається на конфіги jsconfig.json і package.json, тому поведінка узгоджується з ними й лишається стабільною для проєктних налаштувань.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
- Свідомо пропускає шляхи: `base/`.
