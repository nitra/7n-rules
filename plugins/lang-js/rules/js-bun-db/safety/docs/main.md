## Огляд

Цей файл виконує перевірку відповідності проєкту правилу `js-bun-db.mdc`, яке виявляє небезпечні патерни при використанні Bun SQL та підтверджує коректне застосування механізмів `LISTEN/NOTIFY` у застосунках, що взаємодіють з `pg`.

## Поведінка

1. Ініціалізація перевірки з використанням `lint` для валідації відповідності проєкту правилу `js-bun-db.mdc`.
2. Визначення всіх шляхів до JS/TS-джерел у репозиторії, повністю ігноруючи вказані каталоги.
3. Сканування знайдених джерел на небезпечні патерни Bun SQL, одночасно збираючи метадані про використання `pg`/LISTEN-NOTIFY.
4. Перевірка залежностей PostgreSQL: для кожного `package.json` з `dependencies.pg` підтверджується наявність використання патерну LISTEN/NOTIFY. Також перевіряється, чи містить кожен файл з імпортом `'pg'` цей патерн, щоб уникнути приховування звичайних SQL-операцій.
5. При виявленні порушень, генеруються деталізовані повідомлення про помилки, що базуються на конкретному типі проблеми (наприклад, відсутність "guard" у списку).
6. Усі фінальні висновки збираються у результат `lint`, який повертається при завершенні перевірки.
