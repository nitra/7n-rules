## Огляд

Поведінка програми полягає у виконанні основної логіки, визначеної в функції `main`. Програма надає функціональність, що не передбачає взаємодії з файловою системою або базами даних. Для верифікації коректності роботи логіки необхідно застосувати тести, які відносяться до `test.mdc`.

## Поведінка

1. Викликається `main` для початку перевірки.
2. Програма шукає всі файли, що відповідають шаблону JS-тесту.
3. Для кожного тестового файлу здійснюється пошук виклику `process.chdir`.
4. Якщо виклик знайдено у тестовому файлі, фіксується його місцезнаходження.
5. Якщо жоден виклик не знайдено, виводиться повідомлення про успішне виконання перевірки (test.mdc).
6. Якщо знайдено виклики, для кожного інциденту виводиться повідомлення про порушення (test.mdc), з обов'язковою згадкою про рекомендацію використання `withTmpDir + явні join + cwd: dir`.

## Публічний API

main — Забороняє зміну робочої директорії (`process.chdir`) у тестових файлах.
