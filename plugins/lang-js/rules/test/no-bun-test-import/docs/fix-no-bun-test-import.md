## Огляд

Цей файл забезпечує автоматичне виправлення імпортів у контексті `test/no-bun-test-import`. Він сканує файли та замінює декларації імпорту з `bun:test` на їхні еквіваленти з `vitest` (такі як `describe`, `test`, `it`, `expect`, `beforeEach`, `beforeAll`, `afterEach`, `afterAll`). Мета полягає в уніфікації тестового середовища за допомогою `vitest`. Зауважте, що імпорти, які містять `mock`, `spyOn` або `jest`, не підлягають автоматичній заміні, оскільки вони вимагають ручного коригування місць виклику API (`vi.fn`/`vi.spyOn`).

## Поведінка

Поведінка:
1. Ініціалізувати процес автофіксу для виявлених порушень, пов'язаних із використанням `bun-test-import`.
2. Ітеративно обробити кожен файл, у якому виявлено такі порушення.
3. Для кожного файлу прочитати його вміст.
4. Виявити імпорти з `bun:test`, які можуть бути замінені на еквівалент у `vitest`.
5. Замінити ці імпорти з кінця файлу до початку, щоб уникнути зсуву позицій наступних матчів.
6. Записати змінений вміст назад у файл.
7. Після обробки всіх файлів повернути перелік змінених файлів та повідомлення про застосовані зміни.
8. Використовувати наданий об'єкт `patterns` для визначення логіки обробки.
