## Огляд

Задає канонічний preview для Vue-компонентних бібліотек, щоб Storybook мав відтворювану базову конфігурацію в межах правил проєкту.

Файл генерується правилом `storybook`: команда `npx @7n/rules fix storybook` відновлює його, якщо файл видалено або канон порушено.

## Поведінка

1. Підключає Storybook до Vue-бібліотеки як канонічне preview-середовище, яке правило `storybook` може відновити для однакового запуску stories у всіх компонентах.

2. Ініціалізує перехоплення мережевих запитів для stories, щоб компоненти могли працювати з контрольованими mock-відповідями.

3. Пропускає очікувані same-origin GET-запити dev-сервера, щоб службовий трафік Storybook і Vite не створював зайвих попереджень.

4. Позначає інші не перехоплені запити попередженням, щоб показати потенційно неописану інтеграцію без зупинки збірки.

5. Встановлює Quasar у Storybook як повноцінний plugin, щоб компоненти бібліотеки бачили ті самі глобальні Quasar-примітиви, що й у реальному застосунку-споживачі.

6. Додає Quasar-плагіни сповіщень і діалогів, щоб stories коректно відтворювали компоненти, які залежать від цих UI-сервісів.

7. Налаштовує icon set для Quasar, щоб внутрішні компоненти й stories мали стабільне відображення іконок поза повним Quasar CLI build.

8. Підключає loader для mock-service-worker перед рендерингом кожної story, щоб мережеві сценарії були готові до показу компонента.

9. Надає стандартні параметри controls для Storybook, щоб UI-редактор коректно розпізнавав поля кольорів і дат.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
