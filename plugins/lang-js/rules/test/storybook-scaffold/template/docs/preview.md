## Огляд

Файл задає канонічний preview для Vue-компонентних бібліотек відповідно до `storybook.mdc` та ADR `канон-storybook-для-vue-компонентних-бібліотек`. Він існує, щоб Storybook мав відтворювану базову конфігурацію для перегляду компонентів у бібліотеках.

Файл згенеровано правилом `storybook`: команда `npx @7n/rules fix storybook` відтворює його, якщо файл видалено або зламано канон.

Файл є read-only: не пише у ФС чи БД.

## Поведінка

1. Готує Storybook як канонічне середовище для Vue-компонентних бібліотек, щоб stories працювали наближено до реального застосунку-споживача.

2. Підключає базові стилі Quasar, щоб компоненти відображалися з очікуваним виглядом у Storybook.

3. Вмикає перехоплення мережевих запитів у Storybook, щоб компоненти могли працювати з контрольованими mock-відповідями.

4. Ігнорує очікувані same-origin GET-запити dev-сервера, зокрема для HMR і статичних файлів, щоб вони не створювали зайвого шуму.

5. Позначає інші мережеві запити, які не підпадають під тихий пропуск, попередженням, а не помилкою збірки, щоб дати м’який сигнал без блокування роботи Storybook.

6. Встановлює Quasar у Storybook повністю, щоб бібліотечні компоненти мали доступ до глобальних Quasar-примітивів так само, як у реальному застосунку.

7. Додає Quasar-плагіни для повідомлень і діалогів, щоб stories могли відтворювати сценарії з користувацькими сповіщеннями та модальними взаємодіями.

8. Налаштовує набір іконок Quasar, щоб компоненти коректно показували матеріальні SVG-іконки поза повним Quasar CLI-середовищем.

9. Передає Storybook декоратор для mock service worker, щоб мережеве оточення застосовувалося до stories централізовано.

10. Налаштовує автоматичне розпізнавання controls-панелі Storybook для кольорів і дат, щоб інтерфейс керування props був зручнішим для перегляду компонентів.
