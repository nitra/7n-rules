## Огляд

Оголошує канонічний Storybook-конфіг для Vue-компонентних бібліотек, зафіксований правилом `storybook.mdc` та ADR `канон-storybook-для-vue-компонентних-бібліотек`. Файл потрібен як відтворювана точка налаштування Storybook: якщо його видалено або зламано канон, команда `npx @7n/rules fix storybook` відновлює очікуваний стан.

## Поведінка

1. Оголошує канонічний Storybook-конфіг для Vue-компонентних бібліотек, який має бути відтворюваним правилом `storybook`.

2. Налаштовує Storybook на збирання Vue через Vite та підключає історії за канонічним шаблонним шляхом.

3. Додає публічний каталог для статичних ресурсів, потрібних preview-оточенню, зокрема для service worker.

4. Примусово спрямовує Storybook-збирач на порожній Vite-конфіг поруч із цим шаблоном, щоб він не підхоплював Vite-конфіг пакета до фінального контрольованого обʼєднання.

5. Під час фінального Vite-налаштування читає Vite-конфіг поточного пакета, щоб успадкувати релевантні налаштування споживача.

6. Розгортає вкладені й асинхронні записи плагінів у фактичний список плагінів, щоб подальша фільтрація працювала з реальними плагінами, а не з обгортками.

7. Відкидає плагіни файлової маршрутизації та layout-інфраструктури застосунку, бо вони зайві для ізольованого рендеру компонентів у Storybook.

8. Відкидає дублікати Vue-трансформації та Quasar-плагіна з конфіга пакета, щоб замінити їх власними екземплярами у фіксованому порядку.

9. Зберігає інші корисні плагіни пакета, зокрема ті, що забезпечують macro-синтаксис або допоміжну автоматизацію бібліотеки.

10. Обʼєднує Storybook-конфіг із успадкованими налаштуваннями резолву, CSS і контрольованим списком плагінів, щоб Storybook працював близько до середовища пакета без конфліктних трансформацій.
