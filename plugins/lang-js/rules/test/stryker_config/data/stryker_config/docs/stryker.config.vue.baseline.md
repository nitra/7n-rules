## Огляд

Огляд
Цей файл відповідає за конфігурацію та запуск мутаційного аналізу коду за допомогою Stryker. Він вимірює якість тестування шляхом порівняння покриття вихідного коду тестами. Для налаштування опирається на `mutation.json` та `incremental.json`.

Поведінка
Тестове середовище налаштовується для використання Vitest відповідно до конфігураційного файлу Vitest. Запуск мутаційного аналізу здійснюється за допомогою Stryker, який обробляє кожен тест окремо для підвищення швидкості порівняно із запуском всього тестового набору. Згенеровані результати зберігаються у текстовому та JSON-форматах, причому JSON-звіт поміщається у `reports/stryker/mutation.json`. Для оптимізації подальших аналізів використовується `reports/stryker/incremental.json` для підтримки стану аналізу. Також реалізовано ігнорування мутацій у Vue-макросах `<script setup>` за допомогою спеціального локального плагіна, що запобігає збоям компіляції SFC.

## Поведінка

1. Визначає середовище виконання тестів як Vitest.
2. Налаштовує відповідність до файлу конфігурації Vitest.
3. Запускає Stryker для аналізу коду, обробляючи кожен тест окремо, що підвищує швидкість порівняно з запуском всього тестового набору.
4. Створює тимчасовий каталог для звітів Stryker.
5. Збирає результати в текстовий та JSON-формати, зберігаючи JSON-звіт у `reports/stryker/mutation.json`.
6. Дозволяє відновлення результатів попередніх запусків за допомогою файлу `reports/stryker/incremental.json`, що прискорює повторні запуски.
7. Використовує спеціальний локальний плагін для ігнорування мутацій у макросах Vue `<script setup>`, щоб уникнути збоїв при компіляції SFC.
8. Визначає мутації Vue-макросів як ігноровані.
