## Огляд

Файл забезпечує виконання тестових сценаріїв для виявлення дефектів у коді. Він використовує `vitest` для тестування та інтегрує Stryker для мутаційного тестування. Процес збору та збереження результатів мутацій відбувається відповідно до конфігурацій `mutation.json` та `incremental.json`.

## Поведінка

1. Використовує `vitest` як рушій для виконання тестів.
2. Визначає конфігураційний файл `vitest.config.mjs` для `vitest`.
3. При зміні коду Stryker запускає лише ті тести, які покривають змінену лінію, що підвищує швидкість порівняно з запуском всього набору тестів для кожного мутанта.
4. Встановлює тимчасову директорію для звітів як `reports/stryker/.tmp`.
5. Виводить результати у форматі JSON та у вигляді чистого тексту.
6. Зберігає деталі мутацій у файлі `reports/stryker/mutation.json`.
7. Увімкнено інкрементальний режим, що дозволяє відновлювати результати після збоїв, забезпечуючи значне прискорення при повторних запусках.
8. Процес роботи спирається на конфігураційні файли `mutation.json` та `incremental.json`.
