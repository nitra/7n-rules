## Огляд

`lint` перевіряє файли на відповідність правилу й повертає результат перевірки для подальшої обробки lint-системою. Секція потрібна, щоб зафіксувати поведінку правила з урахуванням файлових обходів, перевірок і передавання знайдених порушень у reporter.

## Поведінка

1. `lint` визначає app-пакети, для яких увімкнено Storybook-перевірки сторінок.

2. Якщо таких app-пакетів немає, перевірка завершується успішно з повідомленням про відсутність цільових пакетів.

3. Для кожного цільового app-пакета перевіряються Vue-сторінки в `src/pages/`.

4. Сторінка вважається покритою smoke-story, якщо в її каталозі є хоча б один story-файл.

5. Якщо поруч зі сторінкою немає story-файла, `lint` фіксує попередження, а не блокує зміни, бо це м’яка хвиля контролю покриття.

6. Повідомлення порушень пов’язують результат із відповідним правилом.

## Публічний API

- lint — Detector concern-а `storybook/page-coverage` (ADR-розширення 2026-07-20, хвиля 2a): для
кожного app-пакета у скоупі (`storybook.detectApps: true`, `collectInScopeVuePackages`,
`type: 'app'`) — кожен `.vue` під `src/pages/` має мати хоча б одну story поряд. Рівень
`warn` (не гейт) — на відміну від бібліотечного скафолду хвилі 1, smoke-покриття
сторінок хвилі 2a свідомо мʼяке.

## Гарантії поведінки

- Перевірка обходить і читає файли сторінок та story-файлів; знайдені порушення передаються в reporter як попередження.
