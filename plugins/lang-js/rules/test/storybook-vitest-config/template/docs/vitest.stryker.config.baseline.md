## Огляд

Файл створює специфічний конфігураційний об'єкт для Stryker, відокремлений від загального налаштування Vitest. Він забезпечує ізольоване середовище для тестування, використовуючи `happy-dom`. Конфігурація визначає, що покриття коду збирається для всіх файлів, що мають розширення `.test.js` або `.test.mjs` у директорії `tests` та у корені проєкту. Вона також налаштовує звітність про покриття, використовуючи провайдер `v8` та виводячи звіти у форматах `lcov` та `text-summary`.

## Поведінка

1. Забезпечує окремий конфігураційний об'єкт для Stryker, що ізольовано від основного конфігу Vitest, який включає проєкти для `unit` тестування та `storybook`.
2. Використовує конфігурацію з середовищем `happy-dom` для виконання тестів.
3. Включає у відповідність усі файли з розширеннями `.test.js` та `.test.mjs` у корені та в директорії `tests`.
4. Налаштовує збір покриття коду з використанням провайдера `v8` та репортерів `lcov` і `text-summary`.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
