## Огляд

Окремий vitest-конфіг для набору тестів, який використовують у перевірках мутацій. Він повторює базові налаштування основного середовища, але додає власні обмеження для цього сценарію. Тут також задаються параметри покриття, щоб результат був придатний для подальшої автоматичної обробки та швидкого перегляду.

## Поведінка

1. Формує окремий vitest-конфіг для запуску Stryker поверх уже зібраного базового тестового набору.
2. Зберігає сумісність із unit-тестами основного середовища, але відсікає storybook-частину, яка для Stryker непридатна через browser-mode.
3. Вмикає виконання тестів у DOM-подібному середовищі, щоб мутування перевірялося в близьких до браузера умовах без справжнього браузера.
4. Обмежує охоплення лише unit-подібними тестами проєкту; storybook-проєкти та інші browser-mode сценарії тут свідомо не проходять.
5. Налаштовує збір покриття так, щоб результат був придатний і для машинної обробки, і для швидкого людського перегляду.
6. Призначений саме для окремого файлу конфігурації Stryker; основний vitest-конфіг має лишатися джерелом спільних налаштувань.
