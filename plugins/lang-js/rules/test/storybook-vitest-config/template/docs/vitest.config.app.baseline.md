## Огляд

Файл задає канонічний `vitest`-конфіг для APP-проєкту, щоб один опис визначав правила запуску та перевірок тестового середовища. Він централізує поведінку для `unit`-сценаріїв і утримує узгодженість тестової конфігурації в межах проєкту.

## Поведінка

1. Формує канонічний `vitest`-конфіг для APP-проєкту в контексті Storybook, щоб один файл задавав узгоджені правила для unit- і storybook-перевірок.
2. Поєднує базовий Vite-конфіг із тестовими налаштуваннями, щоб зберегти спільний каркас проєкту й не дублювати загальні правила.
3. Встановлює два логічні контури запуску: `unit` для звичайних тестів і `storybook` для перевірок історій.
4. Для `unit` залишає стандартний набір тестів без додаткових browser-налаштувань, щоб ізолювати звичайну перевірку коду від UI-оточення.
5. Для `storybook` вмикає окремий набір плагінів, щоб Storybook-тести працювали з тим самим UI-стеком, що й застосунок.
6. Підключає підтримку Storybook-конфігурації з каталогу `.storybook`, щоб перевірки були прив’язані до локального джерела налаштувань історій.
7. Додає UI-плагіни для Quasar, автоматичних імпортів і сторінок, щоб storybook-проєкт бачив ті самі застосункові можливості, що й основний код.
8. Запускає storybook-перевірки в browser-режимі через Chromium у headless-форматі, щоб отримувати поведінковий результат, наближений до реального користувацького сценарію.
9. Обмежує storybook-частину окремим списком історій, щоб перевіряти лише визначений набір story-файлів.
10. Підключає окремий setup-файл для storybook-проєкту, щоб підготувати середовище саме для UI-перевірок.
11. Не виконує власних операцій запису у ФС або БД; файл лише описує поведінку тестового запуску.
