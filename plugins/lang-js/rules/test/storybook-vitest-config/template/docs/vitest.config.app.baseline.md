## Огляд

Файл задає Vitest-конфіг APP-проєкту, який поєднує основний `export default` із Storybook-скоупом через `test.projects`. Він потрібен, щоб APP-тести й перевірки Storybook-історій мали окремо позначені межі виконання та читались у конфігу як явні секції для Storybook, браузерного запуску, провайдера, інстансів і setup-файлів.

## Поведінка

1. Формує конфігурацію тестів для APP-проєкту, поєднуючи базові налаштування застосунку з правилами Vitest.

2. Визначає стандартний набір unit-тестів для файлів тестів у проєкті та запускає їх у DOM-подібному середовищі.

3. Вмикає звітність покриття коду, щоб команда бачила підсумок перевіреності застосунку тестами.

4. Розділяє тестовий запуск на два named projects: окремо звичайні unit-тести й окремо Storybook-тести.

5. Для Storybook-скоупу підключає браузерний запуск у Chromium, щоб історії компонентів перевірялися в середовищі, ближчому до реального UI.

6. Додає до Storybook-проєкту власний набір плагінів застосунку, потрібний для коректної роботи компонентів із Quasar, auto-imports і сторінками.

7. Обмежує Storybook-тести заданим шаблонним набором історій, який підставляється під час генерації конфіга для конкретного пакета.

8. Підключає Storybook setup-файл, щоб перед запуском історій застосовувались спільні тестові налаштування Storybook.

## Гарантії поведінки

- Файл лише описує тестову конфігурацію; побічні ефекти можливі під час виконання імпортованих модулів або запуску інструментів, які цю конфігурацію споживають.
