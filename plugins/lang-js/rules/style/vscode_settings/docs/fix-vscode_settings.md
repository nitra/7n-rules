## Огляд

Файл описує autofix-сценарій `patterns` для підтримання `.vscode/settings.json` через деклараційний `template-deep-merge`. Він потрібен, щоб проєкт отримував базові VS Code налаштування за відсутності файлу або доповнював наявні налаштування лише відсутніми канонічними полями без втрати локальних значень.

## Поведінка

1. `patterns` оголошує єдиний autofix-сценарій для підтримання канонічних VS Code налаштувань проєкту через `settings.json`.
2. Якщо файл налаштувань відсутній, сценарій має створити його з шаблону правила, щоб проєкт отримав базову узгоджену конфігурацію редактора.
3. Якщо файл уже існує, сценарій має додати лише відсутні канонічні поля, щоб зберегти локальні налаштування розробника без перезапису.

## Публічний API

- patterns — Фікс-патерни концерну: один template-deep-merge запис для `.vscode/settings.json`.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
