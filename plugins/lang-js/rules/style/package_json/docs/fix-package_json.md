## Огляд

`T0-autofix` для `style/package_json` описує `template-deep-merge`: створення відсутнього `package.json` із канонічного шаблону правила або доповнення наявного файлу лише бракуючими канонічними полями. Публічна функція `patterns` потрібна, щоб проєкти автоматично отримували поля, очікувані правилом, без втрати локальних налаштувань.

## Поведінка

1. `patterns` оголошує єдиний autofix для правила стилю, що відповідає за наявність і канонічне наповнення `package.json`.
2. Якщо файл відсутній, autofix має створити його з шаблону правила, щоб проєкт отримав базову узгоджену структуру.
3. Якщо файл уже існує, autofix має додати лише відсутні канонічні поля, зберігаючи локальні значення та проєктні доповнення.
4. Така поведінка потрібна, щоб автоматично вирівнювати мінімальні вимоги до метаданих пакета без перезапису ручних налаштувань.

## Публічний API

- patterns — Фікс-патерни концерну: один template-deep-merge запис для `package.json`.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
