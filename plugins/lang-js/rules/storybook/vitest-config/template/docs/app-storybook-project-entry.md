## Огляд

Файл задає `storybook`-проєкт для app-пакетів у спільній тестовій конфігурації. Він також передбачає каркас для специфічних для проєкту доповнень поверх базової структури `vitest`-конфіга.

## Поведінка

1. Додає канонічний `storybook`-проєкт для app-пакетів із browser-mode на Chromium, щоб Storybook- історії запускалися в тому ж тестовому контурі, що й інші проєкти.
2. Підключає набір плагінів, без яких app-storybook-історії не зможуть коректно збиратися: Quasar для стилів і компонентного контексту, auto-import для глобально доступних композиційних і helper-API, Pages для обробки сторінкових маршрутних блоків.
3. Залишає місце для проєкт-специфічних доповнень поверх базового набору, щоб пакет міг додати власні storybook-інтеграції без зміни канонічного каркаса.
4. Вказує окремий `setupFiles` для Storybook, щоб підготувати тестове середовище саме для storybook-запуску.
5. Працює як частина спільного vitest-каркаса, а не як самостійний приклад застосунку.
