## Огляд

Файл декларує autofix-сценарій для `npm-module/root_package_json`, який приводить кореневий `package.json` до канонічної декларації npm-модуля. Він потрібен, щоб за відсутності файлу створювати базовий шаблон, а за наявності — доповнювати його канонічними полями без втрати локальних налаштувань.

## Поведінка

1. `patterns` оголошує один autofix-сценарій для правила `npm-module/root_package_json`, який відповідає за приведення кореневого `package.json` до канонічної структури npm-модуля.
2. Якщо `package.json` відсутній, сценарій має створити його з шаблону правила, щоб проєкт отримав базову декларацію npm-модуля.
3. Якщо `package.json` уже існує, сценарій має доповнити його лише канонічними полями з шаблону, зберігаючи локальні налаштування проєкту.
4. Файл лише декларує доступний autofix-патерн і не виконує власних операцій запису.

## Публічний API

- patterns — Фікс-патерни концерну: один template-deep-merge запис для `package.json`.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
