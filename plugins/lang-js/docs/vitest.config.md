## Огляд

Файл задає Vitest-режим для плагіна lang-js: він підтягує env-канон ядра та обмежує запуск до потрібного набору тестів і службових перевірок. Шлях `node_modules` свідомо не береться до уваги, щоб конфіг не виходив за межі потрібного набору тестів.

## Поведінка

1. Обмежує запуск лише тестів і пов’язаних перевірок плагіна lang-js, щоб не зачіпати сторонні частини репозиторію.
2. Застосовує однаковий для ядра набір середовищних налаштувань, щоб запуск тестів був стабільним і не створював зайвого trace-шуму.
3. Використовує окремий тимчасовий шлях для LLM-trace, щоб ізолювати службові артефакти від робочих файлів.
4. Працює в node-оточенні з fork-пулом, щоб тести виконувалися передбачувано й незалежно між собою.
5. Свідомо ігнорує `node_modules`, щоб не витрачати ресурси на залежності, які не є частиною продуктового коду.

## Гарантії поведінки

- Свідомо пропускає шляхи: `node_modules`.
