## Огляд

Надає JS/TS-провайдер для ядра правил: визначає, чи стосується файл JS/TS/Vue-екосистеми, і делегує роботу сценаріям для unit-тестів, mutation-перевірок і Storybook stories. Файл існує як адаптер маршрутизації: передає запити ядра на генерацію тестового покриття, створення stories, виправлення тестів або обробку survived-мутантів, не виконуючи власного запису даних.

## Поведінка

1. Надає JS/TS-провайдер покриття для ядра правил, щоб однаково оцінювати line coverage, mutation coverage і окремий Storybook-вимір для компонентів.

2. Делегує виявлення проєкту, збір загального покриття та збір покриття по файлах спеціалізованим модулям екосистеми.

3. Для файлів із недостатнім unit-покриттям спершу уточнює, чи справді потрібні тести, а потім передає до генерації лише ті файли, де потребу підтверджено.

4. Для Vue-компонентів із недостатнім Storybook-покриттям запускає догенерацію stories, які далі мають перевірятися тестовим Storybook-проєктом споживача.

5. Для survived-мутантів запускає batch-виправлення через агентний механізм і повертає перелік файлів, які були змінені під час виправлення.

6. Для падаючих тестів запускає виправлення на основі звіту тестового раннера й повертає перелік переписаних тестових файлів.

7. Не виконує власних операцій запису у цьому файлі: зміни виконують підключені fix-модулі, а провайдер лише передає їм контроль записів і часові обмеження.

8. Не завантажує fix-модулі під час read-only збору покриття; вони підключаються лише тоді, коли ядро просить виконати відповідне виправлення.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
