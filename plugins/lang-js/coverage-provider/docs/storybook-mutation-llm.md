## Огляд

Файл додає до `storybook-mutation.mjs` LLM-джерело контекстних bug-like мутантів, які важко отримати детермінованими операторами. `proposeLlmMutants` існує, щоб розширити набір кандидатів на мутації без LLM-суддівства: LLM лише пропонує зміни, а приймаються тільки пропозиції з покритого сторі рядка, точним збігом `original` і валідним парсингом через `parseAst` для подальшого browser-mode прогону.

LLM-мутанти вимикаються через `N_7N_TEST_NO_LLM_MUTANTS=1`. Якщо LLM-виклик недоступний або завершується помилкою, джерело повертає `[]` з одноразовим попередженням, щоб детерміновані мутанти продовжували працювати.

## Поведінка

1. `proposeLlmMutants` запитує LLM про контекстні bug-like мутанти для одного файлу, щоб доповнити детерміновані оператори випадками на кшталт off-by-one, підмінених fallback-значень або переплутаних аргументів.

2. LLM використовується лише як джерело пропозицій: рішення про те, чи мутант убитий або вижив, залишається за реальним browser-mode прогоном у Storybook mutation executor-і.

3. Кожна пропозиція проходить жорстке приймання перед передачею executor-у: рядок має існувати, бути покритим сторі, містити точний вихідний фрагмент, а результат заміни має залишати код синтаксично придатним.

4. Невалідні, непокриті або синтаксично зламані пропозиції відкидаються, щоб помилки компіляції не зараховувалися як убиті мутанти й не завищували mutation score.

5. За успішного сценарію `proposeLlmMutants` повертає лише валідні мутанти у форматі, придатному для стандартного генератора мутантів; якщо придатних пропозицій немає, результат порожній.

6. Помилки LLM-виклику не мають зупиняти mutation-перевірку загалом: обгортка виклику перетворює такі збої на порожній список із одноразовим попередженням, а детерміновані мутанти продовжують працювати.

7. Генерацію LLM-мутантів можна вимкнути через `N_7N_TEST_NO_LLM_MUTANTS=1`, коли потрібен повністю детермінований або offline-прогін.

## Публічний API

- proposeLlmMutants — Пропонує LLM-мутанти для одного файлу: один callText-виклик (окремий chain),
  парс JSON-відповіді, жорстка валідація кожної пропозиції (рядок/підрядок/
  покриття/синтаксис). Кидає помилки LLM-виклику нагору — graceful-обгортка
  (warn + []) живе на боці виклику (defaultRunner у js-collector.mjs).
