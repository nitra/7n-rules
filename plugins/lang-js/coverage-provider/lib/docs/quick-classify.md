## Огляд

`quickClassify` швидко й детерміновано класифікує вміст коду для концерну `coverage`: чи потрібні unit-тести, чи файл є лише module-wiring, чи потрібне подальше LLM-довизначення. Евристика витягнута з `assess-need.mjs` `@7n/test`, spec `2026-07-22 absorb-7n-test`, щоб детермінована частина жила у провайдері як delta-gate концерну `coverage` без I/O та LLM, а неоднозначні файли передавались у fix-шлях.

## Поведінка

1. `quickClassify` отримує вміст файла та готує його до локальної оцінки, ігноруючи коментарі, щоб службові пояснення не впливали на вердикт.

2. Якщо після очищення файл складається лише з module-wiring, `quickClassify` повертає рішення, що unit-тести не потрібні, бо у файлі немає тестовної поведінки.

3. Якщо файл містить виконувану логіку з розгалуженнями всередині функціональних блоків, `quickClassify` повертає рішення, що unit-тести потрібні, бо така поведінка має перевірятися автоматично.

4. Якщо локальних ознак недостатньо для впевненого рішення, `quickClassify` не виносить вердикт. Такий випадок передається на подальше LLM-довизначення у fix-шляху концерну coverage.

5. `quickClassify` працює детерміновано, без I/O, без запису даних і без звернення до LLM.

## Публічний API

- quickClassify — Класифікує очевидні випадки; неоднозначні → `null` (вирішує LLM у fix-шляху).

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
