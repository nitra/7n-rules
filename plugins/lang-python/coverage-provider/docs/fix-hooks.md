## Огляд

Модуль надає інтерфейс для вдосконалення тестового покриття Python-коду та підвищення його стійкості до мутацій. Він дозволяє створювати специфічні промпти (`buildGenTestsPrompt`, `buildFixSurvivedPrompt`) для агентних сесій, які автоматично генерують pytest-тести для файлів, що мають низький рівень покриття (`generatePythonTests`), а також тестують виявлення мутантів, які вижили після аналізу (`fixPythonSurvived`). Вся робота відбувається в межах агентних fix-сесій, що керуються відповідно до ladder-контракту ядра покриття.

## Поведінка

buildGenTestsPrompt генерує промпт для агентної сесії з інструкціями щодо написання pytest-тестів для Python-файлів, які мають низький рівень покриття, використовуючи список файлів та їхній відсоток покриття як вхідні дані. buildFixSurvivedPrompt формує промпт для агента з вимогами щодо написання тестів, здатних виявити мутанти, що пережили попередні тести, базуючись на групування файлів та мутантів. generatePythonTests викликає runSession, передаючи промпт, згенерований buildGenTestsPrompt, для запуску сесії з метою створення нових тестів для файлів з низьким покриттям. fixPythonSurvived також викликає runSession, але з промптом, згенерованим buildFixSurvivedPrompt, для запуску сесії, яка спрямована на покриття виживших мутантів. Усі ці функції використовують внутрішню асинхронну функцію runSession для керування реалізацією агентної сесії, яка отримує контекст та викликає відповідні дії, повертаючи список фактично змінених файлів.

## Публічний API

- buildGenTestsPrompt — Промпт догенерації pytest-тестів для файлів нижче порогу покриття.
- buildFixSurvivedPrompt — Промпт написання тестів, що вбивають survived-мутанти mutmut.
- generatePythonTests — Догенерація pytest-тестів для файлів нижче порогу (hook `generateTests`).
- fixPythonSurvived — Тести проти survived-мутантів mutmut (hook `fixSurvived`).

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
