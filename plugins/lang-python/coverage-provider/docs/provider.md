## Огляд

Модуль збирає line coverage через `uv run pytest --cov` із `pytest-cov` та `lcov`-звітом, а також додає mutmut 4.x як окремий вимір, якщо проєкт це підтримує через `[tool.mutmut].source_paths` у `pyproject.toml`. Його роль — віддати ядру `test` уже зведений Python coverage без власної CLI-оркестрації.

Якщо `uv` недоступний, модуль чесно пропускає запуск із одноразовим hint. Якщо в `pyproject.toml` немає `[tool.mutmut].source_paths`, mutational вимір пропускається з warning, але line coverage усе одно збирається.

## Поведінка

1. `defaultRunner` запускає зовнішні Python-інструменти для збору звіту покриття й, за наявності підтримки mutmut 4.x, для мутаційного виміру.
2. Якщо в проєкті немає `uv`, модуль чесно пропускає збір даних і один раз показує підказку, без помилкового падіння всього процесу.
3. Для line coverage він проходить по Python-коренях під `cwd`, збирає покриття через `pytest-cov` у lcov-форматі та повертає загальні й файлові показники відносно кожного Python-кореня.
4. Для mutmut він спочатку перевіряє, чи в корені Python-пакета є коректна конфігурація `mutmut` 4.x; якщо секції `source_paths` немає, мутаційний вимір для цього кореня пропускається з попередженням, але line coverage продовжує збиратися.
5. Коли mutmut доступний, модуль виконує повний цикл збору мутаційних результатів, рахує загальну кількість і survived-мутації та групує їх по файлах у форматі, придатному для звіту покриття.
6. Усі шляхи приводяться до меж відповідного Python-кореня, а потім — до `cwd`, щоб звіт був узгодженим для багато-пакетних репозиторіїв.
