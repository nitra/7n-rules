## Огляд

Файл перевіряє, чи цільові файли мають очікувані документаційні коментарі, і повідомляє про порушення без зміни перевірюваних файлів. Публічні точки входу `isDocCommentTarget`, `checkFileDocComments` і `lint` потрібні для визначення цілей перевірки, перевірки окремого файлу та запуску правила в lint-системі.

Шляхи `node_modules`, `.venv`, `venv`, `__pycache__` і `.worktrees` свідомо пропускаються, щоб не перевіряти сторонній або службовий код.

## Поведінка

`lint` формує набір Python-файлів для перевірки: під час повного обходу бере файли з робочого дерева, свідомо оминаючи `node_modules`, `.venv`, `venv`, `__pycache__` і `.worktrees`, а під час дельта-перевірки працює з уже переданим списком. Далі кожен шлях проходить через `isDocCommentTarget`, щоб залишити лише файли, на які поширюється правило docstring-ів.

Для кожного відібраного файлу `lint` читає вміст і передає його до `checkFileDocComments`. Ця перевірка повертає порушення для модуля без опису наміру або для публічних top-level функцій і класів без docstring. Якщо над публічним оголошенням є придатний суцільний коментарний блок, результат позначається як такий, що може бути автоматично підвищений до docstring.

Усі знайдені порушення накопичуються в межах одного прогону й повертаються як результат `lint`. Файл сам не змінює перевірювані Python-джерела: він лише читає їх і повідомляє, де документаційний контракт не виконано.

## Публічний API

- isDocCommentTarget — Чи підпадає файл під вимогу docstring-ів.
- checkFileDocComments — Перевіряє один `.py`-файл: module-docstring + docstring над кожним top-level
публічним def/class.
- lint — Detector python/doc_comments: per-file (дельта) або повний обхід.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає.
- Свідомо пропускає шляхи: `node_modules`, `.venv`, `venv`, `__pycache__`, `.worktrees`.
