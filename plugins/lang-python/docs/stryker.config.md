## Огляд

Цей файл конфігурує параметри для мутаційного тестування, використовуючи Vitest як тестовий раннер для прискорення виконання тестів. Він визначає директорії для звітів Stryker та налаштовує виведення результатів у форматах JSON і clear-text. Встановлюється режим інкрементального виконання, що забезпечує відновлення стану виконання за допомогою конфігурацій `mutation.json` та `incremental.json`.

## Поведінка

def mutator_config
1. Визначає тестовий раннер як Vitest, посилаючись на відповідний конфігураційний файл.
2. Встановлює аналіз покриття для виконання індивідуально для кожного тесту, що прискорює процес.
3. Визначає тимчасову директорію для звітів Stryker.
4. Налаштовує репортери для виведення результатів у форматах JSON та clear-text.
5. Специфічно конфігурує JSON-репортер, визначаючи шлях збереження вихідних даних мутацій.
6. Вмикає режим інкрементального виконання, що дозволяє відновлювати роботу між збоями.
7. Встановлює файл для зберігання інкрементальних даних, що є ключовим для відновлення процесу.
Mutation.json та incremental.json використовуються для збереження результатів Stryker.
end
