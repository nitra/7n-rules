## Огляд

Vitest-конфіг для плагіна lang-python: спирається на env-канон ядра, але обмежує запуск лише тестами цього плагіна. Свідомо пропускає `node_modules`, щоб не втягувати сторонній код у scope перевірки та тримати прогін зосередженим на поведінці самого плагіна.

## Поведінка

1. Підбирає лише тести плагіна lang-python і не змішує їх із тестами інших частин репозиторію.
2. Працює в node-середовищі з однаковим базовим набором runtime-налаштувань, щоб запуск тестів був стабільним і передбачуваним.
3. Вимикає git trace-стеження та задає окремий тимчасовий шлях для LLM trace, щоб тестовий прогін не створював зайвого шуму й не конфліктував із глобальним оточенням.
4. Обмежує час на виконання тестів, щоб завислі перевірки не блокували пайплайн надовго.
5. Запускає тести у forks-режимі, щоб ізолювати виконання та зменшити взаємний вплив тестів.
6. Свідомо пропускає `node_modules`, щоб не витрачати ресурси на сторонній код і не включати його в перевірки.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Свідомо пропускає шляхи: `node_modules`.
