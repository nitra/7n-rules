# main.js

## Огляд

Файл `demo/src/main.js` — точка входу (entry point) демонстраційного Vue-застосунку. Файл імпортує кореневий single-file component `App` з модуля `./App.vue` та монтує створений Vue-застосунок до DOM-елемента з ідентифікатором `#app` в HTML-документі.

Повний код файлу `demo/src/main.js`:

```js
import App from './App.vue'

createApp(App).mount('#app')
```

Файл складається лише з двох виконуваних інструкцій: ES-module import та одного ланцюжкового виклику `createApp(App).mount('#app')`. Файл не містить власних експортів, не визначає функцій, класів чи змінних — він лише ініціалізує застосунок під час завантаження модуля.

## Експорти / API

Файл `demo/src/main.js` **не містить жодних експортів** (немає інструкцій `export`, `export default`, `export const`, `export function` тощо). Це side-effect-only модуль: під час імпорту/завантаження він просто виконує інструкцію `createApp(App).mount('#app')`.

Публічного програмного API файл `demo/src/main.js` не надає. Єдина його зовнішня поведінка — змонтувати Vue-застосунок до DOM під час виконання.

## Функції

Файл `demo/src/main.js` **не визначає власних функцій** (немає `function`-декларацій, стрілкових функцій, методів, класів). Файл лише викликає сторонню функцію `createApp` із результатом-методом `mount`.

### Виклик `createApp(App)`

- **Сигнатура (зі сторони Vue 3):** `createApp(rootComponent: Component, rootProps?: object): App`
- **Параметри, що передаються у файлі `demo/src/main.js`:**
  - `App` — кореневий компонент, імпортований з `./App.vue` (тип: Vue Single-File Component).
- **Що повертає:** інстанс Vue-застосунку (об’єкт з API на кшталт `.mount`, `.use`, `.component`, `.directive`, `.provide`, `.config` тощо). У файлі `demo/src/main.js` цей інстанс не зберігається у змінну — на ньому одразу викликається метод `.mount('#app')`.
- **Side effects виклику `createApp(App)`:** створює внутрішній стан Vue-застосунку (інстанс), але сам по собі ще не модифікує DOM.

### Виклик `.mount('#app')`

- **Сигнатура (зі сторони Vue 3):** `mount(rootContainer: string | Element): ComponentPublicInstance`
- **Параметри, що передаються у файлі `demo/src/main.js`:**
  - `'#app'` — CSS-селектор кореневого DOM-контейнера, до якого монтується Vue-застосунок.
- **Що повертає:** інстанс кореневого компонента (повертається, але у файлі `demo/src/main.js` не зберігається й не використовується).
- **Side effects виклику `.mount('#app')`:**
  - Знаходить у документі елемент за селектором `#app`.
  - Рендерить шаблон компонента `App` всередину цього елемента.
  - Підключає реактивну систему Vue до DOM (оновлення стану призводять до повторного рендерингу).
  - Якщо елемент `#app` відсутній у DOM на момент виконання — Vue видасть попередження/помилку у консоль (поведінка залежить від білда Vue).

## Залежності

### Імпорти у файлі `demo/src/main.js`

- `import App from './App.vue'` — імпорт default-експорту із сусіднього файлу `./App.vue` (відносний шлях від `demo/src/main.js` — це `demo/src/App.vue`). `App` — це Vue-компонент, що використовується як кореневий.

### Неявні / зовнішні залежності

- **Функція `createApp`** — у файлі `demo/src/main.js` використовується **без явного імпорту**. Вона має бути доступною в глобальній області або наданою через build-tooling/auto-import-плагін (наприклад, `unplugin-auto-import` для Vite). Без авто-імпорту або глобального оголошення цей код у нормальному середовищі модулів ES викине `ReferenceError: createApp is not defined`. У контексті проєкту `demo/` передбачається, що `createApp` надає Vue 3 (`vue`) через механізм автоматичного імпорту.
- **DOM API:** код покладається на наявність у середовищі виконання браузерного DOM з елементом `<div id="app">` (зазвичай оголошується в `index.html` проєкту Vite/Vue).

### Транзитивно

- Vue 3 runtime (`vue`) — постачає `createApp` та реалізацію `.mount`.
- Збирач/девсервер (Vite/Webpack/тощо) — обробляє `.vue` SFC та, ймовірно, налаштовує авто-імпорт `createApp`.

## Потік виконання / Використання

Послідовність дій під час завантаження файлу `demo/src/main.js`:

1. Завантажувач модулів виконує `import App from './App.vue'` — компілює/підвантажує SFC `demo/src/App.vue` і присвоює його default-експорт локальній bind-змінній `App`.
2. Виконується синхронний вираз `createApp(App).mount('#app')`:
   - `createApp(App)` створює новий інстанс Vue-застосунку з `App` у ролі кореневого компонента.
   - `.mount('#app')` шукає в DOM елемент із селектором `#app`, рендерить у нього шаблон компонента `App` і запускає реактивний рендеринг-цикл Vue.
3. Подальша поведінка застосунку повністю визначається компонентом `App` із `demo/src/App.vue` (його `template`, `script`, `style`, props, state, lifecycle hooks тощо). Файл `demo/src/main.js` після монтування ніяких подальших дій не виконує.

### Як використовувати

- Файл `demo/src/main.js` зазвичай вказують як `entry` у конфігурації збирача (наприклад, у Vite — у полі `build.rollupOptions.input` або через `<script type="module" src="/src/main.js"></script>` в `index.html`).
- HTML-шаблон, що завантажує `demo/src/main.js`, мусить містити кореневий контейнер `<div id="app"></div>` до моменту виконання `.mount('#app')`.
- Для розширення застосунку (плагіни, глобальні компоненти, директиви, конфіг) ланцюжок переписується через проміжну змінну, наприклад:
  ```js
  const app = createApp(App)
  app.use(somePlugin)
  app.mount('#app')
  ```
  У поточному файлі `demo/src/main.js` такого розширення немає — лише голий старт із кореневим компонентом `App`.

### Side effects файлу `demo/src/main.js` сумарно

- Імпорт і компіляція SFC `./App.vue`.
- Створення Vue-інстансу.
- Модифікація DOM-піддерева `#app` (рендеринг кореневого компонента `App`).
- Підписки реактивної системи Vue на залежності всередині `App` (далі — за внутрішньою логікою `App.vue`).
