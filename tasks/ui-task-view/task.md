## Task

Розробити веб-інтерфейс для перегляду стану задач у системі `n-cursor graph`.

Інтерфейс має відображати рекурсивний граф вузлів, відновлений зі сканування файлової структури `tasks/`, з актуальними станами та можливістю розгортати/згортати вузли.

## Done when

- [ ] Сторінка показує дерево вузлів зі станами (`waiting` / `running` / `resolved` / `failed` / `invalidated`)
- [ ] Для кожного вузла доступна панель деталей: `task.md`, список `run_NNN.md` з полями `actor`, `result`, `created_at`, посилання на `outputs_NNN.md`
- [ ] Граф оновлюється без перезавантаження сторінки (polling або SSE)
- [ ] Вузли у стані `failed` та `invalidated` виділені візуально
- [ ] Інтерфейс зчитує дані через `graph scan --json` або REST-прошарок над ним

## Inputs

### context

Архітектура системи та контракт файлів описані у `docs/думка.MD`.

### constraints

- Технологічний стек: Vue 3 + Vite (поточний стек проєкту)
- Бекенд: мінімальний Node/Bun HTTP-сервер або CLI-команда `graph scan --json` як джерело даних
- Без зовнішніх залежностей для рендерингу графу — стандартне CSS-дерево або SVG
