# slugify.mjs

## Огляд

Модуль `slugify.mjs` надає одну чисту функцію `slugify`, яка перетворює довільний рядок на URL-безпечний slug. Функція виконує нормалізацію регістру, видаляє символи, що не належать до «слівних» (word) символів і пробілів, замінює пробіли на дефіси, об'єднує послідовні дефіси й обрізає результат до максимум 64 символів.

Файл написаний як ESM-модуль (`.mjs`) і не має жодних залежностей від інших модулів — лише вбудовані можливості JavaScript (`String.prototype.trim`, `toLowerCase`, `replaceAll`, `slice`, регулярні вирази).

Файл-джерело: `/Users/vitaliytv/www/nitra/cursor/.worktrees/main-docgen/benchmarks/runner-comparison/demo/src/slugify.mjs`.

## Експорти / API

| Експорт | Тип | Опис |
| --- | --- | --- |
| `slugify` | named export, функція `(input: string) => string` | Перетворює вхідний текст на URL-безпечний slug, обмежений 64 символами |

Інших експортів (default, re-export) модуль `slugify.mjs` не містить.

## Константи модуля (внутрішні)

Перед оголошенням функції `slugify` модуль `slugify.mjs` визначає три приватні (не експортовані) регулярні вирази на верхньому рівні модуля:

| Ім'я | Значення | Прапори | Призначення |
| --- | --- | --- | --- |
| `NON_WORD` | `/[^\w\s-]/g` | `g` (глобальний) | Збігається з будь-яким символом, який НЕ є word-символом (`\w` — `[A-Za-z0-9_]`), НЕ є whitespace-символом (`\s`) і НЕ є дефісом (`-`). Тобто будь-яка пунктуація, символ Unicode за межами ASCII word, спецсимвол тощо |
| `SPACES` | `/\s+/g` | `g` (глобальний) | Збігається з одним або кількома послідовними whitespace-символами (пробіл, табуляція, новий рядок тощо) |
| `DASHES` | `/-+/g` | `g` (глобальний) | Збігається з одним або кількома послідовними дефісами `-` |

Усі три регулярні вирази оголошені через `const` на рівні модуля, тож вони компілюються один раз під час завантаження модуля `slugify.mjs` і повторно використовуються в кожному виклику `slugify`.

## Функції

### `slugify(input)`

JSDoc у файлі `slugify.mjs`:

```
Convert text into a URL-safe slug.
@param {string} input source text
@returns {string} slug limited to 64 characters
```

#### Сигнатура

```js
export function slugify(input)
```

#### Параметри

| Параметр | Тип (за JSDoc) | Обов'язковий | Опис |
| --- | --- | --- | --- |
| `input` | `string` | так | Вихідний текст, який потрібно перетворити на slug |

#### Повертає

| Тип | Опис |
| --- | --- |
| `string` | URL-безпечний slug довжиною не більше 64 символів. Порожній рядок `''`, якщо `input` не є рядком |

#### Алгоритм (покроково)

1. Перевірка типу: якщо `typeof input !== 'string'`, функція `slugify` одразу повертає порожній рядок `''`. Це обробляє `undefined`, `null`, числа, об'єкти, масиви, булеві тощо — без кидання винятку.
2. Створюється локальна змінна `s` зі значенням `input.trim().toLowerCase()` — спершу обрізаються провідні/кінцеві whitespace-символи, потім увесь рядок переводиться в нижній регістр.
3. `s = s.replaceAll(NON_WORD, '')` — глобальна заміна всіх символів, що не є word/whitespace/дефіс, на порожній рядок (видалення). Це прибирає пунктуацію, лапки, дужки, не-ASCII букви тощо.
4. `s = s.replaceAll(SPACES, '-')` — кожна послідовність whitespace-символів (один або більше) замінюється на один дефіс `-`.
5. `s = s.replaceAll(DASHES, '-')` — кожна послідовність дефісів (один або більше) згортається до одного дефіса. Цей крок нормалізує, зокрема, ситуації, коли в оригіналі вже були дефіси перед/після пробілів, які тепер також стали дефісами.
6. Обмеження довжини: якщо `s.length > 64`, повертається `s.slice(0, 64)` — перші 64 символи; інакше повертається `s` як є.

#### Side effects

Жодних. Функція `slugify`:

- не змінює аргумент `input` (рядки в JavaScript іммутабельні);
- не пише в файлову систему, не звертається до мережі, не використовує `console`, `globalThis`, таймери;
- не залежить від часу, випадкових чисел чи зовнішнього стану;
- є детермінованою: за однаковим `input` завжди повертає однаковий результат.

#### Граничні випадки (behavior matrix)

| Вхід `input` | Результат |
| --- | --- |
| `undefined`, `null`, число, об'єкт, масив, булеве, `Symbol`, `BigInt` | `''` (через перевірку `typeof input !== 'string'`) |
| `''` | `''` |
| `'   '` (лише пробіли) | `''` (після `trim()` рядок стає порожнім, далі заміни не змінюють нічого) |
| `'Hello, World!'` | `'hello-world'` (кома й знак оклику видаляються, пробіл стає дефісом) |
| `'Hello---World'` | `'hello-world'` (`DASHES` згортає трикратний дефіс до одного) |
| `'Hello  World'` (кілька пробілів) | `'hello-world'` (`SPACES` згортає до одного дефіса) |
| Рядок з не-ASCII літерами (наприклад, кирилиця) | Не-ASCII літери видаляються `NON_WORD`, оскільки `\w` у JS — це `[A-Za-z0-9_]` без прапора `u` і без Unicode-розширень. Наприклад, `'Привіт світ'` → `'-'` (літери видалені, пробіл став дефісом, після `DASHES` лишається один дефіс) |
| Рядок з підкресленнями `'foo_bar'` | `'foo_bar'` (символ `_` входить у `\w` і зберігається) |
| Рядок з цифрами `'Title 42'` | `'title-42'` |
| Рядок довший за 64 символи | Обрізається до перших 64 символів через `s.slice(0, 64)` |
| Рядок рівно 64 символи | Повертається повністю (умова `s.length > 64` хибна) |

## Залежності

### Зовнішні модулі

Жодних. Модуль `slugify.mjs` не містить інструкцій `import` і не залежить від npm-пакетів чи інших файлів проєкту.

### Платформенні вимоги

Функція `slugify` використовує `String.prototype.replaceAll`, яка є стандартом ECMAScript 2021 (ES12). Тобто середовище виконання має підтримувати ES2021+:

- Node.js >= 15.0.0
- Bun (будь-яка актуальна версія)
- Сучасні браузери (Chrome 85+, Firefox 77+, Safari 13.1+)

Файл має розширення `.mjs`, тож Node.js трактує його як ESM-модуль незалежно від `"type"` у `package.json`.

## Потік виконання / Використання

### Імпорт

```js
import { slugify } from './slugify.mjs'
```

Шлях слід коригувати відповідно до розташування модуля-споживача відносно файлу `slugify.mjs` (у демо-проєкті він лежить у `benchmarks/runner-comparison/demo/src/slugify.mjs`).

### Типовий виклик

```js
import { slugify } from './slugify.mjs'

const title = 'Hello, World! Welcome to 2026.'
const slug = slugify(title)
// slug === 'hello-world-welcome-to-2026'
```

### Сценарій із обрізанням до 64 символів

```js
import { slugify } from './slugify.mjs'

const long = 'a'.repeat(100)
const slug = slugify(long)
// slug.length === 64
// slug === 'a'.repeat(64)
```

### Сценарій із не-рядковим вхідним значенням

```js
import { slugify } from './slugify.mjs'

slugify(undefined) // ''
slugify(null)      // ''
slugify(123)       // ''
slugify({})        // ''
```

### Послідовність трансформацій (внутрішня)

Для запиту `slugify('  Hello,  World!! ')` крокова трансформація змінної `s` всередині функції `slugify`:

1. `input` = `'  Hello,  World!! '`
2. Після `trim().toLowerCase()`: `'hello,  world!!'`
3. Після `replaceAll(NON_WORD, '')` (видалено `,`, `!`, `!`): `'hello  world'`
4. Після `replaceAll(SPACES, '-')` (два пробіли → один дефіс): `'hello-world'`
5. Після `replaceAll(DASHES, '-')` (немає послідовних дефісів — без змін): `'hello-world'`
6. Довжина 11 ≤ 64 → повертається `'hello-world'`.
