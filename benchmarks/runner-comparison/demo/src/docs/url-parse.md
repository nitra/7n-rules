# url-parse.mjs

## Огляд

Модуль `url-parse.mjs` — невелика самодостатня бібліотека для роботи з URL query string у форматі `key=value&key2=value2`. Містить дві чисті (pure) функції без зовнішніх залежностей:

- `parseQuery(qs)` — розбирає рядок query string на об'єкт пар ключ-значення.
- `buildQuery(params)` — будує закодований query string з об'єкта параметрів.

Обидві функції використовують стандартні браузерні/Node.js глобальні функції `decodeURIComponent` та `encodeURIComponent` для URL-кодування. Side effects відсутні: функції не мутують вхідні аргументи й не звертаються до зовнішнього стану.

Файл розташований у демо-частині бенчмарку `runner-comparison` (`benchmarks/runner-comparison/demo/src/url-parse.mjs`) і призначений насамперед як набір цілей для тестування різних JS-раннерів.

## Експорти / API

Модуль експортує два іменованих експорти (ESM, формат `.mjs`):

| Експорт         | Тип                                              | Опис                                              |
| --------------- | ------------------------------------------------ | ------------------------------------------------- |
| `parseQuery`    | `(qs: string) => Record<string, string>`         | Парсить query string у словник параметрів.        |
| `buildQuery`    | `(params: Record<string, unknown>) => string`    | Серіалізує об'єкт параметрів у query string.      |

Default-експорту немає.

## Функції

### `parseQuery(qs)`

Сигнатура:

```js
export function parseQuery(qs: string): Record<string, string>
```

Параметри:

- `qs` — `string`. Query string з опційним початковим знаком питання `?`. Приклади валідних вхідних значень: `"?a=1&b=2"`, `"a=1&b=2"`, `"flag&x=y"`, `""`.

Повертає:

- `Record<string, string>` — звичайний об'єкт (літерал `{}`), де кожен ключ — декодований ім'я параметра, а значення — декодоване значення параметра (або порожній рядок `''`, якщо в парі не було знаку `=`).

Поведінка та алгоритм `parseQuery`:

1. Якщо `qs` не є рядком (`typeof qs !== 'string'`) або має довжину `0` — функція повертає порожній об'єкт `{}`.
2. Ініціалізує локальний об'єкт-результат `out = {}`.
3. Якщо `qs` починається з символу `?`, відрізає його: `clean = qs.slice(1)`. Інакше `clean = qs`.
4. Розбиває `clean` за роздільником `&` на масив пар: `clean.split('&')`.
5. Для кожної пари `pair`:
   - Якщо `pair.length === 0` (порожня пара, наприклад через подвійний `&&` або кінцевий `&`) — пропускає її через `continue`.
   - Шукає індекс першого символу `=`: `eq = pair.indexOf('=')`.
   - Якщо `eq === -1` (знак `=` відсутній) — встановлює `out[decodeURIComponent(pair)] = ''` (ключ без значення трактується як ключ із порожнім рядком).
   - Інакше:
     - Бере ключ як підрядок до `eq`: `k = decodeURIComponent(pair.slice(0, eq))`.
     - Бере значення як підрядок після `eq`: `v = decodeURIComponent(pair.slice(eq + 1))`.
     - Записує `out[k] = v`. У випадку повторюваних ключів виграє останнє значення (overwrite).
6. Повертає `out`.

Особливості:

- Використовується `indexOf('=')`, тому значення може містити додаткові `=` (вони залишаються частиною значення).
- Якщо `decodeURIComponent` отримає некоректну %-послідовність, він викине `URIError` — `parseQuery` цю помилку не перехоплює.
- Жодних інших типів (масиви, числа, `null`, `undefined`) не трактує спеціально — будь-який нерядок повертає `{}`.

Side effects: відсутні. Функція не модифікує `qs` і не звертається до глобального стану.

### `buildQuery(params)`

Сигнатура:

```js
export function buildQuery(params: Record<string, unknown>): string
```

Параметри:

- `params` — об'єкт з парами ключ-значення. Значення можуть бути будь-якого типу; функція приводить їх до рядка через `String(v)`. Допускаються типи `string`, `number`, `boolean`, `bigint`, об'єкти з власним `toString()` тощо.

Повертає:

- `string` — закодований query string у форматі `k1=v1&k2=v2`. Без початкового `?`. Якщо вхідні параметри відсутні або всі значення відфільтровані — повертає порожній рядок `''`.

Поведінка та алгоритм `buildQuery`:

1. Якщо `params` falsy (`null`, `undefined`, `0`, `""`, `false`) або не є об'єктом (`typeof params !== 'object'`) — повертає `''`.
2. Ініціалізує локальний масив `parts = []`.
3. Ітерує по `Object.entries(params)`:
   - Якщо `v === undefined` або `v === null` — пропускає пару (такі значення не включаються в результат).
   - Інакше додає в `parts` рядок ``` `${encodeURIComponent(k)}=${encodeURIComponent(String(v))}` ```.
4. Повертає результат `parts.join('&')`.

Особливості:

- Ключ завжди кодується через `encodeURIComponent`.
- Значення приводиться до рядка через `String(v)` ще перед кодуванням, тож `true` → `"true"`, `42` → `"42"`, `{a:1}` → `"[object Object]"` тощо.
- Перевірка `typeof params !== 'object'` пропускає масиви (масив — це теж `'object'`), тому масив на вході оброблятиметься як об'єкт з числовими ключами (індекси `0`, `1`, ... перетворяться на рядки `"0"`, `"1"`, ...).
- Порядок ключів у результаті відповідає порядку перебору `Object.entries` (порядок вставки для рядкових ключів).

Side effects: відсутні.

## Залежності

- Зовнішні npm-залежності: немає.
- Внутрішні залежності проєкту: немає.
- Глобальні API стандартного середовища:
  - `decodeURIComponent` — використовується в `parseQuery` для декодування ключів і значень.
  - `encodeURIComponent` — використовується в `buildQuery` для кодування ключів і значень.
  - `String` (конструктор-приводник до рядка) — у `buildQuery`.
  - `Object.entries` — у `buildQuery`.
- Формат модуля: ESM (`.mjs`), іменовані експорти `export function ...`.

## Потік виконання / Використання

Типовий сценарій імпорту:

```js
import { parseQuery, buildQuery } from './url-parse.mjs'
```

Приклади парсингу з `parseQuery`:

```js
parseQuery('?a=1&b=2')
// → { a: '1', b: '2' }

parseQuery('a=1&b=2')
// → { a: '1', b: '2' }

parseQuery('flag&x=y')
// → { flag: '', x: 'y' }

parseQuery('a=1&&b=2&')
// → { a: '1', b: '2' }    (порожні пари пропускаються)

parseQuery('a=hello%20world')
// → { a: 'hello world' }

parseQuery('')
// → {}

parseQuery(null)
// → {}                    (нерядок → порожній об'єкт)

parseQuery('a=1=2')
// → { a: '1=2' }          (тільки перший '=' є роздільником)
```

Приклади побудови з `buildQuery`:

```js
buildQuery({ a: 1, b: 'hello world' })
// → 'a=1&b=hello%20world'

buildQuery({ a: 1, b: null, c: undefined, d: 2 })
// → 'a=1&d=2'             (null і undefined відкидаються)

buildQuery({ a: true, b: 0, c: false })
// → 'a=true&b=0&c=false'  (всі приводяться до рядка)

buildQuery({})
// → ''

buildQuery(null)
// → ''

buildQuery('not an object')
// → ''
```

Симетрія між функціями `parseQuery` і `buildQuery` зберігається лише для значень, які є рядками без `null`/`undefined`: круговий прохід `parseQuery(buildQuery(obj))` поверне об'єкт з тими ж ключами, але всі значення будуть рядками (бо `buildQuery` явно викликає `String(v)`).
