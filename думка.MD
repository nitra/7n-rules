# Архітектура: Динамічний Самомодифікований Граф Задач

## Назва структури

**Рекурсивний складений ОАГ** (орієнтований ациклічний граф) — із динамічним розкладом вузлів та файловим сховищем стану.

---

## Вузол

Кожен вузол або є атомарним, або розкладається на підграф — рішення приймається **динамічно при запуску** на основі вхідних даних.

```
Вузол
  ├── реалізація:
  │     ├── Атомарний  — fn(вхідні) → вихідні
  │     └── Складений  — Граф{ вхід, вузли[], ребра[], виходи[] }
  ├── стан: очікує | виконується | вирішено | помилка | невирішуваний
  ├── вхідні:  Map<портId, Значення>
  └── вихідні: Map<портId, Значення>   ← заповнюється при вирішено
```

Для батьківського вузла інтерфейс однаковий: він чекає `стан = вирішено` не знаючи що всередині (**інкапсуляція чорної скриньки**).

---

## Граф

- **ОАГ** — орієнтований, без циклів
- **Ребра** — потік даних: виходи одного вузла стають входами наступного
- **Вхідний вузол** — отримує вхідні дані батьківського складеного вузла
- **Вихідні вузли** — кілька, їх виходи зливаються у виходи батька
- Складений вузол може бути замінений атомарним і навпаки без змін у батьківському графі

```
СкладенийВузол
  └── Граф
        ├── вхід    → отримує вхідні дані батька
        ├── вузли[]: Вузол[]
        ├── ребра[]: { від: ВузолId+портId, до: ВузолId+портId }
        └── виходи[]: Вузол[]   ← кілька, злиття → виходи батька
```

## Файлова структура (сховище стану)

Граф зберігається у файловій системі — кожен вузол пише свої файли сам. Граф відновлюється скануванням.

```
tasks/
  <ідВузла>/
    meta.json            ← тип, стан, ребра
    inputs.json
    outputs.json         ← з'являється при вирішено
    error.json           ← з'являється при помилці
    repair_history.json  ← пам'ять інженера на цьому вузлі
    patches/
      patch_001.json     ← зміни від агента-інженера
    graph/               ← якщо складений вузол
      <ідДочірнього>/
        ...
```

Граф має **часовий вимір** — це послідовність знімків:
```
ОАГ_v1 → [ПатчІнженера] → ОАГ_v2 → [ПатчІнженера] → ОАГ_v3
```

---

## Самовідновлення: агент-інженер

Інженер — **мета-рівень поза графом**, без власного стану. Викликається при `стан = помилка`.

```
АгентІнженер(помилка, повний_шлях_від_кореня, repair_history) →
  ПатчГрафу:
    ├── замінити вузол (атомарний → складений або навпаки)
    ├── вставити вузли (додати проміжні кроки)
    ├── перепідключити ребра
    └── змінити вхідні дані
```

Може змінювати **будь-який рівень** — від вузла що впав до кореня.

### Каскад інвалідації

Зміна вузла інвалідує всіх наступників:
```
патч(вузол_6) →
  вузол_6.вихідні = застарілі →
    вузол_7, вузол_8 = знедійснені →
      їх наступники = знедійснені → ...
```

### Пам'ять

Пам'ять живе **на вузлі**, де інженер вносить зміни:
```json
repair_history.json
[
  {
    "attempt": 1,
    "engineer_reasoning": "...",
    "patch_applied": {},
    "result": "failed",
    "failure_reason": "..."
  }
]
```
При зміні батьківського вузла через помилку дочірнього:
- `батько.repair_history` ← записується патч
- `дочірній.repair_history` ← записується `{ triggered_parent_patch: id }`

## Збіжність: часовий бюджет

Замість ліміту спроб — **часовий бюджет**:

```
repair_context.json
  ├── крайній_термін: мітка_часу   ← встановлюється один раз
  ├── розпочато_о:   мітка_часу
  ├── бюджет_сек: 600              ← 10 хвилин
  └── спроби: []                   ← необмежено в межах часу
```

Інженер адаптує стратегію до залишку часу:
- багато часу → складна спроба
- мало часу → швидке виправлення
- `залишок ≤ 0` → `стан = невирішуваний`, ескалація

---

## Ієрархія ескалації

Кожен рівень отримує **свіжий** часовий бюджет:

```
вузол впав
  → агент-інженер (бюджет: 10хв)
      → час вийшов → ескалація
          → батько: агент-інженер (свіжий бюджет: 10хв)
              → час вийшов → ескалація
                  → дідусь: агент-інженер (свіжий бюджет: 10хв)
                      → ...
                          → корінь: час вийшов
                              → сповіщення старшого інженера (людина)
```

**Максимальний час до людини:** `глибина × бюджет`

### Сповіщення старшого інженера

```json
{
  "failed_node": "root/project_A/workflow_X/node_7",
  "escalation_chain": [
    { "level": "node_7",     "attempts": [], "time_spent": 600 },
    { "level": "workflow_X", "attempts": [], "time_spent": 600 },
    { "level": "project_A",  "attempts": [], "time_spent": 600 }
  ],
  "current_graph_snapshot": "...",
  "suggested_next_steps": []
}
```
