## Огляд

Огляд
Цей файл налаштовує середовище для запуску тестів, використовуючи Vitest. Він інтегрує Stryker для аналізу покриття коду шляхом мутаційного тестування, що спирається на конфігурації `mutation.json` та `incremental.json`. Він забезпечує виконання перевірки лише змінених рядків коду та зберігає детальні звіти мутацій у визначені файли.

## Поведінка

1. Визначає середовище для виконання тестів як Vitest, використовуючи конфігураційний файл `vitest.config.mjs`.
2. Налаштовує Stryker на запуск лише тих тестів, які покривають мутовану лінію, що оптимізує швидкість виконання порівняно із повним прогоном тестового набору.
3. Визначає тимчасову директорію для звітів мутацій у `reports/stryker/.tmp`.
4. Налаштовує генерацію звітів у форматі JSON та у вигляді чистого тексту, зберігаючи детальний звіт у `reports/stryker/mutation.json`.
5. Вмикає інкрементальне збереження результатів мутацій у `reports/stryker/incremental.json`, що дозволяє відновлювати роботу після збоїв.
6. Підтримує конфігураційні файли `mutation.json` та `incremental.json`.
