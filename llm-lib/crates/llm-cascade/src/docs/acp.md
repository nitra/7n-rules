## Огляд

ACP-бекенд для Zed, що працює через особисту підписку: спавнить уже залогінений локальний агентський CLI (`agent login` / `codex login` виконано заздалегідь власником) і спілкується з ним по stdio/JSON-RPC. Для цього шляху крейт не виставляє жодних API-ключів. `AcpAgentKind` і `one_shot_acp` дають окремий ACP-шлях доступу до моделей, а за збою він поводиться fail-fast без вбудованого retry; якщо ACP недоступний, подальший fallback на `crate::local_cloud` вирішує викликач.

## Поведінка

- `AcpAgentKind` — описує ACP-бекенд для запуску вже залогіненого локального CLI через підписку: Cursor або Codex; використовується для вибору цільового агента, а не API-ключа.
- `one_shot_acp` — виконує один ACP-запит у вказаному робочому каталозі, спавнить обраний агент, надсилає prompt і повертає текст відповіді; якщо ACP-шлях недоступний, повертає помилку, щоб викликач міг перейти на `crate::local_cloud`.

## Публічний API

- AcpAgentKind — задає, з яким ACP-агентом працювати; перелік можна розширювати новими варіантами без зміни протоколу.
- one_shot_acp — запускає один ACP-запит: піднімає агента, відкриває сесію в потрібному `cwd`, надсилає `prompt` і дочитує відповідь до завершення ходу; повертає помилку, якщо агент недоступний, не авторизований або падає під час виконання.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
