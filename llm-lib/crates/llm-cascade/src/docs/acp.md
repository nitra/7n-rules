## Огляд

ACP-бекенди (Agent Client Protocol, Zed) — доступ до потужних моделей через **особисту підписку**, а не API-ключ: спавниться вже залогінений локально агентський CLI (`agent login` / `codex login` виконано заздалегідь власником), спілкування йде по stdio/JSON-RPC.

## Поведінка

`one_shot_acp(agent, prompt)` виконує один виклик: спавнить агента, відкриває сесію в поточній директорії, надсилає prompt і читає повний текст відповіді до кінця ходу. Якщо CLI не встановлений, не залогінений підпискою або процес завершився з помилкою — повертається `CascadeError::Provider`, і викликач сам вирішує, чи падати далі на local/cloud рунги.

Крейт не виставляє жодних API-ключів для цього шляху.

## Публічний API

- `AcpAgentKind` — ціль ACP-підключення; список відкритий, протокол однаковий для всіх:
  - `Cursor` — Cursor CLI у нативному ACP-режимі (`agent acp`), без стороннього моста.
  - `Codex` — OpenAI Codex через офіційний міст `@agentclientprotocol/codex-acp` (наступник задеприкейченого `@zed-industries/codex-acp`).
- `one_shot_acp(agent, prompt) -> Result<String, CascadeError>` — один виклик через ACP.

## Гарантії поведінки

- Fail-fast без retry: недоступний чи незалогінений агент провалюється швидко з `Err`, а не висить (перевірено тестом зі свідомо неіснуючим бінарником під 5-секундним таймаутом) — драбина Cursor → Codex → local → cloud на це покладається.
- `Codex` цілиться саме в офіційний міст, не в задеприкейчений пакет (закріплено тестом).
