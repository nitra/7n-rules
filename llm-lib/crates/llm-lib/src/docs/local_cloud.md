## Огляд

[`genai`] для one-shot викликів через [`crate::one_shot_local_or_cloud`]: окремо підтримує локальний tier з кастомним OpenAI-сумісним ендпоінтом і cloud tier зі стандартною автентифікацією `genai` через змінні середовища провайдера. Публічні точки входу — `LocalProvider`, `LocalCloud`, `new` і `one_shot` — дають один fail-fast шлях без retry: один HTTP-виклик, а помилки перехоплюються й не виходять назовні; для окремих збоїв повертається порожнє значення, коли результат потрібно безпечно згортати в керовану відповідь.

## Поведінка

LocalProvider задає конфігурацію локального OpenAI-сумісного провайдера як мапу між префіксом моделі та його ендпоінтом і, за потреби, ключем доступу. Для локальних тирових викликів базовий URL має бути із завершальним слешем, щоб адреса не обрізалася під час збирання запиту; це особливо важливо для `http://127.0.0.1:8000/v1/` і подібних адрес. Значення ключа може братися з локального профілю провайдера, зокрема через `settings.json`, якщо сервер очікує перевірку авторизації.

LocalCloud тримає спільний стан маршрутизації між локальними й хмарними тирами: один набір локальних провайдерів, окремий клієнт для них і окремий клієнт для стандартних cloud-викликів. Уся поведінка побудована на fail-fast принципі: один запит, без retry, без прихованого повторення або накопичення проміжного стану.

new перетворює вхідну мапу локальних провайдерів на внутрішній реєстр і одразу готує правила резолву моделі для локальних тирових викликів. Після цього LocalCloud готовий розрізняти, чи модель належить локальному провайдеру, чи має піти в cloud-шлях.

one_shot бере цільовий tier, визначає модель для нього, потім розбирає її на provider/model і вибирає маршрут виконання. Якщо provider є серед локальних, запит іде через local-client у кастомний OpenAI-сумісний ендпоінт; якщо ні — через cloud-client, де модель передається без префіксу й далі розпізнається стандартною логікою провайдера. Результат зчитується як перший текст відповіді; якщо модель нічого не повернула або стався збій виклику, помилка перетворюється на кероване значення LlmError без виходу винятку назовні.

## Публічний API

- LocalProvider — Конфіг одного локального/кастомного OpenAI-сумісного провайдера.
- LocalCloud — Клієнт для local/cloud тирів. Локальні провайдери (напр. `omlx`) — кастомний OpenAI-сумісний ендпоінт із `local_providers`; будь-який інший provider-префікс трактується як відомий genai хмарний провайдер (модель передається без префіксу — genai сам розпізнає адаптер за іменем моделі).
- new — `local_providers`: мапа `provider-префікс → конфіг`, напр. `{"omlx": LocalProvider { base_url: "http://127.0.0.1:8000/v1", api_key: Some(key) }}`.
- one_shot — Один виклик чату для абстрактного тиру: резолвить `"provider/model-id"` через [`resolve_model`], б'є в local- чи cloud-клієнт залежно від того, чи `provider` є в `local_providers`.  # Errors [`LlmError::NoModelConfigured`] якщо для тиру не задано жодної env-змінної; [`LlmError::Provider`] на помилку самого виклику.
