## Огляд

Виконує one-shot виклики до LLM через `genai`: локальні тири спрямовує на кастомний OpenAI-сумісний endpoint, а хмарні тири — на провайдери зі стандартною автентифікацією через змінні середовища. Файл є єдиною точкою для `crate::one_shot_local_or_cloud` з fail-fast контрактом: один HTTP-виклик без retry, як у `runOneShot` з `@7n/llm-lib`, з перехопленням помилок без винятків назовні та з порожнім значенням, наприклад `null`, для частини помилкових сценаріїв.

## Поведінка

`LocalProvider` задає дані для локального OpenAI-сумісного провайдера: базову адресу та опційний ключ авторизації. Для коректного маршруту базова адреса має зберігати семантику каталогу; наприклад, `http://127.0.0.1:8000/v1/` не еквівалентний `http://127.0.0.1:8000/v1` для побудови кінцевого endpoint. Якщо локальний сервер перевіряє Bearer key, значення узгоджується з його `settings.json`.

`new` створює `LocalCloud` зі спільним набором локальних провайдерів і двома шляхами виконання: локальним для відомих provider-префіксів та хмарним для решти моделей. Цей стан надалі використовується всіма one-shot викликами без власних операцій запису.

`one_shot` приймає абстрактний тір, перетворює його на model-spec через конфігурацію оточення і передає виконання в `one_shot_with_spec`. Якщо для тиру немає налаштованої моделі, результатом є контрольована помилка, а не повторні спроби.

`one_shot_with_spec` є спільною точкою відправлення запиту: формує chat request із системним і користувацьким повідомленням, обирає локальний або хмарний клієнт за provider-префіксом і виконує рівно один HTTP-виклик. Для локального провайдера model-spec іде через локальний resolver, для хмарного — у провайдерський клієнт передається ім’я моделі без префікса, щоб стандартна автентифікація провайдера спрацювала через середовище.

Результат з `LocalCloud` повертається як перший текст відповіді моделі. Некоректний model-spec, помилка провайдера або порожня відповідь перетворюються на явний `LlmError`, що зберігає fail-fast поведінку без retry.

## Публічний API

- LocalProvider — Конфіг одного локального/кастомного OpenAI-сумісного провайдера.
- LocalCloud — Клієнт для local/cloud тирів. Локальні провайдери (напр. `omlx`) — кастомний OpenAI-сумісний ендпоінт із `local_providers`; будь-який інший provider-префікс трактується як відомий genai хмарний провайдер (модель передається без префіксу — genai сам розпізнає адаптер за іменем моделі).
- new — `local_providers`: мапа `provider-префікс → конфіг`, напр. `{"omlx": LocalProvider { base_url: "http://127.0.0.1:8000/v1", api_key: Some(key) }}`.
- one_shot — Один виклик чату для абстрактного тиру: резолвить `"provider/model-id"` через [`resolve_model`], б'є в local- чи cloud-клієнт залежно від того, чи `provider` є в `local_providers`.  # Errors [`LlmError::NoModelConfigured`] якщо для тиру не задано жодної env-змінної; [`LlmError::Provider`] на помилку самого виклику.
- one_shot_with_spec — Той самий один виклик чату, що й [`Self::one_shot`], але з явним `"provider/model-id"` замість тиру (задача T5, napi `oneShotLocalCloud`: приймає або тір, або явний model-spec — тут другий шлях, без жодного звернення до [`resolve_model`]/env).  # Errors [`LlmError::InvalidModelSpec`] якщо `spec` не парситься; [`LlmError::Provider`] на помилку самого виклику.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
