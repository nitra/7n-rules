## Огляд

`LlmError` задає спільний формат збоїв для каскадного доступу до LLM у Rust-аналогі env-контракту `@7n/llm-lib` (`model-tiers.mjs`), щоб викликач міг однаково реагувати на відсутню конфігурацію, помилки конкретного провайдера або ACP-агента й будувати власну драбину ескалації поверх одноразових викликів без вбудованого retry. Шар охоплює local/cloud тири через `genai` і ACP-бекенди особистих підписок, зокрема Codex та Cursor CLI.

## Поведінка

1. `LlmError` описує єдиний формат помилок для каскаду викликів LLM, щоб викликач міг обробляти збій без прив’язки до конкретного backend або ACP-агента.
2. Якщо для вибраного тиру не задано жодної відповідної env-змінної, повертається помилка про відсутню конфігурацію моделі для цього тиру.
3. Якщо переданий model spec не розбирається, повертається помилка про невалідний model spec.
4. Якщо збій стався під час самого звернення до провайдера, ACP-хендшейку або процесу, повертається помилка з текстом цього збою.
5. Додаткові рівні retry, ескалації чи fallback не вбудовані; їх має будувати зовнішній викликач, комбінуючи примітиви крейта.

## Публічний API

- LlmError — Помилка каскаду. Навмисно плоска — деталі провайдера/ACP-агента вже в тексті, без вкладеної типізації для кожного backend-у.
