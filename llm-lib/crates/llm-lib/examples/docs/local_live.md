## Огляд

Живий smok-test local-бекенда через genai проти omlx для швидкого підтвердження інтеграції.

Запуск: `N_LOCAL_MIN_MODEL=omlx/<model> cargo run --example local_live`

`N_LOCAL_MIN_MODEL=omlx/<model>` — конфігураційний маркер мінімальної моделі для цього smok-test.

Працює як fail-safe точка запуску для автоматичних перевірок: будь-який збій не піднімає виняток назовні.

## Поведінка

1. Готує локальне підключення до omlx і використовує базову адресу http://127.0.0.1:8000/v1/ як ціль для смок-тесту.
2. Запускає один мінімальний запит до local-бекенда, щоб швидко підтвердити, що інтеграція відповідає на просту бізнес-команду.
3. Повертає успіх лише тоді, коли отримує текстову відповідь; у цьому разі виводить результат у консоль як підтвердження працездатності.
4. У разі будь-якої помилки не піднімає її далі, а фіксує збій у stderr і завершує процес із кодом помилки, щоб сценарій був fail-safe для автоматичних перевірок.
5. Не виконує власних операцій запису у файлову систему чи базу даних; межа відповідальності файлу — лише перевірка доступності й відповіді local-бекенда.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
