## Огляд

Жива обкатка композитної драбини Cursor → Codex → local → cloud для короткого запиту: кожна ланка викликається реально, без моків, а результатом є друк того, яка саме ланка відповіла. Запуск: `cargo run --example ladder_live`. Помилки перехоплюються fail-safe, тож вони не виходять назовні як винятки; за окремих збоїв замість них повертається порожнє значення, наприклад `null`.

## Поведінка

1. Стартує живу перевірку драбини вибору провайдера для короткого запиту й фіксує час до першої успішної відповіді.
2. Пробує канали по черзі: спершу Cursor, далі Codex, потім локальний рівень, і лише після цього — хмарний рівень; кожен наступний крок запускається лише якщо попередній не дав результат.
3. Для локального рівня використовує доступний провайдер `omlx` з базовою адресою http://127.0.0.1:8000/v1/ та бере ключ доступу з середовища, якщо він заданий.
4. Якщо будь-яка ланка повертає відповідь, одразу повідомляє, яка саме ланка спрацювала, і виводить текст відповіді разом із затраченим часом.
5. Якщо ланка не спрацьовує, фіксує причину пропуску в stderr і переходить до наступної.
6. Якщо жодна ланка не дала результату, завершує виконання як невдале без винесення винятку назовні.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
