## Огляд

Бенч-калібрування дефолтів [`llm_lib::batch`] для задачі T6 на локальному omlx: запускається через `N_LOCAL_MIN_MODEL=omlx/<model-id> cargo run --release --example batch_bench`, бере модель з `N_LOCAL_MIN_MODEL` і працює з endpoint `http://127.0.0.1:8000/v1/` та `OMLX_API_KEY` у контракті, який очікує `LocalCloud::new` від `~/.omlx/settings.json`-налаштованого сервера. Проганяє N коротких промптів через [`llm_lib::batch::submit`] для кількох комбінацій `chunk_size` і `concurrency`, щоб відкалібрувати ці дефолти за часом виконання та кількістю успішних і помилкових відповідей.

## Поведінка

1. Перевіряє наявність моделі для локального запуску; якщо значення не задано, зупиняє виконання з зрозумілим повідомленням.
2. Готує локальний провайдер для omlx із endpoint `http://127.0.0.1:8000/v1/` та ключем доступу з середовища.
3. Створює фіксований набір коротких тестових запитів і позначає кожен окремим business ID.
4. Показує в консолі обрану модель, розмір набору та таблицю метрик для подальшого порівняння.
5. Послідовно проганяє тестовий набір через кілька комбінацій `chunk_size` і `concurrency`, щоб порівняти режими batch-обробки.
6. Для кожної комбінації запускає batch-виконання через `submit`, збирає результати та рахує успішні й помилкові відповіді.
7. Вимірює тривалість кожного прогону і виводить її разом із кількістю успішних та неуспішних результатів.
