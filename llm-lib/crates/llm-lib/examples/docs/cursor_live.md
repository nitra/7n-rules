## Огляд

`cargo run --example cursor_live` запускає живий смок-тест ACP-бекенда через уже залогінений Cursor CLI (`agent acp`) і робить реальний виклик підписки до сервісу. Це потрібно, щоб вручну перевірити, чи інтеграція з поточною Cursor-сесією працює в реальних умовах. Обробка помилок fail-safe: збої не пробиваються назовні як винятки.

## Поведінка

1. Перевіряє, чи доступний поточний робочий каталог для запуску живого звернення до вже залогіненого Cursor CLI.
2. Ініціює один реальний запит до ACP-бекенда від імені Cursor із короткою перевіркою працездатності.
3. Очікує відповідь і в разі успіху виводить підтвердження разом із отриманим текстом.
4. Якщо запит або відповідь завершується помилкою, фіксує її в stderr і завершує процес з ненульовим кодом.
5. Не виконує власних записів у файлову систему чи базу даних.
6. Не є автотестом: використовується як ручний живий смок-тест і витрачає реальну квоту підписки.

## Гарантії поведінки

- Власних операцій запису (ФС/БД) у файлі немає; виклики імпортованих модулів можуть писати.
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
