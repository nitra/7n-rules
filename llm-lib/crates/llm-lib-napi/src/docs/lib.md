## Огляд

Файл надає native-шар для `@7n/llm-lib`, який з’єднує JS-виклики з Rust-бібліотекою `llm-lib` для одноразових ACP-запитів, вибору моделі за тиром і local/cloud chat-викликів. Він існує, щоб JS-обгортка отримувала результати й помилки у форматі `napi`, не дублюючи ACP, tiers або local_cloud-поведінку.

## Поведінка

`one_shot_acp`, `resolve_model` і `one_shot_local_cloud` утворюють тонкий native-шар між JS-пакетом `@7n/llm-lib` і Rust-бібліотекою `llm-lib`: дані надходять із JS, приводяться до внутрішніх понять бібліотеки, передаються в `llm-lib`, а результат повертається назад у JS як готове текстове значення або помилка `napi`.

`resolve_model` працює як попередній безмережевий етап для вибору моделі: він розв’язує абстрактний тир у конкретний ідентифікатор provider/model за змінними оточення та не запускає LLM-запит. Це дає JS-обгортці змогу окремо перевірити, яку модель буде використано, перед фактичним викликом через local/cloud-каскад.

`one_shot_acp` передає одноразовий запит до підтриманого ACP-агента з особистою підпискою, використовуючи робочий каталог проєкту-викликача як контекст запуску. `one_shot_local_cloud` приймає конфігурацію локальних provider-ів із JS, поєднує її з вибраним тиром і делегує одноразовий chat-виклик каскаду local/cloud.

Спільний стан у цьому файлі не зберігається: кожен виклик самостійно перетворює вхідні JS-дані, делегує роботу в `llm-lib` і повертає результат. Файл не виконує власних операцій запису та не дублює ACP, tiers або local_cloud-логіку.

Усі помилки з нижнього шару мапляться у помилки `napi`, щоб JS-сторона отримувала узгоджений формат відмов. Невідомі назви ACP-агента або тиру зупиняють виклик до делегування в `llm-lib`; відсутність моделі для тиру в `resolve_model` повертається як помилка.

## Публічний API

- one_shot_acp — Один виклик через ACP-агента з особистою підпискою (`cursor`/`codex`). `cwd` — робочий каталог проєкту-викликача (не process cwd).
- resolve_model — Каскадне розв'язання абстрактного тиру (`min`/`avg`/`max`) у `"provider/model-id"` за `N_LOCAL_*`/`N_CLOUD_*` env — чиста функція, без мережевого виклику.
- one_shot_local_cloud — Один chat-виклик local/cloud тиру. `local_providers` — JSON-мапа `{ "<provider>": { "baseUrl": "...", "apiKey": "..." | null } }`.

- Помилки повертаються назовні у форматі `napi`.
- Помилкові стани не перетворюються на порожнє значення.
