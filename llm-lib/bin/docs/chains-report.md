## Огляд

Для створення секції "Огляд" описується функціональність, яка має наступні характеристики:

Операція виконується лише у режимі читання; система не вносить жодних змін до файлової системи чи баз даних. У разі виникнення помилок, система перехоплює їх (fail-safe), гарантуючи відсутність винятків, які можуть бути поширені назовні. Механізм кешування у цій функціональності відсутній.

## Поведінка

1. Зчитує лог-файл з трасуванням LLM-викликів, якщо він доступний. Якщо файл відсутній, процес завершується без помилок.
2. Парсить вміст лог-файлу для отримання структурованих даних про ланцюжки.
3. Обчислює агреговані звіти про ланцюжки, застосовуючи фільтр за часовим діапазоном, якщо він вказаний.
4. Виводить загальний підсумок за всіма обробленими ланцюжками.
5. Друк табличного звіту, згрупованого за типом ланцюжка, відображаючи кількісні показники успішності, відхилень, середню кількість кроків та використання хмарних ресурсів.
6. Друк табличного звіту, згрупованого за конкретним правилом, показуючи аналогічні кількісні показники.
7. Якщо визначено ланцюжки, які завжди ескалюються або повністю залежать від хмарних викликів, виводить список цих кандидатів.
8. Повідомляє про кількість незакритих ланцюжків, що завершилися помилкою або були примусово зупинені, які не враховуються в розрахунках швидкості.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
